The Configuration API surface manages **who** Fineract talks to for credit
checks (`CreditBureau`), **as which tenant alias** (`OrganisationCreditBureau`,
"OCB"), **with what URLs and credentials** (`CreditBureauConfiguration`), and
**for which loan products** (`CreditBureauLoanProductMapping`). All entities
sit under `org.apache.fineract.infrastructure.creditbureau.domain` in
`fineract-provider` and the JAX-RS resource is
`org.apache.fineract.infrastructure.creditbureau.api.CreditBureauConfigurationApiResource`
mounted at `/v1/CreditBureauConfiguration`.

## Resource summary

Source: `api/CreditBureauConfigurationApiResource.java`.

```java
@Path("/v1/CreditBureauConfiguration")
@Component
@Tag(name = "Credit Bureau Configuration", description = "")
@RequiredArgsConstructor
public class CreditBureauConfigurationApiResource {

    private static final Set<String> RESPONSE_DATA_PARAMETERS = new HashSet<>(
            Arrays.asList("creditBureauId", "alias", "country", "creditBureauProductId",
                          "startDate", "endDate", "isActive"));
    private static final String RESOURCE_NAME_FOR_PERMISSIONS = "CreditBureau";
```

All `GET` operations require `READ_CreditBureau` (or `ALL_FUNCTIONS_READ`)
and use `apiRequestParameterHelper` to honour standard `?fields=...`
projection. All `POST`/`PUT` operations route through the
[command pipeline](/command/overview) via `CommandWrapperBuilder`.

## CreditBureau entity

Source: `domain/CreditBureau.java`. Table `m_creditbureau`. The "catalogue"
row for a bureau implementation — one row per supported provider.

```java
@Entity
@Table(name = "m_creditbureau")
@Getter @Setter @NoArgsConstructor @Accessors(chain = true)
public class CreditBureau extends AbstractPersistableCustom<Long> {

    private String name;
    private String product;
    private String country;

    @Column(name = "implementation_key")
    private String implementationKey;

    public static CreditBureau fromJson(final JsonCommand command) {
        final String tname = command.stringValueOfParameterNamed("name");
        final String tproduct = command.stringValueOfParameterNamed("product");
        final String tcountry = command.stringValueOfParameterNamed("country");
        final String timplementationKey = command.stringValueOfParameterNamed("implementationKey");

        return new CreditBureau()
                .setName(tname).setProduct(tproduct)
                .setCountry(tcountry).setImplementationKey(timplementationKey);
    }
}
```

| Column | Type | Notes |
| --- | --- | --- |
| `id` | `BIGINT` | PK |
| `name` | `VARCHAR` | Display name, e.g. "Myanmar Credit Bureau" |
| `product` | `VARCHAR` | Bureau product, e.g. "CreditReport" |
| `country` | `VARCHAR` | 2-letter or country name |
| `implementation_key` | `VARCHAR` | Logical key the integration code switches on; the ThitsaWorks integration matches `THITSAWORKS` |

## OrganisationCreditBureau entity

Source: `domain/OrganisationCreditBureau.java`. Table
`m_organisation_creditbureau`. Binds a `CreditBureau` to the tenant under a
human-readable `alias`, with an `is_active` flag and the list of loan-product
mappings cascading from it.

```java
@Entity
@Table(name = "m_organisation_creditbureau")
public class OrganisationCreditBureau extends AbstractPersistableCustom<Long> {

    private String alias;

    @OneToOne
    @JoinColumn(name = "creditbureau_id", nullable = false)
    private CreditBureau creditbureau;

    @Column(name = "is_active")
    private boolean isActive;

    @OneToMany(mappedBy = "organisation_creditbureau", cascade = CascadeType.ALL)
    private List<CreditBureauLoanProductMapping> creditBureauLoanProductMapping = new ArrayList<>();

    public static OrganisationCreditBureau fromJson(final JsonCommand command,
                                                    CreditBureau creditbureau) {
        final String alias  = command.stringValueOfParameterNamed("alias");
        final boolean isActive = command.booleanPrimitiveValueOfParameterNamed("isActive");
        return new OrganisationCreditBureau(alias, creditbureau, isActive, null);
    }
}
```

| Column | Type | Notes |
| --- | --- | --- |
| `id` | `BIGINT` | PK |
| `alias` | `VARCHAR` | Tenant-readable name |
| `creditbureau_id` | `BIGINT` | FK to `m_creditbureau.id`, one-to-one |
| `is_active` | `BOOLEAN` | Controls whether the OCB participates in lookups |

## CreditBureauLoanProductMapping entity

Source: `domain/CreditBureauLoanProductMapping.java`. Table
`m_creditbureau_loanproduct_mapping`. Decides whether a loan product
requires a credit check from a given OCB and for how long a cached report
remains acceptable.

```java
@Entity
@Table(name = "m_creditbureau_loanproduct_mapping")
public class CreditBureauLoanProductMapping extends AbstractPersistableCustom<Long> {

    @Column(name = "is_credit_check_mandatory")
    private boolean creditCheckMandatory;

    @Column(name = "skip_credit_check_in_failure")
    private boolean skipCreditCheckInFailure;

    @Column(name = "stale_period")
    private int stalePeriod;

    @Column(name = "is_active")
    private boolean active;

    @ManyToOne
    private OrganisationCreditBureau organisation_creditbureau;

    @OneToOne
    @JoinColumn(name = "loan_product_id")
    private LoanProduct loanProduct;
```

| Column | Type | Notes |
| --- | --- | --- |
| `id` | `BIGINT` | PK |
| `is_credit_check_mandatory` | `BOOLEAN` | Block disbursal if no fresh report |
| `skip_credit_check_in_failure` | `BOOLEAN` | Allow continuing on bureau HTTP failure |
| `stale_period` | `INT` | Days after which a saved report must be refetched |
| `is_active` | `BOOLEAN` | Soft-disable |
| `organisation_creditbureau_id` | `BIGINT` | FK to `m_organisation_creditbureau.id` |
| `loan_product_id` | `BIGINT` | FK to `m_product_loan.id` (1:1) |

`fromJson` defends against null booleans by reading the parameters with
`booleanPrimitiveValueOfParameterNamed` and falling back to `false` /
`-1`:

```java
Boolean isCreditCheckMandatory = false;
Boolean skipCreditCheckInFailure = false;
Integer stalePeriod = -1;
Boolean isActive = false;
if ((Boolean) command.booleanPrimitiveValueOfParameterNamed("isCreditcheckMandatory") != null) {
    isCreditCheckMandatory = command.booleanPrimitiveValueOfParameterNamed("isCreditcheckMandatory");
}
// ... etc
```

Note the JSON casing — the parameter is `isCreditcheckMandatory`
(lower-case "c") in the request body.

## CreditBureauConfiguration entity

Source: `domain/CreditBureauConfiguration.java`. Table
`m_creditbureau_configuration`. Generic key/value rows scoped to an OCB.

```java
@Entity
@Table(name = "m_creditbureau_configuration")
public class CreditBureauConfiguration extends AbstractPersistableCustom<Long> {

    @Column(name = "configkey")
    private String configurationKey;

    @Column(name = "value")
    private String value;

    @Column(name = "description")
    private String description;

    @ManyToOne
    @JoinColumn(name = "organisation_creditbureau_id")
    private OrganisationCreditBureau organisationCreditbureau;
```

Standard config keys (see [`CreditBureauConfigurations`](/creditbureau/overview#creditbureauconfigurations-keys))
are `USERNAME`, `PASSWORD`, `SUBSCRIPTIONID`, `SUBSCRIPTIONKEY`, `TOKENURL`,
`SEARCHURL`, `CREDITREPORTURL`. Custom integrations may add more, e.g.
`addCreditReporturl` is read directly in
`ThitsaWorksCreditBureauIntegrationWritePlatformServiceImpl`:

```java
CreditBureauConfiguration addReportURL =
    this.configDataRepository.getCreditBureauConfigData(creditBureauId, "addCreditReporturl");
String url = addReportURL.getValue();
```

The `update` method maps the JSON command into actual changes for three
fields:

```java
public Map<String, Object> update(final JsonCommand command) {
    final Map<String, Object> actualChanges = new LinkedHashMap<>(1);

    final String configurationKey = "configkey";
    if (command.isChangeInStringParameterNamed(configurationKey, this.configurationKey)) {
        final String newValue = command.stringValueOfParameterNamed(configurationKey);
        actualChanges.put(configurationKey, newValue);
        this.configurationKey = StringUtils.defaultIfEmpty(newValue, null);
    }
    // value, description ...
    return actualChanges;
}
```

The `description` comparison contains a known bug — it compares against
`this.configurationKey` rather than `this.description` — but the code is
preserved here verbatim.

## CreditBureauToken entity

Source: `domain/CreditBureauToken.java`. Table `m_creditbureau_token`. The
cached bearer token (see [Integration API](/creditbureau/integration-api)
for usage).

| Column | Type | Notes |
| --- | --- | --- |
| `id` | `BIGINT` | PK |
| `username` | `VARCHAR` | Echoed from request |
| `token` | `VARCHAR` | Bearer access token |
| `token_type` | `VARCHAR` | OAuth token_type |
| `expires_in` | `VARCHAR` | Lifetime in seconds (string) |
| `issued` | `VARCHAR` | `.issued` from response |
| `expiry_date` | `DATE` | Parsed `.expires` |

```java
DateTimeFormatter dateformat = new DateTimeFormatterBuilder()
        .appendPattern("EEE, dd MMM yyyy kk:mm:ss zzz").toFormatter();
LocalDate expires = LocalDate.parse(expiry, dateformat);
```

Note that only the **date** (not the full instant) is persisted, so token
freshness is checked at day-granularity:
`if (DateUtils.isBefore(getExpiryDate, current)) { ... delete token ... }`.

## CreditReport entity

Source: `domain/CreditReport.java`. Table `m_creditreport`.

```java
@Entity
@Table(name = "m_creditreport")
public class CreditReport extends AbstractPersistableCustom<Long> {

    @Column(name = "credit_bureau_id")
    private Long creditBureauId;

    @Column(name = "national_id")
    private String nationalId;

    @Basic(fetch = FetchType.LAZY)
    @Column(name = "credit_reports")
    private byte[] creditReports;
```

A persisted report. The natural key is `(credit_bureau_id, national_id)`.
The `byte[]` payload is lazy-loaded to avoid pulling large blobs on
listing.

## Serializers / validators

| Class | Validates |
| --- | --- |
| `CreditBureauCommandFromApiJsonDeserializer` | Create/update of OCB-level data (`alias`, `isActive`, `creditBureauId`); permitted params are `alias`, `isActive`, `creditBureauId` |
| `CreditBureauConfigurationCommandFromApiJsonDeserializer` | Create/update of config rows; permitted params are `configkey`, `value`, `description`, each capped at 100 chars |
| `CreditBureauLoanProductCommandFromApiJsonDeserializer` | Mapping rows |
| `CreditBureauTokenCommandFromApiJsonDeserializer` | Parses token-grant responses (`access_token`, `token_type`, `expires_in`, `.issued`, `.expires`) |

All four deserializers reject unsupported parameters via
`fromApiJsonHelper.checkForUnsupportedParameters` and throw
`PlatformApiDataValidationException` on validation errors.

## Sample request bodies

### Add OCB

```http
POST /api/v1/CreditBureauConfiguration/organisationCreditBureau/3
Content-Type: application/json

{
  "alias":   "MCB - Production",
  "isActive": true
}
```

`3` is the catalogue `creditBureauId`. `alias` is required (≤ 100 chars).

### Add configuration row

```http
POST /api/v1/CreditBureauConfiguration/configuration/3
Content-Type: application/json

{
  "configkey":   "TOKENURL",
  "value":       "https://uat.mcix.com.mm/security/connect/token",
  "description": "OAuth token endpoint (UAT)"
}
```

All three fields are required (`notBlank`, ≤ 100 chars). The
[Integration API](/creditbureau/integration-api) consumes these values via
`getCreditBureauConfiguration(creditBureauId, "TOKENURL")`.

### Create loan product mapping

```http
POST /api/v1/CreditBureauConfiguration/mappings/2
Content-Type: application/json

{
  "loanProductId":           7,
  "isCreditcheckMandatory":  true,
  "skipCreditcheckInFailure": false,
  "stalePeriod":             30,
  "isActive":                true
}
```

`2` is the `organisationCreditBureauId`. Note the lower-case "c" in
`isCreditcheckMandatory` / `skipCreditcheckInFailure`.
