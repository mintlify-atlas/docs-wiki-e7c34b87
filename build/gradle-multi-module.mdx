Apache Fineract is built with Gradle as a multi-module project. The repository contains 30+ first-party modules (`fineract-core`, `fineract-loan`, `fineract-provider`, â€¦) plus a dynamically discovered tree of customer-owned modules under `custom/<company>/<category>/<module>/`. The orchestration of this graph lives in three files: `settings.gradle` (module inclusion), `gradle.properties` (JVM args and feature toggles), and the root `build.gradle` (plugins applied to every subproject). This page walks through each.

## Settings.gradle: module inclusion

`settings.gradle` does three things: configures Gradle Develocity (build cache, scans), enables the local build cache, and declares all Gradle subprojects with `include`. The static include list defines the Apache-published modules:

```groovy
rootProject.name='fineract'
include ':fineract-core'
include ':fineract-security'
include ':fineract-cob'
include ':fineract-validation'
include ':fineract-command'
include ':fineract-accounting'
include ':fineract-provider'
include ':fineract-branch'
include ':fineract-document'
include ':fineract-investor'
include ':fineract-rates'
include ':fineract-charge'
include ':fineract-tax'
include ':fineract-loan-origination'
include ':fineract-loan'
include ':fineract-savings'
include ':fineract-report'
include ':fineract-mix'
include ':fineract-war'
include ':integration-tests'
include ':twofactor-tests'
include ':oauth2-tests'
include ':fineract-client'
include ':fineract-client-feign'
include ':fineract-doc'
include ':fineract-avro-schemas'
include ':fineract-e2e-tests-core'
include ':fineract-e2e-tests-runner'
include 'fineract-progressive-loan'
include 'fineract-progressive-loan-embeddable-schedule-generator'
include 'fineract-working-capital-loan'
```

A separate `include ':custom:docker'` adds the fat-image assembly project, and a Groovy block walks the `custom/` directory to dynamically register every customer module:

```groovy
file("${rootDir}/custom").eachDir { companyDir ->
    if('build' != companyDir.name && 'docker' != companyDir.name) {
        file("${rootDir}/custom/${companyDir.name}").eachDir { categoryDir ->
            if('build' != categoryDir.name) {
                file("${rootDir}/custom/${companyDir.name}/${categoryDir.name}").eachDir { moduleDir ->
                    if('build' != moduleDir.name) {
                        include ":custom:${companyDir.name}:${categoryDir.name}:${moduleDir.name}"
                    }
                }
            }
        }
    }
}
```

The pattern is fixed at three levels deep: `custom/<company>/<category>/<module>/`. The `build` directory name is skipped (it is the Gradle output directory). See [custom modules](/build/custom-modules) for the complete authoring pattern.

### Develocity build scans

```groovy
plugins {
    id 'com.gradle.develocity' version '3.18.2'
    id 'com.gradle.common-custom-user-data-gradle-plugin' version '2.0.2'
}

develocity {
    projectId = "fineract"
    server = "https://develocity.apache.org"
    buildScan {
        uploadInBackground = !isCI
        publishing.onlyIf { it.authenticated }
        obfuscation { ipAddresses { it.collect { "0.0.0.0" } } }
    }
}
```

ASF members can sign in to [develocity.apache.org](https://develocity.apache.org) to inspect cached task outcomes, dependency resolution, and test results from CI runs. Jenkins (`JENKINS_URL` env var) is detected so background upload is disabled in CI to ensure the scan ships before the worker is torn down.

### Local build cache

```groovy
buildCache {
    local { enabled = true }
}
```

This is paired with `org.gradle.caching=true` in `gradle.properties` so configuration-cache friendly tasks can reuse outputs across builds.

## Gradle version and wrapper

The wrapper is checked in. From `gradle/wrapper/gradle-wrapper.properties`:

```properties
distributionUrl=https\://services.gradle.org/distributions/gradle-8.14.3-bin.zip
distributionBase=GRADLE_USER_HOME
distributionPath=wrapper/dists
networkTimeout=10000
validateDistributionUrl=true
```

Always invoke `./gradlew` (or `gradlew.bat` on Windows) â€” never a globally installed `gradle`. The wrapper downloads Gradle 8.14.3 on first run and pins all contributors to the same toolchain. The `validateDistributionUrl=true` flag protects against tampered distribution URLs.

Java toolchain is hard-pinned to **JDK 21** in the root `build.gradle`:

```groovy
java {
    toolchain { languageVersion = JavaLanguageVersion.of(21) }
    withSourcesJar()
    withJavadocJar()
}
```

If you do not have JDK 21 on `PATH`, Gradle's toolchain provisioning will attempt to download it via the configured provisioners. The Develocity / Foojay toolchain resolver plugin is not configured, so JDK 21 must be installed locally (e.g. via SDKMAN) for development.

```properties
  --add-exports jdk.compiler/com.sun.tools.javac.tree=ALL-UNNAMED \
  --add-exports jdk.compiler/com.sun.tools.javac.util=ALL-UNNAMED \
  --add-exports=java.naming/com.sun.jndi.ldap=ALL-UNNAMED \
  --add-opens=java.base/java.lang=ALL-UNNAMED \
  --add-opens=java.base/java.lang.invoke=ALL-UNNAMED \
  --add-opens=java.base/java.io=ALL-UNNAMED \
  --add-opens=java.base/java.security=ALL-UNNAMED \
  --add-opens=java.base/java.util=ALL-UNNAMED \
  --add-opens=java.management/javax.management=ALL-UNNAMED \
  --add-opens=java.naming/javax.naming=ALL-UNNAMED \
  --add-opens=java.rmi/sun.rmi.transport=null

buildType=BUILD
org.gradle.caching=true
org.gradle.parallel=true
org.gradle.daemon.idletimeout=10800000
# Temporarily disabled until we fix configuration cache compatibility
#org.gradle.configuration-cache=true
org.gradle.vfs.watch=true
```

| Property | Purpose |
| --- | --- |
| `-Xmx12g` | Heap for the Gradle daemon. The Spotbugs + ErrorProne + EclipseLink static-weaving pipeline is memory-hungry. |
| `--add-exports jdk.compiler/...` | Required by Spotbugs/ErrorProne which use internal `javac` APIs on JDK 17+. |
| `--add-opens java.base/...` | EclipseLink reflective access (also needed at runtime â€” see `fineract-common.env`). |
| `org.gradle.caching=true` | Enables the local cache; combines with Develocity remote cache when authenticated. |
| `org.gradle.parallel=true` | Lets independent subprojects compile in parallel. |
| `org.gradle.daemon.idletimeout=10800000` | 3-hour daemon idle timeout (3 Ă— 3600 Ă— 1000 ms) â€” keeps the daemon warm across edit-build cycles. |
| `org.gradle.vfs.watch=true` | Filesystem watching for incremental rebuilds. |
| `org.gradle.configuration-cache` | **Disabled** today. A comment notes it will be re-enabled once incompatible plugins are fixed. |

## Root build.gradle

The root build script does the heavy lifting. It defines three project groupings, applies common plugins, configures Spotless / RAT / Checkstyle / SpotBugs / ErrorProne / Modernizer, and wires the static-weaving and dependency-management conventions.

### Project groupings

Three `ext` arrays partition the subprojects:

| Group | Membership | Treatment |
| --- | --- | --- |
| `fineractJavaProjects` | All modules in the static include list **except** `fineract-war`, `fineract-doc`, `:custom:docker`. Custom modules are added dynamically. | Receive Java + Checkstyle + SpotBugs + ErrorProne + Spotless + License + RAT + JaCoCo configuration. |
| `fineractCustomProjects` | Every `custom/...` leaf module. | Get a slimmer dependency baseline (no Spring Boot starter test by default) and Lombok / MapStruct annotation processors. |
| `fineractPublishProjects` | Modules that are published to Maven Central / Apache snapshots: `fineract-core`, `fineract-security`, `fineract-loan`, `fineract-client`, etc. | Apply `maven-publish` and (unless `-PnoSign`) `signing`. |

The classification block from `build.gradle`:

```groovy
fineractJavaProjects = subprojects.findAll{
    [
        'fineract-core', 'fineract-security', 'fineract-cob',
        'fineract-validation', 'fineract-command', 'fineract-accounting',
        'fineract-provider', 'fineract-branch', 'fineract-document',
        'fineract-investor', 'fineract-charge', 'fineract-rates',
        'fineract-tax', 'fineract-loan-origination', 'fineract-loan',
        'fineract-savings', 'fineract-report', 'fineract-mix',
        'integration-tests', 'twofactor-tests', 'oauth2-tests',
        'fineract-client', 'fineract-client-feign',
        'fineract-avro-schemas',
        'fineract-e2e-tests-core', 'fineract-e2e-tests-runner',
        'fineract-progressive-loan',
        'fineract-progressive-loan-embeddable-schedule-generator',
        'fineract-working-capital-loan'
    ].contains(it.name)
}
```

Note that `fineract-war` is intentionally excluded â€” it only assembles a WAR archive and applies `apply plugin: 'war'` directly. Likewise `fineract-doc` produces Asciidoctor output and does not need Java compilation conventions.

### Versioning

```groovy
version = '0.0.0-SNAPSHOT'

gitVersioning.apply {
    refs {
        considerTagsOnBranches = true
        describeTagPattern = '.*(\\d+\\.\\d+\\.\\d+).*'
        describeTagFirstParent = false

        branch("release\\/\\d+\\.\\d+\\.\\d+") {
            version = '${describe.tag.version.major}.${describe.tag.version.minor}.${describe.tag.version.patch}'
        }
        branch("maintenance\\/\\d+\\.\\d+") {
            version = '${describe.tag.version.major}.${describe.tag.version.minor}.${describe.tag.version.patch}'
        }
        tag("(\\d+\\.\\d+\\.\\d+)") {
            version = '${describe.tag.version.major}.${describe.tag.version.minor}.${describe.tag.version.patch}'
        }
    }
    rev {
        version = '${describe.tag.version.major}.${describe.tag.version.minor.next}.0-SNAPSHOT'
    }
}
```

`me.qoomon.git-versioning` derives the project version from Git refs. On `release/X.Y.Z` and `maintenance/X.Y` branches the exact tag version is used; on arbitrary commits the next minor `-SNAPSHOT` is computed. This means the Docker image tag, the Maven artifact version, and the Swagger spec version all follow Git automatically.

### Common configuration applied to every project

The `allprojects { â€¦ }` block:

- Sets `group = 'org.apache.fineract'` and `repositories { mavenCentral() }`.
- Excludes `commons-logging` globally (jcl-over-slf4j replaces it).
- Substitutes vulnerable `org.lz4:lz4-java` with `at.yawk.lz4:lz4-java:1.10.1` (CVE-2025-12183).
- Applies `com.adarshr.test-logger`, `com.diffplug.spotless`, `com.github.hierynomus.license`, `net.ltgt.errorprone`, `org.nosphere.apache.rat`, and the license-report plugin.
- Configures Spotless for Markdown, properties, YAML, XML, JSON, SQL, gitignore, and `*.gradle` files.
- Configures the Apache Release Audit Tool (RAT) with a long excludes list for binary blobs, generated docs, and license-bearing third-party assets.

### Common configuration applied to `fineractJavaProjects`

The `configure(project.fineractJavaProjects) { â€¦ }` block adds Java-specific conventions:

```groovy
apply plugin: 'java'
apply plugin: 'idea'
apply plugin: 'eclipse'
apply plugin: 'checkstyle'
apply plugin: 'jacoco'
apply plugin: 'com.github.spotbugs'
apply plugin: 'com.github.andygoossens.modernizer'
apply from: "${rootDir}/buildSrc/src/main/groovy/org.apache.fineract.dependencies.gradle"
```

It also pins ASM 9.5 inside the SpotBugs classpath (for JDK 21 support), turns on incremental + forked Java compilation, sets `-parameters` and `-Xlint:*`, configures Lombok as `compileOnly`/`annotationProcessor`, and wires the `check` task to also run `rat`, `licenseMain`, and `licenseTest`. See [build-src](/build/build-src) for the dependency convention details.

### Static weaving fan-out

Right at the bottom of the common block:

```groovy
// Apply static weaving configuration to all subprojects
subprojects { subproject ->
    apply from: rootProject.file('static-weaving.gradle')
}
```

This pulls `static-weaving.gradle` (at the repo root) into every subproject. The script itself is a no-op for non-Java projects and for modules without a `src/main/resources/jpa/static-weaving/module/<name>/persistence.xml` file. See [static weaving](/build/static-weaving) for the EclipseLink details.

## Common Gradle tasks

| Task | Purpose |
| --- | --- |
| `./gradlew build` | Compile, run tests, run Checkstyle / SpotBugs / Modernizer / RAT / license / Spotless checks. |
| `./gradlew check` | Same quality gates as `build` minus the `assemble` step. Includes `cucumber` for `fineract-provider`. |
| `./gradlew :fineract-provider:bootRun` | Run the Spring Boot application from source. Picks up local code changes after a recompile. |
| `./gradlew :fineract-provider:bootJar` | Build the executable Spring Boot fat JAR at `fineract-provider/build/libs/`. |
| `./gradlew :fineract-provider:jibDockerBuild` | Build the Docker image into the local daemon as `fineract:latest`. See [Dockerfile & Jib](/build/dockerfile-and-jib). |
| `./gradlew :fineract-provider:jib` | Push the Docker image to the registry configured in `jib.to.image`. |
| `./gradlew :custom:docker:jibDockerBuild` | Build the `fineract-custom:latest` image with **all** custom modules layered in. |
| `./gradlew :fineract-provider:resolve` | Run the Swagger plugin to generate `fineract.json` / `fineract.yaml`. |
| `./gradlew :fineract-client:buildJavaSdk` | Regenerate the Java client SDK from the OpenAPI spec. |
| `./gradlew :fineract-provider:checkBreakingChanges` | Run swagger-brake against the committed baseline. See [swagger-brake](/build/swagger-brake). |
| `./gradlew spotlessApply` | Auto-format every formattable file. |
| `./gradlew licenseFormat` | Add the Apache license header to any source file missing one. |
| `./gradlew :integration-tests:test` | Run the Rest-Assured integration suite against a running server (`8443`). |
| `./gradlew printSourceSetInformation` | Diagnostic â€” prints source dirs, output dirs, and compile classpath per source set. |
| `./gradlew dependencies` | Inspect the dependency tree. Combine with `--configuration runtimeClasspath` to focus. |
| `./gradlew taskInfo --task=<name>` | Inspect inputs/outputs/dependencies of a single task (from `org.barfuin.gradle.taskinfo`). |

### `devRun` and quick iteration

The root description string sets a hint:

```
gradle clean bootRun
```

In practice the standard "dev run" is `./gradlew :fineract-provider:bootRun -x test -x spotlessCheck -x licenseMain -x rat` against a database started with one of the [docker-compose profiles](/build/docker-compose-profiles). Enable JRebel by adding `-PjrebelEnabled=true` if you have a licence â€” the `org.zeroturnaround.gradle.jrebel` plugin is wired in but applied lazily.

### Property flags

`build.gradle` honours a few `-P<name>=<value>` Gradle properties:

| Property | Effect |
| --- | --- |
| `-Pwarnings=fail` | Adds `-Werror` to javac. |
| `-Pwarnings=none` | Adds `-nowarn`. |
| `-PexcludeTests=<pattern>` | Excludes tests matching the given Gradle test filter. |
| `-PnoSign` | Skips `signing` plugin application on publish-eligible projects (use in CI without GPG keys). |
| `-PapiBaseline=<path>` | Points swagger-brake at a custom baseline file. See [swagger-brake](/build/swagger-brake). |
| `-Penv=eclipse` | Reroutes the `main` source set output to `bin/main` so Eclipse-built classes are picked up by Gradle. |

## Configuration cache caveat

`gradle.properties` keeps `org.gradle.configuration-cache` commented out. Multiple plugins in the stack (most notably some swagger / openapi tasks and the legacy `gradle-cargo-plugin`) still touch live `Project` instances at execution time. Do not enable it locally unless you are willing to debug the resulting cache misses â€” the team will flip it on once compatibility is achieved.
