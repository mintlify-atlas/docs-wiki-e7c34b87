`fineract-provider` ships its `@Configuration` classes under
`org.apache.fineract.infrastructure.core.config` and a handful of
neighbouring packages. Together they replace **six** Spring Boot
autoconfigurations, define the bean graph that wires Jersey, EclipseLink,
HikariCP, Spring Security, Spring Batch, Micrometer, and the in‑process
task executors, and validate the runtime mode at startup.

This page catalogues every class, its conditions, the `@Bean` methods it
declares, and the property keys it consumes. See
[`/runtime/server-application`](/runtime/server-application) for how
`ServerApplication` activates these classes, and
[`/overview/runtime-modes`](/overview/runtime-modes) for the broader
read/write/COB mode story.

## Autoconfiguration exclusions

`FineractWebApplicationConfiguration` excludes the following so Fineract
can supply its own beans without conflict:

```java FineractWebApplicationConfiguration.java
@EnableAutoConfiguration(exclude = {
        DataSourceAutoConfiguration.class,
        HibernateJpaAutoConfiguration.class,
        DataSourceTransactionManagerAutoConfiguration.class,
        GsonAutoConfiguration.class,
        JdbcTemplateAutoConfiguration.class,
        LiquibaseAutoConfiguration.class })
```

| Excluded autoconfiguration | Replaced by | Reason |
| --- | --- | --- |
| `DataSourceAutoConfiguration` | `HikariCpConfig` + `RoutingDataSource` | Spring's default would bind one global pool; Fineract needs a master tenants pool **plus** dynamic per‑tenant pools. |
| `HibernateJpaAutoConfiguration` | `JPAConfig` w/ `EclipseLinkJpaVendorAdapter` | Fineract uses EclipseLink with static weaving, not Hibernate. |
| `DataSourceTransactionManagerAutoConfiguration` | `ExtendedJpaTransactionManager` (registered in `ScheduledJobRunnerConfig`) | A `DataSourceTransactionManager` cannot honour JPA flush‑mode semantics and lifecycle callbacks; see [`/runtime/transaction-management`](/runtime/transaction-management). |
| `GsonAutoConfiguration` | `GoogleGsonSerializerHelper` & per‑adapter `JsonSerializer`s | Boot's autoconfig would register Gson as a Spring HTTP message converter; Fineract registers Gson manually inside the JAX‑RS pipeline. |
| `JdbcTemplateAutoConfiguration` | `JdbcConfig#jdbcTemplate(RoutingDataSource)` | The template must wrap the **routing** datasource, not the master Hikari pool directly. |
| `LiquibaseAutoConfiguration` | `TenantDatabaseUpgradeService` | Boot would run a single global change log; Fineract runs one per tenant DB. |

Everything else from `spring-boot-autoconfigure` is still in force: Web MVC,
Validation, Actuator + Micrometer, Jackson, Spring Batch (with
`spring.batch.job.enabled=false`), Spring Security, Spring Integration,
Spring Cache, etc.

## Bean‑override

```properties application.properties (line 462)
spring.main.allow-bean-definition-overriding=true
```

This is intentional because `RoutingDataSource` is declared
`@Service(value = "dataSource") @Primary` and Spring would otherwise warn
about the alias clashing with the Hikari master pool exposed as
`hikariTenantDataSource`.

## Per‑class detail

### `HikariCpConfig`

```java HikariCpConfig.java
@Configuration
@ConditionalOnExpression("#{ systemEnvironment['fineract_tenants_driver'] == null }")
public class HikariCpConfig {

    @Bean(initMethod = "getConnection", destroyMethod = "close")
    @ConfigurationProperties(prefix = "spring.datasource.hikari")
    public HikariDataSource hikariTenantDataSource() {
        return new HikariDataSource();
    }
}
```

| Aspect | Detail |
| --- | --- |
| Bean name | `hikariTenantDataSource` |
| Property prefix | `spring.datasource.hikari.*` (declared in `application.properties` lines 404‑428) |
| Activation | Only when the legacy env var `fineract_tenants_driver` is **not** set; otherwise `CompatibilityConfig` wins. |
| `initMethod = "getConnection"` | Forces eager pool initialisation by acquiring a connection at startup so misconfiguration fails fast. |
| `destroyMethod = "close"` | Closes the pool on container shutdown (paired with `server.shutdown=graceful`). |

The keys bound under `spring.datasource.hikari.*` include:

| Key | Default | Env var |
| --- | --- | --- |
| `driverClassName` | `org.mariadb.jdbc.Driver` | `FINERACT_HIKARI_DRIVER_SOURCE_CLASS_NAME` |
| `jdbcUrl` | `jdbc:mariadb://localhost:3306/fineract_tenants` | `FINERACT_HIKARI_JDBC_URL` |
| `username` | `root` | `FINERACT_HIKARI_USERNAME` |
| `minimumIdle` | `3` | `FINERACT_HIKARI_MINIMUM_IDLE` |
| `maximumPoolSize` | `10` | `FINERACT_HIKARI_MAXIMUM_POOL_SIZE` |
| `connectionTestquery` | `SELECT 1` | `FINERACT_HIKARI_TEST_QUERY` |
| `autoCommit` | `true` | `FINERACT_HIKARI_AUTO_COMMIT` |
| `transactionIsolation` | `TRANSACTION_REPEATABLE_READ` | `FINERACT_HIKARI_TRANSACTION_ISOLATION` |

Every other `dataSourceProperties['…']` entry maps to MySQL/MariaDB JDBC
optimisations (`cachePrepStmts`, `useServerPrepStmts`, etc.).

### `JdbcConfig`

```java JdbcConfig.java
@Configuration
public class JdbcConfig {
    @Bean public JdbcTemplate jdbcTemplate(RoutingDataSource dataSource) {
        return new JdbcTemplate(dataSource);
    }
    @Bean public NamedParameterJdbcTemplate namedParameterJdbcTemplate(RoutingDataSource dataSource) {
        return new NamedParameterJdbcTemplate(dataSource);
    }
}
```

Both templates wrap `RoutingDataSource` — they always go through tenant
routing, never directly to the master pool. See
[`/runtime/datasource-and-connection-pooling`](/runtime/datasource-and-connection-pooling).

### `JPAConfig`

`JPAConfig extends JpaBaseConfiguration` so it can override
`createJpaVendorAdapter()` (to return `EclipseLinkJpaVendorAdapter`),
`getVendorProperties(DataSource)` (to inject EclipseLink‑specific
properties), and `entityManagerFactory(...)` (to set the persistence unit
name to `jpa-pu` and use static weaving). It also registers
`AuditorAware<Long>` for `@CreatedBy` / `@LastModifiedBy` and a default
`TransactionTemplate`.

```java JPAConfig.java
@Override
protected Map<String, Object> getVendorProperties(DataSource dataSource) {
    Map<String, Object> vendorProperties = new HashMap<>();
    vendorProperties.put(PersistenceUnitProperties.WEAVING, "static");
    vendorProperties.put(PersistenceUnitProperties.PERSISTENCE_CONTEXT_CLOSE_ON_COMMIT, "true");
    vendorProperties.put(PersistenceUnitProperties.CACHE_SHARED_DEFAULT, "false");
    emFactoryCustomizers.forEach(c -> vendorProperties.putAll(c.additionalVendorProperties()));
    return vendorProperties;
}
```

The persistence unit post‑processor is
`DatabaseSelectingPersistenceUnitPostProcessor`, which adds
`PersistenceUnitProperties.TARGET_DATABASE` based on the driver class
detected by `DatabaseTypeResolver` (MariaDB / MySQL / PostgreSQL).

`@EnableJpaRepositories(basePackages = {"org.apache.fineract.**.domain", "org.apache.fineract.**.repository"})`
scans for Spring Data repository interfaces across every module.
`@EnableJpaAuditing` activates Spring's auditing hook (timestamps and
auditor IDs on `AbstractAuditableCustom`).

### `CacheConfig`

`CacheConfig` builds a `JCacheCacheManager` backed by EhCache 3, scans the
classpath for `@Cacheable` annotations and method/class-level
`@CacheConfig` via Reflections, and wraps the whole thing in a
`TransactionBoundCacheManager` so cache puts are only visible after the
enclosing transaction commits. The default template is sized via
`fineract.cache.default-template.maximum-entries` and
`fineract.cache.default-template.ttl`.

### `SecurityConfig`

```java SecurityConfig.java (excerpt)
@Configuration
@ConditionalOnProperty("fineract.security.basicauth.enabled")
@EnableMethodSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http.securityMatcher(API_MATCHER.matcher("/api/**"))
            .authorizeHttpRequests(auth -> { ... long list of permission rules ... })
            .httpBasic(hb -> hb.authenticationEntryPoint(basicAuthenticationEntryPoint()))
            .csrf(AbstractHttpConfigurer::disable)
            .sessionManagement(smc -> smc.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
            .addFilterBefore(tenantAwareBasicAuthenticationFilter(), SecurityContextHolderFilter.class)
            .addFilterAfter(requestResponseFilter(), ExceptionTranslationFilter.class)
            .addFilterAfter(correlationHeaderFilter(), RequestResponseFilter.class)
            .addFilterAfter(fineractInstanceModeApiFilter(), CorrelationHeaderFilter.class);
        ...
    }
}
```

`SecurityConfig` is the **single source of truth** for who can call which
URI. The filter chain order it builds — `TenantAwareBasicAuthenticationFilter`
→ `RequestResponseFilter` → `CorrelationHeaderFilter` →
`FineractInstanceModeApiFilter` → optional `LoanCOBApiFilter` /
`IdempotencyStoreFilter` / `ProgressiveLoanModelCheckerFilter` / 2FA — is
covered in [`/security/overview`](/security/overview).

### `SpringConfig`

Three responsibilities:

```java SpringConfig.java
@Bean(name = "fineractEventExecutor")
public ThreadPoolTaskExecutor fineractEventExecutor(...) {
    int cpus = Runtime.getRuntime().availableProcessors();
    int smartCore = cpus * 2;
    int smartMax = cpus * 5;
    int coreSize = configuredCore > 0 ? configuredCore : smartCore;
    int rawMaxSize = configuredMax > 0 ? configuredMax : smartMax;
    int queueCapacity = configuredQueueCapacity >= 0 ? configuredQueueCapacity : DEFAULT_QUEUE_CAPACITY;
    ...
    executor.setRejectedExecutionHandler(new ThreadPoolExecutor.CallerRunsPolicy());
    executor.setWaitForTasksToCompleteOnShutdown(true);
    executor.setAwaitTerminationSeconds(60);
    return executor;
}

@Bean
public MethodInvokingFactoryBean overrideSecurityContextHolderStrategy() {
    MethodInvokingFactoryBean factoryBean = new MethodInvokingFactoryBean();
    factoryBean.setTargetClass(SecurityContextHolder.class);
    factoryBean.setTargetMethod("setStrategyName");
    factoryBean.setArguments(SecurityContextHolder.MODE_INHERITABLETHREADLOCAL);
    return factoryBean;
}

@Bean
@DependsOn("overrideSecurityContextHolderStrategy")
public SimpleApplicationEventMulticaster applicationEventMulticaster(
        @Qualifier("fineractEventExecutor") ThreadPoolTaskExecutor taskExecutor) {
    SimpleApplicationEventMulticaster multicaster = new SimpleApplicationEventMulticaster();
    multicaster.setTaskExecutor(new DelegatingSecurityContextAsyncTaskExecutor(taskExecutor));
    return multicaster;
}
```

| Bean | Notes |
| --- | --- |
| `fineractEventExecutor` | Per‑CPU smart sizing (`cpus * 2` core, `cpus * 5` max) overridable by `spring.task.execution.pool.*`. `CallerRunsPolicy` backpressure. Drain on shutdown for 60s. |
| `overrideSecurityContextHolderStrategy` | Switches Spring Security to `InheritableThreadLocal` so child threads in `fineractEventExecutor` inherit the principal. |
| `applicationEventMulticaster` | Async multicaster wrapping the event executor in a `DelegatingSecurityContextAsyncTaskExecutor` (transports `SecurityContext` to event threads). |

### `OkHttp3Config`

```java OkHttp3Config.java
@Bean
public OkHttpClient okHttpClient() throws Exception {
    var okBuilder = new OkHttpClient.Builder()
            .connectTimeout(Duration.ofSeconds(fineractProperties.getClientConnectTimeout()))
            .readTimeout(Duration.ofSeconds(fineractProperties.getClientReadTimeout()))
            .writeTimeout(Duration.ofSeconds(fineractProperties.getClientWriteTimeout()));

```

| Property | Default | Purpose |
| --- | --- | --- |
| `fineract.client-connect-timeout` | `30` (seconds) | OkHttp `connectTimeout`. |
| `fineract.client-read-timeout` | `30` | OkHttp `readTimeout`. |
| `fineract.client-write-timeout` | `30` | OkHttp `writeTimeout`. |

### `FineractStartupValidationConfig`

```java FineractStartupValidationConfig.java
@Component
@Order(Ordered.HIGHEST_PRECEDENCE)
@Conditional(FineractValidationCondition.class)
public class FineractStartupValidationConfig implements InitializingBean {

    private final ApplicationContext applicationContext;

    @Override
    public void afterPropertiesSet() throws Exception {
        terminateApplication();
    }

    private void terminateApplication() {
        log.error("The application startup fails on validations. Please check the log above for the details");
        ((ConfigurableApplicationContext) applicationContext).close();
    }
}
```

If `FineractValidationCondition` matches, Spring instantiates this
component **first** (because of `HIGHEST_PRECEDENCE`) and its
`afterPropertiesSet()` immediately closes the context — the JVM exits with
a non‑zero status and the actual validation failure (logged by one of the
nested conditions below) becomes the boot's final word.

```java FineractValidationCondition.java
public class FineractValidationCondition extends AnyNestedCondition {

    public FineractValidationCondition() {
        super(ConfigurationPhase.PARSE_CONFIGURATION);
    }

    @Conditional(FineractModeValidationCondition.class)
    static class FineractModeValidation {}

    @Conditional(FineractPartitionJobConfigValidationCondition.class)
    static class FineractPartitionedJobValidation {}

    @Conditional(FineractRemoteJobMessageHandlerCondition.class)
    static class FineractRemoteJobMessageHandlerValidation {}

    @Conditional(FineractExternalEventConfigCondition.class)
    static class FineractExternalEventConfigValidation {}
}
```

It is an `AnyNestedCondition` — **any** of the four nested validation
conditions matching is enough to abort startup:

| Nested condition | Failure scenario |
| --- | --- |
| `FineractModeValidationCondition` | `fineract.mode.read-enabled=false` AND `fineract.mode.write-enabled=false`, i.e. the node would be unable to serve any traffic. |
| `FineractPartitionJobConfigValidationCondition` | A `fineract.partitioned-job.partitioned-job-properties[i].*` entry is missing required keys. |
| `FineractRemoteJobMessageHandlerCondition` | Both `fineract.remote-job-message-handler.jms.enabled` and `fineract.remote-job-message-handler.kafka.enabled` are true, or both are false in a node running in batch‑manager mode without `spring-events`. |
| `FineractExternalEventConfigCondition` | `fineract.events.external.enabled=true` but no producer (JMS, Kafka, Spring events) is enabled. |

The configurations the validation guards (read/write mode, partitioned
jobs, external events) are all driven by `FineractProperties` (under
`fineract-core`). The phase is `PARSE_CONFIGURATION` so the gate fires
before any other `@Bean` evaluates.

## Property files consumed

The active property source is
`fineract-provider/src/main/resources/application.properties` (725 lines).
Spring Boot also recognises `application-<profile>.properties` for profiles
like `liquibase-only` (none ship by default), and CLI overrides via
`--key=value`.

| Prefix | Bound to | Used by |
| --- | --- | --- |
| `fineract.*` | `FineractProperties` (`fineract-core`) | every feature flag, tenant‑config override, timeout, executor sizing |
| `spring.datasource.hikari.*` | `HikariCpConfig#hikariTenantDataSource()` | master tenants pool |
| `spring.jpa.*` | `JpaProperties` (autoconfig) | OSIV disabled, vendor name |
| `spring.liquibase.*` | `LiquibaseProperties` + `TenantDatabaseUpgradeService` | tenants DB migrations and per‑tenant parameters |
| `spring.task.execution.pool.*` | `SpringConfig#fineractEventExecutor` | event multicaster executor sizing |
| `spring.batch.*` | Spring Batch autoconfig | `spring.batch.job.enabled=false` to suppress autostart |
| `spring.servlet.multipart.*` | Servlet autoconfig | file upload sizes (`5MB` / `10MB`) |
| `server.*` | `ServerProperties` | Tomcat tuning, TLS, shutdown |
| `management.*` | Actuator | endpoints exposure, Prometheus, OTLP, CloudWatch |
| `logging.*` | Logback via `logback-spring.xml` | pattern, level (`FINERACT_LOGGING_LEVEL`) |
| `springdoc.*` | springdoc‑openapi | `/api-docs`, Swagger UI |
