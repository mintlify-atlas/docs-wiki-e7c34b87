`custom/acme/note/` is the Apache Fineract reference implementation for **replacing a portfolio service**. It exposes `AcmeNoteReadPlatformService` and `AcmeNoteWritePlatformService` as `@ConditionalOnProperty("acme.note.enabled")` beans that implement the same `NoteReadPlatformService` / `NoteWritePlatformService` interfaces the platform uses, and demonstrates the pattern for shipping a replacement schema (`acme_note_dummy` table) alongside it. The bodies are stubs — the read service returns empty lists and the write service throws `UnsupportedOperationException` — but the wiring is the real thing.

## The replacement write service

`com.acme.fineract.portfolio.note.service.AcmeNoteWritePlatformService`:

```java
@Slf4j
@Service
@ConditionalOnProperty("acme.note.enabled")
public class AcmeNoteWritePlatformService implements NoteWritePlatformService, InitializingBean {

    @Override
    public void afterPropertiesSet() throws Exception {
        log.warn("Note Write Service: '{}'", getClass().getCanonicalName());
    }

    @Override
    public NoteCreateResponse createNote(NoteCreateRequest request) {
        throw new UnsupportedOperationException("createNote() is not yet implemented.");
    }

    @Override
    public NoteUpdateResponse updateNote(NoteUpdateRequest request) {
        throw new UnsupportedOperationException("updateNote() is not yet implemented.");
    }

    @Override
    public NoteDeleteResponse deleteNote(NoteDeleteRequest request) {
        throw new UnsupportedOperationException("deleteNote() is not yet implemented.");
    }
}
```

Key contract points:

| Member | Purpose |
| --- | --- |
| `@Service` | Spring registers the class as a bean. |
| `@ConditionalOnProperty("acme.note.enabled")` | The bean is only created when the operator flips on `acme.note.enabled=true`. With the property absent or `false`, the stock `NoteWritePlatformServiceImpl` continues to win. |
| `implements NoteWritePlatformService` | Same interface as the stock implementation, so any autowire of that interface picks up the ACME bean transparently when it is active. |
| `implements InitializingBean` | Optional — used here to log a "Yes, I'm wired" message at boot. Helpful for diagnosing whether the conditional fired. |
| Method bodies | Stubs. A real customisation would persist to a custom table (`acme_note_dummy` or a richer schema). |

## The replacement read service

`AcmeNoteReadPlatformService` mirrors the write service:

```java
@Slf4j
@Service
@ConditionalOnProperty("acme.note.enabled")
public class AcmeNoteReadPlatformService implements NoteReadPlatformService, InitializingBean {

    @Override
    public void afterPropertiesSet() throws Exception {
        log.warn("Note Read Service: '{}'", getClass().getCanonicalName());
    }

    @Override
    public NoteData retrieveNote(Long noteId, Long resourceId, Integer noteTypeId) {
        return null;
    }

    @Override
    public List<NoteData> retrieveNotesByResource(Long resourceId, Integer noteTypeId) {
        return Collections.emptyList();
    }
}
```

Same `@ConditionalOnProperty` gate. The stub returns `null` / empty list, demonstrating that the platform tolerates "I don't know this note" responses from the read service.

## Liquibase changelog: `acme_note_dummy`

`custom/acme/note/starter/src/main/resources/db/custom-changelog/0001_acme_note_initial.xml`:

```xml
<changeSet author="acme" id="1">
    <createTable tableName="acme_note_dummy">
        <column autoIncrement="true" name="id" type="BIGINT">
            <constraints nullable="false" primaryKey="true"/>
        </column>
        <column name="name" type="VARCHAR(100)">
            <constraints unique="true"/>
        </column>
        <column name="description" type="VARCHAR(500)"/>
    </createTable>
</changeSet>
```

This is the canonical pattern: a custom module that needs schema ships its own Liquibase changelog under `db/custom-changelog/`. Fineract scans the classpath for those changelogs at boot, so just bundling the JAR is enough. The table is created unconditionally (Liquibase has no equivalent of `@ConditionalOnProperty`) but no application code writes to it unless `acme.note.enabled=true`.

## Test harness

The starter ships a tiny Cucumber harness whose only job is to assert the bean substitution actually works.

### `TestSuite`

```java
@CucumberContextConfiguration
@ExtendWith(SpringExtension.class)
@TestPropertySource("classpath:application-test.properties")
@WebAppConfiguration
public class TestSuite {}
```

`@CucumberContextConfiguration` tells `cucumber-spring` how to bootstrap the Spring context for the feature.

### `TestDefaultConfiguration` vs `TestOverrideConfiguration`

Two Spring configurations let the feature parametrise the test over "stock" and "ACME" wiring.

`TestOverrideConfiguration` (excerpt):

```java
@ComponentScan("com.acme.fineract")
class TestOverrideConfiguration {

    @Bean RoutingDataSourceServiceFactory routingDataSourceServiceFactory() { return mock(RoutingDataSourceServiceFactory.class); }
    @Bean RoutingDataSource              routingDataSource()                 { return mock(RoutingDataSource.class); }
    @Bean JdbcTemplate                   jdbcTemplate()                      { return mock(JdbcTemplate.class); }
    @Bean NoteRepository                 noteRepository()                    { return mock(NoteRepository.class); }
    @Bean ClientRepositoryWrapper        clientRepository()                  { return mock(ClientRepositoryWrapper.class); }
    @Bean GroupRepository                groupRepository()                   { return mock(GroupRepository.class); }
    @Bean LoanRepositoryWrapper          loanRepository()                    { return mock(LoanRepositoryWrapper.class); }
    @Bean LoanTransactionRepository      loanTransactionRepository()         { return mock(LoanTransactionRepository.class); }
}
```

This is the "no JPA, no JDBC, no datasource" minimum the platform needs to bring up the note services. Mockito mocks stand in for every repository / template the dependency graph would otherwise drag in.

`TestDefaultConfiguration` does not `@ComponentScan("com.acme.fineract")` — it brings up the stock platform services with the same mocked repository scaffold.

### Step definitions

`AcmeNoteServiceStepDefinitions` uses Spring Boot's `ApplicationContextRunner`:

```java
public AcmeNoteServiceStepDefinitions() {
    Given("/^A service configuration (.*)$/", (String configurationClassName) -> {
        contextRunner = new ApplicationContextRunner()
            .withPropertyValues("acme.note.enabled", "true")
            .withUserConfiguration(Class.forName(configurationClassName.trim()));
    });

    When("/^The user retrieves the service of interface class (.*)$/", (String interfaceClassName) -> {
        contextRunner.run(ctx -> {
            this.interfaceClass = Class.forName(interfaceClassName.trim());
            assertThat(this.interfaceClass.isInterface()).isTrue();
            assertThat(ctx).hasSingleBean(this.interfaceClass);
            this.implementationClass = ctx.getBean(interfaceClass).getClass();
        });
    });
    // …
}
```

### Feature file

`features/note.feature`:

```gherkin
Feature: Note Feature

  @modules
  Scenario Outline: Verify that the dummy service returns the correct message
    Given A service configuration <configurationClass>
    When The user retrieves the service of interface class <interfaceClass>
    Then The service class should match <serviceClass>

    Examples:
      | configurationClass                                                 | interfaceClass                                                      | serviceClass                                                            |
      | com.acme.fineract.portfolio.note.starter.TestDefaultConfiguration  | org.apache.fineract.portfolio.note.service.NoteReadPlatformService  | org.apache.fineract.portfolio.note.service.NoteReadPlatformServiceImpl  |
      | com.acme.fineract.portfolio.note.starter.TestDefaultConfiguration  | org.apache.fineract.portfolio.note.service.NoteWritePlatformService | org.apache.fineract.portfolio.note.service.NoteWritePlatformServiceImpl |
      | com.acme.fineract.portfolio.note.starter.TestOverrideConfiguration | org.apache.fineract.portfolio.note.service.NoteReadPlatformService  | com.acme.fineract.portfolio.note.service.AcmeNoteReadPlatformService    |
      | com.acme.fineract.portfolio.note.starter.TestOverrideConfiguration | org.apache.fineract.portfolio.note.service.NoteWritePlatformService | com.acme.fineract.portfolio.note.service.AcmeNoteWritePlatformService   |
```

Reading this as a table: when the test brings up the platform via `TestDefaultConfiguration`, the platform's read/write services resolve to `NoteReadPlatformServiceImpl` / `NoteWritePlatformServiceImpl`. When it brings up `TestOverrideConfiguration` (which `@ComponentScan`s `com.acme.fineract` and is run with `acme.note.enabled=true`), the platform's read/write services resolve to the ACME implementations instead.

That is exactly the assertion the test is supposed to make: the feature flag + scan combination substitutes the implementation atomically and without changing any caller.

## Operator workflow

| Step | Action |
| --- | --- |
| 1 | Build the WAR with the custom JAR on the classpath. |
| 2 | Set `acme.note.enabled=true`. |
| 3 | Boot Fineract. The `acme_note_dummy` table is created. The ACME note services come up; the stock `NoteReadPlatformServiceImpl` / `NoteWritePlatformServiceImpl` are still on the classpath but the platform autowire resolves the ACME beans by virtue of the conditional. |
| 4 | Existing `/api/v1/clients/<id>/notes/**` calls now hit the ACME services (which currently throw / return empty — replace them with real bodies in a fork). |

## Conventions and gotchas

- **`@ConditionalOnProperty` on every bean, not just the auto-configuration.** Putting the conditional only on `AcmeNoteAutoConfiguration` would still hit `@ComponentScan` and pick the `@Service` classes up; gating each individual bean is the safety belt that guarantees the bean is *not* created when the flag is off.
- **Bean type, not bean name.** Spring resolves the bean by type (`NoteReadPlatformService`). The stock impl and the ACME impl must therefore not coexist as eligible candidates, or you'll get a `NoUniqueBeanDefinitionException`. The conditional achieves this.
- **Liquibase runs unconditionally.** `acme_note_dummy` will exist even when the flag is off. Drop the changelog if you don't want that.
- **The stub throws.** The reference implementation deliberately throws on write so callers know they need to fill the bodies in.
- **Test override sees the real services.** Even though `TestOverrideConfiguration` mocks repositories, it uses the real ACME service classes — that is the point of the test.
