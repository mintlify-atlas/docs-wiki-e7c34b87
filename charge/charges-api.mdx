`ChargesApiResource` is the Apache Fineract Jersey resource at `/v1/charges` that exposes the catalog-level `Charge` entity to HTTP clients. The class lives at `fineract-charge/src/main/java/org/apache/fineract/portfolio/charge/api/ChargesApiResource.java` and is a thin wrapper: GETs go directly to `ChargeReadPlatformService`, writes build a `CommandWrapper` via `CommandWrapperBuilder` and dispatch through `PortfolioCommandSourceWritePlatformService` to the matching `@CommandType` handler.

For the underlying entity see [Charge domain](/charge/charge-domain); for the JSON validator and the handler bodies see [Charge serialization and handlers](/charge/charge-serialization-and-handlers).

## Class declaration

```java
@Path("/v1/charges")
@Component
@Tag(name = "Charges", description = """
        Its typical for MFIs to add extra costs for their financial products. These are typically Fees or Penalties.

        A Charge on fineract platform is what we use to model both Fees and Penalties.

        At present we support defining charges for use with Client accounts and both loan and saving products.""")
@RequiredArgsConstructor
public class ChargesApiResource {

    private static final String RESOURCE_NAME_FOR_PERMISSIONS = "CHARGE";

    private final PlatformSecurityContext context;
    private final ChargeReadPlatformService readPlatformService;
    private final DefaultToApiJsonSerializer<ChargeData> toApiJsonSerializer;
    private final ApiRequestParameterHelper apiRequestParameterHelper;
    private final PortfolioCommandSourceWritePlatformService commandsSourceWritePlatformService;
    // ...
}
```

Every endpoint guards on the literal `CHARGE` resource through `context.authenticatedUser().validateHasReadPermission("CHARGE")`. Write permissions are evaluated downstream by `SynchronousCommandProcessingService` against `m_permission` rows seeded by Liquibase (`CREATE_CHARGE`, `UPDATE_CHARGE`, `DELETE_CHARGE`, plus the `*_CHECKER` variants when maker-checker is enabled).

## Endpoint summary

| HTTP | Path | Method | Permission | Purpose |
| --- | --- | --- | --- | --- |
| `GET` | `/v1/charges` | `retrieveAllCharges` | `READ_CHARGE` | List every non-deleted catalog row. |
| `GET` | `/v1/charges/{chargeId}` | `retrieveCharge` | `READ_CHARGE` | Fetch a single charge. Optional `?template=true` merges dropdown options. |
| `GET` | `/v1/charges/template` | `retrieveNewChargeDetails` | `READ_CHARGE` | Standalone template for create forms. |
| `POST` | `/v1/charges` | `createCharge` | `CREATE_CHARGE` | Insert a new catalog row. |
| `PUT` | `/v1/charges/{chargeId}` | `updateCharge` | `UPDATE_CHARGE` | Partial update; rejects mutation of `chargeAppliesTo`. |
| `DELETE` | `/v1/charges/{chargeId}` | `deleteCharge` | `DELETE_CHARGE` | Soft-deletes (`is_deleted=true`, name rewritten). |

The Swagger schema names live in `ChargesApiResourceSwagger` — for example `PostChargesResponse`, `PutChargesChargeIdResponse`, `DeleteChargesChargeIdResponse`, `GetChargesResponse`. They are referenced from the resource via `@ApiResponse(... content = @Content(schema = @Schema(implementation = ...)))`.

## `GET /v1/charges`

```java
@GET
@Consumes({ MediaType.APPLICATION_JSON })
@Produces({ MediaType.APPLICATION_JSON })
@Operation(summary = "Retrieve Charges", description = """
        Returns the list of defined charges.

        Example Requests:

        charges""")
public List<ChargeData> retrieveAllCharges() {
    context.authenticatedUser().validateHasReadPermission(RESOURCE_NAME_FOR_PERMISSIONS);
    return readPlatformService.retrieveAllCharges();
}
```

Returns every catalog row that is **not soft-deleted**. The implementation is JDBC-driven (`ChargeReadPlatformService.retrieveAllCharges()`) and includes the income GL account, the tax group lookup, and the resolved currency.

## `GET /v1/charges/{chargeId}`

```java
@GET
@Path("{chargeId}")
@ApiResponse(responseCode = "200", description = "OK",
    content = @Content(schema = @Schema(implementation = ChargesApiResourceSwagger.GetChargesResponse.class)))
public ChargeData retrieveCharge(@PathParam("chargeId") final Long chargeId,
                                 @Context final UriInfo uriInfo) {
    context.authenticatedUser().validateHasReadPermission(RESOURCE_NAME_FOR_PERMISSIONS);
    final ApiRequestJsonSerializationSettings settings = apiRequestParameterHelper.process(uriInfo.getQueryParameters());
    ChargeData charge = readPlatformService.retrieveCharge(chargeId);
    if (settings.isTemplate()) {
        final ChargeData templateData = readPlatformService.retrieveNewChargeDetails();
        charge = ChargeData.withTemplate(charge, templateData);
    }
    return charge;
}
```

When `?template=true` is appended, the dropdown options from the template endpoint are merged onto the returned `ChargeData` — useful for "edit charge" UIs that want to render the form selectors and the current selections in a single round-trip.

`readPlatformService.retrieveCharge(...)` throws `ChargeNotFoundException` (mapped to HTTP 404) when the id does not resolve.

## `GET /v1/charges/template`

```java
@GET
@Path("template")
@Operation(summary = "Retrieve Charge Template", description = """
        This is a convenience resource. ...
        charges/template
        """)
public ChargeData retrieveNewChargeDetails() {
    context.authenticatedUser().validateHasReadPermission(RESOURCE_NAME_FOR_PERMISSIONS);
    return readPlatformService.retrieveNewChargeDetails();
}
```

Returns an empty `ChargeData` shell whose nested `*Options` lists are the dropdowns populated by `ChargeDropdownReadPlatformService`:

- `chargeAppliesToOptions` — `ChargeAppliesTo.LOAN, SAVINGS, CLIENT, SHARES`.
- `chargeTimeTypeOptions` — `ChargeTimeType.validLoanValues()` ∪ `validSavingsValues()` ∪ `validClientValues()` ∪ `validShareValues()`.
- `chargeCalculationTypeOptions` — `ChargeCalculationType.validValuesForLoan()` ∪ `validValuesForSavings()` ∪ `validValuesForClients()`.
- `chargePaymentModeOptions` — `REGULAR`, `ACCOUNT_TRANSFER`.
- `currencyOptions` — every row from `m_organisation_currency`.
- `feeFrequencyOptions` — `PeriodFrequencyType` constants.
- `incomeOrLiabilityAccountOptions` — `GLAccount` rows filtered by `accounting/glaccount`.
- `taxGroupOptions` — `m_tax_group` lookups from [Tax module](/tax/overview).

The same payload is returned by `retrieveCharge(...)` when `?template=true`, but `template` here returns the shell **without** an existing row preloaded.

## `POST /v1/charges`

```java
@POST
@Consumes({ MediaType.APPLICATION_JSON })
@Produces({ MediaType.APPLICATION_JSON })
@Operation(summary = "Create/Define a Charge", description = "Define a new charge that can later be associated with loans and savings through their respective product definitions or directly on each account instance.")
@RequestBody(required = true, content = @Content(schema = @Schema(implementation = ChargeRequest.class)))
@ApiResponse(responseCode = "200", description = "OK",
    content = @Content(schema = @Schema(implementation = ChargesApiResourceSwagger.PostChargesResponse.class)))
public CommandProcessingResult createCharge(@Parameter(hidden = true) ChargeRequest chargeRequest) {
    final CommandWrapper commandRequest = new CommandWrapperBuilder()
            .createCharge()
            .withJson(toApiJsonSerializer.serialize(chargeRequest))
            .build();
    return commandsSourceWritePlatformService.logCommandSource(commandRequest);
}
```

The path through `CommandWrapperBuilder.createCharge()` produces a `CommandWrapper` whose `actionName="CREATE"` and `entityName="CHARGE"`. `PortfolioCommandSourceWritePlatformService.logCommandSource(...)` persists the `m_command_source` audit row and dispatches to `CreateChargeDefinitionCommandHandler.processCommand(JsonCommand)`. The handler delegates to `ChargeWritePlatformService.createCharge(command)`, which:

1. Runs `ChargeDefinitionCommandFromApiJsonDeserializer.validateForCreate(json)`.
2. Resolves the optional `incomeAccountId`, `taxGroupId`, `paymentTypeId` via repository wrappers.
3. Calls `Charge.fromJson(...)` to instantiate.
4. Saves through `ChargeRepository`.

### `ChargeRequest` DTO

`fineract-charge/.../request/ChargeRequest.java` defines the POST/PUT body. The supported field set mirrors the parameters accepted by `ChargeDefinitionCommandFromApiJsonDeserializer`:

| Field | Type | Notes |
| --- | --- | --- |
| `name` | string | Required, ≤ 100 chars, must be unique. |
| `currencyCode` | string | Required, ISO 4217. |
| `amount` | BigDecimal | Required, positive. |
| `locale` | string | Required for numeric/locale parsing. |
| `chargeAppliesTo` | int | Required; one of `ChargeAppliesTo.validValues()`. |
| `chargeTimeType` | int | Required, must be valid for `chargeAppliesTo`. |
| `chargeCalculationType` | int | Required, must be valid for `chargeAppliesTo`. |
| `chargePaymentMode` | int | Required for loan charges. |
| `feeOnMonthDay`, `monthDayFormat`, `feeInterval`, `feeFrequency` | mixed | Recurring savings fee anchors. |
| `penalty`, `active` | boolean | Defaults to false / true. |
| `minCap`, `maxCap` | BigDecimal | For percentage-based charges. |
| `enableFreeWithdrawalCharge`, `freeWithdrawalFrequency`, `restartCountFrequency`, `countFrequencyType` | mixed | Withdrawal-fee waiver config. |
| `enablePaymentType`, `paymentTypeId` | mixed | Restrict the charge to a single `PaymentType`. |
| `incomeAccountId` | long | Optional GL account FK; mandatory for client charges. |
| `taxGroupId` | long | Optional tax group FK. |

The exact validation matrix is reproduced in [Charge serialization and handlers](/charge/charge-serialization-and-handlers).

  "name":                  "Late installment penalty",
  "currencyCode":          "USD",

## `PUT /v1/charges/{chargeId}`

```java
@PUT
@Path("{chargeId}")
@RequestBody(required = true, content = @Content(schema = @Schema(implementation = ChargeRequest.class)))
@ApiResponse(responseCode = "200", description = "OK",
    content = @Content(schema = @Schema(implementation = ChargesApiResourceSwagger.PutChargesChargeIdResponse.class)))
public CommandProcessingResult updateCharge(@PathParam("chargeId") final Long chargeId,
                                            @Parameter(hidden = true) ChargeRequest chargeRequest) {
    final CommandWrapper commandRequest = new CommandWrapperBuilder()
            .updateCharge(chargeId)
            .withJson(toApiJsonSerializer.serialize(chargeRequest))
            .build();
    return commandsSourceWritePlatformService.logCommandSource(commandRequest);
}
```

`CommandWrapperBuilder.updateCharge(chargeId)` produces a wrapper with `actionName="UPDATE"`, `entityName="CHARGE"`, `entityId=chargeId`. `UpdateChargeDefinitionCommandHandler` then calls `ChargeWritePlatformService.updateCharge(command.entityId(), command)`. That service runs `ChargeDefinitionCommandFromApiJsonDeserializer.validateForUpdate(json)` and then `charge.update(command)`, which:

- Treats the body as a **partial** update — only `command.isChangeInXxxParameterNamed(...)` deltas are applied.
- Rejects mutation of `chargeAppliesTo` with `ChargeParameterUpdateNotSupportedException` ("Update of Charge applies to is not supported").
- Rejects reassigning a non-null `taxGroup` via `failWithCode("modification.not.supported")`.
- Re-validates `(penalty, chargeTimeType)` combinations and throws `ChargeDueAtDisbursementCannotBePenaltyException` / `ChargeMustBePenaltyException` on conflict.

The response payload is the standard `CommandProcessingResult`:

```json
{
  "officeId": null,
  "resourceId": 17,
  "changes": {
    "amount": 30,
    "locale": "en"
  }
}
```

The `changes` map is exactly what `Charge.update(...)` returned to the handler.

## `DELETE /v1/charges/{chargeId}`

```java
@DELETE
@Path("{chargeId}")
@Produces({ MediaType.APPLICATION_JSON })
@Operation(summary = "Delete a Charge", description = "Deletes a Charge.")
@ApiResponse(responseCode = "200", description = "OK",
    content = @Content(schema = @Schema(implementation = ChargesApiResourceSwagger.DeleteChargesChargeIdResponse.class)))
public CommandProcessingResult deleteCharge(@PathParam("chargeId") final Long chargeId) {
    final CommandWrapper commandRequest = new CommandWrapperBuilder().deleteCharge(chargeId).build();
    return commandsSourceWritePlatformService.logCommandSource(commandRequest);
}
```

`DeleteChargeDefinitionCommandHandler` delegates to `ChargeWritePlatformService.deleteCharge(command.entityId())`. Three things happen:

1. The write service loads the row via `ChargeRepositoryWrapper.findOneWithNotFoundDetection(chargeId)`.
2. It checks references — if any `LoanProduct`, `SavingsProduct`, `ShareProduct`, `LoanCharge`, `SavingsAccountCharge`, `ClientCharge` or `ShareAccountCharge` still points to the catalog row, `ChargeCannotBeDeletedException` is thrown.
3. Otherwise `Charge.delete()` flips `is_deleted=true` **and rewrites `name`** to `<id>_<originalName>` so the unique constraint on `m_charge.name` is freed up for re-creation.

## Errors

| HTTP | Cause |
| --- | --- |
| 400 | `PlatformApiDataValidationException` — every rule that the deserializer enforces; the response body contains the `errors[]` array with `parameterName` and `userMessageGlobalisationCode`. |
| 400 | `ChargeDueAtDisbursementCannotBePenaltyException`, `ChargeMustBePenaltyException` — disbursement charges cannot be penalties; overdue installment charges must be. |
| 400 | `ChargeParameterUpdateNotSupportedException` — attempt to change `chargeAppliesTo` or reassign an existing `taxGroup`. |
| 400 | `ChargeCannotBeDeletedException` — catalog row still referenced by a product or account. |
| 401 / 403 | Missing `READ_CHARGE` / `CREATE_CHARGE` / `UPDATE_CHARGE` / `DELETE_CHARGE` permission. |
| 404 | `ChargeNotFoundException` — `chargeId` does not exist (or has been soft-deleted). |
| 409 | Unique constraint violation on `m_charge.name` — duplicate active name. |
