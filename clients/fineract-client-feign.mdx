`fineract-client-feign` is the second Java SDK for Apache Fineract. It is generated from the same `fineract.json` spec as [`fineract-client`](/clients/fineract-client) but uses the openapi-generator's `library: feign` mode, producing Feign-style interfaces backed by Apache HttpClient 5 (with an OkHttp fallback). Methods return values directly (`List<ClientData>` instead of `Call<List<ClientData>>`) and serialization is Jackson — both choices align the SDK with how Spring Cloud / Spring Boot apps usually call HTTP services. This page covers the build wiring, the `FineractFeignClient` / `FineractFeignClientConfig` builders, the connection-pool and TLS options, error decoding, and when to prefer Feign over Retrofit.

## Build configuration

From `fineract-client-feign/build.gradle`:

```groovy
apply plugin: 'org.openapi.generator'
description = 'Fineract Client with Feign'

openApiMeta {
    generatorName = 'Fineract-Feign'
    packageName   = 'org.apache.fineract.client.feign'
    outputFolder  = "$buildDir/meta".toString()
}

tasks.register('buildJavaSdk', org.openapitools.generator.gradle.plugin.tasks.GenerateTask) {
    generatorName    = 'java'
    library          = 'feign'
    inputSpec        = "file:///$swaggerFile"
    outputDir        = "$buildDir/generated/temp-java".toString()
    templateDir      = "$projectDir/src/main/resources/templates/java"
    groupId          = 'org.apache.fineract'
    apiPackage       = 'org.apache.fineract.client.feign.services'
    invokerPackage   = 'org.apache.fineract.client.feign'
    modelPackage     = 'org.apache.fineract.client.models'
    configOptions = [
        dateLibrary             : 'java8',
        library                 : 'feign',
        feignVersion            : '13.6',
        feignApacheHttpClient   : 'true',
        useFeign13              : 'true',
        useFeignApacheHttpClient: 'true',
        hideGenerationTimestamp : 'true',
        containerDefaultToNull  : 'true',
        oauth2Implementation    : 'none',
        useJakartaEe            : 'true'
    ]
    dependsOn(':fineract-provider:resolve')
}

sourceSets {
    main {
        java {
            srcDirs = [
                new File(buildDir, "generated/java/src/main/java"),
                "$projectDir/src/main/java"
            ]
        }
    }
}
```

Key choices:

| Option                       | Value          | Effect                                                                                                               |
|------------------------------|----------------|---------------------------------------------------------------------------------------------------------------------|
| `library`                    | `feign`        | OpenAPI Generator emits `@RequestLine(...)`-style Feign interfaces.                                                  |
| `feignVersion`               | `13.6`         | Pins to OpenFeign 13.x.                                                                                              |
| `feignApacheHttpClient`      | `true`         | Picks the Apache HttpClient 5 transport adapter over the OkHttp one.                                                 |
| `useFeign13` / `useJakartaEe`| `true`         | Use Feign 13 APIs and Jakarta EE namespaces (matches Spring Boot 3 / Java 17+).                                       |
| `oauth2Implementation`       | `none`         | No generated OAuth client; you plug your own interceptor.                                                            |
| `containerDefaultToNull`     | `true`         | Null vs empty collection distinction preserved.                                                                      |
| `apiPackage`                 | `org.apache.fineract.client.feign.services` | All `*Api` interfaces land here.                                                          |
| `modelPackage`               | `org.apache.fineract.client.models`         | Same model package as `fineract-client`. The model classes themselves are **not** literally shared, but the structures match. |
| `invokerPackage`             | `org.apache.fineract.client.feign`          | Configuration/auth classes generated here.                                                |

There is **no `cleanupGeneratedJavaFiles` task** in this module — Feign's mustache template is healthier than Retrofit's for the patterns the spec uses, so post-processing isn't needed.

The `sourceSets` declaration mixes the generated output with the hand-written code under `src/main/java`:


## Generated API style

A generated Feign interface looks like:

```java
public interface ClientApi {

    @RequestLine("GET /v1/clients")
    @Headers({ "Accept: application/json" })
    GetClientsResponse retrieveAll21(@QueryMap RetrieveAll21QueryParams queryParameters);

    @RequestLine("POST /v1/clients")
    @Headers({ "Accept: application/json", "Content-Type: application/json" })
    PostClientsResponse create7(PostClientsRequest postClientsRequest);

    @RequestLine("GET /v1/clients/{clientId}")
    @Headers({ "Accept: application/json" })
    GetClientsClientIdResponse retrieveOne8(
            @Param("clientId") Long clientId,
            @QueryMap RetrieveOne8QueryParams queryParameters);

    // … more endpoints
}
```

Compared to Retrofit:

- Return type is the **decoded model**, not `Call<T>`. The HTTP call is synchronous; throw on non-2xx.
- Query parameters with multiple values are grouped into a `*QueryParams` holder class.
- Headers are declared inline.

## FineractFeignClient (hand-written wrapper)

```java
public final class FineractFeignClient {

    private final FineractFeignClientConfig config;

    private FineractFeignClient(Builder builder) {
        this.config = builder.configBuilder.build();
    }

    public static Builder builder() { return new Builder(); }

    public <T> T create(Class<T> apiType) {
        return config.createClient(apiType);
    }

    // Convenience accessors for each generated API:
    public AccountNumberFormatApi    accountNumberFormat()   { return create(AccountNumberFormatApi.class); }
    public AccountTransfersApi       accountTransfers()      { return create(AccountTransfersApi.class); }
    public ClientApi                 clients()               { return create(ClientApi.class); }
    public LoansApi                  loans()                 { return create(LoansApi.class); }
    public SavingsAccountApi         savingsAccount()        { return create(SavingsAccountApi.class); }
    // … 120+ of these
}
```

`create(Class<T>)` is the entry point — both the convenience accessors and any custom Feign interface go through it.

### Builder

```java
public static class Builder {
    private final FineractFeignClientConfig.Builder configBuilder =
            FineractFeignClientConfig.builder();

    public Builder baseUrl(String baseUrl)                                 { configBuilder.baseUrl(baseUrl); return this; }
    public Builder credentials(String username, String password)           { configBuilder.credentials(username, password); return this; }
    public Builder tenantId(String tenantId)                               { configBuilder.tenantId(tenantId); return this; }

    public Builder connectTimeout(int t, TimeUnit unit)                    { configBuilder.connectTimeout(t, unit); return this; }
    public Builder readTimeout   (int t, TimeUnit unit)                    { configBuilder.readTimeout   (t, unit); return this; }
    public Builder connectionTimeToLive       (long t, TimeUnit unit)      { configBuilder.connectionTimeToLive       (t, unit); return this; }
    public Builder idleConnectionEvictionTime (long t, TimeUnit unit)      { configBuilder.idleConnectionEvictionTime (t, unit); return this; }
    public Builder maxConnections             (int  m)                     { configBuilder.maxConnTotal(m); return this; }
    public Builder maxConnectionsPerRoute     (int  m)                     { configBuilder.maxConnPerRoute(m); return this; }

    public Builder debug(boolean enabled)                                  { configBuilder.debugEnabled(enabled); return this; }
    public Builder disableSslVerification(boolean disable)                 { configBuilder.disableSslVerification(disable); return this; }
    public Builder httpClientType(FineractFeignClientConfig.HttpClientType ct) { configBuilder.httpClientType(ct); return this; }

    public FineractFeignClient build() { return new FineractFeignClient(this); }
}
```

The connection-pool levers (`connectionTimeToLive`, `idleConnectionEvictionTime`, `maxConnections`, `maxConnectionsPerRoute`) are not present in the Retrofit client — exposing them is one of the main reasons to pick Feign + Apache HC5 for high-throughput downstream services. See the comparison table at the end.

## FineractFeignClientConfig

The config materializes the Feign instance per API type:

```java
public <T> T createClient(Class<T> apiType) {
    JacksonEncoder jacksonEncoder = new JacksonEncoder(ObjectMapperFactory.getShared());
    Encoder multipartEncoder = new FineractMultipartEncoder(jacksonEncoder);

    return Feign.builder()
            .client(getOrCreateHttpClient())
            .encoder(multipartEncoder)
            .decoder(new ApiResponseDecoder(new JacksonDecoder(ObjectMapperFactory.getShared())))
            .errorDecoder(new FineractErrorDecoder())
            .options(new Request.Options(connectTimeout, TimeUnit.MILLISECONDS,
                                         readTimeout,    TimeUnit.MILLISECONDS, true))
            .retryer(Retryer.NEVER_RETRY)
            .requestInterceptor(new BasicAuthRequestInterceptor(username, password))
            .requestInterceptor(new TenantIdRequestInterceptor(tenantId))
            .logger(new Slf4jLogger(apiType))
            .logLevel(debugEnabled ? feign.Logger.Level.FULL : feign.Logger.Level.BASIC)
            .target(apiType, baseUrl);
}
```

Pieces:

| Component                                | Purpose                                                                                                       |
|------------------------------------------|---------------------------------------------------------------------------------------------------------------|
| `JacksonEncoder` / `JacksonDecoder`      | JSON serialization; the shared `ObjectMapper` (see `ObjectMapperFactory`) has the project's adapters wired.   |
| `FineractMultipartEncoder`               | Wraps the Jackson encoder to handle the `multipart/form-data` document upload endpoints.                       |
| `ApiResponseDecoder`                     | Wraps the Jackson decoder; understands the platform's standard envelopes.                                     |
| `FineractErrorDecoder`                   | Translates a non-2xx into `FeignException` and ultimately `CallFailedRuntimeException`.                       |
| `Retryer.NEVER_RETRY`                    | No automatic retries — caller decides.                                                                        |
| `BasicAuthRequestInterceptor`            | Adds `Authorization: Basic <base64>`.                                                                          |
| `TenantIdRequestInterceptor`             | Adds `Fineract-Platform-TenantId: <tenant>`.                                                                  |
| `Slf4jLogger`                            | All wire-level logging goes through SLF4J on the per-API logger.                                              |

### HTTP client choice

```java
public enum HttpClientType { APACHE, OKHTTP }

private Client createHttpClient() {
    switch (clientType) {
        case APACHE:
            // Build a PoolingHttpClientConnectionManager + CloseableHttpClient, then wrap as ApacheHttp5Client
            ...
        case OKHTTP:
            // ...
    }
}
```

The Apache HC5 path builds a `PoolingHttpClientConnectionManager` whose pool size, route limits, TTL, and idle-eviction timer are taken from the builder. The result is wrapped in `feign.hc5.ApacheHttp5Client`. The OkHttp path is provided for callers who prefer not to ship the Apache stack.

The client instance is cached via double-checked locking inside `getOrCreateHttpClient()` — one pool per `FineractFeignClient`, shared across every `create(...)` call.

### TLS bypass

`disableSslVerification(true)` installs an all-trusting `X509TrustManager` and a permissive `HostnameVerifier`. Same warning as the Retrofit client: **never on production**.

## End-to-end usage

```java
FineractFeignClient fineract = FineractFeignClient.builder()
        .baseUrl("https://localhost:8443/fineract-provider/api/v1")
        .credentials("mifos", "password")
        .tenantId("default")
        .connectTimeout(5_000, TimeUnit.MILLISECONDS)
        .readTimeout(30_000, TimeUnit.MILLISECONDS)
        .maxConnections(50)
        .maxConnectionsPerRoute(20)
        .connectionTimeToLive(5, TimeUnit.MINUTES)
        .idleConnectionEvictionTime(60, TimeUnit.SECONDS)
        .disableSslVerification(true)       // dev only
        .debug(true)
        .build();

// Direct, no Calls.ok wrapper needed
GetClientsClientIdResponse client = fineract.clients().retrieveOne8(1L, false);
System.out.println(client.getDisplayName());

// Create a new client
PostClientsRequest req = new PostClientsRequest();
req.setFirstname("Alice");
req.setLastname("Mwangi");
req.setOfficeId(1L);
req.setDateFormat("dd MMMM yyyy");
req.setLocale("en");
req.setActive(false);
PostClientsResponse created = fineract.clients().create7(req);
System.out.println("Created clientId=" + created.getClientId());
```

A non-2xx response is decoded by `FineractErrorDecoder` into a Feign exception (a `FineractException` subclass) which carries the HTTP status and the parsed problem body. Wrap calls in `try`/`catch` as needed.

## Side-by-side comparison with fineract-client

| Aspect                       | fineract-client (Retrofit2)                                                | fineract-client-feign (Feign)                                                |
|------------------------------|-----------------------------------------------------------------------------|------------------------------------------------------------------------------|
| Transport                    | OkHttp                                                                      | Apache HttpClient 5 (default) or OkHttp                                       |
| Serialization                | Gson                                                                        | Jackson                                                                      |
| Return type                  | `Call<T>` (manual `.execute()` or `Calls.ok(...)`)                          | `T` directly                                                                  |
| Connection pool tuning       | OkHttp defaults (no explicit knobs on builder)                              | TTL, eviction, max-total, max-per-route all on the builder                    |
| Auth                         | `HttpBasicAuth` interceptor                                                 | `BasicAuthRequestInterceptor`                                                 |
| Tenant header                | `ApiKeyAuth` interceptor                                                    | `TenantIdRequestInterceptor`                                                  |
| TLS skip                     | `.insecure(true)`                                                           | `.disableSslVerification(true)`                                               |
| Multipart                    | Custom `Parts` utility                                                      | `FineractMultipartEncoder`                                                    |
| Error wrapping               | `CallFailedRuntimeException`                                                | `FeignException` + `FineractErrorDecoder` + `CallFailedRuntimeException`      |
| Reactive                     | Could swap for RxJava2 (currently `false`)                                  | Sync only                                                                    |
| Jakarta EE namespace         | n/a                                                                         | Yes (`useJakartaEe: 'true'`)                                                  |
| OpenAPI generator library    | `retrofit2`                                                                 | `feign`                                                                       |
| Generated package            | `org.apache.fineract.client.{services,models}`                              | `org.apache.fineract.client.feign.{services,models}` + shared `client.models` |
| Code volume                  | Larger (Calls helper, Parts, JSON, custom adapter)                          | Smaller (more delegated to Feign)                                             |

**When to prefer Feign:**

- You already use Spring Cloud OpenFeign elsewhere.
- You need fine-grained connection-pool control (high-throughput batch).
- You want Jackson (matches your existing serialization stack).
- You want synchronous calls with direct return types.

**When to prefer Retrofit:**

- You want `Call<T>` for explicit response inspection and headers.
- You're integrating with an Android client (Retrofit is the lingua franca).
- You don't want Apache HC5 on the classpath.

## Regenerating after a server change

```bash
./gradlew :fineract-provider:resolve
./gradlew :fineract-client-feign:buildJavaSdk --rerun-tasks
./gradlew :fineract-client-feign:compileJava
```

If you change `templateDir`'s `api.mustache`, also `--rerun-tasks` to bypass the build cache.

## Spring Boot integration

Because the Feign interfaces are plain Java interfaces, they integrate cleanly with `@FeignClient` from Spring Cloud — but you don't *need* Spring Cloud. The simplest wiring:

```java
@Configuration
public class FineractConfig {

    @Bean
    public FineractFeignClient fineractFeignClient(@Value("${fineract.base-url}") String baseUrl,
                                                   @Value("${fineract.tenant-id}") String tenantId,
                                                   @Value("${fineract.username}")  String u,
                                                   @Value("${fineract.password}")  String p) {
        return FineractFeignClient.builder()
                .baseUrl(baseUrl)
                .tenantId(tenantId)
                .credentials(u, p)
                .maxConnections(64)
                .maxConnectionsPerRoute(16)
                .build();
    }

    @Bean public ClientApi  clientApi (FineractFeignClient c) { return c.clients(); }
    @Bean public LoansApi   loansApi  (FineractFeignClient c) { return c.loans();   }
    @Bean public SavingsAccountApi savingsApi(FineractFeignClient c) { return c.savingsAccount(); }
}
```

Then inject `ClientApi` directly into any `@Service`.

## Common pitfalls

| Symptom                                                                                | Cause                                                                                                | Fix                                                                                       |
|----------------------------------------------------------------------------------------|-------------------------------------------------------------------------------------------------------|--------------------------------------------------------------------------------------------|
| `feign.FeignException$NotFound: status 404`                                            | `baseUrl` does not include the version segment (`/api/v1`)                                          | Use `https://host:8443/fineract-provider/api/v1`                                           |
| `RetryableException: Connect to host:8443 [host/ip] failed`                            | TLS misconfigured or remote unreachable                                                              | Confirm cert; for local dev `.disableSslVerification(true)`                                |
| `Connections held > maxConnections`, HTTP retried slowly                              | Pool too small for the workload                                                                       | Increase `.maxConnections(...)` and `.maxConnectionsPerRoute(...)`                         |
| Sporadic `NoHttpResponseException` after server restart                                | Stale pooled connections                                                                              | Set `.connectionTimeToLive(...)` to a value below the server's keep-alive timeout         |
| `FeignException` body is empty                                                         | `ApiResponseDecoder` consumed the InputStream during error path                                      | Use `FineractErrorDecoder` (default) and inspect `responseBody()` rather than `body()`     |
| `JsonMappingException: Unrecognized field`                                              | New server response field; downstream library mismatch                                                | Rebuild the SDK against the latest `fineract.json`                                         |
| `IllegalStateException` from multipart upload                                          | `FineractMultipartEncoder` requires `MultipartFile` payloads to be wrapped correctly                  | See `FineractMultipartEncoder` source and the `DocumentsApiFixed` example                  |
