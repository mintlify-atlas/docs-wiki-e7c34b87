Loan-specific Spring Batch jobs in Apache Fineract are configured under `org.apache.fineract.portfolio.loanaccount.jobs.*`. Each job follows a uniform pattern: a `*Config` `@Configuration` declares a single `Step` and `Job` keyed off a `JobName` enum value; the `*Tasklet` runs the work. Most of these jobs were originally cron-driven but now have COB-step equivalents that run them inside the daily `LOAN_COB` chain — the standalone jobs remain for backward compatibility, legacy installs, and out-of-COB recovery.

This page lists every loan job, its `JobName` constant, its config and tasklet classes, and the matching COB business step (where one exists). The `JobName` enum is in `fineract-core` ([source](#)) and the COB steps are documented at [Loan COB business steps](/cob/loan-cob-business-steps).

## `JobName` constants (loan-related)

From `fineract-core/.../infrastructure/jobs/service/JobName.java`:

```java
public enum JobName {
    UPDATE_LOAN_ARREARS_AGEING("Update Loan Arrears Ageing"),
    APPLY_HOLIDAYS_TO_LOANS("Apply Holidays To Loans"),
    TRANSFER_FEE_CHARGE_FOR_LOANS("Transfer Fee For Loans From Savings"),
    APPLY_CHARGE_TO_OVERDUE_LOAN_INSTALLMENT("Apply penalty to overdue loans"),
    ADD_ACCRUAL_ENTRIES("Add Accrual Transactions"),
    ADD_PERIODIC_ACCRUAL_ENTRIES("Add Periodic Accrual Transactions"),
    RECALCULATE_INTEREST_FOR_LOAN("Recalculate Interest For Loans"),
    GENERATE_LOANLOSS_PROVISIONING("Generate Loan Loss Provisioning"),
    ADD_PERIODIC_ACCRUAL_ENTRIES_FOR_LOANS_WITH_INCOME_POSTED_AS_TRANSACTIONS(
            "Add Accrual Transactions For Loans With Income Posted As Transactions"),
    LOAN_COB("Loan COB"),
    LOAN_DELINQUENCY_CLASSIFICATION("Loan Delinquency Classification"),
    ACCRUAL_ACTIVITY_POSTING("Accrual Activity Posting"),
    WORKING_CAPITAL_LOAN_COB_JOB("Working Capital Loan COB"),
    // ...
}
```

The string value (e.g. `"Update Loan Arrears Ageing"`) is the human-readable name visible in `GET /v1/jobs`; the enum name (`UPDATE_LOAN_ARREARS_AGEING`) is the value stored in `job_name` on `job` and `job_run_history` tables and used to look up the Spring `Job` bean.

## Job catalog

<Note>
Jobs in **`fineract-loan`** are part of the core loan module and run on every Fineract topology. Jobs in **`fineract-provider`** are wired in the runtime module — same packaging, more dependencies (event publishing, accounting GL).
</Note>

### 1. `UPDATE_LOAN_ARREARS_AGEING`

| Aspect | Value |
| --- | --- |
| `JobName` | `UPDATE_LOAN_ARREARS_AGEING` |
| Config | `portfolio/loanaccount/jobs/updateloanarrearsageing/UpdateLoanArrearsAgeingConfig.java` (`fineract-loan`) |
| Tasklet | `UpdateLoanArrearsAgeingTasklet` |
| Handler | `LoanArrearsAgeingUpdateHandler` |
| COB step | [`UPDATE_LOAN_ARREARS_AGING`](/cob/loan-cob-business-steps) (`UpdateLoanArrearsAgingBusinessStep`) |

```java
@Bean
public Job updateLoanArrearsAgeingJob() {
    return new JobBuilder(JobName.UPDATE_LOAN_ARREARS_AGEING.name(), jobRepository)
            .start(updateLoanArrearsAgeingStep())
            .incrementer(new RunIdIncrementer()).build();
}
```

Refreshes `m_loan_arrears_aging` rows: for each open loan it computes overdue principal, interest, fees, penalties and number of days past due, then upserts the aggregate. See [Arrears and re-aging](/loan/loan-arrears-and-reaging).

### 3. `ADD_PERIODIC_ACCRUAL_ENTRIES`

| Aspect | Value |
| --- | --- |
| `JobName` | `ADD_PERIODIC_ACCRUAL_ENTRIES` |
| Config | `portfolio/loanaccount/jobs/addperiodicaccrualentries/AddPeriodicAccrualEntriesConfig.java` (`fineract-loan`) |
| Tasklet | `AddPeriodicAccrualEntriesTasklet` |
| Service | `LoanAccrualsProcessingService` |
| COB step | [`ADD_PERIODIC_ACCRUAL_ENTRIES`](/cob/loan-cob-business-steps) (`AddPeriodicAccrualEntriesBusinessStep`) |

```java
@Configuration @RequiredArgsConstructor
public class AddPeriodicAccrualEntriesConfig {
    private final JobRepository jobRepository;
    private final PlatformTransactionManager transactionManager;
    private final LoanAccrualsProcessingService loanAccrualsProcessingService;

    @Bean
    public Job addPeriodicAccrualEntriesJob() {
        return new JobBuilder(JobName.ADD_PERIODIC_ACCRUAL_ENTRIES.name(), jobRepository)
                .start(addPeriodicAccrualEntriesStep())
                .incrementer(new RunIdIncrementer()).build();
    }

    @Bean
    public AddPeriodicAccrualEntriesTasklet addPeriodicAccrualEntriesTasklet() {
        return new AddPeriodicAccrualEntriesTasklet(loanAccrualsProcessingService);
    }
}
```

Creates accrual transactions on each loan up to *today* for products with `accountingRule = ACCRUAL_PERIODIC`. See [Accounting overview](/accounting/overview).

### 12. `LOAN_COB` (orchestrator)

| Aspect | Value |
| --- | --- |
| `JobName` | `LOAN_COB` |
| Config | `cob/loan/LoanCOBConfig.java` (`fineract-provider`) |
| COB step chain | All steps in [Loan COB business steps](/cob/loan-cob-business-steps) |

`LOAN_COB` is the *master* job that orchestrates the per-loan business-step chain. It does not classify or accrue on its own — instead it iterates open loans and invokes each enabled `LoanCOBBusinessStep` in configured order. The other jobs above remain available as standalone fallbacks.

## Pattern: `*Config` template

Every loan job follows this pattern:

```java
@Configuration
@RequiredArgsConstructor
public class FooConfig {
    private final JobRepository jobRepository;
    private final PlatformTransactionManager transactionManager;
    private final FooService fooService; // domain dependency

    @Bean
    protected Step fooStep() {
        return new StepBuilder(JobName.FOO.name(), jobRepository)
                .tasklet(fooTasklet(), transactionManager).build();
    }

    @Bean
    public Job fooJob() {
        return new JobBuilder(JobName.FOO.name(), jobRepository)
                .start(fooStep())
                .incrementer(new RunIdIncrementer()).build();
    }

    @Bean
    public FooTasklet fooTasklet() {
        return new FooTasklet(fooService);
    }
}
```

The `RunIdIncrementer` lets the framework launch a fresh execution of the same `Job` on each schedule tick. Step and job both share `JobName.FOO.name()` — the Fineract job scheduler (in `fineract-core/.../infrastructure/jobs/`) discovers `Job` beans by this name when matching `m_jobs.name` to runnable Spring beans.

## Cron vs. COB

Historically all loan operations ran as cron jobs:

```
0 0 5 * * ? UPDATE_LOAN_ARREARS_AGEING
0 0 5 * * ? LOAN_DELINQUENCY_CLASSIFICATION
0 0 5 * * ? ADD_PERIODIC_ACCRUAL_ENTRIES
0 0 5 * * ? APPLY_CHARGE_TO_OVERDUE_LOAN_INSTALLMENT
```

The COB framework now lets the same logic execute inside a per-loan transaction in a deterministic order. Two ways to migrate:

1. **Disable the standalone job** (`PUT /v1/jobs/{id} { "active": false }`) and add the matching step to the active `LOAN_COB` chain.
2. **Run both** — safe for idempotent jobs (e.g. arrears aging just overwrites the same row).

See [COB batch jobs](/cob/cob-batch-jobs) for the full picture of how `LOAN_COB` interacts with the rest of the batch system.

## Job execution observability

- `m_job_run_history` records each execution's start/end time, success/failure and trigger type.
- Failed `Tasklet` invocations push errors onto the configured Slack / email channel (`JobExecutionListenerImpl`).
- `GET /v1/jobs/{jobId}/job-history` returns the rolling history.

## Manual trigger

```bash
POST /fineract-provider/api/v1/jobs/{jobId}?command=executeJob
```

For loan jobs, this is useful when:

- Recovering after a COB failure mid-day.
- Backfilling `m_loan_arrears_aging` after a manual schedule rewrite.
- Re-classifying delinquency immediately after creating a new `DelinquencyBucket`.
