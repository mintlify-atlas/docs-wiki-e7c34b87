# Client Address

Fineract stores postal addresses in a small dedicated subdomain shared by
the `client` and (potentially) other party-modules. The `Address` entity
holds the address itself, `ClientAddress` is the typed many-to-many link
between a `Client` and an `Address`, and `FieldConfiguration` controls
which address fields are required at the UI layer.

| Concern | Module | Package |
| --- | --- | --- |
| `Address` entity | `fineract-provider` | `org.apache.fineract.portfolio.address.domain` |
| `ClientAddress` entity (join + type + active flag) | `fineract-provider` | `org.apache.fineract.portfolio.client.domain` |
| `FieldConfiguration` entity | `fineract-provider` | `org.apache.fineract.portfolio.address.domain` |
| `AddressData`, `FieldConfigurationData` DTOs | `fineract-core` / `fineract-provider` | `…portfolio.address.data` |
| `ClientAddressApiResource` | `fineract-provider` | `org.apache.fineract.portfolio.client.api` |
| `EntityFieldConfigurationApiResource` | `fineract-provider` | `org.apache.fineract.portfolio.address.api` |
| `AddressWritePlatformService(Impl)`, `AddressReadPlatformService(Impl)` | `fineract-provider` | `org.apache.fineract.portfolio.address.service` |
| `ClientAddressRepository(Wrapper)` | `fineract-provider` | `org.apache.fineract.portfolio.client.domain` |
| Handlers: `AddClientAddressCommandHandler`, `UpdateClientAddressCommandHandler` | `fineract-provider` | `org.apache.fineract.portfolio.client.handler` |

See [Clients](/portfolio/clients) for the owning party. The address
sub-system also feeds the bulk-import flow and the standard reports.

## Address entity

`Address` maps to `m_address`:

```java
@Entity
@Table(name = "m_address")
public class Address extends AbstractPersistableCustom<Long> {

    @OneToMany(mappedBy = "address", cascade = CascadeType.ALL)
    private Set<ClientAddress> clientaddress;

    @Column(name = "street")          private String street;
    @Column(name = "address_line_1")  private String addressLine1;
    @Column(name = "address_line_2")  private String addressLine2;
    @Column(name = "address_line_3")  private String addressLine3;
    @Column(name = "town_village")    private String townVillage;
    @Column(name = "city")            private String city;
    @Column(name = "county_district") private String countyDistrict;

    @ManyToOne @JoinColumn(name = "state_province_id")
    private CodeValue stateProvince;

    @ManyToOne @JoinColumn(name = "country_id")
    private CodeValue country;

    @Column(name = "postal_code") private String postalCode;
    @Column(name = "latitude")    private BigDecimal latitude;
    @Column(name = "longitude")   private BigDecimal longitude;

    @Column(name = "created_by") private String createdBy;
    @Column(name = "created_on") private LocalDate createdOn;
    @Column(name = "updated_by") private String updatedBy;
    @Column(name = "updated_on") private LocalDate updatedOn;
}
```

Two CodeValue lookups are wired in:

- `state_province_id` → `STATE` (or `ADDRESS_TYPE_STATE` depending on the
  install) — the dropdown of provinces.
- `country_id` → `COUNTRY` — ISO list of countries.

The `created_by` / `updated_by` columns here are **string** copies for
historical reasons (the address sub-system pre-dated the unified audit
mixin).

## ClientAddress entity

```java
@Entity
@Table(name = "m_client_address")
public class ClientAddress extends AbstractPersistableCustom<Long> {

    @ManyToOne private Client client;
    @ManyToOne private Address address;

    @ManyToOne @JoinColumn(name = "address_type_id")
    private CodeValue addressType;

    @Column(name = "is_active")
    private boolean isActive;

    public static ClientAddress fromJson(final boolean isActive, final Client client,
            final Address address, final CodeValue address_type) {
        return new ClientAddress(client, address, address_type, isActive);
    }
}
```

The `address_type_id` is a CodeValue against `ADDRESS_TYPE`. Standard
seeded values include `Permanent Address`, `Current Address`, `Office
Address`, `Other` — institutions can extend the code freely.

The `is_active` flag denotes the "active address of this type" — when
two `Permanent Address` rows exist for the same client, only one should
have `is_active = true`. The write service flips the old one to false
when a new active address of the same type is uploaded.

## FieldConfiguration

```java
@Entity
@Table(name = "m_field_configuration")
public class FieldConfiguration extends AbstractPersistableCustom<Long> {

    private String entity;
    private String table;
    private String field;
    private boolean isEnabled;

    public FieldConfiguration(final String entity, final String table,
                              final String field, final boolean is_enabled) {
        this.entity = entity;
        this.table = table;
        this.field = field;
        this.isEnabled = is_enabled;
    }
}
```

`m_field_configuration` is a tiny lookup: each row says "for entity X,
field Y on table Z is `enabled` (or not)". The platform ships rows for
the address fields; the UI honours `isEnabled` to decide whether a field
should be rendered.

| `entity` | Example |
| --- | --- |
| `ADDRESS` | `m_address.street`, `m_address.country_id`, etc. |
| `CLIENT` | reserved for future use (per-field client toggles) |

## ClientAddressApiResource

Mounted at `/v1/client`. Note the singular path (not `/v1/clients`),
which is a legacy quirk preserved for backwards compatibility.

| Method | Path | Operation | Permission |
| --- | --- | --- | --- |
| `GET` | `/v1/client/addresses/template` | Template (CodeValue lookups for state, country, addressType) | `READ_CLIENTADDRESS` |
| `POST` | `/v1/client/{clientid}/addresses?type={addressTypeId}` | Add a typed address | `CREATE_CLIENTADDRESS` |
| `GET` | `/v1/client/{clientid}/addresses?status=…&type=…` | List addresses | `READ_CLIENTADDRESS` |
| `PUT` | `/v1/client/{clientid}/addresses` | Update | `UPDATE_CLIENTADDRESS` |

Note there is no `DELETE` here — addresses are toggled inactive via the
`is_active` flag on `ClientAddress` rather than removed.

### Add an address

```http
POST /v1/client/{clientid}/addresses?type=15
Content-Type: application/json
```

```json
{
  "street": "Av. Insurgentes 100",
  "addressLine1": "Col. Roma",
  "city": "Mexico City",
  "countyDistrict": "Cuauhtémoc",
  "stateProvinceId": 808,
  "countryId": 2021,
  "postalCode": "06700",
  "latitude": 19.4326,
  "longitude": -99.1332,
  "isActive": true,
  "createdBy": "loanofficer1",
  "createdOn": "27 January 2025",
  "dateFormat": "dd MMMM yyyy",
  "locale": "en"
}
```

Flow:

1. `ClientAddressApiResource` builds a `CommandWrapper` for
   `addClientAddress`.
2. `AddClientAddressCommandHandler` calls
   `AddressWritePlatformServiceImpl.addClientAddress(clientId, addressTypeId, command)`.
3. The service:
   - Resolves the `addressType` CodeValue.
   - Resolves `stateProvince` and `country` CodeValues.
   - Persists an `Address`.
   - Persists a `ClientAddress(client, address, addressType, isActive)`.
   - If `isActive = true`, flips any existing active `ClientAddress`
     with the same `addressType` to inactive.

### Update an address

`PUT /v1/client/{clientid}/addresses` takes the addressId in the body
(not the URL):

```json
{
  "addressId": 42,
  "addressTypeId": 15,
  "street": "Av. Insurgentes 200",
  "isActive": true,
  "updatedBy": "supervisor3",
  "updatedOn": "27 January 2025",
  "dateFormat": "dd MMMM yyyy",
  "locale": "en"
}
```

The change-detection is field-by-field; only mutated columns are written.

### List

`GET /v1/client/{clientid}/addresses` accepts:

- `status=active|inactive|all` — filter by `is_active`.
- `type={codeValueId}` — restrict to a specific `addressType`.

The response is a flat array of `AddressData`.

## EntityFieldConfigurationApiResource

```java
@Path("/v1/fieldconfiguration/{entity}")
public class EntityFieldConfigurationApiResource {
    @GET
    public List<FieldConfigurationData> getAddresses(
            @PathParam("entity") @Parameter(description = "entity") final String entityname) {
        ...
    }
}
```

So:

```
GET /v1/fieldconfiguration/ADDRESS
```

returns the enabled / disabled list of all address fields:

```json
[
  { "fieldConfigurationId": 1, "entity": "ADDRESS", "table": "m_address",
    "field": "street", "is_enabled": true },
  { "fieldConfigurationId": 2, "entity": "ADDRESS", "table": "m_address",
    "field": "addressLine1", "is_enabled": true },
  { "fieldConfigurationId": 3, "entity": "ADDRESS", "table": "m_address",
    "field": "addressLine2", "is_enabled": false }
]
```

UIs typically fetch this once at startup and cache the per-field
visibility decisions.

## Read path

`ClientAddressReadPlatformServiceImpl` and
`AddressReadPlatformServiceImpl` together build `AddressData`:

```sql
SELECT a.id, a.street, a.address_line_1, a.address_line_2,
       a.city, a.county_district, a.postal_code, a.latitude, a.longitude,
       sp.code_value AS stateName, c.code_value AS countryName,
       at.code_value AS addressType, ca.is_active
FROM m_address a
LEFT JOIN m_client_address ca ON ca.address_id = a.id
LEFT JOIN m_code_value sp ON sp.id = a.state_province_id
LEFT JOIN m_code_value c  ON c.id  = a.country_id
LEFT JOIN m_code_value at ON at.id = ca.address_type_id
WHERE ca.client_id = ?
```

(Approximate — the actual SQL lives in
`AddressReadPlatformServiceImpl#retrieveAddress`.)

## Validation and field-configuration toggles

`AddressWritePlatformServiceImpl` consults `m_field_configuration` to
decide which fields are mandatory at validation time:

```java
if (fieldConfig.isMandatory("street")) {
    baseDataValidator.reset().parameter("street").value(street).notBlank();
}
```

Disabling a field via `FieldConfiguration` therefore both hides it from
the UI and stops the validator from rejecting empty values.

## CodeValue lookups

| Code | Purpose |
| --- | --- |
| `ADDRESS_TYPE` | The address-type column (Permanent, Current, …) |
| `COUNTRY` | `Address.country` |
| `STATE` | `Address.stateProvince` |

The template endpoint pre-fetches all three for UI consumption.

## Common exceptions

| Exception | When |
| --- | --- |
| `AddressNotFoundException` | `addressId` does not resolve |
| `CodeValueNotFoundException` | `addressTypeId`, `stateProvinceId` or `countryId` invalid |
| `PlatformDataIntegrityException` | DB constraint violation (e.g. stale `addressId` on update) |
