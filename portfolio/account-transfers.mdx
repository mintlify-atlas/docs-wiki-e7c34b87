# Account Transfers

The account-transfer subsystem moves money between savings and loan
accounts within Apache Fineract. A transfer always produces a pair of
side-effects (a `SavingsAccountTransaction` withdrawal + a
`SavingsAccountTransaction` deposit, or a `SavingsAccountTransaction`
withdrawal + a `LoanTransaction` repayment), bundled into one
`AccountTransferDetails` row with a typed `AccountTransferType`.

| --- | --- | --- |

| `AccountAssociations`, `AccountAssociationType` | `fineract-provider` | `org.apache.fineract.portfolio.account.domain` |

## PortfolioAccountType

The integer-coded discriminator that drives the rest of the subsystem:

```java
public enum PortfolioAccountType {
    INVALID(0,  "accountType.invalid"),
    LOAN(1,     "accountType.loan"),
    SAVINGS(2,  "accountType.savings");
}
```

The `from_account_type` / `to_account_type` columns on
`AccountTransferDetails` carry these codes.

## AccountTransferType

```java
public enum AccountTransferType {
    INVALID(0,           "accountTransferType.invalid"),
    ACCOUNT_TRANSFER(1,  "accountTransferType.account.transfer"),
    LOAN_REPAYMENT(2,    "accountTransferType.loan.repayment"),
    CHARGE_PAYMENT(3,    "accountTransferType.charge.payment"),
    INTEREST_TRANSFER(4, "accountTransferType.interest.transfer"),
    LOAN_DOWN_PAYMENT(5, "accountTransferType.loan.downpayment");
}
```

| Code | Direction | Use |
| --- | --- | --- |
| 1 | Savings â†’ Savings | Generic transfer between savings accounts |
| 2 | Savings â†’ Loan | Repayment of a loan from a savings account |
| 3 | Savings â†’ Loan (charge) | Pay a loan-charge from savings |
| 4 | Savings â†’ Savings (interest) | Interest posting transfer |
| 5 | Savings â†’ Loan (down payment) | Down-payment leg of an originated loan |

There is no Loan â†’ Savings type â€” loans never originate a transfer.
Disbursement to savings is handled by the loan disbursement flow, which
internally uses `AccountTransferType = ACCOUNT_TRANSFER` from a special
"loan-portfolio" virtual savings.

## AccountTransferRecurrenceType

```java
public enum AccountTransferRecurrenceType {
    INVALID(0,    "accountTransferRecurrenceType.invalid"),
    PERIODIC(1,   "accountTransferRecurrenceType.periodic"),
    AS_PER_DUES(2,"accountTransferRecurrenceType.as.per.dues");
}
```

Used by standing instructions: `PERIODIC` runs on a fixed recurrence,
`AS_PER_DUES` runs whenever the linked loan has an instalment due.

## AccountTransferDetails

`m_account_transfer_details` is the parent table â€” one row per logical
transfer endpoint pair:

```java
@Entity
@Table(name = "m_account_transfer_details")
public class AccountTransferDetails extends AbstractPersistableCustom<Long> {

    @ManyToOne @JoinColumn(name = "from_office_id",  nullable = false)
    private Office fromOffice;
    @ManyToOne @JoinColumn(name = "from_client_id",  nullable = false)
    private Client fromClient;
    @ManyToOne @JoinColumn(name = "from_savings_account_id")
    private SavingsAccount fromSavingsAccount;

    @ManyToOne @JoinColumn(name = "to_office_id",  nullable = false)
    private Office toOffice;
    @ManyToOne @JoinColumn(name = "to_client_id",  nullable = false)
    private Client toClient;
    ...
}
```

The columns continue with `from_loan_account_id`,
`to_savings_account_id`, `to_loan_account_id`, `transfer_type`,
`is_reversed` and `account_transfer_standing_instructions_id`.

## AccountTransferTransaction

`m_account_transfer_transaction` is the journal of each *execution* of
a transfer detail (a standing instruction may produce many of these
across time):

```java
@Entity
@Table(name = "m_account_transfer_transaction")
@Getter
public class AccountTransferTransaction extends AbstractPersistableCustom<Long> {

    @ManyToOne @JoinColumn(name = "account_transfer_details_id")
    private AccountTransferDetails accountTransferDetails;

    @ManyToOne @JoinColumn(name = "from_savings_transaction_id")
    private SavingsAccountTransaction fromSavingsTransaction;
    @ManyToOne @JoinColumn(name = "to_savings_transaction_id")
    private SavingsAccountTransaction toSavingsTransaction;

    @ManyToOne @JoinColumn(name = "to_loan_transaction_id")
    private LoanTransaction toLoanTransaction;
    @ManyToOne @JoinColumn(name = "from_loan_transaction_id")
    private LoanTransaction fromLoanTransaction;

    @Embedded private Money amount;
    @Column(name = "transaction_date") private LocalDate transactionDate;
}
```

Each row links the two underlying transactions, so reversing a transfer
becomes "reverse both child transactions then mark this row reversed".

## AccountAssociations

`m_portfolio_account_associations` is a separate table that records
*permanent* associations between accounts independently of any
individual transfer â€” e.g. the savings account that should automatically
collect a loan's instalments. `AccountAssociationType` enumerates the
possible kinds (`LINKED_ACCOUNT_ASSOCIATION`,
`SAVINGS_LINKED_ASSOCIATION`, etc.).

## AccountTransfersApiResource

Mounted at `/v1/accounttransfers`:

| Method | Path | Operation | Permission |
| --- | --- | --- | --- |
| `GET` | `/v1/accounttransfers/template?fromAccountType=&fromOfficeId=&fromClientId=&fromAccountId=&toOfficeId=&toClientId=&toAccountId=&toAccountType=` | Template (dropdown population for the next transfer) | `READ_ACCOUNTTRANSFER` |
| `POST` | `/v1/accounttransfers` | Create transfer | `CREATE_ACCOUNTTRANSFER` |
| `GET` | `/v1/accounttransfers?externalId=&offset=&limit=&orderBy=&sortOrder=&accountDetailId=` | Paged list | `READ_ACCOUNTTRANSFER` |
| `GET` | `/v1/accounttransfers/{transferId}` | Single | `READ_ACCOUNTTRANSFER` |
| `GET` | `/v1/accounttransfers/templateRefundByTransfer?â€¦` | Template for refund-by-transfer | `READ_ACCOUNTTRANSFER` |
| `POST` | `/v1/accounttransfers/refundByTransfer` | Refund an active loan by transferring out to savings | `CREATE_ACCOUNTTRANSFER` |

There is no `PUT` and no `DELETE` â€” once posted, a transfer is
immutable. To undo a transfer, you must reverse the underlying
`SavingsAccountTransaction` / `LoanTransaction` (which automatically
sets the `is_reversed` flag on `AccountTransferTransaction`).

### Create request

```http
POST /v1/accounttransfers
Content-Type: application/json
```

```json
{
  "fromOfficeId": 1,
  "fromClientId": 109,
  "fromAccountType": 2,
  "fromAccountId": 77,

  "toOfficeId": 1,
  "toClientId": 109,
  "toAccountType": 1,
  "toAccountId": 445,

  "transferAmount": 250.00,
  "transferDate": "27 January 2025",
  "transferDescription": "Manual repayment from savings",
  "dateFormat": "dd MMMM yyyy",
  "locale": "en"
}
```

Behaviour (in `AccountTransfersWritePlatformServiceImpl.create`):

1. Resolve the source and target accounts via the right
   `PortfolioAccountReadPlatformService` (savings or loan).
2. Validate currencies match.
3. Open a transaction; let `AccountTransferAssembler` create:
   - A `SavingsAccountTransaction` withdrawal on the source.
   - Either a `SavingsAccountTransaction` deposit (transfer) or a
     `LoanTransaction` repayment / charge-payment (loan repayment).
4. Persist an `AccountTransferDetails(â€¦, transferType=â€¦)` row.
5. Persist an `AccountTransferTransaction` linking both child
   transactions.
6. Return the new transfer id.

### Template

`GET /v1/accounttransfers/template?fromAccountType=2&fromOfficeId=1&fromClientId=109`
returns the lists needed for the next step â€” eligible savings/loan
accounts, offices, clients â€” without doing any state changes. The
`AccountTransSearchParam` `@BeanParam` carries all of `fromOfficeId`,
`fromClientId`, `fromAccountId`, `fromAccountType`, `toOfficeId`,
`toClientId`, `toAccountId`, `toAccountType`.

### List

`GET /v1/accounttransfers` returns a Spring `Page<AccountTransferData>`.
Supported query params: `externalId`, `offset`, `limit`, `orderBy`,
`sortOrder`, `accountDetailId`. `accountDetailId` filters the list to
transactions on one `AccountTransferDetails` row â€” useful for showing
"transfers under standing instruction X".

### Refund by transfer

```http
POST /v1/accounttransfers/refundByTransfer
```

Same body shape as `create`, but the platform validates that the source
is a **loan with overpayment** and the destination is a **savings
account**. Used to return excess money paid against a loan to the
client's savings.

## Loan â†’ Savings disbursement

When a loan disburses **to savings** (configured on the loan product),
the loan write-service calls
`LoanAccountTransfersServiceImpl.transferFunds` which builds an
`AccountTransferDTO`:

```java
new AccountTransferDTO(
    transactionDate, transactionAmount,
    /*fromAccountType*/ LOAN,  /*toAccountType*/ SAVINGS,
    fromAccountId, toAccountId,
    "Loan disbursement", null, null, null, null,
    AccountTransferType.ACCOUNT_TRANSFER, null, null,
    /*recurrenceType*/ null, null, null,
    ExternalId.empty(), null, null,
    fromSavingsAccount, /*isRegularTransaction*/ true,
    /*isExceptionForBalanceCheck*/ false);
```

This is the same DTO used by the standing-instruction tasklet â€” see
[Standing Instructions](/portfolio/standing-instructions).

## Read service

`AccountTransfersReadPlatformServiceImpl` exposes:

- `retrieveTemplate(...)` â€” pre-create lookups.
- `retrieveAll(SearchParameters, accountDetailId)` â€” paged history.
- `retrieveOne(transferId)` â€” single transfer detail.
- `retrieveRefundByTransferTemplate(...)` â€” refund pre-checks.

## Currency

Both source and target must share the same `currencyCode`. The
write-service throws a typed exception otherwise; cross-currency
transfers are not supported.

## Common exceptions

| Exception | When |
| --- | --- |
| `AccountTransferNotFoundException` | Bad `transferId` |
| `DifferentCurrenciesException` | Source / target currency mismatch |
| `InsufficientAccountBalanceException` | Source savings has insufficient balance |
| `InvalidAccountTypeException` | `fromAccountType` / `toAccountType` is `INVALID` |
| `PlatformDataIntegrityException` | DB-level constraint violation |

## Events emitted

- `AccountTransferCreatedBusinessEvent` (parent of the typed variants)
- `LoanRepaymentBusinessEvent` (when the loan side fires)
- `SavingsDepositBusinessEvent`, `SavingsWithdrawalBusinessEvent` (on
  the savings side)
