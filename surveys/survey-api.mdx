Fineract exposes two parallel survey REST surfaces. This page documents both
in depth: the typed `org.apache.fineract.spm.api.SpmApiResource` at
`/v1/surveys` backed by JPA entities (`Survey`, `Component`, `Question`,
`Response`) and the datatable-based `org.apache.fineract.infrastructure.survey.api.SurveyApiResource`
at `/v1/survey` that fulfils registered PPI datatables. Both are part of the
`fineract-provider` module and are routed through Jersey.

## SpmApiResource — typed survey CRUD

Source: `fineract-provider/.../spm/api/SpmApiResource.java`. The class is
annotated `@Path("/v1/surveys")` and declared as a Spring
`@Component`. Constructor injection provides
`PlatformSecurityContext securityContext` and `SpmService spmService`.

```java
@Path("/v1/surveys")
@Component
@Tag(name = "Spm-Surveys", description = "")
@RequiredArgsConstructor
public class SpmApiResource {

    private final PlatformSecurityContext securityContext;
    private final SpmService spmService;
```

### Endpoint matrix

| Method | Path | Operation | Returns |
| --- | --- | --- | --- |
| `GET` | `/v1/surveys` | `fetchAllSurveys?isActive=...` | `List<SurveyData>` |
| `GET` | `/v1/surveys/{id}` | `findSurvey` | `SurveyData` |
| `POST` | `/v1/surveys` | `createSurvey` | `{"resourceId": Long}` JSON |
| `PUT` | `/v1/surveys/{id}` | `editSurvey` | `{"resourceId": Long}` JSON |
| `POST` | `/v1/surveys/{id}?command=activate` | `activateOrDeactivateSurvey` | `void` |
| `POST` | `/v1/surveys/{id}?command=deactivate` | `activateOrDeactivateSurvey` | `void` |

Activation routes the query parameter through:

```java
if (command != null && command.equalsIgnoreCase("activate")) {
    this.spmService.activateSurvey(id);
} else if (command != null && command.equalsIgnoreCase("deactivate")) {
    this.spmService.deactivateSurvey(id);
} else {
    throw new UnrecognizedQueryParamException("command", command);
}
```

### List filter

When `isActive=true`, `SpmService.fetchValidSurveys()` is used (typically
filters by `validFrom <= today <= validTo`). Otherwise `fetchAllSurveys()`
returns everything. Each `Survey` is converted to a `SurveyData` by
`SurveyMapper.map(survey)`.

## Survey entity

Source: `spm/domain/Survey.java`. Maps to `m_surveys`.

```java
@Entity
@Table(name = "m_surveys")
public class Survey extends AbstractPersistableCustom<Long> {

    @OneToMany(mappedBy = "survey", fetch = FetchType.EAGER,
               cascade = CascadeType.ALL, orphanRemoval = true)
    @OrderBy("sequenceNo")
    private List<Component> components;

    @OneToMany(mappedBy = "survey", fetch = FetchType.EAGER,
               cascade = CascadeType.ALL, orphanRemoval = true)
    @OrderBy("sequenceNo")
    private List<Question> questions;

    @Column(name = "a_key", length = 32)        private String key;
    @Column(name = "a_name", length = 255)      private String name;
    @Column(name = "description", length = 4096) private String description;
    @Column(name = "country_code", length = 2)  private String countryCode;
    @Column(name = "valid_from")                private LocalDate validFrom;
    @Column(name = "valid_to")                  private LocalDate validTo;
}
```

Notes:

- Both child collections are eagerly fetched and orphan-removed: a survey owns
  its components and questions outright.
- `setQuestions` clears the existing list before adding to preserve JPA managed
  state on update.
- The 2-character `country_code` allows surveys to be scoped per market.

### Survey table columns

| Column | Type | Notes |
| --- | --- | --- |
| `id` | `BIGINT` | PK |
| `a_key` | `VARCHAR(32)` | Unique business key, used by mappers |
| `a_name` | `VARCHAR(255)` | Display name |
| `description` | `VARCHAR(4096)` | Long description |
| `country_code` | `CHAR(2)` | ISO country code |
| `valid_from` | `DATE` | Start of validity window |
| `valid_to` | `DATE` | End of validity window |

## Question entity

Source: `spm/domain/Question.java`. Maps to `m_survey_questions`.

```java
@Entity
@Table(name = "m_survey_questions")
public class Question extends AbstractPersistableCustom<Long> {

    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "survey_id")
    private Survey survey;

    @OneToMany(mappedBy = "question", fetch = FetchType.EAGER, cascade = CascadeType.ALL)
    @OrderBy("sequenceNo")
    private List<Response> responses;

    @Column(name = "component_key", length = 32) private String componentKey;
    @Column(name = "a_key", length = 32)         private String key;
    @Column(name = "a_text", length = 255)       private String text;
    @Column(name = "description", length = 4096) private String description;
    @Column(name = "sequence_no", precision = 4) private Integer sequenceNo;
}
```

A `Question` may be grouped under a `Component` (denormalised by
`componentKey` — there is no FK). Responses are ordered by `sequenceNo` and
cascaded.

## Response entity

Source: `spm/domain/Response.java`. Maps to `m_survey_responses`.

```java
@Entity
@Table(name = "m_survey_responses")
public class Response extends AbstractPersistableCustom<Long> {

    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "question_id")
    private Question question;

    @Column(name = "a_text", length = 255)       private String text;
    @Column(name = "a_value", precision = 4)     private Integer value;
    @Column(name = "sequence_no", precision = 4) private Integer sequenceNo;
}
```

Each response carries an integer `value`. Scoring sums these up: see
[SPM Scorecards](/surveys/spm-scorecards). `sequenceNo` controls display
order.

## Component entity

Source: `spm/domain/Component.java`. Maps to `m_survey_components`.

```java
@Entity
@Table(name = "m_survey_components")
public class Component extends AbstractPersistableCustom<Long> {

    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "survey_id")
    private Survey survey;

    @Column(name = "a_key", length = 32)         private String key;
    @Column(name = "a_text", length = 255)       private String text;
    @Column(name = "description", length = 4096) private String description;
    @Column(name = "sequence_no", precision = 4) private Integer sequenceNo;
}
```

A `Component` groups related questions under a `key`. Questions are not
mapped via a foreign key; they reference `Component` via `componentKey`
string equality.

## SurveyData DTO

Source: `spm/data/SurveyData.java`.

| Field | Type | Description |
| --- | --- | --- |
| `id` | `Long` | Server-assigned id, present in responses |
| `componentDatas` | `List<ComponentData>` | Component groups |
| `questionDatas` | `List<QuestionData>` | Questions with nested responses |
| `key` | `String` | Business key |
| `name` | `String` | Display name |
| `description` | `String` | Description |
| `countryCode` | `String` | ISO country code |
| `validFrom` / `validTo` | `LocalDate` | Validity window |

The `createSurvey` operation declares the mandatory fields explicitly in its
Swagger description:

```
Mandatory Fields
countryCode, key, name, questions, responses, sequenceNo, text, description
```

## SurveyValidator

`spm/domain/SurveyValidator.java` is the read-time validator used by
`SpmService` before persisting a survey, enforcing presence of these
mandatory fields and the integrity of the validity window.

## SurveyApiResource — PPI datatable surface

Source: `fineract-provider/.../infrastructure/survey/api/SurveyApiResource.java`.
Annotated `@Path("/v1/survey")`. Unlike the `spm.*` surface, this one calls
the `CommandWrapperBuilder` for writes so they run through
[the command pipeline](/command/overview).

```java
@Path("/v1/survey")
@Component
@Tag(name = "Survey", description = "")
@RequiredArgsConstructor
public class SurveyApiResource {
    private final DefaultToApiJsonSerializer<SurveyData> toApiJsonSerializer;
    private final DefaultToApiJsonSerializer<ClientScoresOverview> toApiJsonClientScoreOverviewSerializer;
    private final PlatformSecurityContext context;
    private final ReadSurveyService readSurveyService;
    private final PortfolioCommandSourceWritePlatformService commandsSourceWritePlatformService;
    private final GenericDataService genericDataService;
```

Note the `SurveyData` type here is the legacy
`org.apache.fineract.infrastructure.survey.data.SurveyData` — separate from
the `spm.data.SurveyData` shown above.

### Endpoint matrix

| Method | Path | Permission | Notes |
| --- | --- | --- | --- |
| `GET` | `/v1/survey` | `READ_Survey` | `retrieveSurveys` → `List<SurveyDataTableData>` |
| `GET` | `/v1/survey/{surveyName}` | `READ_Survey` | `retrieveSurvey` → `SurveyDataTableData` |
| `POST` | `/v1/survey/{surveyName}/{apptableId}` | via command | Insert PPI entry through `CommandWrapperBuilder.fullFilSurvey(...)` |
| `GET` | `/v1/survey/{surveyName}/{clientId}` | `READ_Survey` | Client scores overview |
| `GET` | `/v1/survey/{surveyName}/{clientId}/{entryId}` | `READ_Survey` | Single fulfilled entry |
| `PUT` | `/v1/survey/register/{surveyName}/{apptable}` | via command | `CommandWrapperBuilder.registerSurvey(...)` |
| `DELETE` | `/v1/survey/{surveyName}/{clientId}/{fulfilledId}` | via command | `CommandWrapperBuilder.deleteDatatableEntry(...)` |

### Permission checks

The legacy resource enforces the `Survey` permission via:

```java
this.context.authenticatedUser()
        .validateHasReadPermission(SurveyApiConstants.SURVEY_RESOURCE_NAME);
```

Where `SurveyApiConstants.SURVEY_RESOURCE_NAME = "Survey"`. See the
[user administration domain](/core/useradministration-domain) for permission
codes.

### Fulfill a survey

```java
final CommandWrapper commandRequest = new CommandWrapperBuilder() //
        .fullFilSurvey(datatable, apptableId) //
        .withJson(apiRequestBodyAsJson) //
        .build();

final CommandProcessingResult result =
        this.commandsSourceWritePlatformService.logCommandSource(commandRequest);

return this.toApiJsonSerializer.serialize(result);
```

`datatable` is the survey/PPI table name (e.g. `ppi_kenya_2010_a`),
`apptableId` is the bound entity id (typically a client id), and the JSON body
contains one column-per-question answer.

### Register a survey

```java
final CommandWrapper commandRequest = new CommandWrapperBuilder()
        .registerSurvey(datatable, apptable)
        .withJson(apiRequestBodyAsJson)
        .build();
```

The `apptable` parameter is the Fineract application table (commonly
`m_client`). The serializer body specifies category, role-based permissions,
and column definitions. See `DataTableApiConstant.CATEGORY_PPI` for the
discriminator value used in `x_registered_table.category`.

### Retrieve all surveys (SQL)

`ReadSurveyServiceImpl.retrieveAllSurveys` runs:

```java
"select application_table_name, cf.enabled, registered_table_name, entity_subtype"
    + " from x_registered_table "
    + " left join c_configuration cf on x_registered_table.registered_table_name = cf.name "
    + " where exists (select 'f' from m_appuser_role ur "
    + "  join m_role r on r.id = ur.role_id"
    + "  left join m_role_permission rp on rp.role_id = r.id"
    + "  left join m_permission p on p.id = rp.permission_id"
    + "  where ur.appuser_id = " + userId
    + "  and (p.code in ('ALL_FUNCTIONS', 'ALL_FUNCTIONS_READ') "
    + "    or p.code = concat('READ_', registered_table_name))) "
    + " and x_registered_table.category = " + DataTableApiConstant.CATEGORY_PPI;
```

Only PPI-category datatables visible to the current user are returned.

## SurveyDataTableData DTO

Source: `infrastructure/survey/data/SurveyDataTableData.java`. Combines a
generic `DatatableData` with an `enabled` flag from `c_configuration`. It is
the response shape of `GET /v1/survey` and `GET /v1/survey/{surveyName}`.

## ClientScoresOverview DTO

Returned by `GET /v1/survey/{surveyName}/{clientId}`. Captures one row per
survey fulfilment, joined with the matching poverty-line/likelihood:

| Field | Source column / value |
| --- | --- |
| `id` | `tz.id` (fulfilment row) |
| `surveyName` | survey datatable name |
| `likelihoodName` | `lkh.name` |
| `likelihoodCode` | `lkh.code` |
| `score` | `tz.score` |
| `povertyLine` | `pvl.poverty_line` |
| `date` | `tz.date` |

The aggregating SQL is shown in
[Overview](/surveys/overview#ppi-datatable-model).

## Mapping between the two surfaces

There is no automatic mapping between `spm.*` surveys and PPI datatables.
Operators choose one or the other per use case. Cross-references include:

- Both surfaces consume `Client` from `portfolio.client` — see
  [portfolio shared domain](/core/portfolio-shared-domain).
- Both depend on `AppUser` and on the security infrastructure — see
  [user administration domain](/core/useradministration-domain).
- The PPI surface uses `CommandWrapperBuilder` and goes through the
  [commands framework](/command/overview); the typed surface bypasses it.
- See [`/api/surveys`](/api/surveys) for the published API page.
