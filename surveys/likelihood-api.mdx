The Likelihood API powers Fineract's **Progress out of Poverty Index (PPI)**
configuration. A "likelihood" is a calibration row that pairs a PPI survey
(by `ppi_name`) with a poverty-likelihood model (by `likelihood_id`) and
records whether that model is currently `enabled` (200) or `disabled` (100).
The REST surface is implemented in
`org.apache.fineract.infrastructure.survey.api.LikelihoodApiResource` under
the path `/v1/likelihood`.

## Resource summary

Source: `infrastructure/survey/api/LikelihoodApiResource.java`.

```java
@Path("/v1/likelihood")
@Component
@Tag(name = "Likelihood", description = "")
@RequiredArgsConstructor
public class LikelihoodApiResource {

    private final DefaultToApiJsonSerializer<LikelihoodData> toApiJsonSerializer;
    private final PlatformSecurityContext context;
    private final PortfolioCommandSourceWritePlatformService commandsSourceWritePlatformService;
    private final ReadLikelihoodService readService;
```

Three endpoints are exposed, all relative to a `{ppiName}` path segment:

| Method | Path | Operation |
| --- | --- | --- |
| `GET` | `/v1/likelihood/{ppiName}` | `retrieveAll(ppiName)` → `List<LikelihoodData>` |
| `GET` | `/v1/likelihood/{ppiName}/{likelihoodId}` | `retrieve(likelihoodId)` → `LikelihoodData` |
| `PUT` | `/v1/likelihood/{ppiName}/{likelihoodId}` | `update(likelihoodId, json)` → `CommandProcessingResult` |

### Authorisation

All three operations call:

```java
this.context.authenticatedUser()
        .validateHasReadPermission(PovertyLineApiConstants.POVERTY_LINE_RESOURCE_NAME);
```

Where `POVERTY_LINE_RESOURCE_NAME = "PovertyLine"`. So both read and write on
the Likelihood resource require the `READ_PovertyLine` permission (or
`ALL_FUNCTIONS` / `ALL_FUNCTIONS_READ`). See
[user administration domain](/core/useradministration-domain) for permission
resolution.

### Write path

The `PUT` operation is the only mutator. It packages the request as a
`CommandWrapper` and dispatches it through the standard command pipeline:

```java
final CommandWrapper commandRequest = new CommandWrapperBuilder() //
        .updateLikelihood(likelihoodId) //
        .withJson(apiRequestBodyAsJson) //
        .build();

final CommandProcessingResult result =
        this.commandsSourceWritePlatformService.logCommandSource(commandRequest);
```

The command then hits `UpdateLikelihoodCommandHandler` (see below) which
delegates to `WriteLikelihoodServiceImpl.update`. See
[commands framework](/command/overview) for how `CommandWrapperBuilder`,
`logCommandSource`, and `@CommandType` interact.

## Likelihood entity

Source: `infrastructure/survey/domain/Likelihood.java`. Maps to
`ppi_likelihoods_ppi`.

```java
@Entity
@Table(name = "ppi_likelihoods_ppi")
@Getter @Setter @NoArgsConstructor
@Accessors(chain = true)
public final class Likelihood extends AbstractPersistableCustom<Long> {

    @Column(name = "ppi_name", nullable = false)
    private String ppiName;

    @Column(name = "likelihood_id", nullable = false)
    private Long likelihoodId;

    @Column(name = "enabled", nullable = false)
    private Long enabled;
```

| Column | Type | Notes |
| --- | --- | --- |
| `id` | `BIGINT` | PK, surrogate |
| `ppi_name` | `VARCHAR` | Datatable name of the PPI survey, e.g. `ppi_kenya_2010_a` |
| `likelihood_id` | `BIGINT` | FK into `ppi_likelihoods` (a row from the global catalogue) |
| `enabled` | `BIGINT` | `200` enabled, `100` disabled |

A single PPI typically has multiple `Likelihood` rows — one per calibration
year / methodology — but only one should be enabled at any time. The
`WriteLikelihoodServiceImpl.update` method enforces this by disabling all
peers when one is activated (see below).

### Update logic

```java
public Map<String, Object> update(final JsonCommand command) {
    final Map<String, Object> actualChanges = new LinkedHashMap<>(7);

    final boolean enabled = command
            .booleanPrimitiveValueOfParameterNamed(LikelihoodApiConstants.ACTIVE);

    Long changeToValue = null;
    if (enabled) {
        changeToValue = LikelihoodStatus.ENABLED;   // 200
    } else {
        changeToValue = LikelihoodStatus.DISABLED;  // 100
    }

    if (!changeToValue.equals(this.enabled)) {
        actualChanges.put(LikelihoodApiConstants.ACTIVE, enabled);
        this.enabled = changeToValue;
    }
    return actualChanges;
}

public boolean isActivateCommand(final JsonCommand command) {
    return command.booleanPrimitiveValueOfParameterNamed(LikelihoodApiConstants.ACTIVE);
}
```

## ReadLikelihoodServiceImpl

Two SQL helpers back the GET endpoints:

```java
// retrieveAll(ppiName)
"SELECT lkp.id, lkh.code , lkh.name, lkp.enabled "
    + " FROM ppi_poverty_line pl "
    + " JOIN ppi_likelihoods_ppi lkp on lkp.id = pl.likelihood_ppi_id "
    + " JOIN ppi_likelihoods lkh on lkp.likelihood_id = lkh.id "
    + " WHERE lkp.ppi_name = ? "
    + " GROUP BY pl.likelihood_ppi_id ";

// retrieve(likelihoodId)
"SELECT lkp.id, lkh.code , lkh.name, lkp.enabled "
    + " FROM ppi_likelihoods lkh "
    + " JOIN ppi_likelihoods_ppi lkp on lkp.likelihood_id = lkh.id "
    + " WHERE lkp.id = ? ";
```

Note that `retrieveAll` joins through `ppi_poverty_line` to ensure only
likelihoods that actually have poverty-line rows are returned, while
`retrieve` resolves a single row by primary key.

## WriteLikelihoodServiceImpl

Source: `infrastructure/survey/service/WriteLikelihoodServiceImpl.java`. The
core update logic:

```java
@Override
public CommandProcessingResult update(Long likelihoodId, JsonCommand command) {

    this.context.authenticatedUser();
    try {
        this.likelihoodDataValidator.validateForUpdate(command);

        final Likelihood likelihood = this.repository.findById(likelihoodId).orElse(null);

        if (!likelihood.update(command).isEmpty()) {
            this.repository.save(likelihood);

            if (likelihood.isActivateCommand(command)) {
                List<Likelihood> likelihoods = this.repository
                        .findByPpiNameAndLikeliHoodId(likelihood.getPpiName(), likelihood.getId());

                for (Likelihood aLikelihood : likelihoods) {
                    aLikelihood.setEnabled(LikelihoodStatus.DISABLED);
                }
                this.repository.saveAll(likelihoods);
            }
        }

        return new CommandProcessingResultBuilder()
                .withCommandId(command.commandId())
                .withEntityId(likelihood.getId())
                .build();
    } catch (final JpaSystemException | DataIntegrityViolationException dve) {
        final Throwable throwable = dve.getMostSpecificCause();
        handleDataIntegrityIssues(throwable, dve);
        return CommandProcessingResult.empty();
    }
}
```

Critical invariant: activating one likelihood **disables all peer
likelihoods** for the same `ppi_name`. The peer lookup uses:

```java
@Query("select liklihood FROM Likelihood liklihood "
     + "WHERE liklihood.ppiName =:ppiName AND liklihood.id <>:id")
List<Likelihood> findByPpiNameAndLikeliHoodId(@Param("ppiName") String ppiName,
                                              @Param("id") Long likeliHoodId);
```

(Yes, the parameter name has a typo — `liklihood` — kept here to match the
source verbatim.)

## LikelihoodDataValidator

Source: `infrastructure/survey/data/LikelihoodDataValidator.java`.

Only the `active` parameter is allowed:

```java
private static final Set<String> UPDATE_LIKELIHOOD_DATA_PARAMETERS =
        new HashSet<>(Arrays.asList(LikelihoodApiConstants.ACTIVE));
```

`validateForUpdate`:

1. Throws `InvalidJsonException` on blank JSON.
2. Calls `fromApiJsonHelper.checkForUnsupportedParameters` to reject unknown
   fields.
3. Validates the `active` value with
   `DataValidatorBuilder.validateForBooleanValue()`.
4. Throws `PlatformApiDataValidationException` if any parameter errors
   accumulated.

## UpdateLikelihoodCommandHandler

Source: `infrastructure/survey/handler/UpdateLikelihoodCommandHandler.java`.

```java
@Service
@CommandType(entity = "LIKELIHOOD", action = "UPDATE")
public class UpdateLikelihoodCommandHandler implements NewCommandSourceHandler {

    private final WriteLikelihoodService writePlatformService;

    @Transactional
    @Override
    public CommandProcessingResult processCommand(final JsonCommand command) {
        return this.writePlatformService.update(command.entityId(), command);
    }
}
```

The `@CommandType(entity = "LIKELIHOOD", action = "UPDATE")` annotation is
the registration key for the command handler registry — see
[Commands framework](/command/overview) and the
`updateLikelihood(Long)` builder method on `CommandWrapperBuilder` whose
`actionName="UPDATE"` and `entityName="LIKELIHOOD"`.

## Sample request

Activate likelihood id 17 for PPI datatable `ppi_kenya_2010_a`:

```http
PUT /api/v1/likelihood/ppi_kenya_2010_a/17
Content-Type: application/json
Fineract-Platform-TenantId: default

{ "active": true }
```

Response:

```json
{
  "commandId": 9876,
  "officeId": null,
  "clientId": null,
  "loanId": null,
  "resourceId": 17,
  "changes": { "active": true }
}
```

After this call:

- `ppi_likelihoods_ppi` row 17 has `enabled = 200`.
- All other `ppi_likelihoods_ppi` rows with the same `ppi_name` have
  `enabled = 100`.
- A new `m_portfolio_command_source` row records the command for audit and
  maker-checker.

## Relationship to scoring

`Likelihood` rows are joined with `ppi_scores` and `ppi_poverty_line` to
compute the per-client poverty likelihood when a survey is fulfilled. See
[Poverty Line API](/surveys/poverty-line-api) for the
`PpiPovertyLineData`/`LikeliHoodPovertyLineData` shapes and the SQL used by
`PovertyLineServiceImpl`, and [Survey API](/surveys/survey-api) for how
`ClientScoresOverview` aggregates the result.
