`RateApiResource` is the JAX-RS resource that exposes Apache Fineract's **named rate catalogue** — simple percentage entries (name + percentage + active flag + `productApply` scope) that can be attached to loan products and to loans themselves. The class lives under the `portfolio/rate` package and was originally contributed by Bowpi GT.

> The task map asked for a `RatesApiResource`; the actual class in the codebase is `RateApiResource`. This page documents that resource at the same `/v1/rates` URL space.

## Source

- **File:** `fineract-provider/src/main/java/org/apache/fineract/portfolio/rate/api/RateApiResource.java`
- **Class path annotation:** `@Path("/v1/rates")`
- **OpenAPI tag:** `Rate`
- **Spring stereotype:** `@Component`

Constructor-injected dependencies:

- `PlatformSecurityContext context` — enforces `RATE` read permission.
- `RateReadService readPlatformService` — read store.
- `DefaultToApiJsonSerializer<RateData> toApiJsonSerializer` — request/response serializer.
- `PortfolioCommandSourceWritePlatformService commandsSourceWritePlatformService` — command bus.

Static configuration:

```java
private static final Set<String> RESPONSE_DATA_PARAMETERS = new HashSet<>(
    Arrays.asList("id", "name", "percentage", "productApply", "active"));
private static final String RESOURCE_NAME_FOR_PERMISSIONS = "RATE";
```

## Endpoints

| Method | Path | Description | Command / Handler | Permission |
| ------ | ---- | ----------- | ----------------- | ---------- |
| GET | `/v1/rates` | List every rate. | `readPlatformService.retrieveAllRates()` | Read permission `RATE` |
| GET | `/v1/rates/{rateId}` | Retrieve a single rate. | `readPlatformService.retrieveOne(rateId)` | Read permission `RATE` |
| POST | `/v1/rates` | Create a new rate. | `CommandWrapperBuilder.createRate()` → `CREATE_RATE` | `CREATE_RATE` (maker-checker capable) |
| PUT | `/v1/rates/{rateId}` | Update a rate. | `CommandWrapperBuilder.updateRate(rateId)` → `UPDATE_RATE` | `UPDATE_RATE` (maker-checker capable) |

There is no `DELETE` endpoint — rates are deactivated by toggling `active` through `PUT`.

## Request / response examples

### Create

`POST /v1/rates`

Request body — `RateRequest`:

```json
{
  "name": "Default Interest Surcharge",
  "percentage": 1.50,
  "productApply": true,
  "active": true,
  "locale": "en"
}
```

Handler:

```java
final CommandWrapper commandRequest = new CommandWrapperBuilder()
    .createRate()
    .withJson(toApiJsonSerializer.serialize(rateRequest))
    .build();
return commandsSourceWritePlatformService.logCommandSource(commandRequest);
```

Response (`CommandProcessingResult`):

```json
{
  "resourceId": 4,
  "changes": {}
}
```

### List

`GET /v1/rates`

```json
[
  {
    "id": 4,
    "name": "Default Interest Surcharge",
    "percentage": 1.50,
    "productApply": true,
    "active": true
  },
  {
    "id": 5,
    "name": "VAT 18%",
    "percentage": 18.00,
    "productApply": false,
    "active": true
  }
]
```

### Retrieve one

`GET /v1/rates/4`

```json
{
  "id": 4,
  "name": "Default Interest Surcharge",
  "percentage": 1.50,
  "productApply": true,
  "active": true
}
```

### Update

`PUT /v1/rates/4`

```json
{
  "percentage": 2.00,
  "active": true,
  "locale": "en"
}
```

```json
{
  "resourceId": 4,
  "changes": {
    "percentage": 2.00
  }
}
```

## Data carriers

- **Request:** `RateRequest` — `name`, `percentage`, `productApply`, `active`, `locale`.
- **Response:** `RateData` — id, name, percentage, productApply, active.
- **Write response:** `CommandProcessingResult` envelope.

## Permissions

All `GET` handlers call:

```java
this.context.authenticatedUser().validateHasReadPermission(RESOURCE_NAME_FOR_PERMISSIONS);
```

with `RESOURCE_NAME_FOR_PERMISSIONS = "RATE"`. Write handlers delegate to `PortfolioCommandSourceWritePlatformService`, which enforces `CREATE_RATE` / `UPDATE_RATE` and routes through maker-checker if enabled.

## How rates plug into loans

- Loan product configuration carries a `rateIds` array — on product create/update the values are joined with `m_rate` to build `LoanProductRate` rows.
- Each disbursement on a loan accrues a charge derived from the active `Rate.percentage` × principal, recorded against `LoanCharge` rows scoped to `RATE` source.
- Toggling `active=false` keeps existing accruals but stops new loans from picking up the rate.

See [/loan/loan-product-api](/loan/loan-product-api) for how loan product `rates` are wired, and [/portfolio/loan-charges](/loan/loan-charges) for downstream accrual semantics.
