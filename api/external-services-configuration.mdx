The External Services Configuration API stores per-service name/value bags that Apache Fineract uses to talk to third-party systems — currently SMTP (email delivery) and Amazon S3 (document storage), with the same model extended for SMS providers and notification gateways in some deployments. The resource class is named `ExternalServicesConfigurationApiResource`; many integrations also call it `ExternalServicesPropertiesApiResource` after the underlying `ExternalServicesProperties` entity.

## Source

| Aspect | Value |
| --- | --- |
| Resource class | `org.apache.fineract.infrastructure.configuration.api.ExternalServicesConfigurationApiResource` |
| File | `fineract-provider/src/main/java/org/apache/fineract/infrastructure/configuration/api/ExternalServicesConfigurationApiResource.java` |
| JAX-RS `@Path` | `/v1/externalservice` |
| Swagger tag | `External Services` |
| Permission code | `ExternalServiceConfigurationApiConstant.EXTERNAL_SERVICE_RESOURCE_NAME` (`EXTERNALSERVICES`) |

## Endpoints

| Method | Path | Description | Command / read handler | Permission |
| --- | --- | --- | --- | --- |
| `GET` | `/v1/externalservice/{servicename}` | Retrieve the name/value bag for `servicename` (e.g. `S3`, `SMTP`). | `ExternalServicesPropertiesReadPlatformService.retrieveOne(serviceName)` | `READ_EXTERNALSERVICES` |
| `PUT` | `/v1/externalservice/{servicename}` | Update properties for the given service. | `CommandWrapperBuilder.updateExternalServiceProperties(serviceName)` → `UPDATE_EXTERNALSERVICES` | `UPDATE_EXTERNALSERVICES` |

## Supported services

| Service name | Keys used by the platform |
| --- | --- |
| `S3` | `s3_access_key`, `s3_secret_key`, `s3_bucket_name` |
| `SMTP` | `username`, `password`, `host`, `port`, `useTLS` |
| `SMS` | Provider-specific (configured by integrations) |
| `NOTIFICATION` | Push-notification provider keys |

The service name parameter is case-sensitive and is forwarded verbatim into the command handler; mis-spelling it raises a `PlatformResourceNotFoundException` from the read service.

## Request body — update

The body is a JSON object whose keys match the property names returned by the GET endpoint:

```json
{
  "username": "noreply@example.org",
  "password": "*****",
  "host": "smtp.example.org",
  "port": "587",
  "useTLS": "true"
}
```

`password` is masked when returned from GET but accepted in plaintext when updating.

## Response — get

```json
[
  { "name": "username", "value": "noreply@example.org" },
  { "name": "password", "value": "*****" },
  { "name": "host", "value": "smtp.example.org" },
  { "name": "port", "value": "587" },
  { "name": "useTLS", "value": "true" }
]
```

## Response — update

```json
{
  "resourceIdentifier": "SMTP",
  "changes": { "host": "smtp.example.org" }
}
```
