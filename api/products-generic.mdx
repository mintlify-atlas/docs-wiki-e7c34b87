`ProductsApiResource` is the type-dispatching JAX-RS resource in Apache Fineract that resolves a *generic* product URL like `/v1/products/share/1` to the typed service for that product family. Today it is wired to the **share** product subsystem (and is the only "generic" entry point for it), but the dispatch pattern is intentionally open so further product types can plug a `ShareProductReadPlatformService`-compatible bean and reuse the same endpoints.

The dedicated loan-product and savings-product resources (`LoanProductsApiResource`, `SavingsProductsApiResource`, `FixedDepositProductsApiResource`, `RecurringDepositProductsApiResource`, `ShareProductApiResource`) live alongside it and are usually preferred for typed access; `ProductsApiResource` is the URL space picked up by the Mifos web UI when it issues product calls without knowing the type at compile time.

## Source

- **File:** `fineract-provider/src/main/java/org/apache/fineract/portfolio/products/api/ProductsApiResource.java`
- **Class path annotation:** `@Path("/v1/products/{type}")`
- **OpenAPI tag:** `Products`
- **Spring stereotype:** `@Component`

Constructor-injected dependencies:

- `ApplicationContext applicationContext` — used to resolve `{type}<READPLATFORM_NAME>` beans.
- `ApiRequestParameterHelper apiRequestParameterHelper`
- `DefaultToApiJsonSerializer<ProductData> toApiJsonSerializer`
- `PlatformSecurityContext platformSecurityContext`
- `PortfolioCommandSourceWritePlatformService commandsSourceWritePlatformService`

The bean lookup key is `{type} + ProductsApiConstants.READPLATFORM_NAME` (where `READPLATFORM_NAME = "_product_read_service"` — the convention each product family follows when registering its service bean). For example, `share_product_read_service` is registered by `ShareProductReadPlatformServiceImpl`.

## Endpoints

| Method | Path | Description | Command / Handler | Permission |
| ------ | ---- | ----------- | ----------------- | ---------- |
| GET | `/v1/products/{type}/template` | Return the create-product template for the typed read service. | `service.retrieveTemplate()` | Authenticated |
| GET | `/v1/products/{type}/{productId}` | Retrieve a single product. | `service.retrieveOne(productId, isTemplate)` | Authenticated |
| GET | `/v1/products/{type}` | List products of a type (paged via `offset`, `limit`). | `service.retrieveAllProducts(offset, limit)` | Authenticated |
| POST | `/v1/products/{type}` | Create a product. | `CommandWrapperBuilder.createProduct(type)` → `CREATE_{TYPE}PRODUCT` | `CREATE_{TYPE}PRODUCT` (e.g. `CREATE_SHAREPRODUCT`) |
| POST | `/v1/products/{type}/{productId}?command=…` | Execute a sub-command against a product (activate, close, etc.). | `CommandWrapperBuilder.createProductCommand(type, command, productId)` | `{COMMAND}_{TYPE}PRODUCT` |
| PUT | `/v1/products/{type}/{productId}` | Update a product. | `CommandWrapperBuilder.updateProduct(type, productId)` → `UPDATE_{TYPE}PRODUCT` | `UPDATE_{TYPE}PRODUCT` |

Where `{type}` is lower-case — currently only `share` resolves cleanly today.

## Dispatch implementation

The shared code lookups the typed bean:

```java
String serviceName = productType + ProductsApiConstants.READPLATFORM_NAME;
try {
    ShareProductReadPlatformService service =
        (ShareProductReadPlatformService) this.applicationContext.getBean(serviceName);
    // ...
} catch (BeansException e) {
    throw new ResourceNotFoundException(e);
}
```

If no Spring bean is registered for `<type>_product_read_service` the resource throws `ResourceNotFoundException`, which the global exception mapper translates to HTTP 404.

## Examples

### List share products

`GET /v1/products/share`

```json
{
  "totalFilteredRecords": 1,
  "pageItems": [
    {
      "id": 1,
      "name": "Equity Class A",
      "shortName": "EQA",
      "totalShares": 1000000,
      "unitPrice": 1.0,
      "currency": { "code": "USD", "decimalPlaces": 2 }
    }
  ]
}
```

### Retrieve a share product template

`GET /v1/products/share/template`

Returns the editable fields plus option lists (`accountingMappingOptions`, `chargeOptions`, ...). Equivalent to `ShareProductReadPlatformService.retrieveTemplate()`.

### Create a share product

`POST /v1/products/share`

```json
{
  "name": "Equity Class A",
  "shortName": "EQA",
  "description": "Common equity",
  "currencyCode": "USD",
  "digitsAfterDecimal": 2,
  "inMultiplesOf": 0,
  "totalShares": 1000000,
  "unitPrice": 1.0,
  "nominalShares": 100,
  "allowDividendCalculationForInactiveClients": false,
  "accountingRule": 1,
  "locale": "en"
}
```

Response:

```json
{ "resourceId": 1, "changes": {} }
```

### Execute a sub-command

`POST /v1/products/share/1?command=close`

Routes through `createProductCommand("share", "close", 1L)` — the actual permission checked is `CLOSE_SHAREPRODUCT`.

### Update

`PUT /v1/products/share/1`

```json
{ "unitPrice": 1.10, "locale": "en" }
```

```json
{
  "resourceId": 1,
  "changes": { "unitPrice": 1.10 }
}
```

## Data carriers

- **Request:** opaque JSON string — the resource passes the body straight into the command wrapper without binding to a DTO. Each typed `*ProductCommandFromApiJsonDeserializer` validates it.
- **Read response:** `ProductData` (abstract; concrete subclass per product type).
- **Write response:** `CommandProcessingResult` envelope.

## Permissions

`platformSecurityContext.authenticatedUser()` is called on `POST` / `PUT`; read endpoints rely on the typed read service to enforce its own permissions when it goes to the DB. Write authorization is performed by `PortfolioCommandSourceWritePlatformService` against the resolved command-action name.

## Notes & gotchas

- The Swagger annotations on the resource describe only the **share** product flow even though the URL is parameterised by `{type}`.
- Adding a new product family means:
  1. Implementing a class assignable to `ShareProductReadPlatformService` (the dispatch's cast target).
  2. Registering it as `<type>_product_read_service`.
  3. Adding a `CommandHandler` for the `CREATE_{TYPE}PRODUCT`, `UPDATE_{TYPE}PRODUCT`, etc., commands.
