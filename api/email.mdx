The Email resource is Apache Fineract's per-message outbox. It exposes the contents of `m_email_messages_outbound` — the queue of individual emails awaiting SMTP delivery, along with already-sent and failed messages. Bulk message generation lives in [Email Campaigns](/api/email-campaigns); SMTP credentials are configured via [Email Configuration](/api/email-configuration).

## Source

- **File**: `fineract-provider/src/main/java/org/apache/fineract/infrastructure/campaigns/email/api/EmailApiResource.java`
- **Base path**: `@Path("/v1/email")`
- **Permission entity**: `Email` (`RESOURCE_NAME_FOR_PERMISSIONS = "Email"`)
- **Tag**: (none — the class has no `@Tag`)

Reads use `EmailReadPlatformService`; writes are command-sourced via `PortfolioCommandSourceWritePlatformService` with the `createEmail`, `updateEmail` and `deleteEmail` builders.

## Endpoints

| Method | Path | Description | Command builder / Handler | Permission |
| ------ | ---- | ----------- | ------------------------- | ---------- |
| GET | `/v1/email` | List every email message | `retrieveAll` | `READ_Email` |
| GET | `/v1/email/pendingEmail` | List queued-but-not-sent messages (paged) | `retrieveAllPending` | `READ_Email` |
| GET | `/v1/email/sentEmail` | List successfully delivered messages (paged) | `retrieveAllSent` | `READ_Email` |
| GET | `/v1/email/failedEmail` | List messages that failed delivery (paged) | `retrieveAllFailed` | `READ_Email` |
| GET | `/v1/email/messageByStatus` | Paged status-filtered list with optional `fromDate`/`toDate` | `retrieveEmailByStatus` | `READ_Email` |
| GET | `/v1/email/{resourceId}` | Retrieve one message | `retrieveOne` | `READ_Email` (no explicit check — relies on prior auth) |
| POST | `/v1/email` | Queue a new email | `createEmail` → entity `EMAIL`, action `CREATE` | `CREATE_Email` |
| PUT | `/v1/email/{resourceId}` | Update a queued message | `updateEmail(resourceId)` → action `UPDATE` | `UPDATE_Email` |
| DELETE | `/v1/email/{resourceId}` | Delete a queued message | `deleteEmail(resourceId)` → action `DELETE` | `DELETE_Email` |

The paged list endpoints accept `offset`, `limit`, `orderBy` and `sortOrder`. `orderBy` and `sortOrder` are passed through `SqlValidator.validate`.

`messageByStatus` additionally accepts:

| Param | Description |
| ----- | ----------- |
| `status` | Integer status code (see below) |
| `fromDate`, `toDate` | Window filter (use with `dateFormat` and `locale`) |

## Status codes

| status | Meaning | Enum constant |
| ------ | ------- | ------------- |
| 100 | Pending | `EmailMessageStatusType.PENDING` |
| 200 | Sent | `EmailMessageStatusType.SENT` |
| 400 | Failed | `EmailMessageStatusType.FAILED` |

Status values come from `org.apache.fineract.infrastructure.campaigns.email.domain.EmailMessageStatusType`. A status id outside `{100, 200, 400}` raises an `InvalidEmailStatusException`.

## Request body — create

`POST /v1/email`

```json
{
  "to": "client@example.org",
  "subject": "Loan disbursement confirmation",
  "message": "Hello, your loan has been disbursed. Reference: 4500-001.",
  "groupId": null,
  "clientId": 41,
  "staffId": 1,
  "campaignName": null
}
```

The platform queues the message; the `EmailMessageJobEmailService` picks it up on the next scheduler tick and attempts SMTP delivery using credentials from [Email Configuration](/api/email-configuration).

Response:

```json
{ "resourceId": 1208, "changes": {} }
```

## Examples

### Pending messages, newest first

`GET /v1/email/pendingEmail?offset=0&limit=20&orderBy=submittedonDate&sortOrder=DESC`

### Failed messages in a date window

`GET /v1/email/messageByStatus?status=300&fromDate=2024-03-01&toDate=2024-03-04&dateFormat=yyyy-MM-dd&locale=en`

### Cancel a queued message

`DELETE /v1/email/1208` → `{ "resourceId": 1208 }`

## Notes

- The retrieveOne endpoint **does not** call `validateHasReadPermission`. Treat reads of `/v1/email/{id}` as authenticated-but-not-authorised, and front-end-enforce additional controls if needed.
- Status transitions are owned by the scheduler — there is no API endpoint to mark a message "sent" manually. Use this resource for **inspection** and **cancellation**, not retries.
- Messages persisted as `failed` retain their last error in the `errorMessage` field of `EmailData`; downstream tooling can poll `messageByStatus?status=400` to alert operators.

## Curl reference

List pending messages:

```bash
curl -u mifos:password \
  -H "Fineract-Platform-TenantId: default" \
  "https://example.org/fineract-provider/api/v1/email/pendingEmail?limit=100&offset=0"
```

Queue a new ad-hoc email:

```bash
curl -u mifos:password \
  -H "Fineract-Platform-TenantId: default" \
  -H "Content-Type: application/json" \
  -X POST https://example.org/fineract-provider/api/v1/email \
  -d '{
    "groupId": null,
    "clientId": 12,
    "staffId": null,
    "emailAddress": "client12@example.org",
    "emailSubject": "Statement ready",
    "message": "Your January statement is available in your portal."
  }'
```

Fetch the failed window for a date range:

```bash
curl -u mifos:password \
  -H "Fineract-Platform-TenantId: default" \
  "https://example.org/fineract-provider/api/v1/email/messageByStatus?status=400&fromDate=2024-01-01&toDate=2024-01-31&dateFormat=yyyy-MM-dd&locale=en"
```

Cancel a queued message:

```bash
curl -u mifos:password \
  -H "Fineract-Platform-TenantId: default" \
  -X DELETE https://example.org/fineract-provider/api/v1/email/4271
```

## EmailData response shape

The DTO returned by the retrieval endpoints (`org.apache.fineract.infrastructure.campaigns.email.data.EmailData`) carries:

| Field | Type | Notes |
| --- | --- | --- |
| `id` | long | Primary key of `m_email_messages_outbound`. |
| `groupId` / `clientId` / `staffId` | long, nullable | Recipient resolution scope. |
| `campaignId` / `campaignName` | long, string, nullable | Set when the row originated from an Email Campaign. |
| `emailAddress` | string | Target SMTP address. |
| `emailSubject` | string | Rendered subject. |
| `message` | string | Rendered body. |
| `statusType` | object | `{ id, code, value }` mapped from `EmailMessageStatusType`. |
| `submittedOnDate` | array of int | `[yyyy, MM, dd]` per the platform date format. |
| `errorMessage` | string, nullable | Populated on `FAILED`; truncated at the column length defined in the entity. |

## Permissions matrix

| Action | Permission | Source check |
| --- | --- | --- |
| List / search | `READ_Email` | `validateHasReadPermission(EmailApiConstants.RESOURCE_NAME)` in each handler. |
| Retrieve one | none beyond authentication | The handler skips `validateHasReadPermission`. Front-end with care. |
| Create | `CREATE_Email` (`+ _CHECKER`) | Command source `CREATE_EMAIL`. |
| Update | `UPDATE_Email` (`+ _CHECKER`) | Command source `UPDATE_EMAIL`. |
| Delete | `DELETE_Email` (`+ _CHECKER`) | Command source `DELETE_EMAIL`. |

## Error responses

| HTTP | Error code | Cause |
| --- | --- | --- |
| 400 | `error.msg.email.invalid.status` | `status` query parameter outside `{100, 200, 400}`. |
| 400 | `error.msg.email.message.cannot.be.blank` | Missing `message` on create. |
| 400 | `error.msg.email.address.invalid.format` | `emailAddress` fails the SMTP format check. |
| 403 | Standard `validateHasReadPermission` failure | Caller lacks `READ_Email`. |
| 404 | `error.msg.email.id.invalid` | `resourceId` does not exist. |

## Operational notes

- The dispatcher is a Quartz job (`Update email outbound with campaign message`) — pause it via [Scheduler Jobs](/api/scheduler-jobs) when running point-in-time migrations.
- Messages do not carry HTML/text discrimination; the body field is plain text. For HTML email, use a campaign template that wraps the merge body.
- The Mifos web UI surfaces this resource under **Communications → Email Outbox**; the SMTP profile it uses is the one configured via [Email Configuration](/api/email-configuration).

## Data model footprint

The outbox is persisted in `m_email_messages_outbound`. Key columns map 1:1 onto `EmailData`:

| Column | DTO field | Notes |
| --- | --- | --- |
| `id` | `id` | Numeric PK. |
| `group_id`, `client_id`, `staff_id` | matching `*Id` fields | Recipient resolution scope. |
| `email_campaign_id` | `campaignId` | Backlink to the campaign that produced the row. |
| `email_address` | `emailAddress` | Normalised recipient. |
| `email_subject`, `message` | `emailSubject`, `message` | Already-rendered content. |
| `status_enum` | `statusType.id` | Stored as int; see status code table. |
| `error_message` | `errorMessage` | Trimmed; populated on `FAILED`. |
| `submittedon_date` | `submittedOnDate` | Set on insert. |
