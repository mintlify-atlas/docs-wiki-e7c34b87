The SMS resource is Apache Fineract's per-message short-message outbox. Each row represents one outbound SMS waiting for (or already handled by) an external SMS gateway (typically delivered through a [Hook](/api/hooks) — `SMS Bridge` — or a configured provider). Mass-message generation lives in [SMS Campaigns](/api/sms-campaigns).

## Source

- **File**: `fineract-provider/src/main/java/org/apache/fineract/infrastructure/sms/api/SmsApiResource.java`
- **Base path**: `@Path("/v1/sms")`
- **Permission entity**: `SMS`
- **Tag**: `SMS`

Reads use `SmsReadPlatformService`; writes are command-sourced via `PortfolioCommandSourceWritePlatformService` with the `createSms`, `updateSms` and `deleteSms` builders (entity name `SMS`).

## Endpoints

| Method | Path | Description | Command builder / Handler | Permission |
| ------ | ---- | ----------- | ------------------------- | ---------- |
| GET | `/v1/sms` | List every SMS message | `SmsReadPlatformService.retrieveAll` | `READ_SMS` |
| POST | `/v1/sms` | Queue a new message | `CommandWrapperBuilder.createSms` → entity `SMS`, action `CREATE`, href `/sms/template` | `CREATE_SMS` |
| GET | `/v1/sms/{resourceId}` | Retrieve one message | `retrieveOne` | authenticated user |
| GET | `/v1/sms/{campaignId}/messageByStatus` | Paged list of messages for a campaign filtered by status and date range | `retrieveSmsByStatus` | `READ_SMS` |
| PUT | `/v1/sms/{resourceId}` | Update a queued message | `CommandWrapperBuilder.updateSms(id)` → action `UPDATE` | `UPDATE_SMS` |
| DELETE | `/v1/sms/{resourceId}` | Delete a queued message | `CommandWrapperBuilder.deleteSms(id)` → action `DELETE` | `DELETE_SMS` |

`POST`, `PUT` and `DELETE` return a `CommandProcessingResult`.

## Status filter — `messageByStatus`

`SmsRequestParam` (`@BeanParam`) collects:

| Query parameter | Description |
| --------------- | ----------- |
| `status` | Integer status. 100 = pending, 200 = sent, 300 = waiting for delivery report, 400 = delivered, 500 = failed, 600 = invalid. |
| `offset`, `limit` | Pagination. |
| `orderBy`, `sortOrder` | Sort. |
| `fromDate`, `toDate` | Window filter, parsed via `DateParam`. |
| `locale`, `dateFormat` | Locale and date pattern for the date parameters. |

Path parameter `{campaignId}` ties the page to one [SMS campaign](/api/sms-campaigns) — pass `0` for unattached/direct messages.

## Request bodies

### Create — `SmsCreationRequest`

`POST /v1/sms`

```json
{
  "groupId": null,
  "clientId": 41,
  "staffId": 1,
  "message": "Your loan payment is due tomorrow.",
  "mobileNo": "+254700000000",
  "providerId": 1
}
```

### Update — `SmsUpdateRequest`

`PUT /v1/sms/1208`

```json
{
  "message": "Your loan payment is due tomorrow at 17:00."
}
```

## Response shape — `SmsData`

```json
{
  "id": 1208,
  "groupId": null,
  "clientId": 41,
  "staffId": 1,
  "campaignName": null,
  "message": "Your loan payment is due tomorrow.",
  "mobileNo": "+254700000000",
  "statusEnum": 100,
  "providerId": 1,
  "submittedOnDate": "2024-03-04",
  "deliveredOnDate": null
}
```

## Examples

### Failed messages for campaign 17 since 2024-03-01

`GET /v1/sms/17/messageByStatus?status=500&fromDate=2024-03-01&dateFormat=yyyy-MM-dd&locale=en&limit=50`

### Cancel a queued message

`DELETE /v1/sms/1208`

```json
{ "resourceId": 1208, "changes": {} }
```

## Subsystem cross-links

- **[SMS Campaigns](/api/sms-campaigns)** — produce the messages this resource manages.
- **[Hooks](/api/hooks)** — the `SMS Bridge`/Twilio templates dispatch the messages.
- **[Email](/api/email)** — equivalent outbox for email channel.
- **[Notifications](/api/notifications)** — in-app counterpart.

## Notes

- `retrieveOne` does **not** call `validateHasReadPermission` — it only requires a valid login.
- Status transitions are written by the SMS bridge / provider callback handlers, not by clients. The `PUT` here is for editing the **outbound payload** (e.g. fixing a typo before send), not for marking a message delivered.
- Direct creates (`POST /v1/sms`) are intended for ad-hoc messages. For volume, use [SMS Campaigns](/api/sms-campaigns).
