`ClientFamilyMembersApiResource` is a small CRUD surface for capturing household composition on a client — spouses, children, dependants. The records live in `m_family_members` and reference the parent client by `clientId`.

## Permissions

Resource string: `FamilyMembers` (`RESOURCE_NAME_FOR_PERMISSIONS`, mixed case in source). Reads use `validateHasReadPermission("FamilyMembers")`. Writes are gated by `CREATE_FAMILYMEMBERS`, `UPDATE_FAMILYMEMBERS`, `DELETE_FAMILYMEMBERS` through the command wrapper.

## Source excerpt

```java
@Path("/v1/clients/{clientId}/familymembers")
@Component
@Tag(name = "Client Family Member")
public class ClientFamilyMembersApiResource {

    private static final String RESOURCE_NAME_FOR_PERMISSIONS = "FamilyMembers";

    @POST
    @Consumes({ MediaType.APPLICATION_JSON })
    @Produces({ MediaType.APPLICATION_JSON })
    @Operation(summary = "Add a client family member", operationId = "createClientFamilyMember")
    public CommandProcessingResult addClientFamilyMembers(
            @PathParam("clientId") final long clientid,
            ClientFamilyMemberRequest clientFamilyMemberRequest) {
        final CommandWrapper commandRequest = new CommandWrapperBuilder()
            .addFamilyMembers(clientid)
            .withJson(toApiJsonSerializer.serialize(clientFamilyMemberRequest)).build();
        return this.commandsSourceWritePlatformService.logCommandSource(commandRequest);
    }
}
```

The `/template` endpoint returns an empty `ClientFamilyMembersData` whose `relationshipIdOptions`, `genderIdOptions`, `maritalStatusIdOptions`, `professionIdOptions` arrays carry the code-value dropdowns sourced from `Relationship`, `Gender`, `MaritalStatus`, `Profession` code-value families.

## Source excerpt — list

```java
@GET
@Consumes({ MediaType.APPLICATION_JSON })
@Produces({ MediaType.APPLICATION_JSON })
@Operation(summary = "List all client family members",
        operationId = "retrieveAllClientFamilyMembers")
public List<ClientFamilyMembersData> getFamilyMembers(
        @PathParam("clientId") final long clientId) {
    this.context.authenticatedUser()
            .validateHasReadPermission(RESOURCE_NAME_FOR_PERMISSIONS);
    return this.readPlatformService.getClientFamilyMembers(clientId);
}
```

The list returns a raw `List<ClientFamilyMembersData>` (Jersey handles JSON serialisation through Jackson's default writer because the method declares a typed return rather than `String`). There is no pagination — the list is normally tiny.

## Source excerpt — single member

```java
@GET
@Path("/{familyMemberId}")
@Consumes({ MediaType.APPLICATION_JSON })
@Produces({ MediaType.APPLICATION_JSON })
@Operation(summary = "Retrieve a client family member",
        operationId = "retrieveOneClientFamilyMember")
public ClientFamilyMembersData getFamilyMember(
        @PathParam("familyMemberId") final Long familyMemberId,
        @PathParam("clientId") final Long clientId) {
    this.context.authenticatedUser()
            .validateHasReadPermission(RESOURCE_NAME_FOR_PERMISSIONS);
    return this.readPlatformService.getClientFamilyMember(familyMemberId);
}
```

Note that the `clientId` path parameter is accepted but the implementation only looks up by `familyMemberId` — there is no cross-check that the member actually belongs to the named client. If the family-member row exists under a different client, the call still returns its data.

## Typed write DTO

Unlike most Fineract resources which take raw JSON, the create and update operations on this resource accept a typed `ClientFamilyMemberRequest` data class:

```java
@Data
@Builder
@FieldNameConstants
public class ClientFamilyMemberRequest implements Serializable {
    private String lastName;
    private String firstName;
    private String middleName;
    private Long clientId;
    private String dateFormat;
    private String mobileNumber;
    private Long genderId;
    private Boolean isDependent;
    private String dateOfBirth;   // string, parsed via dateFormat/locale
    private Long relationshipId;
    private String locale;
    private String familyMembers;
    private String qualification;
    private Long maritalStatusId;
    private Long id;
    private Long age;
    private Long professionId;
}
```

Source: `fineract-core/src/main/java/org/apache/fineract/portfolio/client/data/ClientFamilyMemberRequest.java`.

The resource serialises the DTO back to JSON before handing it to the `CommandWrapperBuilder`, so the command pipeline sees the same JSON shape as if the client had POSTed it directly.
