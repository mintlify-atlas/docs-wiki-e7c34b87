`OfficeTransactionsApiResource` exposes a small CRUD surface for **head-office** cash movements between branches. It records that office A handed off `X` USD to office B on a given date, optionally citing a description, and it backs the standard journal-entry pipeline so the GL balances stay consistent.

## Permissions

Resource string: `OFFICETRANSACTION` (`RESOURCE_NAME_FOR_READ_PERMISSIONS`). Reads call `validateHasReadPermission("OFFICETRANSACTION")`. Writes use `CREATE_OFFICETRANSACTION`, `DELETE_OFFICETRANSACTION`.

## Endpoint inventory

| HTTP | Path | Description | Command / Read service |
| --- | --- | --- | --- |
| `GET` | `/v1/officetransactions` | List all transactions | `readPlatformService.retrieveAllOfficeTransactions()` |
| `GET` | `/v1/officetransactions/template` | Office + currency dropdowns | `readPlatformService.retrieveNewOfficeTransactionDetails()` |
| `POST` | `/v1/officetransactions` | Record a transaction | `createOfficeTransaction` |
| `DELETE` | `/v1/officetransactions/{transactionId}` | Delete a transaction | `deleteOfficeTransaction(transactionId)` |

There is no PUT — to amend, delete and re-create.

## Source excerpt — create

```java
@POST
@Consumes(MediaType.APPLICATION_JSON)
@Produces(MediaType.APPLICATION_JSON)
public String createOfficeTransaction(final String apiRequestBodyAsJson) {
    final CommandWrapper commandRequest = new CommandWrapperBuilder()
        .createOfficeTransaction()
        .withJson(apiRequestBodyAsJson).build();
    final CommandProcessingResult result =
        commandsSourceWritePlatformService.logCommandSource(commandRequest);
    return toApiJsonSerializer.serialize(result);
}
```

## Canonical curl

```bash
# Record a transfer of 5000 USD from Head Office (1) to Anytown Branch (5)
curl -k -u mifos:password \
  -H "Fineract-Platform-TenantId: default" \
  -H "Content-Type: application/json" \
  -X POST https://localhost:8443/fineract-provider/api/v1/officetransactions \
  -d '{
    "fromOfficeId": 1,
    "toOfficeId": 5,
    "transactionDate": "04 March 2024",
    "transactionAmount": 5000.0,
    "currencyCode": "USD",
    "description": "Branch float top-up",
    "locale": "en",
    "dateFormat": "dd MMMM yyyy"
  }'
```

Sample response:

```json
{
  "officeId": 1,
  "resourceId": 17,
  "resourceIdentifier": "17",
  "changes": null
}
```

## Read DTO

`org.apache.fineract.organisation.office.data.OfficeTransactionData`:

```json
{
  "id": 17,
  "transactionDate": [2024, 3, 4],
  "fromOfficeId": 1,
  "fromOfficeName": "Head Office",
  "toOfficeId": 5,
  "toOfficeName": "Anytown Branch",
  "currencyCode": "USD",
  "transactionAmount": 5000.0,
  "description": "Branch float top-up"
}
```

The list endpoint returns a flat array of `OfficeTransactionData`. There is no paging.

## Request body — POST

| Field | Required | Notes |
| --- | --- | --- |
| `fromOfficeId` | one of `fromOfficeId`/`toOfficeId` is required | Source office |
| `toOfficeId` | as above | Destination office |
| `transactionDate` | yes | |
| `transactionAmount` | yes | Positive number |
| `currencyCode` | yes | Must be a permitted currency (see [Currencies](/api/currencies)) |
| `description` | no | Free text |
| `locale`, `dateFormat` | yes | |

Either `fromOfficeId` or `toOfficeId` can be left null to represent a one-sided injection or withdrawal (e.g. injecting cash into Head Office from outside the system).

## GL postings

Each office transaction triggers a journal entry through `JournalEntryWritePlatformService.createJournalEntriesForOfficeTransaction`. The accounts posted are configured on the source/destination offices via the [Financial Activity Account mappings](/accounting/financial-activity-accounts). If those mappings are missing, the request fails with `error.msg.organisation.financial.activity.not.mapped`.

Deleting an office transaction also reverses the underlying journal entries — but only if they have not been further reconciled.
