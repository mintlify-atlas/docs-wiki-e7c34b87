The Loan Charges API attaches one-time and recurring fees/penalties to an Apache Fineract loan account. It supports creating, listing, updating and deleting loan charges, plus executing the standard `pay`, `waive`, `adjustment` commands. Every endpoint exists in four variants: by loan id + charge id, by loan id + charge external id, by loan external id + charge id, and by both external ids.

## Endpoints

### List & template

| Method | Path | Description | Operation id | Permission |
| --- | --- | --- | --- | --- |
| `GET` | `/v1/loans/{loanId}/charges` | List all charges on a loan. | `retrieveAllLoanCharges` | `READ_LOANCHARGE` |
| `GET` | `/v1/loans/external-id/{loanExternalId}/charges` | Same, addressing the loan by external id. | `retrieveAllLoanChargesByLoanExternalId` | `READ_LOANCHARGE` |
| `GET` | `/v1/loans/{loanId}/charges/template` | Charge-creation template (allowed charge options + dropdowns). | `retrieveTemplateLoanCharge` | `READ_LOANCHARGE` |
| `GET` | `/v1/loans/external-id/{loanExternalId}/charges/template` | External-id variant. | `retrieveTemplateLoanChargeByLoanExternalId` | `READ_LOANCHARGE` |

### Retrieve a single charge

| Method | Path | Operation id |
| --- | --- | --- |
| `GET` | `/v1/loans/{loanId}/charges/{loanChargeId}` | `retrieveLoanCharge` |
| `GET` | `/v1/loans/{loanId}/charges/external-id/{loanChargeExternalId}` | `retrieveLoanChargeByChargeExternalId` |
| `GET` | `/v1/loans/external-id/{loanExternalId}/charges/{loanChargeId}` | `retrieveLoanChargeByLoanExternalId` |
| `GET` | `/v1/loans/external-id/{loanExternalId}/charges/external-id/{loanChargeExternalId}` | `retrieveLoanChargeByLoanAndChargeExternalId` |

All four resolve to `LoanChargeReadPlatformService.retrieveLoanChargeDetails(loanChargeId, loanId)` after external-id resolution.

### Create / pay / waive / adjustment

`POST /v1/loans/{loanId}/charges` accepts an optional `?command=` query parameter. Without a command, the body is treated as a **create charge** request. With one of `pay`, `waive`, or `adjustment`, the body is consumed by the matching action.

| Method | Path | Operation id | Command handler |
| --- | --- | --- | --- |
| `POST` | `/v1/loans/{loanId}/charges` | `executeLoanCharge` | `createLoanCharge(loanId)` (no command) or `payLoanCharge(loanId, ...)` (`command=pay`) |
| `POST` | `/v1/loans/external-id/{loanExternalId}/charges` | `executeLoanChargeByLoanExternalId` | same, after external-id resolution |
| `POST` | `/v1/loans/{loanId}/charges/{loanChargeId}` | `executeLoanChargeOnExistingCharge` | `payLoanCharge(loanId, loanChargeId)` / `waiveLoanCharge(loanId, loanChargeId)` / `adjustmentForLoanCharge(loanId, loanChargeId)` |
| `POST` | `/v1/loans/{loanId}/charges/external-id/{loanChargeExternalId}` | `executeLoanChargeByChargeExternalId` | same |
| `POST` | `/v1/loans/external-id/{loanExternalId}/charges/{loanChargeId}` | `executeLoanChargeByLoanExternalIdOnExistingCharge` | same |
| `POST` | `/v1/loans/external-id/{loanExternalId}/charges/external-id/{loanChargeExternalId}` | `executeLoanChargeByLoanAndChargeExternalId` | same |

The accepted commands (sourced from the dispatch in `executeLoanCharge`):

- `pay` — pay the charge from a configured payment type. Builder: `CommandWrapperBuilder.payLoanCharge(loanId, loanChargeId)`.
- `waive` — waive the outstanding amount. Builder: `CommandWrapperBuilder.waiveLoanCharge(loanId, loanChargeId)`.
- `adjustment` — adjust the recognised charge amount. Builder: `CommandWrapperBuilder.adjustmentForLoanCharge(loanId, loanChargeId)`.

## Request / response shapes

### Create a charge

`POST /v1/loans/{loanId}/charges`:

```json
{
  "chargeId": 14,
  "amount": 100.00,
  "dueDate": "27 March 2026",
  "locale": "en",
  "dateFormat": "dd MMMM yyyy"
}
```

For instalment-fee charges add `chargeTimeType`, `chargeCalculationType` and `chargePaymentMode` only when overriding the `Charge` defaults.

### Pay a charge

`POST /v1/loans/{loanId}/charges/{loanChargeId}?command=pay`:

```json
{
  "transactionDate": "01 April 2026",
  "locale": "en",
  "dateFormat": "dd MMMM yyyy"
}
```

### Waive a charge

`POST /v1/loans/{loanId}/charges/{loanChargeId}?command=waive`:

```json
{ "locale": "en", "dateFormat": "dd MMMM yyyy" }
```

### Charge adjustment

`POST /v1/loans/{loanId}/charges/{loanChargeId}?command=adjustment`:

```json
{
  "amount": 10.00,
  "externalId": "adj-001",
  "paymentTypeId": 1,
  "note": "Partial reversal",
  "locale": "en"
}
```

### Update a charge

`PUT /v1/loans/{loanId}/charges/{loanChargeId}`:

```json
{
  "amount": 120.00,
  "dueDate": "30 March 2026",
  "locale": "en",
  "dateFormat": "dd MMMM yyyy"
}
```

### Response envelope (write endpoints)

```json
{
  "officeId": 1,
  "clientId": 42,
  "loanId": 123,
  "resourceId": 88,
  "subResourceId": null,
  "changes": { "amount": 120.00 }
}
```

## Permissions and security context

Each handler calls `PlatformSecurityContext.authenticatedUser()`:

- GETs invoke `validateHasReadPermission("LOANCHARGE")`.
- Create / pay / waive / adjustment / update / delete are routed through `PortfolioCommandSourceWritePlatformService.logCommandSource(...)` which applies the appropriate `CREATE_/UPDATE_/DELETE_/PAY_/WAIVE_LOANCHARGE` permission code based on the `CommandWrapperBuilder` action code.

External-id resolution happens via `LoanReadPlatformService.getResolvedLoanId(externalId)` and `LoanChargeReadPlatformService.getResolvedLoanChargeId(externalId)` before the builder is invoked.
