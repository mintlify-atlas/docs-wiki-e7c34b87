`InterestRateChartSlabsApiResource` is the JAX-RS resource that manages **chart slabs** (a.k.a. *interest bands*) — the rows that map a deposit-period range to an annual interest rate within a parent `InterestRateChart`. Each slab is shaped `(periodType, fromPeriod, toPeriod, annualInterestRate[, currency, fromAmount, toAmount, description])` and can carry per-incentive overrides.

The parent chart resource is documented in [Interest Rate Charts](/api/interest-rate-charts).

## Source

- **File:** `fineract-provider/src/main/java/org/apache/fineract/portfolio/interestratechart/api/InterestRateChartSlabsApiResource.java`
- **Class path annotation:** `@Path("/v1/interestratecharts/{chartId}/chartslabs")`
- **OpenAPI tag:** *Interest Rate Slab (A.K.A interest bands)*
- **Class-level annotations:** `@Consumes({ MediaType.APPLICATION_JSON })`, `@Produces({ MediaType.APPLICATION_JSON })`, `@Component`

Constructor-injected dependencies:

- `InterestRateChartSlabsReadService interestRateChartSlabsReadService` — read store.
- `CommandPipeline commandPipeline` — typed command dispatcher.

## Endpoints

| Method | Path | Description | Command / Handler | Permission |
| ------ | ---- | ----------- | ----------------- | ---------- |
| GET | `/v1/interestratecharts/{chartId}/chartslabs/template` | Return the slab template (period-type options, currency options, incentive options). | `interestRateChartSlabsReadService.retrieveTemplate()` | Authenticated |
| GET | `/v1/interestratecharts/{chartId}/chartslabs` | List slabs attached to a chart. | `interestRateChartSlabsReadService.retrieveAll(chartId)` | Authenticated |
| GET | `/v1/interestratecharts/{chartId}/chartslabs/{chartSlabId}` | Retrieve a single slab. | `interestRateChartSlabsReadService.retrieveOne(chartId, chartSlabId)` | Authenticated |
| POST | `/v1/interestratecharts/{chartId}/chartslabs` | Create a slab. Mandatory: `periodType`, `fromPeriod`, `annualInterestRate`. | `commandPipeline.send(new InterestRateChartSlabsCreateCommand(request))` | `CREATE_INTERESTRATECHARTSLAB` |
| PUT | `/v1/interestratecharts/{chartId}/chartslabs/{chartSlabId}` | Update a slab. | `commandPipeline.send(new InterestRateChartSlabsUpdateCommand(request))` | `UPDATE_INTERESTRATECHARTSLAB` |
| DELETE | `/v1/interestratecharts/{chartId}/chartslabs/{chartSlabId}` | Delete a slab. | `commandPipeline.send(new InterestRateChartSlabsDeleteCommand(...))` | `DELETE_INTERESTRATECHARTSLAB` |

The resource stamps both path params (`chartId`, `chartSlabId`) into the request object before dispatching:

```java
request.setChartId(chartId);
request.setChartSlabId(chartSlabId);
```

### Create

`POST /v1/interestratecharts/12/chartslabs`

`InterestRateChartSlabsCreateRequest`:

```json
{
  "periodType": 2,
  "fromPeriod": 1,
  "toPeriod": 11,
  "annualInterestRate": 3.0,
  "currencyCode": "USD",
  "description": "Short term",
  "locale": "en"
}
```

Handler:

```java
request.setChartId(chartId);
final var command = new InterestRateChartSlabsCreateCommand();
command.setPayload(request);
return commandPipeline.send(command).get();
```

Response — `InterestRateChartSlabsCreateResponse`:

```json
{
  "resourceId": 101,
  "officeId": null,
  "changes": {}
}
```

### Update

`PUT /v1/interestratecharts/12/chartslabs/101`

```json
{
  "annualInterestRate": 3.25,
  "locale": "en"
}
```

Response — `InterestRateChartSlabsUpdateResponse`:

```json
{
  "resourceId": 101,
  "changes": { "annualInterestRate": 3.25 }
}
```

### Delete

`DELETE /v1/interestratecharts/12/chartslabs/101`

```json
{
  "resourceId": 101,
  "changes": {}
}
```

The delete fails if the slab is the only one attached to its chart and the parent product has live accounts using that chart.

## Data carriers

- **Requests:** `InterestRateChartSlabsCreateRequest`, `InterestRateChartSlabsUpdateRequest`, `InterestRateChartSlabsDeleteRequest`.
- **Read response:** `InterestRateChartSlabData` — projects period type, period range, currency, optional amount range, annual rate and incentive list.
- **Write responses:** `InterestRateChartSlabsCreateResponse`, `InterestRateChartSlabsUpdateResponse`, `InterestRateChartSlabsDeleteResponse`.

## Validation rules

Enforced by the command-handler chain (`InterestRateChartSlabsCommandValidator` + `InterestRateChartAssembler#updateChartSlabs`):

- `periodType` must be one of `Days/Weeks/Months/Years`.
- `fromPeriod < toPeriod` when `toPeriod` is non-null; `toPeriod=null` means *open-ended* (must be the last slab).
- Period ranges must not overlap within the parent chart.
- `annualInterestRate` is a percentage and validated against `currency.decimalPlaces`.
- If `fromAmount`/`toAmount` are supplied, the same overlap rules apply across the amount axis.
