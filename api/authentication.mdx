The Authentication API is Apache Fineract's HTTP Basic credential-verification endpoint. A client POSTs a username and password; the resource runs them through the configured `DaoAuthenticationProvider`, and on success returns the user's roles, permissions, office, staff record, a base64-encoded `Authorization` header value, and a flag that tells the client whether two-factor authentication is required for subsequent calls. It is enabled by the `fineract.security.basicauth.enabled` property; the OAuth2 counterpart lives at [User Details](/api/user-details).

## Source

| Aspect | Value |
| --- | --- |
| Resource class | `org.apache.fineract.infrastructure.security.api.AuthenticationApiResource` |
| File | `fineract-security/src/main/java/org/apache/fineract/infrastructure/security/api/AuthenticationApiResource.java` |
| JAX-RS `@Path` | `/v1/authentication` |
| Swagger tag | `Authentication HTTP Basic` |
| Conditional bean | `@ConditionalOnProperty("fineract.security.basicauth.enabled")` |
| Authentication provider | `DaoAuthenticationProvider` bean `customAuthenticationProvider` |
| Response serializer | `ToApiJsonSerializer<AuthenticatedUserData>` |
| Two-factor flag | `${fineract.security.2fa.enabled}` |

## Endpoints

| Method | Path | Description | Command / handler | Permission |
| --- | --- | --- | --- | --- |
| `POST` | `/v1/authentication` | Verify a username/password pair and return the user's profile, roles, permissions, and base64 auth key. | `DaoAuthenticationProvider.authenticate(UsernamePasswordAuthenticationToken)` | None (the call *is* the auth check) |

The resource exposes only one operation; the corresponding swagger schema is `AuthenticationApiResourceSwagger.PostAuthenticationRequest`/`PostAuthenticationResponse`.

## Request

The body is parsed manually with GSON (see the FINERACT-819 / FINERACT-726 references in the source) so it can be received as a raw string:

```bash
curl -X POST \
  -H 'fineract-platform-tenantid: default' \
  -H 'Content-Type: application/json' \
  'https://fineract.example.org/fineract-provider/api/v1/authentication' \
  -d '{"username":"mifos","password":"password"}'
```

Both `username` and `password` are mandatory; supplying `null` for either raises `IllegalArgumentException`. The class defines an inner DTO:

```java
public static class AuthenticateRequest {
    public String username;
    public String password;
}
```

## Response

```json
{
  "username": "mifos",
  "userId": 1,
  "base64EncodedAuthenticationKey": "bWlmb3M6cGFzc3dvcmQ=",
  "authenticated": true,
  "officeId": 1,
  "officeName": "Head Office",
  "staffId": 2,
  "staffDisplayName": "Mifos Staff",
  "organisationalRole": {
    "id": 1,
    "code": "loanOfficer",
    "value": "Loan Officer"
  },
  "roles": [
    { "id": 1, "name": "Super user", "description": "Has all the permissions" }
  ],
  "permissions": ["ALL_FUNCTIONS"],
  "shouldRenewPassword": false,
  "isTwoFactorAuthenticationRequired": false
}
```

The `base64EncodedAuthenticationKey` is the value the client should put in subsequent `Authorization: Basic <key>` headers — Fineract computes it as `Base64(username + ":" + password)`. The original password is *not* stored anywhere; the resource only echoes it back in this encoded form so the client doesn't have to do the base64 itself.

## Status codes

| Status | Trigger |
| --- | --- |
| `200 OK` | Authentication succeeded. |
| `400 Bad Request` | Missing/invalid JSON, missing `username`/`password`. |
| `401/403` | Authentication failed (raised by Spring Security upstream). |
| `403 Forbidden` | Password reset required (`PasswordResetRequiredException`) — the body still contains the `AuthenticatedUserData` so the client can prompt for a new password. |

## Two-factor handshake

When `fineract.security.2fa.enabled=true`, the response sets `isTwoFactorAuthenticationRequired=true` unless the principal holds the `TwoFactorConstants.BYPASS_TWO_FACTOR_PERMISSION`. The flag tells the client to follow up with the [Two Factor](/api/two-factor) endpoints — request an OTP delivery, validate the OTP, and use the returned token for further calls.

## Password expiry

If `SpringSecurityPlatformSecurityContext.doesPasswordHasToBeRenewed(principal)` is true, the resource throws `PasswordResetRequiredException` carrying a minimal `AuthenticatedUserData`:

```json
{
  "username": "mifos",
  "userId": 1,
  "base64EncodedAuthenticationKey": "bWlmb3M6cGFzc3dvcmQ=",
  "authenticated": true,
  "shouldRenewPassword": true,
  "isTwoFactorAuthenticationRequired": false
}
```

The client should then call `POST /v1/users/{userId}/pwd` ([Users](/api/users)) to set a new password before retrying.

## When the bean is not loaded

Setting `fineract.security.basicauth.enabled=false` causes the `@ConditionalOnProperty` guard to skip the bean. Jersey then has no resource registered at `/v1/authentication` and any POST returns `404 Not Found`. Use this configuration when the deployment relies exclusively on OAuth2 ([User Details](/api/user-details)).
