The Loan Transactions API is the gateway through which every Apache Fineract loan-side money movement is recorded: repayments, refunds, charge-backs, write-offs, foreclosures, recoveries, and the progressive-loan specific re-age / re-amortize actions. Every endpoint exists in id and external-id variants for both the loan and the transaction.

## Source

| Aspect | Value |
| --- | --- |

| JAX-RS `@Path` | `/v1/loans` (transactions under `{loanId}/transactions` or `external-id/{loanExternalId}/transactions`) |
| Swagger tag | `Loan Transactions` |

## Read endpoints

| Method | Path | Description |
| --- | --- | --- |
| `GET` | `/v1/loans/{loanId}/transactions/template` | Transaction template, `?command={cmd}` chooses the prefilled action (`repayment`, `prepayLoan`, `foreclosure`, `chargeRefund`, …). |
| `GET` | `/v1/loans/external-id/{loanExternalId}/transactions/template` | External-id variant. |
| `GET` | `/v1/loans/{loanId}/transactions/{transactionId}` | Retrieve one transaction. |
| `GET` | `/v1/loans/{loanId}/transactions/external-id/{externalTransactionId}` | By transaction external id. |
| `GET` | `/v1/loans/external-id/{loanExternalId}/transactions/{transactionId}` | By loan external id. |
| `GET` | `/v1/loans/external-id/{loanExternalId}/transactions/external-id/{externalTransactionId}` | Both external ids. |
| `GET` | `/v1/loans/{loanId}/transactions` | List/paginate transactions on a loan. |
| `GET` | `/v1/loans/external-id/{loanExternalId}/transactions` | External-id variant. |

`LoanReadPlatformService.retrieveLoanTransaction(transactionId)` powers the single-transaction view; the list view uses `retrieveLoanTransactionsByLoanId(loanId, pageable)`.

## Write endpoints — execute a transaction

`POST /v1/loans/{loanId}/transactions?command={cmd}` is dispatched in `executeLoanTransaction(...)` via `CommandParameterUtil.is(commandParam, ...)`:

| `command` | Command handler |
| --- | --- |
| `repayment` | `CommandWrapperBuilder.loanRepaymentTransaction(loanId)` |
| `merchantIssuedRefund` | `loanMerchantIssuedRefundTransaction(loanId)` |
| `payoutRefund` | `loanPayoutRefundTransaction(loanId)` |
| `goodwillCredit` | `loanGoodwillCreditTransaction(loanId)` |
| `interestPaymentWaiver` | `loanInterestPaymentWaiverTransaction(loanId)` |
| `chargeRefund` | `chargeRefund(loanId)` |
| `waiveinterest` | `waiveInterestOnLoan(loanId)` |
| `writeoff` | `writeOffLoan(loanId)` |
| `close-rescheduled` | `closeLoanAsRescheduled(loanId)` |
| `close` | `closeLoan(loanId)` |
| `undowriteoff` | `undoWriteOffLoan(loanId)` |
| `recoverypayment` | `loanRecoveryPaymentTransaction(loanId)` |
| `refundByCash` | `refundLoanCharge(loanId)` |
| `foreclosure` | `forecloseLoan(loanId)` |
| `creditBalanceRefund` | `creditBalanceRefund(loanId)` |
| `charge-off` (constant `CHARGE_OFF_COMMAND_VALUE`) | `chargeOff(loanId)` |
| `undo-charge-off` (constant `UNDO_CHARGE_OFF_COMMAND_VALUE`) | `undoChargeOff(loanId)` |
| `downPayment` (constant `DOWN_PAYMENT`) | `downPayment(loanId)` |
| `reAge` (constant `REAGE`) | `reAge(loanId)` |
| `undoReAge` (constant `UNDO_REAGE`) | `undoReAge(loanId)` |
| `reAmortize` (constant `REAMORTIZE`) | `reAmortize(loanId)` |
| `undoReAmortize` (constant `UNDO_REAMORTIZE`) | `undoReAmortize(loanId)` |
| `capitalizedIncome` (constant `CAPITALIZED_INCOME`) | `capitalizedIncomeTransaction(loanId)` |
| `buyDownFee` (constant `LoanApiConstants.BUY_DOWN_FEE_COMMAND`) | `buyDownFeeTransaction(loanId)` |

Unknown commands throw `UnrecognizedQueryParamException("command", ...)`.

The `external-id` variant exposes the same dispatcher and additionally accepts these legacy aliases inside `executeLoanTransactionByLoanExternalId`:

- `disburse` → `disburseLoanApplication(loanId)` (only when used through the transactions endpoint to push a disbursal as a transaction)
- `disburseToSavings` → `disburseLoanToSavingsApplication(loanId)`
- `prepayLoan` → `loanPrepayLoanTransaction(loanId)`
- `refundbycash` / `refundbytransfer` → `refundLoanByCash(loanId)` / `refundLoanByTransfer(loanId)`

| Method | Path | Operation id |
| --- | --- | --- |
| `POST` | `/v1/loans/{loanId}/transactions` | `executeLoanTransaction` |
| `POST` | `/v1/loans/external-id/{loanExternalId}/transactions` | `executeLoanTransactionByLoanExternalId` |

## Adjust an existing transaction

`POST /v1/loans/{loanId}/transactions/{transactionId}` adjusts (chargeback / reversal / correction) an existing transaction. The `?command=` query parameter selects the action: typical values are `chargeback` and (when omitted) plain adjustment.

| Method | Path | Operation id | Builder |
| --- | --- | --- | --- |
| `POST` | `/v1/loans/{loanId}/transactions/{transactionId}` | `adjustLoanTransaction` | `adjustLoanTransaction(loanId, transactionId)` / `chargeback(loanId, transactionId)` |
| `POST` | `/v1/loans/external-id/{loanExternalId}/transactions/{transactionId}` | `adjustLoanTransactionByLoanExternalId` | same |
| `POST` | `/v1/loans/{loanId}/transactions/external-id/{externalTransactionId}` | `adjustLoanTransactionByTransactionExternalId` | same |
| `POST` | `/v1/loans/external-id/{loanExternalId}/transactions/external-id/{externalTransactionId}` | `adjustLoanTransactionByLoanAndTransactionExternalId` | same |

## Re-Age / Re-Amortize previews

Progressive-loan only. These are GET endpoints that return the new schedule that *would* be produced by a re-age or re-amortize without persisting it.

| Method | Path | Operation id |
| --- | --- | --- |
| `GET` | `/v1/loans/{loanId}/transactions/reage-preview` | `previewReAgeSchedule` |
| `GET` | `/v1/loans/external-id/{loanExternalId}/transactions/reage-preview` | `previewReAgeScheduleByLoanExternalId` |
| `GET` | `/v1/loans/{loanId}/transactions/reamortization-preview` | `previewReAmortizationSchedule` |
| `GET` | `/v1/loans/external-id/{loanExternalId}/transactions/reamortization-preview` | `previewReAmortizationScheduleByLoanExternalId` |

The corresponding *write* endpoints (`reAge`, `undoReAge`, `reAmortize`, `undoReAmortize`) are submitted through the standard `POST /v1/loans/{loanId}/transactions?command={cmd}` dispatcher with command constants exposed on the class:

```java
public static final String DOWN_PAYMENT       = "downPayment";
public static final String REAGE              = "reAge";
public static final String UNDO_REAGE         = "undoReAge";
public static final String REAMORTIZE         = "reAmortize";
public static final String UNDO_REAMORTIZE    = "undoReAmortize";
public static final String CAPITALIZED_INCOME = "capitalizedIncome";
```

## Request / response shapes

### Generic transaction request

`POST /v1/loans/{loanId}/transactions?command=repayment`:

```json
{
  "transactionDate": "01 April 2026",
  "transactionAmount": 250.00,
  "paymentTypeId": 1,
  "externalId": "tx-2026-04-01-001",
  "locale": "en",
  "dateFormat": "dd MMMM yyyy",
  "note": "Customer cash payment"
}
```

### Adjust a transaction

`POST /v1/loans/{loanId}/transactions/{transactionId}`:

```json
{
  "transactionDate": "02 April 2026",
  "transactionAmount": 200.00,
  "paymentTypeId": 1,
  "locale": "en",
  "dateFormat": "dd MMMM yyyy",
  "note": "Reversed amount overpaid"
}
```

The adjusted amount creates a counter-transaction of type `Reversal` and posts a fresh transaction matching the new figures.

### Standard write response

```json

  "clientId": 42,
  "loanId": 123,
  "resourceId": 555,
  "changes": {}
}
```

## Permissions

Read endpoints call `validateHasReadPermission("LOAN")`. Write endpoints rely on the `CommandWrapperBuilder` action code, which is mapped by `PortfolioCommandSourceWritePlatformService` to permission codes such as `REPAYMENT_LOAN`, `WAIVEINTERESTPORTION_LOAN`, `WRITEOFF_LOAN`, `CLOSE_LOAN`, `RECOVERYPAYMENT_LOAN`, `FORECLOSURE_LOAN`, `CREDITBALANCEREFUND_LOAN`, `ADJUST_LOANTRANSACTION`, `CHARGEBACK_LOANTRANSACTION`, etc.
