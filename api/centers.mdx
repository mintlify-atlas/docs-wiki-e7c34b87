`CentersApiResource` exposes the parent aggregate above [groups](/api/groups). A center is a federation of groups that meets on a common schedule; collection sheets typically are generated at the center level so all member groups appear together on the loan officer's tablet.

## Permissions

Resource string: `CENTER` (`GroupingTypesApiConstants.CENTER_RESOURCE_NAME`). Reads call `validateHasReadPermission("CENTER")`. Writes are gated by `CREATE_CENTER`, `UPDATE_CENTER`, `DELETE_CENTER`, `ACTIVATE_CENTER`, `CLOSE_CENTER`, `ASSOCIATEGROUPS_CENTER`, `DISASSOCIATEGROUPS_CENTER`, `SAVECOLLECTIONSHEET_CENTER`.

## The `command=` dispatch table

| `command=` | CommandWrapperBuilder call | Effect |
| --- | --- | --- |
| `activate` | `activateCenter(centerId)` | Pending → Active |
| `close` | `closeCenter(centerId)` | Active → Closed |
| `associateGroups` | `associateGroupsToCenter(centerId)` | Add member groups (body: `groupMembers`) |
| `disassociateGroups` | `disassociateGroupsFromCenter(centerId)` | Remove member groups |
| `generateCollectionSheet` | (read-only) | Produces a `JLGCollectionSheetData` snapshot for the meeting calendar |
| `saveCollectionSheet` | `saveCenterCollectionSheet(centerId)` | Persists collected payments across all member groups |

Unrecognised value → `UnrecognizedQueryParamException`.

## Source excerpt — collection sheet generation

```java
@POST
@Path("{centerId}")
public String activate(
        @PathParam("centerId") final Long centerId,
        @QueryParam("command") final String commandParam,
        final String apiRequestBodyAsJson,
        @Context final UriInfo uriInfo) {
    // ...
    if (is(commandParam, "generateCollectionSheet")) {
        final JsonElement parsedQuery = fromJsonHelper.parse(apiRequestBodyAsJson);
        final JsonQuery query = JsonQuery.from(apiRequestBodyAsJson, parsedQuery, fromJsonHelper);
        final JLGCollectionSheetData collectionSheet =
            collectionSheetReadPlatformService.generateCenterCollectionSheet(centerId, query);
        final ApiRequestJsonSerializationSettings settings =
            apiRequestParameterHelper.process(uriInfo.getQueryParameters());
        return toApiJsonSerializer.serialize(settings, collectionSheet,
            GroupingTypesApiConstants.COLLECTIONSHEET_DATA_PARAMETERS);
    }
    // ...
}
```

Note: `generateCollectionSheet` does **not** call `commandsSourceWritePlatformService.logCommandSource(...)` — it is a read endpoint dressed up as a POST because the body carries a JSON query (with `calendarId`, `transactionDate`, `locale`, `dateFormat`). To persist the edited sheet you must call back with `command=saveCollectionSheet`.

## Canonical curl

```bash
# Create a center
curl -k -u mifos:password \
  -H "Fineract-Platform-TenantId: default" \
  -H "Content-Type: application/json" \
  -X POST https://localhost:8443/fineract-provider/api/v1/centers \
  -d '{
    "officeId": 1,
    "name": "Anytown Center",
    "active": true,
    "activationDate": "04 March 2024",
    "groupMembers": [42, 43],
    "locale": "en",
    "dateFormat": "dd MMMM yyyy"
  }'

# Generate a collection sheet snapshot
curl -k -u mifos:password \
  -H "Fineract-Platform-TenantId: default" \
  -H "Content-Type: application/json" \
  -X POST 'https://localhost:8443/fineract-provider/api/v1/centers/9?command=generateCollectionSheet' \
  -d '{ "calendarId": 9, "transactionDate": "04 March 2024", "locale": "en", "dateFormat": "dd MMMM yyyy" }'
```
