`ClientTransactionsApiResource` exposes the audit trail of monetary movements at the client level — primarily entries generated by [client charge](/api/client-charges) `pay`, `waive`, and `inactivate` commands. It supports listing, fetching a single transaction, and reversing (`undo`) a transaction. All routes are mirrored on the `externalId` side for both the client and the transaction.

## Permissions

Reads use `validateHasReadPermission(ClientApiConstants.CLIENT_CHARGES_RESOURCE_NAME)` — i.e. the `CLIENTCHARGE` resource string (transactions share the charge permission). The `undo` write is gated by `UNDO_CLIENTCHARGE`.

## Endpoint inventory

The class hosts **eight** routes — four numeric-id × two external-id mirrors. The path-level `@Path` is `/v1/clients` and each method adds its own sub-path.

### Numeric-id routes

| HTTP | Path | Description | Command / Read service |
| --- | --- | --- | --- |
| `GET` | `/v1/clients/{clientId}/transactions` | List paged transactions | `retrieveAllTransactions(clientId, offset, limit)` |
| `GET` | `/v1/clients/{clientId}/transactions/{transactionId}` | One transaction by id | `retrieveTransaction(clientId, transactionId)` |
| `POST` | `/v1/clients/{clientId}/transactions/{transactionId}?command=undo` | Reverse the transaction | `undoClientTransaction(clientId, transactionId)` |
| `GET` | `/v1/clients/{clientId}/transactions/external-id/{transactionExternalId}` | One transaction by its external id | `retrieveTransaction(clientId, externalId)` |
| `POST` | `/v1/clients/{clientId}/transactions/external-id/{transactionExternalId}?command=undo` | Reverse by transaction external id | `undoClientTransaction(clientId, externalId)` |

### External-id-client routes

| HTTP | Path | Description |
| --- | --- | --- |
| `GET` | `/v1/clients/external-id/{clientExternalId}/transactions` | List for client by external id |
| `GET` | `/v1/clients/external-id/{clientExternalId}/transactions/{transactionId}` | One transaction (numeric tx id, external client id) |
| `POST` | `/v1/clients/external-id/{clientExternalId}/transactions/{transactionId}?command=undo` | Undo (numeric tx id, external client id) |
| `GET` | `/v1/clients/external-id/{clientExternalId}/transactions/external-id/{transactionExternalId}` | One transaction (both external) |
| `POST` | `/v1/clients/external-id/{clientExternalId}/transactions/external-id/{transactionExternalId}?command=undo` | Undo (both external) |

So **every** combination of (numeric-or-external client) × (numeric-or-external transaction) is reachable.

## Source excerpt — undo

```java
@POST
@Path("{clientId}/transactions/{transactionId}")
@Consumes(MediaType.APPLICATION_JSON)
@Produces(MediaType.APPLICATION_JSON)
public String undoClientTransaction(
        @PathParam("clientId") final Long clientId,
        @PathParam("transactionId") final Long transactionId,
        @QueryParam("command") final String commandParam,
        final String apiRequestBodyAsJson) {

    final CommandWrapper commandRequest = new CommandWrapperBuilder()
        .undoClientTransaction(clientId, transactionId)
        .withJson(apiRequestBodyAsJson).build();

    return toApiJsonSerializer.serialize(
        commandsSourceWritePlatformService.logCommandSource(commandRequest));
}
```

The `command=` value must be `undo`. Other values throw `UnrecognizedQueryParamException`.

## Canonical curl

List:

```bash
curl -k -u mifos:password \
  -H "Fineract-Platform-TenantId: default" \
  'https://localhost:8443/fineract-provider/api/v1/clients/187/transactions?offset=0&limit=50'
```

Undo:

```bash
curl -k -u mifos:password \
  -H "Fineract-Platform-TenantId: default" \
  -H "Content-Type: application/json" \
  -X POST 'https://localhost:8443/fineract-provider/api/v1/clients/187/transactions/901?command=undo' \
  -d '{}'
```

## Transaction types

`ClientTransactionType` enum values:

| id | code | meaning |
| --- | --- | --- |
| 1 | `clientTransactionType.payCharge` | Charge payment |
| 2 | `clientTransactionType.waiveCharge` | Charge waiver |

A reversal flips `reversed: true` on the row but does not delete it; the books still show the credit and matching debit.
