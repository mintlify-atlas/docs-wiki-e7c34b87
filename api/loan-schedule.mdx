The Loan Schedule API exposes the variable-instalment editing capability Apache Fineract calls *Loan Rescheduling* on the pre-approval phase. A single endpoint accepts a `?command=` switch that lets a caller preview a recalculated schedule, add variations, or remove all variations from a loan that has not yet been approved.

## Source

| Aspect | Value |
| --- | --- |
| Resource class | `org.apache.fineract.portfolio.loanaccount.api.LoanScheduleApiResource` |
| File | `fineract-loan/src/main/java/org/apache/fineract/portfolio/loanaccount/api/LoanScheduleApiResource.java` |
| JAX-RS `@Path` | `/v1/loans/{loanId}/schedule` |
| Swagger tag | `Loan Rescheduling` |
| Permission resource | `LOAN` (calculate) / per-command codes for writes |
| Read service | `LoanScheduleCalculationPlatformService` |
| Command source | `PortfolioCommandSourceWritePlatformService` |

The constructor wires `PlatformSecurityContext`, `DefaultToApiJsonSerializer<LoanScheduleData>`, `PortfolioCommandSourceWritePlatformService`, `LoanScheduleCalculationPlatformService`, and `ApiRequestParameterHelper`.

## Endpoint

| Method | Path | `?command=` | Description | Handler |
| --- | --- | --- | --- | --- |
| `POST` | `/v1/loans/{loanId}/schedule` | `calculateLoanSchedule` | Recompute and return a `LoanScheduleData` previewing the supplied exceptions; nothing is persisted. | `LoanScheduleCalculationPlatformService.generateLoanScheduleForVariableInstallmentRequest(loanId, body)` |
| `POST` | `/v1/loans/{loanId}/schedule` | `addVariations` | Persist a set of term variations against the loan. | `CommandWrapperBuilder.createScheduleExceptions(loanId)` |
| `POST` | `/v1/loans/{loanId}/schedule` | `deleteVariations` | Remove **all** existing term variations. | `CommandWrapperBuilder.deleteScheduleExceptions(loanId)` |

If `?command=` is omitted (or unrecognised) the handler quietly falls through and `commandRequest` remains `null`, causing a `NullPointerException` downstream — callers must supply one of the three values.

## Request shape

`POST /v1/loans/{loanId}/schedule?command=addVariations` (also used for `calculateLoanSchedule`). The body type is `LoanScheduleApiResourceSwagger.PostLoansLoanIdScheduleRequest`:

```json
{
  "locale": "en",
  "dateFormat": "dd MMMM yyyy",
  "exceptions": {
    "modifiedinstallments": [
      { "dueDate": "01 June 2026", "installmentAmount": 250.00 }
    ],
    "newinstallments": [
      { "dueDate": "15 June 2026", "installmentAmount": 250.00 }
    ],
    "deletedinstallments": [
      { "dueDate": "01 July 2026" }
    ]
  }
}
```

Mandatory fields: `exceptions`, `locale`, `dateFormat`.

For `deleteVariations` the body is ignored.

## Response shapes

### `calculateLoanSchedule`

Returns the recomputed `LoanScheduleData` (the same DTO returned by `GET /v1/loans/{loanId}?associations=repaymentSchedule`):

```json
{
  "currency": { "code": "USD" },
  "loanTermInDays": 365,
  "totalPrincipalDisbursed": 5000.00,
  "totalPrincipalExpected": 5000.00,
  "totalInterestCharged": 425.00,
  "periods": [
    { "period": 1, "dueDate": "2026-05-01", "principalDue": 200, "interestDue": 25, "totalDueForPeriod": 225 },
    { "period": 2, "dueDate": "2026-06-01", "principalDue": 250, "interestDue": 20, "totalDueForPeriod": 270 }
  ]
}
```

### `addVariations` / `deleteVariations`

A standard `CommandProcessingResult`:

```json
{ "officeId": 1, "clientId": 42, "loanId": 123, "resourceId": 123, "changes": { } }
```

## Permissions

- `calculateLoanSchedule` explicitly calls `validateHasReadPermission("LOAN")`.
- `addVariations` and `deleteVariations` route through `PortfolioCommandSourceWritePlatformService.logCommandSource(...)`, which maps the builder action codes to `CREATE_SCHEDULEEXCEPTIONS` and `DELETE_SCHEDULEEXCEPTIONS` permission codes respectively.

## Behaviour notes

- The endpoint is only meaningful for loans whose `transactionProcessingStrategyCode` allows variable instalments and whose status is still `Submitted and pending approval`.
- For full reschedule of an *approved or active* loan, use the [Reschedule Loans](/api/reschedule-loans) resource instead — that one creates a `LoanRescheduleRequest` workflow.
