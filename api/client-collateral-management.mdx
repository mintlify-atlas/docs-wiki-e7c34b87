`ClientCollateralManagementApiResource` exposes the **per-client inventory** layer of Apache Fineract's modern collateral subsystem. Each row in `m_client_collateral_management` links a `Client` to a `CollateralManagement` product template with a `quantity` owned. A loan can later reserve part of that quantity through `LoanCollateralManagement` rows ([see the loan-side resource](/api/loan-collateral-management)).

## Source

- **File:** `fineract-provider/src/main/java/org/apache/fineract/portfolio/collateralmanagement/api/ClientCollateralManagementApiResource.java`
- **Class path annotation:** `@Path("/v1/clients/{clientId}/collaterals")`
- **OpenAPI tag:** `Client Collateral Management`
- **Spring stereotype:** `@Component`

Dependencies (constructor injected):

- `DefaultToApiJsonSerializer<ClientCollateralManagementData>` — for serialising request DTOs into the `CommandWrapper.json` body.
- `DefaultToApiJsonSerializer<LoanCollateralTemplateData>` — for the `template` endpoint.
- `PortfolioCommandSourceWritePlatformService` — command bus.
- `PlatformSecurityContext` — read-permission gate.
- `ClientCollateralManagementReadPlatformService` — read store.

## Endpoints

| Method | Path | Description | Command / Handler | Permission |
| ------ | ---- | ----------- | ----------------- | ---------- |
| GET | `/v1/clients/{clientId}/collaterals` | List a client's collateral inventory; optional `prodId` filter. | `clientCollateralManagementReadPlatformService.getClientCollaterals(clientId, prodId)` | Read permission `CLIENT_COLLATERAL_PRODUCT` |
| GET | `/v1/clients/{clientId}/collaterals/{clientCollateralId}` | Retrieve a single inventory row. | `clientCollateralManagementReadPlatformService.getClientCollateralManagementData(collateralId)` | Read permission `CLIENT_COLLATERAL_PRODUCT` |
| GET | `/v1/clients/{clientId}/collaterals/template` | Return collateral products plus current per-client inventory for a UI loan-application screen. | `clientCollateralManagementReadPlatformService.getLoanCollateralTemplate(clientId)` | Authenticated |
| POST | `/v1/clients/{clientId}/collaterals` | Add a new collateral item to the client's inventory. | `CommandWrapperBuilder.addClientCollateralProduct(clientId)` → `CREATE_CLIENT_COLLATERAL_PRODUCT` | `CREATE_CLIENT_COLLATERAL_PRODUCT` (maker-checker) |
| PUT | `/v1/clients/{clientId}/collaterals/{collateralId}` | Update the inventory row (e.g. quantity owned). | `CommandWrapperBuilder.updateClientCollateralProduct(clientId, collateralId)` → `UPDATE_CLIENT_COLLATERAL_PRODUCT` | `UPDATE_CLIENT_COLLATERAL_PRODUCT` (maker-checker) |
| DELETE | `/v1/clients/{clientId}/collaterals/{collateralId}` | Remove an inventory row. | `CommandWrapperBuilder.deleteClientCollateralProduct(collateralId, clientId)` → `DELETE_CLIENT_COLLATERAL_PRODUCT` | `DELETE_CLIENT_COLLATERAL_PRODUCT` (maker-checker) |

Permission strings come from `CollateralManagementJsonInputParams.CLIENT_COLLATERAL_PRODUCT_READ_PERMISSION`.

## Request / response examples

### List a client's inventory

`GET /v1/clients/55/collaterals?prodId=17`

```json
[
  {
    "id": 7,
    "name": "Gold Bar 1oz",
    "quantity": 5.000000,
    "total": 10000.00,
    "totalCollateral": 8000.00,
    "clientId": 55,
    "collateralId": 17
  }
]
```

### Add a collateral item

`POST /v1/clients/55/collaterals`

```json
{
  "collateralId": 17,
  "quantity": "5",
  "locale": "en"
}
```

Handler logic:

```java
new CommandWrapperBuilder()
    .addClientCollateralProduct(clientId)
    .withJson(apiJsonSerializerDataService.serialize(clientCollateralRequest))
    .build();
```

Response:

```json
{
  "officeId": null,
  "clientId": 55,
  "resourceId": 7,
  "changes": {}
}
```

### Template (for loan application UI)

`GET /v1/clients/55/collaterals/template`

```json
[
  {
    "id": 17,
    "name": "Gold Bar 1oz",
    "basePrice": 2000.00,
    "pctToBase": 80.00,
    "quantity": 5.000000,
    "total": 10000.00,
    "totalCollateral": 8000.00,
    "available": 3.000000
  }
]
```

This shape (`LoanCollateralTemplateData`) is the input the front-end uses on the loan-application screen — note `available = quantity - quantityFreedFromOtherLoans` is included.

### Update quantity

`PUT /v1/clients/55/collaterals/7`

```json
{
  "quantity": "6",
  "locale": "en"
}
```

### Delete

`DELETE /v1/clients/55/collaterals/7`

```json
{
  "clientId": 55,
  "resourceId": 7,
  "changes": {}
}
```

The delete handler will fail if any `m_loan_collateral_management` row still references the inventory row (foreign-key + business validation in `ClientCollateralManagementWritePlatformServiceImpl#deleteClientCollateral`).

## Data carriers

- **Request DTOs:** `ClientCollateralRequest` (POST), `UpdateClientCollateralRequest` (PUT).
- **Read responses:** `ClientCollateralManagementData`, `LoanCollateralTemplateData`.
- **Write responses:** `CommandProcessingResult` containing `clientId` and `resourceId`. Swagger types live in `ClientCollateralManagementApiResourceSwagger` (`PostClientCollateralResponse`, `PutClientCollateralResponse`, `DeleteClientCollateralResponse`).

## Permissions

All `GET` handlers (except `template`) call:

```java
this.context.authenticatedUser()
    .validateHasReadPermission(
        CollateralManagementJsonInputParams.CLIENT_COLLATERAL_PRODUCT_READ_PERMISSION.getValue());
```

Write handlers delegate authorization to `PortfolioCommandSourceWritePlatformService.logCommandSource`. Maker-checker is honoured.
