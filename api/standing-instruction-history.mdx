The Standing Instructions History API surfaces the run log produced by the Apache Fineract scheduler job that executes due standing instructions. Each row captures the parent instruction, the executed amount, the run date, the resulting account-transfer id (if successful), and an error message when the run failed. The endpoint is read-only and paged.

## Source

| Aspect | Value |
| --- | --- |
| Resource class | `org.apache.fineract.portfolio.account.api.StandingInstructionHistoryApiResource` |
| File | `fineract-provider/src/main/java/org/apache/fineract/portfolio/account/api/StandingInstructionHistoryApiResource.java` |
| JAX-RS `@Path` | `/v1/standinginstructionrunhistory` |
| Swagger tag | `Standing Instructions History` |
| Permission code | `StandingInstructionApiConstants.STANDING_INSTRUCTION_RESOURCE_NAME` (`STANDINGINSTRUCTION`) |
| Read service | `StandingInstructionHistoryReadPlatformService` |
| SQL guard | `SqlValidator` on `orderBy`, `sortOrder`, `externalId` |

## Endpoints

| Method | Path | Description | Read handler | Permission |
| --- | --- | --- | --- | --- |
| `GET` | `/v1/standinginstructionrunhistory` | Paginated run history with rich filtering. | `StandingInstructionHistoryReadPlatformService.retrieveAll(StandingInstructionDTO)` | `READ_STANDINGINSTRUCTION` |

## Query parameters

| Parameter | Type | Notes |
| --- | --- | --- |
| `externalId` | string | Match the parent instruction by external id. |
| `offset`, `limit` | int | Pagination. |
| `orderBy`, `sortOrder` | string | Validated via `SqlValidator`. |
| `transferType` | int | `1` Account Transfer / `2` Dues amount transfer. |
| `clientName`, `clientId` | string / long | Payer scope. |
| `fromAccountId`, `fromAccountType` | long / int | Payer account scope (`PortfolioAccountType`). |
| `locale`, `dateFormat` | string | Provide both when supplying `fromDate` / `toDate`. |
| `fromDate`, `toDate` | DateParam | Execution-date range; parsed via the supplied `dateFormat`/`locale`. |

The handler combines these into a `SearchParameters` object plus a `StandingInstructionDTO` and forwards them to the read service.

## SQL safety

`orderBy`, `sortOrder`, and `externalId` are individually validated by `SqlValidator.validate(...)` before being baked into the underlying JDBC query.
