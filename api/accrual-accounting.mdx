The Periodic Accrual Accounting API exposes a single trigger that Apache Fineract uses to accrue interest, fees, and penalties on active loans up to a caller-specified date (or the latest business date when omitted). The same handler runs from the scheduler job `Add Periodic Accrual Transactions`; the REST endpoint is the manual replay surface.

## Source

| Aspect | Value |
| --- | --- |
| Resource class | `org.apache.fineract.accounting.accrual.api.AccrualAccountingApiResource` |
| File | `fineract-accounting/src/main/java/org/apache/fineract/accounting/accrual/api/AccrualAccountingApiResource.java` |
| JAX-RS `@Path` | `/v1/runaccruals` |
| Swagger tag | `Periodic Accrual Accounting` |
| Read service | None — the resource only triggers the command pipeline |
| Builder | `CommandWrapperBuilder.excuteAccrualAccounting()` (note the original spelling) |

## Endpoints

| Method | Path | Description | Command handler | Permission |
| --- | --- | --- | --- | --- |
| `POST` | `/v1/runaccruals` | Run periodic accrual postings up to `tillDate`. | `CommandWrapperBuilder.excuteAccrualAccounting()` → `EXECUTE_PERIODICACCRUALACCOUNTING` | `EXECUTE_PERIODICACCRUALACCOUNTING` |

The handler is dispatched through `PortfolioCommandSourceWritePlatformService.logCommandSource` so that the call participates in maker-checker just like any other command.

## Request body

The payload is deserialised into `AccrualAccountRequest`:

```json
{
  "tillDate": "01 March 2024",
  "locale": "en",
  "dateFormat": "dd MMMM yyyy"
}
```

If `tillDate` is omitted the underlying job uses `DateUtils.getBusinessLocalDate()`.

## Response

A `CommandProcessingResult`:

```json
{
  "commandId": 1024,
  "resourceId": 0,
  "changes": {}
}
```

## Behaviour

For every loan whose product has periodic accrual accounting enabled, the platform writes journal entries for:

- Interest accrued from the last recorded accrual date to `tillDate`.
- Fees and penalties due in the same window.
- Receivable counterparts on the asset side, income on the credit side, matching the product's GL mappings.
