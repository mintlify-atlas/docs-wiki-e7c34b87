`ClientAddressApiResource` is the address book for a client record. It allows creating multiple addresses per client, each tagged with an address type (home, work, postal, â€¦), and listing or updating them by type. Note the **singular** `/v1/client` prefix â€” historically distinct from the plural `/v1/clients` used by the other client APIs.

## Permissions

Resource string: `ADDRESS` (`RESOURCE_NAME_FOR_PERMISSIONS`). Reads use `validateHasReadPermission("ADDRESS")`. Writes are gated by `CREATE_ADDRESS`, `UPDATE_ADDRESS`.

## Endpoint inventory

| HTTP | Path | Description | Command / Read service |
| --- | --- | --- | --- |
| `GET` | `/v1/client/addresses/template` | Country / state / address-type dropdowns | `readPlatformService.retrieveTemplate()` |
| `GET` | `/v1/client/{clientid}/addresses` | List addresses for a client, optionally filtered by type and status | `readPlatformService.retrieveAddressbyTypeAndStatus(...)` or `retrieveAddressbyType(...)` or `retrieveAddressbyClientId(clientid)` |
| `POST` | `/v1/client/{clientid}/addresses?type={addressTypeId}` | Add a new address of the given type | `addClientAddress(clientid, addressTypeId)` |
| `PUT` | `/v1/client/{clientid}/addresses` | Update an existing address | `updateClientAddress(clientid)` |

## Source excerpt â€” create

```java
@POST
@Path("/{clientid}/addresses")
@Consumes(MediaType.APPLICATION_JSON)
@Produces(MediaType.APPLICATION_JSON)
public CommandProcessingResult createClientAddress(
        @QueryParam("type") final long addressTypeId,
        @PathParam("clientid") final long clientid,
        final String apiRequestBodyAsJson) {
    final CommandWrapper commandRequest = new CommandWrapperBuilder()
        .addClientAddress(clientid, addressTypeId)
        .withJson(apiRequestBodyAsJson).build();
    return commandsSourceWritePlatformService.logCommandSource(commandRequest);
}
```

The `addressTypeId` is **a query parameter, not a path segment** â€” sources lookup the value in `Address Type` code-value definitions.

## Source excerpt â€” list with filters

```java
@GET
@Path("/{clientid}/addresses")
public List<AddressData> getAddresses(
        @QueryParam("status") final String status,
        @QueryParam("type") final Long typeId,
        @PathParam("clientid") final long clientid) {
    context.authenticatedUser().validateHasReadPermission(RESOURCE_NAME_FOR_PERMISSIONS);
    if (status != null && typeId != null) {
        return readPlatformService.retrieveAddressbyTypeAndStatus(clientid, typeId, status);
    }
    if (typeId != null) {
        return readPlatformService.retrieveAddressbyType(clientid, typeId);
    }
    return readPlatformService.retrieveAddressbyClientId(clientid);
}
```

## Path quirk

This is the **only** client-scoped resource that uses the singular `/v1/client` prefix; everything else uses `/v1/clients`. When generating SDKs from the OpenAPI document make sure that the address client class is registered under the `ClientAddress` tag (it does not share a router with `Clients`).
