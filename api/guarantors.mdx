The Guarantors API attaches personal or external guarantors — and optional savings-account guarantee funding — to an Apache Fineract loan. `GuarantorsApiResource` exposes CRUD endpoints plus the templates required to populate UI dropdowns and an Excel-based bulk-import flow.

## Request / response shapes

### Create

`POST /v1/loans/{loanId}/guarantors`. Body type `GuarantorsRequest`:

```json
{
  "guarantorTypeId": 2,
  "clientRelationshipTypeId": 12,
  "firstname": "Jane",
  "lastname": "Doe",
  "dob": "01 January 1990",
  "addressLine1": "10 High Street",
  "city": "Springfield",
  "country": "USA",
  "mobileNumber": "+1 555 123 4567",
  "savingsId": 88,
  "amount": 1000.00,
  "locale": "en",
  "dateFormat": "dd MMMM yyyy"
}
```

`guarantorTypeId` is one of the enum codes returned by the template:

- `1` — `Customer` (an existing client used as a guarantor)
- `2` — `External` (free-form details)
- `3` — `Staff`

When `savingsId` is supplied, an associated `GuarantorFundingDetails` is created and the amount is held on that savings account.

### Update

`PUT /v1/loans/{loanId}/guarantors/{guarantorId}` — same envelope; usually used to update demographics or the funded amount.

### Delete with funding

`DELETE /v1/loans/{loanId}/guarantors/{guarantorId}?guarantorFundingId=14` removes only the named funding entry, leaving the guarantor itself if it has other fundings.

### Standard write response

```json
{
  "officeId": 1,
  "loanId": 123,
  "resourceId": 9,
  "changes": { "amount": 1000.00 }
}
```

## Account-linking template

`GET /v1/loans/{loanId}/guarantors/accounts/template?clientId={clientId}` is used to drive the savings-account dropdown when posting a guarantee. The handler returns `null` for `accountLinkingOptions` if the loan product does not require guarantees (`LoanReadPlatformService.isGuaranteeRequired(loanId)` is false), so the UI knows to hide the section.

## Permissions

All endpoints use the `GUARANTOR` resource. Read endpoints invoke `validateHasReadPermission("GUARANTOR")` directly; writes go through `PortfolioCommandSourceWritePlatformService.logCommandSource(...)` which maps `createGuarantor`/`updateGuarantor`/`deleteGuarantor` to `CREATE_GUARANTOR`, `UPDATE_GUARANTOR`, `DELETE_GUARANTOR`.
