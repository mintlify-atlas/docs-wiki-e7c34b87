Buy-down fees are a progressive-loan capability that let a counter-party prepay a portion of the interest to "buy down" the effective rate. The `LoanBuyDownFeeApiResource` is a read-only endpoint set that exposes the amortization schedule of each buy-down-fee entry on a loan, plus the per-transaction allocation data that the GL needs to recognise income.

## Source

| Aspect | Value |
| --- | --- |
| Resource class | `org.apache.fineract.portfolio.loanaccount.api.LoanBuyDownFeeApiResource` |
| File | `fineract-progressive-loan/src/main/java/org/apache/fineract/portfolio/loanaccount/api/LoanBuyDownFeeApiResource.java` |
| JAX-RS `@Path` | `/v1/loans` (buy-down sub-paths under `{loanId}/buydown-fees` or `external-id/{loanExternalId}/buydown-fees`) |
| Swagger tag | `Loan Buy Down Fees` |
| Permission resource | `LOAN` |
| Read services | `BuyDownFeeReadPlatformService`, `LoanAmortizationAllocationService`, `LoanReadPlatformService` |

The constructor injects `PlatformSecurityContext`, `BuyDownFeeReadPlatformService`, `LoanAmortizationAllocationService`, `DefaultToApiJsonSerializer<LoanAmortizationAllocationData>`, `ApiRequestParameterHelper` and `LoanReadPlatformService`.

The whitelisted response data set is `loanId`, `loanExternalId`, `baseLoanTransactionId`, `baseLoanTransactionDate`, `baseLoanTransactionAmount`, `unrecognizedAmount`, `chargedOffAmount`, `adjustmentAmount`, `amortizationMappings`.

## Endpoints

### Amortization details for a loan

| Method | Path | Description |
| --- | --- | --- |
| `GET` | `/v1/loans/{loanId}/buydown-fees` | List every `BuyDownFeeAmortizationDetails` recorded for the loan. |
| `GET` | `/v1/loans/external-id/{loanExternalId}/buydown-fees` | External-id variant. |

Both return `List<BuyDownFeeAmortizationDetails>` via `BuyDownFeeReadPlatformService.retrieveLoanBuyDownFeeAmortizationDetails(loanId)`. External ids are resolved through `LoanReadPlatformService.getResolvedLoanId(externalId)` first.

### Per-transaction allocation data

| Method | Path | Description |
| --- | --- | --- |
| `GET` | `/v1/loans/{loanId}/buydown-fees/{loanTransactionId}` | Allocation data for one buy-down-fee transaction. |
| `GET` | `/v1/loans/external-id/{loanExternalId}/buydown-fees/{loanTransactionId}` | Loan external id + transaction id. |
| `GET` | `/v1/loans/{loanId}/buydown-fees/external-id/{loanTransactionExternalId}` | Loan id + transaction external id. |
| `GET` | `/v1/loans/external-id/{loanExternalId}/buydown-fees/external-id/{loanTransactionExternalId}` | Both external ids. |

All four call the shared `retrieveBuyDownFeesAllocationData(...)` private which:

1. Validates `LOAN` read permission.
2. Resolves both loan id and transaction id (`LoanReadPlatformService.getResolvedLoanId(...)`, `getResolvedLoanTransactionId(...)`).
3. Invokes `LoanAmortizationAllocationService.retrieveLoanAmortizationAllocationsForBuyDownFeeTransaction(resolvedLoanTransactionId, resolvedLoanId)`.
4. Honours `?fields=` query filtering via `ApiRequestParameterHelper`.

## Response shapes

### List

`GET /v1/loans/{loanId}/buydown-fees` returns `List<BuyDownFeeAmortizationDetails>`:

```json
[
  {
    "loanId": 123,
    "transactionId": 444,
    "transactionDate": "2026-04-01",
    "feeAmount": 250.00,
    "unrecognizedAmount": 100.00,
    "chargedOffAmount": 0.00,
    "adjustmentAmount": 0.00,
    "amortizationMappings": [
      { "installment": 1, "amount": 50.00, "recognitionDate": "2026-05-01" },
      { "installment": 2, "amount": 50.00, "recognitionDate": "2026-06-01" }
    ]
  }
]
```

### Allocation data

`GET /v1/loans/{loanId}/buydown-fees/{loanTransactionId}` returns a `LoanAmortizationAllocationData`:

```json
{
  "loanId": 123,
  "loanExternalId": "loan-001",
  "baseLoanTransactionId": 444,
  "baseLoanTransactionDate": "2026-04-01",
  "baseLoanTransactionAmount": 250.00,
  "unrecognizedAmount": 100.00,
  "chargedOffAmount": 0.00,
  "adjustmentAmount": 0.00,
  "amortizationMappings": [
    { "installment": 1, "amount": 50.00, "recognitionDate": "2026-05-01" },
    { "installment": 2, "amount": 50.00, "recognitionDate": "2026-06-01" }
  ]
}
```

Example queries documented in the Swagger annotations:

```
/loans/1/buydown-fees/1
/loans/1/buydown-fees/1?fields=baseLoanTransaction,unrecognizedAmount
/loans/external-id/1/buydown-fees/1
```

## Permissions

Every handler invokes `context.authenticatedUser().validateHasReadPermission("LOAN")`. There are no write endpoints on this resource — buy-down-fee transactions themselves are created via the standard [Loan Transactions](/api/loan-transactions) `POST .../transactions?command=...` dispatcher with the buy-down-fee command (when configured on the product).
