The Fixed Deposit Account Transactions API records money movements on an Apache Fineract fixed-deposit account — the initial deposit, manual withdrawals (if the product allows them) and the standard undo/adjust corrections.

## Source

| Aspect | Value |
| --- | --- |
| Resource class | `org.apache.fineract.portfolio.savings.api.FixedDepositAccountTransactionsApiResource` |
| File | `fineract-provider/src/main/java/org/apache/fineract/portfolio/savings/api/FixedDepositAccountTransactionsApiResource.java` |
| JAX-RS `@Path` | `/v1/fixeddepositaccounts/{fixedDepositAccountId}/transactions` |
| Swagger tag | `Fixed Deposit Account Transactions` |
| Permission resource | `FIXEDDEPOSITACCOUNT` |
| Read service | `SavingsAccountReadPlatformService` (shared with savings) |

## Endpoints

| Method | Path | Operation id | Description |
| --- | --- | --- | --- |
| `GET` | `/v1/fixeddepositaccounts/{fixedDepositAccountId}/transactions/template` | `retrieveTemplateFixedDepositAccountTransaction` | Transaction template (`?command=deposit|withdrawal`). |
| `GET` | `/v1/fixeddepositaccounts/{fixedDepositAccountId}/transactions/{transactionId}` | `retrieveOneFixedDepositAccountTransaction` | Retrieve a single transaction. |
| `POST` | `/v1/fixeddepositaccounts/{fixedDepositAccountId}/transactions` | (default) | Execute a transaction by `?command=`. |
| `POST` | `/v1/fixeddepositaccounts/{fixedDepositAccountId}/transactions/{transactionId}` | (default) | Adjust an existing transaction by `?command=`. |

## Transaction commands

`POST /v1/fixeddepositaccounts/{fixedDepositAccountId}/transactions?command={cmd}` dispatched in `transaction(...)`:

| `command` | Builder |
| --- | --- |
| `deposit` | `fixedDepositAccountDeposit(fixedDepositAccountId)` |
| `withdrawal` | `fixedDepositAccountWithdrawal(fixedDepositAccountId)` |

Unknown commands raise `UnrecognizedQueryParamException`.

## Adjust commands

`POST /v1/fixeddepositaccounts/{fixedDepositAccountId}/transactions/{transactionId}?command={cmd}` dispatched in `adjustTransaction(...)`:

| `command` | Builder |
| --- | --- |
| `undo` | `undoSavingsAccountTransaction(fixedDepositAccountId, transactionId)` |
| `modify` | `adjustSavingsAccountTransaction(fixedDepositAccountId, transactionId)` |

The shared `undoSavingsAccountTransaction` / `adjustSavingsAccountTransaction` builders are reused — `SavingsAccount` is the parent class of `FixedDepositAccount`.

## Request shapes

### Deposit

`POST /v1/fixeddepositaccounts/{fixedDepositAccountId}/transactions?command=deposit`:

```json
{
  "transactionDate": "01 March 2026",
  "transactionAmount": 10000,
  "paymentTypeId": 1,
  "locale": "en",
  "dateFormat": "dd MMMM yyyy"
}
```

### Withdrawal

`POST /v1/fixeddepositaccounts/{fixedDepositAccountId}/transactions?command=withdrawal`:

```json
{
  "transactionDate": "15 September 2026",
  "transactionAmount": 200,
  "paymentTypeId": 1,
  "locale": "en",
  "dateFormat": "dd MMMM yyyy"
}
```

### Undo / modify

`POST /v1/fixeddepositaccounts/{fixedDepositAccountId}/transactions/{transactionId}?command=undo` accepts an empty body. For `modify` provide the updated `transactionDate` / `transactionAmount` / `paymentTypeId`.

## Permissions

Read endpoints invoke `validateHasReadPermission("FIXEDDEPOSITACCOUNT")`. Writes route through `PortfolioCommandSourceWritePlatformService.logCommandSource(...)` which maps the builder action codes to `DEPOSIT_FIXEDDEPOSITACCOUNT`, `WITHDRAWAL_FIXEDDEPOSITACCOUNT`, `UNDOTRANSACTION_SAVINGSACCOUNT`, `ADJUSTTRANSACTION_SAVINGSACCOUNT` (the last two share the savings codes because the implementation is shared).
