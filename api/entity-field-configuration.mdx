`EntityFieldConfigurationApiResource` is a thin read-only JAX-RS resource that exposes Apache Fineract's **field configuration** table. Each row in `m_field_configuration` belongs to an *entity* (in practice almost always `ADDRESS`, but the API is generic) and records:

- `field_name`
- whether the field is enabled
- whether it is mandatory
- an optional regex used by the UI for client-side validation

The Mifos web UI calls this endpoint to know whether to render an *Address Line 1* / *State* / *Country* field, mark it as required, etc.

## Source

- **File:** `fineract-provider/src/main/java/org/apache/fineract/portfolio/address/api/EntityFieldConfigurationApiResource.java`
- **Class path annotation:** `@Path("/v1/fieldconfiguration/{entity}")`
- **OpenAPI tag:** `Entity Field Configuration`
- **Spring stereotype:** `@Component`

Constructor-injected dependencies:

- `PlatformSecurityContext context`
- `FieldConfigurationReadPlatformService readPlatformServicefld`

Static configuration:

```java
private static final String RESOURCE_NAME_FOR_PERMISSIONS = "Address";
```

## Endpoints

| Method | Path | Description | Command / Handler | Permission |
| ------ | ---- | ----------- | ----------------- | ---------- |
| GET | `/v1/fieldconfiguration/{entity}` | Retrieve every field-configuration row for an entity. | `readPlatformServicefld.retrieveFieldConfiguration(entityname)` | Read permission `Address` |

There are no write endpoints — field configuration is seeded by Liquibase changelogs (`m_field_configuration` migrations).

## Code

```java
@GET
@Consumes({ MediaType.APPLICATION_JSON })
@Produces({ MediaType.APPLICATION_JSON })
public List<FieldConfigurationData> getAddresses(
        @PathParam("entity") final String entityname) {
    this.context.authenticatedUser()
        .validateHasReadPermission(RESOURCE_NAME_FOR_PERMISSIONS);
    return this.readPlatformServicefld.retrieveFieldConfiguration(entityname);
}
```

The handler method name (`getAddresses`) is historic — the resource was extracted from the address subsystem and never renamed.

## Request / response example

`GET /v1/fieldconfiguration/ADDRESS`

```json
[
  {
    "fieldConfigurationId": 1,
    "entity": "ADDRESS",
    "subentity": "country",
    "field": "country",
    "is_enabled": true,
    "is_mandatory": true,
    "validation_regex": null
  },
  {
    "fieldConfigurationId": 2,
    "entity": "ADDRESS",
    "subentity": "state",
    "field": "state",
    "is_enabled": true,
    "is_mandatory": false,
    "validation_regex": null
  },
  {
    "fieldConfigurationId": 3,
    "entity": "ADDRESS",
    "subentity": "postalCode",
    "field": "postalCode",
    "is_enabled": true,
    "is_mandatory": false,
    "validation_regex": "^[0-9]{4,10}$"
  }
]
```

Notes:

- The `entity` path param is case-sensitive — the seed data uses `ADDRESS`. Querying `/v1/fieldconfiguration/Address` returns an empty list.
- `subentity` is used by the UI to bind the field configuration to the right form section (`country`, `state`, `street`, ...).
- `validation_regex` is intended for client-side validation; the platform does **not** apply it server-side.

## Data carriers

- **Response:** `FieldConfigurationData` — flat record with `fieldConfigurationId`, `entity`, `subentity`, `field`, `is_enabled`, `is_mandatory`, `validation_regex`.

## Permissions

```java
this.context.authenticatedUser().validateHasReadPermission(RESOURCE_NAME_FOR_PERMISSIONS);
```

resolves against the `Address` permission row. Anyone with `READ_Address` can read field configuration for **any** entity name (the path param is not validated against an enum).

## Where it fits

- **Producer:** Liquibase scripts under `fineract-provider/src/main/resources/db/changelog/tenant/parts/*.xml` (search for `m_field_configuration`).
- **Consumer:** Mifos UI fetches it once at boot and uses it to lay out client-address forms.
- **Sibling API:** [Client addresses](/portfolio/client-address) writes against the same entity model.
