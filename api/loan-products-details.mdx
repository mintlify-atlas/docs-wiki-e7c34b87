The Loan Products Details API is the lightweight read counterpart to the full [Loan Products](/api/loan-products) resource. It exposes a single endpoint that lists every loan product as `LoanProductBasicDetailsData`, intended for dropdowns and UI selection lists where the full product DTO would be wasteful.

## Source

| Aspect | Value |
| --- | --- |
| Resource class | `org.apache.fineract.portfolio.loanproduct.api.LoanProductsDetailsApiResource` |
| File | `fineract-loan/src/main/java/org/apache/fineract/portfolio/loanproduct/api/LoanProductsDetailsApiResource.java` |
| JAX-RS `@Path` | `/v1/loanproducts` |
| Swagger tag | `Loan Products Details` |
| Permission resource | `LOANPRODUCT` |
| Read service | `List<LoanProductReadBasicDetailsService>` (Spring collects every registered implementation) |

The constructor injects `PlatformSecurityContext` and a `List<LoanProductReadBasicDetailsService>`. By taking a list, the resource is extensible — any module (progressive loans, working-capital loans, custom products) that ships a `LoanProductReadBasicDetailsService` bean automatically contributes its products to the returned collection.

## Endpoint

| Method | Path | Description | Handler | Permission |
| --- | --- | --- | --- | --- |
| `GET` | `/v1/loanproducts/basic-details` | List every loan product as a minimal `LoanProductBasicDetailsData`. | Iterates every `LoanProductReadBasicDetailsService.retrieveProducts()` | `READ_LOANPRODUCT` |

The handler is:

```java
public Collection<LoanProductBasicDetailsData> fetchProducts(@Context final UriInfo uriInfo) {
    this.context.authenticatedUser().validateHasReadPermission(RESOURCE_NAME_FOR_PERMISSIONS);

    Collection<LoanProductBasicDetailsData> products = new ArrayList<>();
    loanProductReadBasicDetailsServices.forEach(service -> products.addAll(service.retrieveProducts()));
    return products;
}
```

## Response shape

Each entry is a `LoanProductBasicDetailsData`:

```json
[
  { "id": 1, "name": "Standard cumulative loan", "shortName": "STD", "fundId": 1, "fundName": "Internal" },
  { "id": 2, "name": "Progressive loan",         "shortName": "PROG", "fundId": null, "fundName": null }
]
```

The exact fields depend on the registered services — at minimum every entry exposes `id`, `name`, and `shortName`. Modules that need additional context (for example progressive loans) may surface extra attributes by returning a subclass.

## When to use which endpoint

| Use case | Endpoint |
| --- | --- |
| Populate a product picker dropdown in UI | `GET /v1/loanproducts/basic-details` |
| Show full product configuration (accounting rules, charges, …) | `GET /v1/loanproducts/{productId}` (see [Loan Products](/api/loan-products)) |
| List products for a given currency or status | `GET /v1/loanproducts` and filter client-side |

## Permissions

Only the `READ_LOANPRODUCT` permission is required. The resource performs the check directly via `PlatformSecurityContext.authenticatedUser().validateHasReadPermission("LOANPRODUCT")`.
