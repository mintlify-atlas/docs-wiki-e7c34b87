`GroupsApiResource` is the write and read surface for joint-liability groups (JLG). It covers the full lifecycle (pending → active → closed), member association/dissociation, staff and role assignment, group-loan account discovery, collection-sheet generation, and inter-group client transfers.

## Permissions

Resource string: `GROUP` (`GroupingTypesApiConstants.GROUP_RESOURCE_NAME`). Reads call `validateHasReadPermission("GROUP")`. Writes go through command-pipeline permission codes: `CREATE_GROUP`, `UPDATE_GROUP`, `DELETE_GROUP`, `ACTIVATE_GROUP`, `CLOSE_GROUP`, `ASSOCIATECLIENTS_GROUP`, `DISASSOCIATECLIENTS_GROUP`, `SAVECOLLECTIONSHEET_GROUP`, etc.

## The `command=` dispatch table

`POST /v1/groups/{groupId}?command=X` (with optional `roleId` query param for role commands):

| `command=` | CommandWrapperBuilder call | Effect |
| --- | --- | --- |
| `activate` | `activateGroup(groupId)` | Pending → Active |
| `close` | `closeGroup(groupId)` | Active → Closed |
| `associateClients` | `associateClientsToGroup(groupId)` | Add clients (body has `clientMembers` array) |
| `disassociateClients` | `disassociateClientsFromGroup(groupId)` | Remove clients |
| `transferClients` | `transferClientsBetweenGroups(groupId)` | Move clients to another group |
| `assignStaff` | `assignGroupStaff(groupId)` | Bind loan officer |
| `unassignStaff` | `unassignGroupStaff(groupId)` | Remove loan officer |
| `assignRole` | `assignRole(groupId)` | Assign role to a client member |
| `unassignRole` | `unassignRole(groupId, roleId)` | Unassign role (needs `roleId` query) |
| `updateRole` | `updateRole(groupId, roleId)` | Update role (needs `roleId` query) |
| `generateCollectionSheet` | (read-only) | Produces a `JLGCollectionSheetData` snapshot; no command logged |
| `saveCollectionSheet` | `saveGroupCollectionSheet(groupId)` | Persists the edited collection sheet |

Unrecognised → `UnrecognizedQueryParamException`.

## Source excerpt — create

```java
@POST
public String create(final String apiRequestBodyAsJson) {
    final CommandWrapper commandRequest = new CommandWrapperBuilder()
        .createGroup()
        .withJson(apiRequestBodyAsJson).build();
    final CommandProcessingResult result =
        commandsSourceWritePlatformService.logCommandSource(commandRequest);
    return toApiJsonSerializer.serialize(result);
}
```

## Canonical curl

```bash
# Create a group with two existing clients
curl -k -u mifos:password \
  -H "Fineract-Platform-TenantId: default" \
  -H "Content-Type: application/json" \
  -X POST https://localhost:8443/fineract-provider/api/v1/groups \
  -d '{
    "officeId": 1,
    "name": "Anytown JLG-3",
    "externalId": "JLG-3",
    "active": true,
    "activationDate": "04 March 2024",
    "clientMembers": [ 187, 188 ],
    "locale": "en",
    "dateFormat": "dd MMMM yyyy"
  }'

# Activate the group
curl -k -u mifos:password \
  -H "Fineract-Platform-TenantId: default" \
  -H "Content-Type: application/json" \
  -X POST 'https://localhost:8443/fineract-provider/api/v1/groups/42?command=activate' \
  -d '{ "activationDate": "04 March 2024", "locale": "en", "dateFormat": "dd MMMM yyyy" }'

# Generate the collection sheet
curl -k -u mifos:password \
  -H "Fineract-Platform-TenantId: default" \
  -H "Content-Type: application/json" \
  -X POST 'https://localhost:8443/fineract-provider/api/v1/groups/42?command=generateCollectionSheet' \
  -d '{ "calendarId": 9, "transactionDate": "04 March 2024", "locale": "en", "dateFormat": "dd MMMM yyyy" }'
```
