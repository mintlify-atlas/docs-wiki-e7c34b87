`TaxGroupApiResource` composes [tax components](/api/tax-components) into named groups that can be attached to loan products, savings products, and charges. Each component membership has its own `startDate` / `endDate` so a group can express "VAT 18% from April 2024 + a 2% cess from June 2024". Once created, groups can only be **extended** — existing component memberships can only have their `endDate` adjusted; you cannot remove or retro-edit them.

## Permissions

Resource string: `TAXGROUP` (`RESOURCE_NAME_FOR_PERMISSIONS`). Reads call `validateHasReadPermission("TAXGROUP")`. Writes use `CREATE_TAXGROUP`, `UPDATE_TAXGROUP`. There is no `DELETE` — groups must persist to keep audit trails on historically taxed transactions.

## Endpoint inventory

| HTTP | Path | Description | Command / Read service |
| --- | --- | --- | --- |
| `GET` | `/v1/taxes/group` | List all groups | `readPlatformService.retrieveAllTaxGroups()` |
| `GET` | `/v1/taxes/group/template` | Tax-component dropdowns for the create form | `readPlatformService.retrieveTaxGroupTemplate()` |
| `GET` | `/v1/taxes/group/{taxGroupId}` | Fetch one group, optionally with `?template=true` | `readPlatformService.retrieveTaxGroupData(...)` or `retrieveTaxGroupWithTemplate(...)` |

| `PUT` | `/v1/taxes/group/{taxGroupId}` | Append components or close existing ones | `updateTaxGroup(taxGroupId)` |

## Source excerpt — single fetch

```java
@GET
@Path("{taxGroupId}")
public TaxGroupData retrieveTaxGroup(
        @PathParam("taxGroupId") final Long taxGroupId,
        @Context final UriInfo uriInfo) {
    context.authenticatedUser().validateHasReadPermission(RESOURCE_NAME_FOR_PERMISSIONS);
    final ApiRequestJsonSerializationSettings settings =
        apiRequestParameterHelper.process(uriInfo.getQueryParameters());
    return settings.isTemplate()
        ? readPlatformService.retrieveTaxGroupWithTemplate(taxGroupId)
        : readPlatformService.retrieveTaxGroupData(taxGroupId);
}
```

## Canonical curl

```bash
# Create a group bundling GST (id=4) and a cess (id=7)
curl -k -u mifos:password \
  -H "Fineract-Platform-TenantId: default" \
  -H "Content-Type: application/json" \
  -X POST https://localhost:8443/fineract-provider/api/v1/taxes/group \
  -d '{
    "name": "Loan Tax Bundle",
    "taxComponents": [
      { "taxComponentId": 4, "startDate": "01 April 2024" },
      { "taxComponentId": 7, "startDate": "01 June 2024" }
    ],
    "locale": "en",
    "dateFormat": "dd MMMM yyyy"
  }'

# Close the cess membership at end of year
curl -k -u mifos:password \
  -H "Fineract-Platform-TenantId: default" \
  -H "Content-Type: application/json" \
  -X PUT https://localhost:8443/fineract-provider/api/v1/taxes/group/1 \
  -d '{
    "taxComponents": [
      { "id": 2, "endDate": "31 December 2024" }
    ],
    "locale": "en",
    "dateFormat": "dd MMMM yyyy"
  }'
```

## Request body — POST

`TaxGroupRequest`:

| Field | Required | Notes |
| --- | --- | --- |
| `name` | yes | Unique among groups |
| `taxComponents` | yes | Array of `{ taxComponentId, startDate?, endDate? }` |
| `locale`, `dateFormat` | when dates present | |

`taxComponents[]` entries on POST:

| Field | Required | Notes |
| --- | --- | --- |
| `taxComponentId` | yes | FK to existing tax component |
| `startDate` | no | Defaults to today |
| `endDate` | no | Open-ended if missing |

## Request body — PUT

You can:

- Add **new** components: omit `id`, supply `taxComponentId` and optional `startDate`.
- Close **existing** components: supply `id` (the mapping id, not the component id) and `endDate`.

You **cannot**:

- Remove a mapping outright.
- Change the `startDate` of an existing mapping.
- Edit the embedded tax component's percentage (use [tax components](/api/tax-components) for that).

With `?template=true`, an extra `taxComponents` array is bundled with all components available for adding.

## How groups are consumed

Loan and savings products carry a nullable `taxGroupId`. At repayment time the [loan engine](/loan/loan-transactions) walks the group's active mappings (those where `startDate ≤ txDate ≤ endDate`), sums their percentages, applies them to the relevant principal/interest/fee components, and produces the corresponding journal entries against each component's `debitAccount` / `creditAccount`.

Charges also accept a `taxGroupId` so a single charge can carry compound tax rates.
