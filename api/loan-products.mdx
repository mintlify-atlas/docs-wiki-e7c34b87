Loan Products are the templates from which every Apache Fineract loan account is instantiated. The `LoanProductsApiResource` exposes CRUD endpoints plus the deeply nested **template** that returns every dropdown a UI needs to author or edit a `LoanProduct` — currencies, funds, charges, transaction-processing strategies, payment types, delinquency buckets, accounting rules, capitalised-income strategies, buy-down-fee strategies, and more.

## Endpoints

| Method | Path | Description | Operation id | Command handler | Permission |
| --- | --- | --- | --- | --- | --- |
| `POST` | `/v1/loanproducts` | Create a new loan product. | (default) | `CommandWrapperBuilder.createLoanProduct()` | `CREATE_LOANPRODUCT` |
| `GET` | `/v1/loanproducts` | List all loan products. | (default) | `LoanProductReadPlatformService.retrieveAllLoanProducts()` | `READ_LOANPRODUCT` |
| `GET` | `/v1/loanproducts/template` | Product-creation template (every dropdown). | `retrieveTemplateLoanProduct` | composite (see below) | `READ_LOANPRODUCT` |
| `GET` | `/v1/loanproducts/{productId}` | Retrieve a single product. | (default) | `LoanProductReadPlatformService.retrieveLoanProduct(productId)` | `READ_LOANPRODUCT` |
| `PUT` | `/v1/loanproducts/{productId}` | Update a loan product. | (default) | `CommandWrapperBuilder.updateLoanProduct(productId)` | `UPDATE_LOANPRODUCT` |
| `GET` | `/v1/loanproducts/external-id/{externalProductId}` | Retrieve by external id. | `retrieveLoanProductDetailsByExternalId` | resolves then `retrieveLoanProduct(...)` | `READ_LOANPRODUCT` |
| `PUT` | `/v1/loanproducts/external-id/{externalProductId}` | Update by external id. | `updateLoanProductByExternalId` | `CommandWrapperBuilder.updateLoanProduct(productId)` | `UPDATE_LOANPRODUCT` |

External-id lookups go through `LoanProductReadPlatformService.getResolvedLoanProductId(ExternalIdFactory.produce(...))` before invoking the standard handler.

## Template composition

`GET /v1/loanproducts/template` aggregates:

- `CurrencyReadPlatformService.retrieveAllowedCurrencies()` — currency options
- `LoanDropdownReadPlatformService` — amortisation, interest type, interest calc period, repayment-frequency, term-frequency, etc.
- `FundReadPlatformService.retrieveAllFunds()` — fund options
- `ChargeReadPlatformService.retrieveLoanProductApplicableCharges(true)` — charge options
- `LoanProductReadPlatformService.transactionProcessingStrategyOptions()` — strategy enum
- `AccountingDropdownReadPlatformService.retrieveAccountingRuleTypeOptions()` and assorted GL account dropdowns
- `CodeValueReadPlatformService.retrieveCodeValuesByCode(...)` for `LoanType`, `LoanPurpose`, etc.
- `DelinquencyReadPlatformService.retrieveAllDelinquencyBuckets()` — delinquency buckets
- `FloatingRatesReadPlatformService.retrieveLookupActive()` — floating-rate templates
- Enum dropdowns for `LoanScheduleType`, `LoanScheduleProcessingType`, `DaysInYearCustomStrategyType`, `LoanChargeOffBehaviour`, `LoanBuyDownFeeStrategy/CalculationType/IncomeType`, `LoanCapitalizedIncomeStrategy/CalculationType/Type`, `AllocationType`

## Permissions

Read endpoints invoke `validateHasReadPermission("LOANPRODUCT")` directly. Writes go through `logCommandSource` which maps the builder action codes to `CREATE_LOANPRODUCT` and `UPDATE_LOANPRODUCT` permissions.
