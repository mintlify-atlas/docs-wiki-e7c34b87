The Bulk Loans API in Apache Fineract is used by branch managers to move every active loan from one loan officer to another in a single command — for example when a staff member leaves or transfers. `BulkLoansApiResource` provides a template that lists candidate loans and a `POST` that fires off the bulk reassignment.

| Read services | `OfficeReadPlatformService`, `StaffReadService`, `BulkLoansReadPlatformService` |

The constructor injects `PlatformSecurityContext`, `StaffReadService`, `OfficeReadPlatformService`, `BulkLoansReadPlatformService`, `DefaultToApiJsonSerializer<BulkTransferLoanOfficerData>`, `ApiRequestParameterHelper` and `PortfolioCommandSourceWritePlatformService`.

The whitelisted response data set is `officeId`, `fromLoanOfficerId`, `assignmentDate`, `officeOptions`, `loanOfficerOptions`, `accountSummaryCollection`.

## Endpoints

| Method | Path | Description | Handler | Permission |
| --- | --- | --- | --- | --- |
| `GET` | `/v1/loans/loanreassignment/template` | Build the reassignment template: list of offices, loan officers in `?officeId=`, and the account summary for `?fromLoanOfficerId=`. | `BulkLoansReadPlatformService.retrieveLoanOfficerAccountSummary(loanOfficerId)` | `READ_LOAN` |
| `POST` | `/v1/loans/loanreassignment` | Execute the bulk reassignment. | `CommandWrapperBuilder.assignLoanOfficersInBulk()` | `BULKREASSIGN_LOAN` |

### `loanReassignmentTemplate(officeId, loanOfficerId)`

```java
final Collection<OfficeData> offices = officeReadPlatformService.retrieveAllOfficesForDropdown();
Collection<StaffData> loanOfficers = officeId == null ? null
        : staffReadPlatformService.retrieveAllLoanOfficersInOfficeById(officeId);
StaffAccountSummaryCollectionData summary = loanOfficerId == null ? null
        : bulkLoansReadPlatformService.retrieveLoanOfficerAccountSummary(loanOfficerId);
return BulkTransferLoanOfficerData.templateForBulk(officeId, loanOfficerId,
        LocalDate.now(DateUtils.getDateTimeZoneOfTenant()),
        offices, loanOfficers, summary);
```

### `loanReassignment(apiRequestBodyAsJson)`

```java
final CommandWrapper commandRequest = new CommandWrapperBuilder()
        .assignLoanOfficersInBulk()
        .withJson(apiRequestBodyAsJson)
        .build();
return toApiJsonSerializer.serialize(commandsSourceWritePlatformService.logCommandSource(commandRequest));
```

## Request / response shapes

### Template response

```json
{
  "officeId": 1,
  "fromLoanOfficerId": 5,
  "assignmentDate": "2026-04-01",
  "officeOptions": [ { "id": 1, "name": "Head Office" }, { "id": 2, "name": "Branch A" } ],
  "loanOfficerOptions": [
    { "id": 5, "displayName": "Alice Smith" },
    { "id": 6, "displayName": "Bob Jones" }
  ],
  "accountSummaryCollection": {
    "loanAccountSummary": [
      { "loanId": 100, "clientId": 42, "loanProductId": 1, "loanStatus": "active" },
      { "loanId": 101, "clientId": 51, "loanProductId": 1, "loanStatus": "active" }
    ],
    "groupAccountSummary": []
  }
}
```

### Bulk reassignment request

`POST /v1/loans/loanreassignment`:

```json
{
  "officeId": 1,
  "fromLoanOfficerId": 5,
  "toLoanOfficerId": 6,
  "assignmentDate": "01 April 2026",
  "loans": [100, 101],
  "groups": [],
  "locale": "en",
  "dateFormat": "dd MMMM yyyy"
}
```

The `loans` and `groups` arrays identify which accounts to transfer; `fromLoanOfficerId` may be omitted when reassigning *unassigned* accounts.

### Response

```json
{ "officeId": 1, "resourceId": null, "changes": { "fromLoanOfficerId": 5, "toLoanOfficerId": 6 } }
```

## Permissions

The template handler explicitly invokes `validateHasReadPermission("LOAN")`. The bulk POST relies on `PortfolioCommandSourceWritePlatformService.logCommandSource(...)`, which maps the `assignLoanOfficersInBulk` action code to the `BULKREASSIGN_LOAN` permission.
