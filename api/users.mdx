The Users API administers the `AppUser` records that authenticate against Apache Fineract — usernames, names, email addresses, office and staff bindings, role assignments, and password lifecycle. It also exposes the bulk-import scaffolding (Excel template + upload) used to provision many users at once.

## Source

| Aspect | Value |
| --- | --- |
| Resource class | `org.apache.fineract.useradministration.api.UsersApiResource` |
| File | `fineract-provider/src/main/java/org/apache/fineract/useradministration/api/UsersApiResource.java` |
| JAX-RS `@Path` | `/v1/users` |
| Swagger tag | `Users` |
| Read service | `AppUserReadPlatformService` |
| Write pipeline | `PortfolioCommandSourceWritePlatformService` (`CommandWrapperBuilder`) |
| Office lookup | `OfficeReadPlatformService` (for template dropdowns) |
| Bulk import | `BulkImportWorkbookPopulatorService` / `BulkImportWorkbookService` |
| Permission constant | `"USER"` |

Constructor injection uses Lombok `@RequiredArgsConstructor`. The response payload set is fixed:

```java
private static final Set<String> RESPONSE_DATA_PARAMETERS = new HashSet<>(Arrays.asList(
    "id", "officeId", "officeName", "username",
    "firstname", "lastname", "email",
    "allowedOffices", "availableRoles", "selectedRoles", "staff"));
```

## Endpoints

| Method | Path | Description | Command / handler | Permission |
| --- | --- | --- | --- | --- |
| `GET` | `/v1/users` | List all users in the requestor's data scope. | `AppUserReadPlatformService.retrieveAllUsers()` | `READ_USER` |
| `GET` | `/v1/users/{userId}` | Retrieve one user. `?template=true` adds office dropdown data. | `AppUserReadPlatformService.retrieveUser(userId)` (+ `AppUserData.template(...)`) | `READ_USER` (scoped to `userId`) |
| `GET` | `/v1/users/template` | Return a blank template for creating a new user. | `AppUserReadPlatformService.retrieveNewUserDetails()` | `READ_USER` |
| `POST` | `/v1/users` | Create a new user. | `CommandWrapperBuilder.createUser()` | `CREATE_USER` |
| `PUT` | `/v1/users/{userId}` | Update an existing user (name, roles, office, etc). | `CommandWrapperBuilder.updateUser(userId)` | `UPDATE_USER` |
| `POST` | `/v1/users/{userId}/pwd` | Change the user's password (requires `password`+`repeatPassword`). | `CommandWrapperBuilder.changeUserPassword(userId)` | `UPDATE_USER` (with self-service exception) |
| `DELETE` | `/v1/users/{userId}` | Delete a user and their role associations. | `CommandWrapperBuilder.deleteUser(userId)` | `DELETE_USER` |
| `GET` | `/v1/users/downloadtemplate` | Download an Excel template for bulk user import. | `BulkImportWorkbookPopulatorService.getTemplate(USERS, officeId, staffId, dateFormat)` | `READ_USER` |
| `POST` | `/v1/users/uploadtemplate` | Upload a filled Excel template to bulk-create users. | `BulkImportWorkbookService.importWorkbook(USERS, ...)` | `CREATE_USER` |

All write paths flow through `PortfolioCommandSourceWritePlatformService.logCommandSource(...)`, so they participate in maker-checker when the matching permission is flagged in [Permissions](/api/permissions) and produce rows in [Audits](/api/audits).

## Listing users

```bash
curl -u mifos:password \
  -H 'fineract-platform-tenantid: default' \
  'https://fineract.example.org/fineract-provider/api/v1/users?fields=id,username,email,officeName'
```

```json
[
  { "id": 1, "username": "mifos", "email": "mifos@example.org", "officeName": "Head Office" },
  { "id": 2, "username": "alice", "email": "alice@example.org", "officeName": "Branch 1" }
]
```

The `?fields=` query parameter is handled by `ApiRequestParameterHelper` — it filters the JSON serialisation to the listed properties.

## Retrieving a single user

```bash
curl -u mifos:password \
  -H 'fineract-platform-tenantid: default' \
  'https://fineract.example.org/fineract-provider/api/v1/users/2?template=true'
```

`template=true` enriches the response with `allowedOffices` (every office the requestor can assign) so the UI can render a dropdown without a second round trip.

## Creating a user

```bash
curl -X POST -u mifos:password \
  -H 'fineract-platform-tenantid: default' \
  -H 'Content-Type: application/json' \
  'https://fineract.example.org/fineract-provider/api/v1/users' \
  -d '{
        "username": "bob",
        "firstname": "Bob",
        "lastname": "Smith",
        "email": "bob@example.org",
        "officeId": 1,
        "roles": [1],
        "sendPasswordToEmail": true,
        "staffId": 4,
        "passwordNeverExpires": false,
        "isLoginRetriesEnabled": true
      }'
```

```json
{
  "officeId": 1,
  "resourceId": 17
}
```

Per the swagger description, mandatory fields are `username`, `firstname`, `lastname`, `email`, `officeId`, `roles`, `sendPasswordToEmail`. Optional fields include `staffId`, `passwordNeverExpires`, `isLoginRetriesEnabled`. Password generation is automatic — Fineract emails the generated password when `sendPasswordToEmail=true`.

## Updating a user

```bash
curl -X PUT -u mifos:password \
  -H 'fineract-platform-tenantid: default' \
  -H 'Content-Type: application/json' \
  'https://fineract.example.org/fineract-provider/api/v1/users/17' \
  -d '{
        "firstname": "Robert",
        "roles": [1, 2]
      }'
```

The handler is symmetric: `CommandWrapperBuilder.updateUser(userId)` + `withJson(...)` + `logCommandSource(...)`. Removing a role from `roles` revokes it; adding one grants it.

## Changing a password

```bash
curl -X POST -u mifos:password \
  -H 'fineract-platform-tenantid: default' \
  -H 'Content-Type: application/json' \
  'https://fineract.example.org/fineract-provider/api/v1/users/17/pwd' \
  -d '{
        "password": "newP@ssw0rd!",
        "repeatPassword": "newP@ssw0rd!"
      }'
```

The handler calls `CommandWrapperBuilder.changeUserPassword(userId)`. The downstream command compares `password` with `repeatPassword` and validates against the global password policy. Resetting your *own* password after a `PasswordResetRequiredException` from [Authentication](/api/authentication) uses this exact endpoint.

## Deleting a user

```bash
curl -X DELETE -u mifos:password \
  -H 'fineract-platform-tenantid: default' \
  'https://fineract.example.org/fineract-provider/api/v1/users/17'
```

```json
{ "officeId": 1, "resourceId": 17 }
```

User deletion cascades into the `m_appuser_role` join table; pending maker-checker commands attributed to the user remain in [Audits](/api/audits) for traceability.

## Bulk import

```bash
# 1. Download an Excel template (filterable by office, staff, locale)
curl -u mifos:password -H 'fineract-platform-tenantid: default' -OJ \
  'https://fineract.example.org/fineract-provider/api/v1/users/downloadtemplate?officeId=1&dateFormat=dd+MMMM+yyyy'

# 2. Upload the filled workbook
curl -X POST -u mifos:password \
  -H 'fineract-platform-tenantid: default' \
  -F 'file=@users.xlsx' \
  -F 'locale=en' \
  -F 'dateFormat=dd MMMM yyyy' \
  'https://fineract.example.org/fineract-provider/api/v1/users/uploadtemplate'
```

The `uploadtemplate` POST returns the `importDocumentId` of the persisted workbook so the import progress can be tracked in the documents subsystem (see [Documents](/api/documents)).

## Permissions cheat sheet

| Permission | Required for |
| --- | --- |
| `READ_USER` | All `GET` endpoints. |
| `CREATE_USER` | `POST /v1/users`, `POST /v1/users/uploadtemplate`. |
| `UPDATE_USER` | `PUT /v1/users/{userId}`, `POST /v1/users/{userId}/pwd`. |
| `DELETE_USER` | `DELETE /v1/users/{userId}`. |
| `ALL_FUNCTIONS` | Wildcard alternative. |

## Curl reference

List users:

```bash
curl -u admin:password \
  -H "Fineract-Platform-TenantId: default" \
  https://example.org/fineract-provider/api/v1/users
```

Create a user with two roles:

```bash
curl -u admin:password \
  -H "Fineract-Platform-TenantId: default" \
  -H "Content-Type: application/json" \
  -X POST https://example.org/fineract-provider/api/v1/users \
  -d '{
    "username": "alice",
    "firstname": "Alice",
    "lastname": "Adekola",
    "email": "alice@example.org",
    "officeId": 1,
    "roles": [3, 5],
    "isSelfServiceUser": false,
    "sendPasswordToEmail": false,
    "password": "Init@l-Pass1",
    "repeatPassword": "Init@l-Pass1"
  }'
```

Reset password:

```bash
curl -u admin:password \
  -H "Fineract-Platform-TenantId: default" \
  -H "Content-Type: application/json" \
  -X POST https://example.org/fineract-provider/api/v1/users/12/pwd \
  -d '{ "password": "Tr@nsient-Pass1", "repeatPassword": "Tr@nsient-Pass1" }'
```

Delete:

```bash
curl -u admin:password \
  -H "Fineract-Platform-TenantId: default" \
  -X DELETE https://example.org/fineract-provider/api/v1/users/12
```

## Error responses

| HTTP | Code | Cause |
| --- | --- | --- |
| 400 | `error.msg.user.password.does.not.match.regexp` | Password fails the policy from `m_password_validation_policy`. |
| 400 | `error.msg.user.duplicate.username` | Username already exists. |
| 400 | `error.msg.user.officeId.cannot.be.blank` | `officeId` missing on create. |
| 403 | Standard | Caller lacks `READ_USER` / `CREATE_USER` / `UPDATE_USER` / `DELETE_USER`. |
| 404 | `error.msg.user.id.invalid` | `userId` not found. |
