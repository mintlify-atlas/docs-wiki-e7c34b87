`WorkingCapitalLoanAmortizationScheduleApiResource` is a single-endpoint JAX-RS resource that returns the **projected amortization schedule** for a Working Capital Loan (WCL) account in Apache Fineract. Unlike the classic loan schedule (which combines historical and future installments), the WCL schedule is purely a *forecast* projection — each row carries expected payments, NPV values, discount factors, expected versus actual amortization, income modifications and deferred balance.

The class shares its `@Path("/v1/working-capital-loans")` with [WorkingCapitalLoanApiResource](/api/working-capital-loans); JAX-RS dispatch picks this method via the `{loanId}/amortization-schedule` sub-path.

## Source

- **File:** `fineract-working-capital-loan/src/main/java/org/apache/fineract/portfolio/workingcapitalloan/api/WorkingCapitalLoanAmortizationScheduleApiResource.java`
- **Class path annotation:** `@Path("/v1/working-capital-loans")`
- **OpenAPI tag:** `Working Capital Loans` — *"Working Capital Loan operations including projected amortization schedule."*
- **Spring stereotype:** `@Component`

Constructor-injected dependency:

- `WorkingCapitalLoanAmortizationScheduleReadService readService`

## Endpoints

| Method | Path | Description | Command / Handler | Permission |
| ------ | ---- | ----------- | ----------------- | ---------- |
| GET | `/v1/working-capital-loans/{loanId}/amortization-schedule` | Retrieve the projected amortization schedule for a WCL account. | `readService.retrieveAmortizationSchedule(loanId)` | Authenticated |

The handler does not call `validateHasReadPermission` directly — authorization is enforced inside `WorkingCapitalLoanAmortizationScheduleReadServiceImpl` (which validates against `WORKINGCAPITALLOAN` read permission and the principal's office hierarchy).

A 404 is returned when either the loan or its schedule projection cannot be found.

## Code

```java
@GET
@Path("{loanId}/amortization-schedule")
@Consumes({ MediaType.APPLICATION_JSON })
@Produces({ MediaType.APPLICATION_JSON })
@Operation(summary = "Retrieve Projected Amortization Schedule", ...)
public ProjectedAmortizationScheduleData retrieveAmortizationSchedule(
        @PathParam("loanId") final Long loanId) {
    return readService.retrieveAmortizationSchedule(loanId);
}
```

`GET /v1/working-capital-loans/1001/amortization-schedule`

```json

      "dueDate": [2025, 2, 4],
      "discountFactor": 0.99205,
      "expectedPayment": 1000.00,
      "forecastPayment": 1000.00,
      "expectedAmortization": 900.00,
      "actualAmortization": 0.00,
      "incomeModification": 0.00,
      "deferredBalance": 100.00,
      "principalBalance": 9100.00,
      "npv": 992.05
    },
    {
      "installmentNumber": 2,
      "dueDate": [2025, 3, 6],
      "discountFactor": 0.98415,
      "expectedPayment": 1000.00,
      "forecastPayment": 1000.00,
      "expectedAmortization": 910.00,
      "actualAmortization": 0.00,
      "incomeModification": 0.00,
      "deferredBalance": 90.00,
      "principalBalance": 8190.00,
      "npv": 984.15
    }
  ],
  "totals": {
    "expectedPayment": 10000.00,
    "expectedAmortization": 9100.00,
    "actualAmortization": 0.00,
    "deferredBalance": 900.00,
    "npv": 9760.50
  }
}
```

Field semantics:

- **`discountFactor`** — per-payment discount factor computed from `npvDayCount` (typically 360 or 365 day basis) and the periodPaymentRate.
- **`expectedPayment` / `forecastPayment`** — original projection vs. the latest reforecast after partial payments. They diverge after any post-disbursement action (charge, write-off).
- **`expectedAmortization` / `actualAmortization`** — projected principal reduction per period and the actually-recorded reduction so far.
- **`incomeModification`** — adjustments due to discount accruals (rebooked on each COB tick).
- **`deferredBalance`** — running deferred / un-recognised income.
- **`npv`** — discounted present value of `forecastPayment`.

## Data carriers

- **Response:** `ProjectedAmortizationScheduleData` and its nested `Payment` rows. See `org.apache.fineract.portfolio.workingcapitalloan.data` for the full DTO.

## Lifecycle

The projection is **regenerated lazily** on read by `WorkingCapitalLoanAmortizationScheduleReadServiceImpl` whenever:

1. The loan is in a state where a schedule exists (`APPROVED`, `ACTIVE`, `CLOSED_OBLIGATIONS_MET`, …); and
2. The cached `m_working_capital_loan_amortization_schedule` rows are out of date with respect to the latest loan event (disbursement, repayment, NPV-rate change).

If neither side has changed since the previous request the result is served straight from the cache.
