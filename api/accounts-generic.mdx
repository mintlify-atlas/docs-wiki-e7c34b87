The generic Accounts API is the type-parameterised facade Apache Fineract exposes under `/v1/accounts/{type}`. The naming suggests cross-product handling but the current implementation services share accounts only: every read delegates to `ShareAccountReadPlatformService` and every write builds a command keyed by the supplied `type` segment (`share`). The same `{type}` URL structure is retained so future product families can be plugged in without breaking clients.

## Endpoints

| Method | Path | Description | Command / read handler | Permission |
| --- | --- | --- | --- | --- |
| `GET` | `/v1/accounts/{type}/template?clientId=&productId=` | Share-account application template (allowed values + defaults). | `ShareAccountReadPlatformService.retrieveTemplate(clientId, productId)` | Authenticated |
| `GET` | `/v1/accounts/{type}/{accountId}` | Retrieve a single share account; `?template=true` enriches with dropdowns. | `ShareAccountReadPlatformService.retrieveOne(accountId, isTemplate)` | Authenticated |
| `GET` | `/v1/accounts/{type}?offset=&limit=` | Paginated list of share applications/accounts. | `ShareAccountReadPlatformService.retrieveAll(offset, limit)` | Authenticated |
| `POST` | `/v1/accounts/{type}` | Submit a new share application. | `CommandWrapperBuilder.createAccount(accountType)` → `CREATE_${TYPE}_ACCOUNT` | Per type |
| `POST` | `/v1/accounts/{type}/{accountId}?command=` | Apply a lifecycle command to the account. | `CommandWrapperBuilder.createAccountCommand(accountType, accountId, commandParam)` | Per command |
| `PUT` | `/v1/accounts/{type}/{accountId}` | Modify a still-pending application. | `updateAccount(accountType, accountId)` → `UPDATE_${TYPE}_ACCOUNT` | Per type |
| `GET` | `/v1/accounts/{type}/downloadtemplate` | Bulk-import Excel workbook for share accounts. | `BulkImportWorkbookPopulatorService.getTemplate(SHARE_ACCOUNTS, ...)` | Authenticated |
| `POST` | `/v1/accounts/{type}/uploadtemplate` | Multipart upload of populated share-accounts workbook. | `BulkImportWorkbookService.importWorkbook(SHARE_ACCOUNTS, ...)` | Authenticated |

## Lifecycle commands

`POST /v1/accounts/{type}/{accountId}?command={cmd}` accepts the following values for `command` (see `AccountsApiResourceSwagger.PostAccountsTypeAccountIdRequest`):

| `command` | Effect |
| --- | --- |
| `approve` | Approve a pending application. |
| `undoapproval` | Revert an approved application to pending. |
| `reject` | Reject a pending application. |
| `activate` | Activate an approved application as a live share account. |
| `close` | Close an active share account. |
| `applyadditionalshares` | Submit an additional-shares purchase request. |
| `approveadditionalshares` | Approve a pending additional-shares request. |
| `rejectadditionalshares` | Reject a pending additional-shares request. |
| `redeemshares` | Redeem some/all shares. |

```

  "dateFormat": "dd MMMM yyyy"

```

```json
{
  "officeId": 1,
  "clientId": 1,
  "resourceId": 21,
  "changes": { "status.id": 100 }
}
```

## Path parameter `type`

Currently only `share` is supported. Passing any other value still routes through `CommandWrapperBuilder.createAccount(accountType)`, which assembles a command code containing the upper-cased type — handlers exist only for share, so other values are rejected by the command-handler registry.
