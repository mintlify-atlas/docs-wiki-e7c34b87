The Loan Account Lock API is a read-only window into the `m_loan_account_locks` table that Apache Fineract's close-of-business (COB) machinery uses to serialise per-loan work. Whenever a loan partition is being processed by the `LOAN_COB` job — or has failed and been parked for retry — a row exists with the loan id, the lock owner, the COB date, and any captured error stack. Operators query this endpoint to verify which loans are still being held by the chunk-step framework before kicking off a [COB Catch-Up](/api/cob-catchup) or releasing stuck locks.

| Module | `fineract-provider` (COB subsystem) |
| JAX-RS `@Path` | `/v1/loans` |
| Swagger tag | `Loan Account Lock` |
| Read service | `LoanAccountLockService.getLockedLoanAccountByPage(page, limit)` |
| Response type | `LoanAccountLockResponseDTO` |

The class is a Lombok `@RequiredArgsConstructor` Spring `@Component` — no permission validation runs inside the handler itself, so the resource is authenticated through Fineract's global security filter chain only.

## Endpoints

| Method | Path | Description | Command / handler | Permission |
| --- | --- | --- | --- | --- |
| `GET` | `/v1/loans/locked` | Page through the loans currently locked by the COB pipeline. | `LoanAccountLockService.getLockedLoanAccountByPage(page, limit)` | Authenticated only (no `validateHasReadPermission`) |

The handler signature:

```java
@GET
@Path("locked")
@Consumes({ MediaType.APPLICATION_JSON })
@Produces({ MediaType.APPLICATION_JSON })
@Operation(summary = "List locked loan accounts", description = "Returns the locked loan IDs")
public LoanAccountLockResponseDTO retrieveLockedAccounts(@QueryParam("page") Integer pageParam,
        @QueryParam("limit") Integer limitParam) {
    int page = Objects.requireNonNullElse(pageParam, 0);
    int limit = Objects.requireNonNullElse(limitParam, 50);

    return new LoanAccountLockResponseDTO(page, limit,
            loanAccountLockService.getLockedLoanAccountByPage(page, limit));
}
```

## Pagination

| Query param | Default | Meaning |
| --- | --- | --- |
| `page` | `0` | Zero-based page index. |
| `limit` | `50` | Page size. |

Defaults are applied with `Objects.requireNonNullElse`. Both parameters are `Integer` to allow omission; supplying a negative value will surface as a Spring-Data exception inside `LoanAccountLockService`.

## Example

```bash
curl -u mifos:password \
  -H 'fineract-platform-tenantid: default' \
  'https://fineract.example.org/fineract-provider/api/v1/loans/locked?page=0&limit=25'
```

```json
{
  "page": 0,
  "limit": 25,
  "content": [
    {
      "loanId": 1042,
      "lockOwner": "LOAN_COB_CHUNK_PROCESSING",
      "lockPlacedOn": "2024-04-15T01:01:14.221Z",
      "errorMessage": null,
      "stackTrace": null
    },
    {
      "loanId": 1078,
      "lockOwner": "LOAN_COB_PARTITIONING",
      "lockPlacedOn": "2024-04-15T01:01:14.301Z",
      "errorMessage": "java.lang.IllegalStateException: ...",
      "stackTrace": "..."
    }
  ]
}
```

A `lockOwner` of `LOAN_INLINE_COB_PROCESSING` indicates an inline run started through [Inline Jobs](/api/inline-jobs); `LOAN_COB_CHUNK_PROCESSING` and `LOAN_COB_PARTITIONING` come from the regular Quartz-driven `LOAN_COB` step. A non-null `errorMessage` means the step parked the loan after a failure — typically inspected via [Internal Loan Information](/api/internal-loan-information) before deciding whether to retry through [COB Catch-Up](/api/cob-catchup).

## Lock model

Locks are persisted as `LoanAccountLock` entities (`org.apache.fineract.cob.domain.LoanAccountLock`):

| Field | Notes |
| --- | --- |
| `loanId` | Foreign key to `m_loan`. |
| `lockOwner` | Enum `LockOwner` — identifies the COB stage. |
| `lockPlacedOn` | Business date (`COB_DATE`) for which the lock was taken. |
| `error` / `stackTrace` | Populated only when the step failed during processing. |

The lock is taken in the partitioning step, held while the chunk step processes the loan, and removed at the end of the chunk transaction. Long-lived locks generally indicate a step that crashed or a JVM that exited mid-chunk.
