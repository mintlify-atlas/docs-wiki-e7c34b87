`ClientIdentifiersApiResource` exposes the CRUD surface for KYC identifier records attached to a client (passport, national id, driving licence, â€¦). Each identifier ties a client to a row in the `Customer Identifier` code (`m_code_value`) with a `documentKey` and optional status. This page enumerates every endpoint from source.

## Permissions

Reads require `validateHasReadPermission(RESOURCE_NAME_FOR_PERMISSIONS)` where the resource string is `CLIENTIDENTIFIER`. Writes are gated by `CREATE_CLIENTIDENTIFIER`, `UPDATE_CLIENTIDENTIFIER`, `DELETE_CLIENTIDENTIFIER` inside their respective command handlers.

## Request body â€” POST

| Field | Required | Notes |
| --- | --- | --- |
| `documentTypeId` | yes | Long â€” points at a `m_code_value` row under `Customer Identifier` |
| `documentKey` | yes | string, unique per `documentType` |
| `description` | no | string |
| `status` | no | string, defaults to `Active` |

## Validation & duplicate detection

Adding two identifiers with the same `documentTypeId` + `documentKey` triggers `DuplicateClientIdentifierException` (HTTP 403, code `error.msg.clientIdentifier.duplicate.documentKey`). Deleting an identifier referenced as the primary KYC document on the client succeeds â€” the foreign key is only soft.

## Source excerpt â€” list

```java
@GET
@Consumes({ MediaType.APPLICATION_JSON })
@Produces({ MediaType.APPLICATION_JSON })
@Operation(summary = "List all Identifiers for a Client",
        operationId = "retrieveAllClientIdentifiers")
public String retrieveAllClientIdentifiers(@Context final UriInfo uriInfo,
        @PathParam("clientId") @Parameter(description = "clientId") final Long clientId) {

    this.context.authenticatedUser().validateHasReadPermission(RESOURCE_NAME_FOR_PERMISSIONS);
    final Collection<ClientIdentifierData> clientIdentifiers =
            this.clientIdentifierReadPlatformService.retrieveClientIdentifiers(clientId);
    final ApiRequestJsonSerializationSettings settings =
            this.apiRequestParameterHelper.process(uriInfo.getQueryParameters());
    return this.toApiJsonSerializer.serialize(settings, clientIdentifiers,
            ClientApiCollectionConstants.CLIENT_IDENTIFIER_RESPONSE_DATA_PARAMETERS);
}
```

The list returns a `Collection<ClientIdentifierData>` directly (not a paged envelope) â€” there is typically a small number of identifiers per client.
