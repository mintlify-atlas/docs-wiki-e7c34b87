`ClientIdentifiersApiResource` exposes the CRUD surface for KYC identifier records attached to a client (passport, national id, driving licence, …). Each identifier ties a client to a row in the `Customer Identifier` code (`m_code_value`) with a `documentKey` and optional status. This page enumerates every endpoint from source.

## Permissions

Reads require `validateHasReadPermission(RESOURCE_NAME_FOR_PERMISSIONS)` where the resource string is `CLIENTIDENTIFIER`. Writes are gated by `CREATE_CLIENTIDENTIFIER`, `UPDATE_CLIENTIDENTIFIER`, `DELETE_CLIENTIDENTIFIER` inside their respective command handlers.

## Endpoint inventory

| HTTP | Path | Description | Command / Read service |
| --- | --- | --- | --- |
| `GET` | `/v1/clients/{clientId}/identifiers` | List all identifiers for the client | `clientIdentifierReadPlatformService.retrieveClientIdentifiers(clientId)` |
| `GET` | `/v1/clients/{clientId}/identifiers/template` | Code-value dropdowns for the create form | `codeValueReadPlatformService.retrieveCodeValuesByCode("Customer Identifier")` |
| `POST` | `/v1/clients/{clientId}/identifiers` | Add a new identifier | `createClientIdentifier(clientId)` → `ClientIdentifierWritePlatformService.addClientIdentifier` |
| `GET` | `/v1/clients/{clientId}/identifiers/{identifierId}` | Fetch one identifier | `clientIdentifierReadPlatformService.retrieveClientIdentifier(clientId, identifierId)` |
| `PUT` | `/v1/clients/{clientId}/identifiers/{identifierId}` | Update an identifier | `updateClientIdentifier(clientId, identifierId)` → `ClientIdentifierWritePlatformService.updateClientIdentifier` |
| `DELETE` | `/v1/clients/{clientId}/identifiers/{identifierId}` | Delete an identifier | `deleteClientIdentifier(clientId, identifierId)` → `ClientIdentifierWritePlatformService.deleteClientIdentifier` |

## Source excerpt — create

```java
@POST
@Consumes(MediaType.APPLICATION_JSON)
@Produces(MediaType.APPLICATION_JSON)
public CommandProcessingResult createClientIdentifier(
        @PathParam("clientId") final Long clientId,
        final String apiRequestBodyAsJson) {

    final CommandWrapper commandRequest = new CommandWrapperBuilder()
        .createClientIdentifier(clientId)
        .withJson(apiRequestBodyAsJson)
        .build();

    return commandsSourceWritePlatformService.logCommandSource(commandRequest);
}
```

Note that this resource returns `CommandProcessingResult` **directly** (not as a serialised String) — Jersey handles the JSON marshalling via the default `MessageBodyWriter`.

```json
    "position": 0
  },
  "documentKey": "P12345678",
  "description": "Passport - valid until 2030",
  "status": "Active"
}
```

The `template` endpoint adds an `allowedDocumentTypes` array (every active value under the `Customer Identifier` code).

## Request body — POST

| Field | Required | Notes |
| --- | --- | --- |
| `documentTypeId` | yes | Long — points at a `m_code_value` row under `Customer Identifier` |
| `documentKey` | yes | string, unique per `documentType` |
| `description` | no | string |
| `status` | no | string, defaults to `Active` |

## Validation & duplicate detection

Adding two identifiers with the same `documentTypeId` + `documentKey` triggers `DuplicateClientIdentifierException` (HTTP 403, code `error.msg.clientIdentifier.duplicate.documentKey`). Deleting an identifier referenced as the primary KYC document on the client succeeds — the foreign key is only soft.
