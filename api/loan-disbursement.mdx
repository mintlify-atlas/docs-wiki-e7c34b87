Apache Fineract supports multi-disbursement loans where the approved principal is released in scheduled tranches. The `LoanDisbursementDetailApiResource` exposes the read and write endpoints used to inspect a planned `DisbursementData` entry, adjust its expected date, or add and remove tranches on the schedule.

## Source

| Aspect | Value |
| --- | --- |
| Resource class | `org.apache.fineract.portfolio.loanaccount.api.LoanDisbursementDetailApiResource` |
| File | `fineract-provider/src/main/java/org/apache/fineract/portfolio/loanaccount/api/LoanDisbursementDetailApiResource.java` |
| JAX-RS `@Path` | `/v1/loans/{loanId}/disbursements` |
| Swagger tag | `Loan Disbursement Details` |
| Permission resource | `LOAN` |
| Read service | `LoanReadPlatformService.retrieveLoanDisbursementDetail(loanId, disbursementId)` |
| Command source | `PortfolioCommandSourceWritePlatformService` |

The constructor injects `DefaultToApiJsonSerializer<DisbursementData>`, `PortfolioCommandSourceWritePlatformService`, `PlatformSecurityContext`, `ApiRequestParameterHelper` and `LoanReadPlatformService`.

The response data set returned by GET is whitelisted to `id`, `expectedDisbursementDate`, `actualDisbursementDate`, `principal`, `approvedPrincipal`.

## Endpoints

| Method | Path | Description | Command handler | Permission |
| --- | --- | --- | --- | --- |
| `GET` | `/v1/loans/{loanId}/disbursements/{disbursementId}` | Retrieve a single disbursement tranche. Supports `?fields=`. | `LoanReadPlatformService.retrieveLoanDisbursementDetail(loanId, disbursementId)` | `READ_LOAN` |
| `PUT` | `/v1/loans/{loanId}/disbursements/{disbursementId}` | Update the expected disbursement date of one tranche. | `CommandWrapperBuilder.updateDisbusementDate(loanId, disbursementId)` | `UPDATE_DISBURSEMENTDATE_LOAN` |
| `PUT` | `/v1/loans/{loanId}/disbursements/editDisbursements` | Add new tranches and/or delete unmade tranches in one transactional call. | `CommandWrapperBuilder.addAndDeleteDisbursementDetails(loanId)` | `UPDATE_DISBURSEMENT_LOAN` |

There are no DELETE or POST endpoints — additions and deletions are batched through `editDisbursements`.

## Request shapes

### Update a single tranche date

`PUT /v1/loans/{loanId}/disbursements/{disbursementId}`:

```json
{
  "expectedDisbursementDate": "15 May 2026",
  "locale": "en",
  "dateFormat": "dd MMMM yyyy"
}
```

### Add and delete tranches

`PUT /v1/loans/{loanId}/disbursements/editDisbursements`. The request body matches `LoanDisbursementDetailApiResourceSwagger.PostAddAndDeleteDisbursementDetailRequest`:

```json
{
  "locale": "en",
  "dateFormat": "dd MMMM yyyy",
  "disbursementData": [
    { "expectedDisbursementDate": "01 June 2026", "principal": 1000 },
    { "expectedDisbursementDate": "01 July 2026", "principal": 1000 }
  ],
  "disbursementDataIdsForDelete": [3, 4]
}
```

`disbursementData` contains the new tranches to insert, `disbursementDataIdsForDelete` lists the ids of unmade tranches to remove. The server validates that referenced tranches have not been disbursed yet.

## Response shapes

### Write endpoints

Standard `CommandProcessingResult`:

```json
{
  "officeId": 1,
  "clientId": 42,
  "loanId": 123,
  "resourceId": 123,
  "changes": { "expectedDisbursementDate": "15 May 2026" }
}
```

### GET response

`DisbursementData` filtered to the whitelisted attributes:

```json
{
  "id": 7,
  "expectedDisbursementDate": "2026-05-15",
  "actualDisbursementDate": null,
  "principal": 1000.0,
  "approvedPrincipal": 1000.0
}
```

## Validation rules

- `updateDisbursementDate` requires the loan to be in `Approved` or `Active` and the specified tranche to be unmade (no `actualDisbursementDate`).
- `addAndDeleteDisbursementDetails` validates that the sum of tranche principals does not exceed the loan's `approvedPrincipal` minus already-disbursed amounts.
- All endpoints rely on `validateHasReadPermission("LOAN")` (GET) or pass through `logCommandSource` (writes), which applies the per-command permission described above.
