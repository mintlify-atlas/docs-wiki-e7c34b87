The Internal Client Information API is a **test-only** read endpoint that returns the four `Auditable` fields (`CREATED_BY`, `CREATED_DATE`, `LAST_MODIFIED_BY`, `LAST_MODIFIED_DATE`) directly from a `Client` JPA entity. Production client reads go through [Clients](/api/clients), which expose a curated DTO — this sidecar lets integration tests assert on the raw timestamps and user ids the JPA auditing layer writes.

<Warning>
This resource is guarded by `@Profile(FineractProfiles.TEST)`. It is only registered when the Spring profile `test` is active, and emits a `DO NOT USE THIS IN PRODUCTION!` banner on startup. Production deployments must omit `test` from `SPRING_PROFILES_ACTIVE`.
</Warning>

## Endpoints

| Method | Path | Description | Command / handler | Permission |
| --- | --- | --- | --- | --- |
| `GET` | `/v1/internal/client/{clientId}/audit` | Return the four auditable fields of the client. | `ClientRepositoryWrapper.findOneWithNotFoundDetection(clientId)` → `getCreatedBy/getCreatedDate/getLastModifiedBy/getLastModifiedDate` | None (relies on `test` profile guard) |

## Handler

```java
@GET
@Path("{clientId}/audit")
public String getClientAuditFields(@Context final UriInfo uriInfo, @PathParam("clientId") Long clientId) {
    final Client client = clientRepositoryWrapper.findOneWithNotFoundDetection(clientId);
    Map<String, Object> auditFields = new HashMap<>(
        Map.of(
            CREATED_BY, client.getCreatedBy().orElse(null),
            CREATED_DATE, client.getCreatedDate().orElse(null),
            LAST_MODIFIED_BY, client.getLastModifiedBy().orElse(null),
            LAST_MODIFIED_DATE, client.getLastModifiedDate().orElse(null)));
    final ApiRequestJsonSerializationSettings settings = apiRequestParameterHelper.process(uriInfo.getQueryParameters());
    return toApiJsonSerializer.serialize(settings, auditFields);
}
```

```bash
curl -u mifos:password \

```

```json
{
  "createdBy":        1,
  "createdDate":      "2024-04-12T09:14:01Z",
  "lastModifiedBy":   2,
  "lastModifiedDate": "2024-04-14T10:08:22Z"
}
```

Unknown ids raise the standard `ClientNotFoundException` (`404`).

## Why a separate test-only endpoint

The public [Clients](/api/clients) read endpoints intentionally omit raw audit fields from the response payload — they expose curated derived fields such as `activationDate` and an `OfficeData` structure. Test suites that need to assert "did the JPA auditing aspect set `lastModifiedBy` to the maker user?" need access to the underlying timestamps. Rather than expand the public DTO and tempt callers to depend on it, Fineract ships this `@Profile("test")` shortcut.

## Field semantics

| Key | Source | Notes |
| --- | --- | --- |
| `createdBy` | `client.getCreatedBy().orElse(null)` | User id from the `AbstractAuditableCustom` listener. |
| `createdDate` | `client.getCreatedDate().orElse(null)` | UTC instant the row was inserted. |
| `lastModifiedBy` | `client.getLastModifiedBy().orElse(null)` | User id of the most recent update. |
| `lastModifiedDate` | `client.getLastModifiedDate().orElse(null)` | UTC instant of the most recent update. |

The `Optional`s come from the auditing API in `AbstractAuditableCustom` — they are empty when the row predates the auditing aspect (rare, mostly relevant to migrated tenants).
