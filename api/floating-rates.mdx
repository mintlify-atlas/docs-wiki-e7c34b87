`FloatingRatesApiResource` is the JAX-RS resource that maintains Apache Fineract's **variable-rate curves**. A floating rate is a named series of `RatePeriods`, each effective from a `fromDate`, optionally flagged as the **Base Lending Rate** for the tenant. Loan products configured with `isLinkedToFloatingInterestRates=true` resolve their effective interest rate against the active floating rate at disbursement (and again at every re-pricing event).

The resource lives in the dedicated `fineract-rates` module so it can be excluded by build profile.

## Source

- **File:** `fineract-rates/src/main/java/org/apache/fineract/portfolio/floatingrates/api/FloatingRatesApiResource.java`
- **Class path annotation:** `@Path("/v1/floatingrates")`
- **OpenAPI tag:** `Floating Rates` — *"It lets you create, list, retrieve and upload the floating rates"*.
- **Spring stereotype:** `@Component`

Constructor-injected dependencies:

- `PlatformSecurityContext context`
- `PortfolioCommandSourceWritePlatformService commandsSourceWritePlatformService`
- `DefaultToApiJsonSerializer<FloatingRateData> toApiJsonSerializer`
- `FloatingRatesReadPlatformService floatingRatesReadPlatformService`

Static configuration:

```java
private static final String RESOURCE_NAME = "FLOATINGRATE";
```

## Endpoints

| Method | Path | Description | Command / Handler | Permission |
| ------ | ---- | ----------- | ----------------- | ---------- |
| POST | `/v1/floatingrates` | Create a new floating rate (mandatory `name`; optional `isBaseLendingRate`, `isActive`, `ratePeriods`). | `CommandWrapperBuilder.createFloatingRate()` → `CREATE_FLOATINGRATE` | `CREATE_FLOATINGRATE` |
| GET | `/v1/floatingrates` | List all floating rates. | `floatingRatesReadPlatformService.retrieveAll()` | Read permission `FLOATINGRATE` |
| GET | `/v1/floatingrates/{floatingRateId}` | Retrieve a single floating rate with its rate periods. | `floatingRatesReadPlatformService.retrieveOne(floatingRateId)` | Read permission `FLOATINGRATE` |
| PUT | `/v1/floatingrates/{floatingRateId}` | Update a floating rate. Past rate periods cannot be modified; future periods are replaced wholesale. | `CommandWrapperBuilder.updateFloatingRate(floatingRateId)` → `UPDATE_FLOATINGRATE` | `UPDATE_FLOATINGRATE` |

There is intentionally no `DELETE`: floating rates that are no longer needed are deactivated via `PUT { "isActive": false }` so historical loan re-pricing can still resolve them.

## Request / response examples

### Create

`POST /v1/floatingrates`

`FloatingRateRequest`:

```json
{
  "name": "BLR-USD",
  "isBaseLendingRate": true,
  "isActive": true,
  "ratePeriods": [
    { "fromDate": "01 January 2025", "interestRate": 5.25, "isDifferentialToBaseLendingRate": false, "locale": "en", "dateFormat": "dd MMMM yyyy" },
    { "fromDate": "01 April 2025", "interestRate": 5.75, "isDifferentialToBaseLendingRate": false, "locale": "en", "dateFormat": "dd MMMM yyyy" }
  ]
}
```

Handler:

```java
final CommandWrapper commandRequest = new CommandWrapperBuilder()
    .createFloatingRate()
    .withJson(toApiJsonSerializer.serialize(floatingRateRequest))
    .build();
return commandsSourceWritePlatformService.logCommandSource(commandRequest);
```

Response (`CommandProcessingResult` → `PostFloatingRatesResponse`):

```json
{ "resourceId": 1 }
```

### List

`GET /v1/floatingrates`

```json
[
  {
    "id": 1,
    "name": "BLR-USD",
    "isBaseLendingRate": true,
    "isActive": true,
    "createdOn": "2025-01-01T00:00:00",
    "modifiedOn": "2025-01-01T00:00:00"
  }
]
```

### Retrieve one

`GET /v1/floatingrates/1`

```json
{
  "id": 1,
  "name": "BLR-USD",
  "isBaseLendingRate": true,
  "isActive": true,
  "ratePeriods": [
    {
      "id": 11,
      "fromDate": [2025,1,1],
      "interestRate": 5.25,
      "isDifferentialToBaseLendingRate": false,
      "isActive": true
    },
    {
      "id": 12,
      "fromDate": [2025,4,1],
      "interestRate": 5.75,
      "isDifferentialToBaseLendingRate": false,
      "isActive": true
    }
  ]
}
```

### Update

`PUT /v1/floatingrates/1`

```json
{
  "ratePeriods": [
    { "fromDate": "01 July 2025", "interestRate": 6.00, "isDifferentialToBaseLendingRate": false, "locale": "en", "dateFormat": "dd MMMM yyyy" }
  ]
}
```

Response:

```json
{
  "resourceId": 1,
  "changes": {
    "ratePeriods": "[...]"
  }
}
```

Validation rules enforced by `FloatingRateWritePlatformServiceImpl`:

- A rate period whose `fromDate` lies **on or before** the business date cannot be modified.
- All supplied `ratePeriods` must be in the future; any in-flight future periods are deactivated and replaced.
- Only one floating rate per tenant may carry `isBaseLendingRate=true`.

## Data carriers

- **Request:** `FloatingRateRequest` (with nested rate-period list).
- **Read response:** `FloatingRateData` (`id`, `name`, `isBaseLendingRate`, `isActive`, `ratePeriods[]`, audit fields).
- **Write response:** `CommandProcessingResult` — see Swagger types `PostFloatingRatesResponse`, `PutFloatingRatesFloatingRateIdResponse`, `GetFloatingRatesResponse` in `FloatingRatesApiResourceSwagger`.

## Permissions

```java
this.context.authenticatedUser().validateHasReadPermission(RESOURCE_NAME);
```

with `RESOURCE_NAME = "FLOATINGRATE"`. Writes delegate to `PortfolioCommandSourceWritePlatformService` which enforces `CREATE_FLOATINGRATE` / `UPDATE_FLOATINGRATE`.

## How loan products consume it

- `LoanProduct` carries `floatingRateId` and a `defaultDifferentialLendingRate` when `isLinkedToFloatingInterestRates=true`.
- At repricing time, `LoanScheduleAssembler` calls `FloatingRatesReadPlatformService` for the period that contains the repricing date and adds `differentialLendingRate`.
- Toggling `isActive=false` blocks new loan products from selecting the floating rate but does not affect already-disbursed loans (they continue to resolve against historical rate periods).
