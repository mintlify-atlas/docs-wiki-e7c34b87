The Loans Point-In-Time API in Apache Fineract is a progressive-loan capability that lets callers retrieve the state of a loan **as of a specific historical date** — useful for audits, accounting roll-forwards, charge-off triggers and external reconciliations. The resource is split into a JAX-RS class plus a delegate, and exposes id, external-id, and bulk variants.

## Source

| Aspect | Value |
| --- | --- |
| Resource class | `org.apache.fineract.portfolio.loanaccount.api.pointintime.LoansPointInTimeApiResource` |
| Delegate | `org.apache.fineract.portfolio.loanaccount.api.pointintime.LoansPointInTimeApiDelegate` |
| Interface | `org.apache.fineract.portfolio.loanaccount.api.pointintime.LoansPointInTimeApi` |
| File | `fineract-provider/src/main/java/org/apache/fineract/portfolio/loanaccount/api/pointintime/LoansPointInTimeApiResource.java` |
| JAX-RS `@Path` | `/v1/loans/at-date` |
| Swagger tag | `LoansPointInTime` |
| Permission resource | `LOAN` |
| Underlying service | `LoanPointInTimeService` |

The handler class is a thin facade — every method delegates to `LoansPointInTimeApiDelegate` which in turn calls `LoanPointInTimeService` and applies external-id resolution. The constructor injects a single `LoansPointInTimeApiDelegate`.

## Endpoints

| Method | Path | Description | Delegate method |
| --- | --- | --- | --- |
| `GET` | `/v1/loans/at-date/{loanId}` | Retrieve loan state at `?date=...`. | `retrieveLoanPointInTime(loanId, dateParam, dateFormat, locale)` |
| `GET` | `/v1/loans/at-date/external-id/{loanExternalId}` | External-id variant. | `retrieveLoanPointInTimeByExternalId(...)` |
| `POST` | `/v1/loans/at-date/search` | Bulk retrieval for many loan ids at the same date. | `retrieveLoansPointInTime(RetrieveLoansPointInTimeRequest)` |
| `POST` | `/v1/loans/at-date/search/external-id` | Bulk retrieval by external ids. | `retrieveLoansPointInTimeByExternalIds(RetrieveLoansPointInTimeExternalIdsRequest)` |

### Query parameters

For the GET endpoints:

- `date` — **required**, the as-of date (`DateParam`).
- `dateFormat` — optional, e.g. `dd MMMM yyyy`. Falls back to the tenant default.
- `locale` — optional, e.g. `en`.

## Request shapes

### Bulk by id

`POST /v1/loans/at-date/search` body type `RetrieveLoansPointInTimeRequest`:

```json
{
  "loanIds": [123, 124, 125],
  "date": "01 January 2026",
  "dateFormat": "dd MMMM yyyy",
  "locale": "en"
}
```

### Bulk by external id

`POST /v1/loans/at-date/search/external-id` body type `RetrieveLoansPointInTimeExternalIdsRequest`:

```json
{
  "loanExternalIds": ["loan-001", "loan-002"],
  "date": "01 January 2026",
  "dateFormat": "dd MMMM yyyy",
  "locale": "en"
}
```

## Response shape

All four endpoints return `LoanPointInTimeData` (or `List<LoanPointInTimeData>` for the bulk variants):

```json
{
  "loanId": 123,
  "loanExternalId": "loan-001",
  "asOfDate": "2026-01-01",
  "status": { "id": 300, "code": "loanStatusType.active" },
  "currency": { "code": "USD" },
  "principalOutstanding": 4500.00,
  "interestOutstanding": 80.00,
  "feeChargesOutstanding": 0.00,
  "penaltyChargesOutstanding": 0.00,
  "totalOutstanding": 4580.00,
  "principalRepaid": 500.00,
  "interestRepaid": 50.00,
  "totalRepaid": 550.00,
  "principalWrittenOff": 0.00,
  "lastTransactionDate": "2025-12-15",
  "delinquentDays": 0
}
```

The bulk variants stream the same DTO inside a JSON array.

## Permissions

Each delegate call enforces the read permission against the `LOAN` resource. No write commands are exposed — point-in-time state is reconstructed read-only from transaction history.

## Behaviour notes

- The implementation replays the loan's transactions up to and including the requested date, applying the same transaction-processor logic as `LoanReadPlatformService.retrieveOne(loanId)`.
- Cancelled and reversed transactions whose effective dates are after the cutoff are honoured (they did not yet exist on the as-of date).
- For multi-disburse loans, the reported outstanding reflects only tranches whose `actualDisbursementDate` is on or before the as-of date.
