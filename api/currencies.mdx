`CurrenciesApiResource` manages the **organisation currency allowlist** — the set of ISO currency codes the institution is allowed to denominate accounts in. Every account (loan, savings, charge, deposit) references one of these codes, so adding a new currency to the list is a prerequisite for adding products in that currency.

Injected collaborators:

- `OrganisationCurrencyReadPlatformService readPlatformService`
- `CommandPipeline commandPipeline` — typed v2-style command pipeline

## Permissions

Required codes: `READ_CURRENCY`, `UPDATE_CURRENCY`. Checks live in the read service and `CurrencyUpdateCommandHandler`.

## Endpoint inventory

| HTTP | Path | Description | Command / Read service |
| --- | --- | --- | --- |
| `GET` | `/v1/currencies` | Permitted currencies + available pool | `readPlatformService.retrieveCurrencyConfiguration()` |
| `PUT` | `/v1/currencies` | Replace the permitted list | `CurrencyUpdateCommand` via `commandPipeline.send(...)` |

No POST or DELETE — the available pool is a hard-coded ISO 4217 catalogue and the permitted list is updated as a single set.

## Source excerpt

```java
@Path("/v1/currencies")
@Component
@Tag(name = "Currency")
@RequiredArgsConstructor
public class CurrenciesApiResource {

    private final OrganisationCurrencyReadPlatformService readPlatformService;
    private final CommandPipeline commandPipeline;

    @GET
    @Consumes(MediaType.APPLICATION_JSON)
    @Produces(MediaType.APPLICATION_JSON)
    public CurrencyConfigurationData retrieveCurrencies() {
        return readPlatformService.retrieveCurrencyConfiguration();
    }

    @PUT
    @Consumes(MediaType.APPLICATION_JSON)
    @Produces(MediaType.APPLICATION_JSON)
    public CurrencyUpdateResponse updateCurrencies(@Valid CurrencyUpdateRequest request) {
        final var command = new CurrencyUpdateCommand();
        command.setPayload(request);
        final Supplier<CurrencyUpdateResponse> response = commandPipeline.send(command);
        return response.get();
    }
}
```

## Canonical curl

```bash
# Fetch the list
curl -k -u mifos:password \
  -H "Fineract-Platform-TenantId: default" \
  https://localhost:8443/fineract-provider/api/v1/currencies

# Allow only USD and INR
curl -k -u mifos:password \
  -H "Fineract-Platform-TenantId: default" \
  -H "Content-Type: application/json" \
  -X PUT https://localhost:8443/fineract-provider/api/v1/currencies \
  -d '{ "currencies": [ "USD", "INR" ] }'
```

## Filter response

The `?fields=` parameter slims the GET response. Common projections:

- `?fields=selectedCurrencyOptions` — just the permitted list.
- `?fields=currencyOptions` — just the unpermitted pool.
- `?fields=selectedCurrencyOptions,currencyOptions` — both arrays, no envelope siblings.

## Request body — PUT

`CurrencyUpdateRequest`:

| Field | Required | Notes |
| --- | --- | --- |
| `currencies` | yes | Array of ISO 4217 codes. **Replaces** the list — not a delta operation |

## Behaviour caveats

- **Replacement semantics** — the PUT replaces the entire permitted list. To remove a currency, omit it; to add one, include both the existing entries and the new one.
- **Cannot remove in-use currencies** — if a product, loan, or savings account references USD, removing USD from the list fails with `error.msg.organisation.currency.in.use`. Close those accounts first.
- **`decimalPlaces` and `inMultiplesOf`** — these are organisation-level *overrides* layered on top of the ISO defaults; they live in `m_organisation_currency` and are mutated when an admin clicks "Save Settings" in the UI. The current REST surface only manages the membership list — to override decimal places you must `UPDATE` the underlying row.
