The External Event Configuration API toggles which Apache Fineract domain events get published to the downstream message channel (Kafka, RabbitMQ, JMS — whichever the `external-event-publisher` is wired to). Each event type — `LoanCreatedBusinessEvent`, `LoanApprovedBusinessEvent`, `SavingsAccountTransactionBusinessEvent`, and so on — has a row in `m_external_event_configuration` with an `enabled` flag. This resource exposes that catalogue and the bulk update path.

## Source

| Aspect | Value |
| --- | --- |
| Resource class | `org.apache.fineract.infrastructure.event.external.api.ExternalEventConfigurationApiResource` |
| File | `fineract-core/src/main/java/org/apache/fineract/infrastructure/event/external/api/ExternalEventConfigurationApiResource.java` |
| JAX-RS `@Path` | `/v1/externalevents/configuration` |
| Swagger tag | `External event configuration` |
| Read service | `ExternalEventConfigurationReadPlatformService` |
| Command pipeline | `CommandPipeline.send(ExternalConfigurationsUpdateCommand)` |
| Request DTO | `ExternalEventConfigurationUpdateRequest` |
| Response DTOs | `ExternalEventConfigurationResponse`, `ExternalEventConfigurationUpdateResponse` |

The class is a Spring `@Component` registered through the standard JAX-RS scanning and uses Lombok's `@RequiredArgsConstructor`.

## Endpoints

| Method | Path | Description | Command / handler | Permission |
| --- | --- | --- | --- | --- |
| `GET` | `/v1/externalevents/configuration` | Return every configured event type and its `enabled` flag. | `ExternalEventConfigurationReadPlatformService.findAllExternalEventConfigurations()` | Authenticated |
| `PUT` | `/v1/externalevents/configuration` | Enable or disable a set of event types in one call. | `CommandPipeline.send(ExternalConfigurationsUpdateCommand)` with the request as payload | Authenticated (command handler may enforce additional checks) |

There is no per-event endpoint — the PUT accepts a map and updates all rows it mentions in a single transaction.

## Listing configurations

```bash
curl -u mifos:password \
  -H 'fineract-platform-tenantid: default' \
  'https://fineract.example.org/fineract-provider/api/v1/externalevents/configuration'
```

```json
{
  "externalEventConfiguration": [
    { "type": "LoanCreatedBusinessEvent",                "enabled": true  },
    { "type": "LoanApprovedBusinessEvent",               "enabled": true  },
    { "type": "LoanStatusChangedBusinessEvent",          "enabled": false },
    { "type": "SavingsAccountTransactionBusinessEvent",  "enabled": true  }
  ]
}
```

The list is exhaustive: every event type the running Fineract version knows about appears. New events introduced by an upgrade arrive disabled by default until an explicit `PUT` toggles them on, which lets operators decide the rollout cadence.

## Bulk toggling events

```bash
curl -X PUT -u mifos:password \
  -H 'fineract-platform-tenantid: default' \
  -H 'Content-Type: application/json' \
  'https://fineract.example.org/fineract-provider/api/v1/externalevents/configuration' \
  -d '{
        "externalEventConfigurations": {
          "LoanStatusChangedBusinessEvent": true,
          "LoanCreatedBusinessEvent": true,
          "LoanCloseBusinessEvent": false
        }
      }'
```

The request body shape is `ExternalEventConfigurationUpdateRequest` — a map of event-type name to boolean. The handler:

```java
final var command = new ExternalConfigurationsUpdateCommand();
command.setPayload(request);
final Supplier<ExternalEventConfigurationUpdateResponse> response = commandPipeline.send(command);
return response.get();
```

Going through `CommandPipeline.send(...)` (the same pattern used by [Business Date](/api/business-date)) means the update is dispatched to a registered handler that validates each event type, applies the changes inside one transaction, and returns the resulting state.

```json
{
  "externalEventConfiguration": [
    { "type": "LoanCreatedBusinessEvent",       "enabled": true  },
    { "type": "LoanStatusChangedBusinessEvent", "enabled": true  },
    { "type": "LoanCloseBusinessEvent",         "enabled": false }
  ]
}
```

Event types omitted from the payload are not touched.

## Interaction with the publishing pipeline

Enabling an event type only means that domain events of that type are written to the **external event outbox** (`m_external_event`). The actual transport happens through the `EXTERNAL_EVENT_PUBLISHING` job — see [Scheduler Jobs](/api/scheduler-jobs) — or its inline counterpart at [Inline Jobs](/api/inline-jobs). To inspect what is currently sitting in the outbox during integration testing, use [Internal External Events](/api/internal-external-events) (TEST profile only).

The `EXTERNAL_EVENT_PUBLISHING_BUSINESS_STEP` referenced under [Configure Business Step](/api/configure-business-step) is the step that drains the outbox at the end of `LOAN_COB`.

## Validation behaviour

- Unknown event-type names cause the command handler to abort the entire payload — no rows are updated.
- Boolean values must be literal `true` / `false`. The Jakarta validation annotation (`@Valid`) on the resource argument rejects malformed JSON before it reaches the handler.
- Re-disabling an event type does not delete already-queued outbox rows; existing rows continue to drain until the publisher catches up.

## Curl reference

List every event configuration row (no paging):

```bash
curl -u mifos:password \
  -H "Fineract-Platform-TenantId: default" \
  https://example.org/fineract-provider/api/v1/externalevents/configuration
```

Toggle a set of events in one PUT:

```bash
curl -u mifos:password \
  -H "Fineract-Platform-TenantId: default" \
  -H "Content-Type: application/json" \
  -X PUT https://example.org/fineract-provider/api/v1/externalevents/configuration \
  -d '{
    "externalEventConfigurations": {
      "LoanDisbursalBusinessEvent": true,
      "LoanRepaymentBusinessEvent": true,
      "ClientCreateBusinessEvent": false
    }
  }'
```

## Response shape — `ExternalEventConfigurationData`

```json
{
  "externalEventConfiguration": [
    { "type": "LoanDisbursalBusinessEvent", "enabled": true },
    { "type": "LoanRepaymentBusinessEvent", "enabled": true },
    { "type": "ClientCreateBusinessEvent", "enabled": false }
  ]
}
```

The list is sorted alphabetically by event type. Each row corresponds to one column in `m_external_event_configuration`.

## Validation behaviour

- Keys not registered as `BusinessEvent` subclasses are silently ignored — the PUT does not 400 on typos.
- Setting a key to `null` or omitting it leaves the existing value untouched.
- The request is processed by `ExternalConfigurationsUpdateCommand` as a single command-source row, so the change is visible in [Audits](/api/audits) as one entry.

## Interaction with the publishing pipeline

- Enabled events are appended to `m_external_event` rows by `ExternalEventService.postEvent(...)`.
- The `EXTERNAL_EVENT_PUBLISHING_BUSINESS_STEP` step (see [Configure Business Step](/api/configure-business-step)) drains the table and forwards to the configured delivery channel.
- Disabling an event stops new rows from being inserted but does **not** purge already-queued rows; those still flow through the publisher.

## Operational notes

- All registered `BusinessEvent` types are inserted by Liquibase migrations as **disabled** by default. New deployments must enable the events they want.
- The list is cached for the duration of a JVM run — clients should always re-read after a PUT to confirm the persisted state.
- The PUT is an upsert in spirit: missing keys keep their previous value, so partial updates are safe.

## Sample state migration

```bash
# Step 1: enable loan disbursal events
curl -u admin:password -H "Fineract-Platform-TenantId: default" \
  -H "Content-Type: application/json" \
  -X PUT https://example.org/fineract-provider/api/v1/externalevents/configuration \
  -d '{ "externalEventConfigurations": { "LoanDisbursalBusinessEvent": true } }'

# Step 2: confirm
curl -u admin:password -H "Fineract-Platform-TenantId: default" \
  https://example.org/fineract-provider/api/v1/externalevents/configuration | jq '.externalEventConfiguration[] | select(.type == "LoanDisbursalBusinessEvent")'

# Step 3: drain the outbox via a Loan COB run that ends with the publishing step
```
