The Standing Instructions API stores recurring transfer orders ("standing orders") in Apache Fineract. A standing instruction directs the platform to debit one account (the payer's savings account) on a recurring schedule and credit a second account (a savings or loan account). The scheduler executes due instructions through a dedicated batch job and writes outcomes into the standing-instruction run history.

## Endpoints

| Method | Path | Description | Command / read handler | Permission |
| --- | --- | --- | --- | --- |
| `GET` | `/v1/standinginstructions/template` | Creation template (allowed payer/payee accounts, `transferType`). | `StandingInstructionReadPlatformService.retrieveTemplate(...)` | `READ_STANDINGINSTRUCTION` |
| `POST` | `/v1/standinginstructions` | Create a standing instruction. | `CommandWrapperBuilder.createStandingInstruction()` → `CREATE_STANDINGINSTRUCTION` | `CREATE_STANDINGINSTRUCTION` |
| `PUT` | `/v1/standinginstructions/{id}?command=update` | Update an instruction. | `updateStandingInstruction(id)` → `UPDATE_STANDINGINSTRUCTION` | `UPDATE_STANDINGINSTRUCTION` |
| `PUT` | `/v1/standinginstructions/{id}?command=delete` | Logically delete (cancel) an instruction. | `deleteStandingInstruction(id)` → `DELETE_STANDINGINSTRUCTION` | `DELETE_STANDINGINSTRUCTION` |
| `GET` | `/v1/standinginstructions` | Paginated list. Filters: `externalId`, `transferType`, `clientName`, `clientId`, `fromAccountId`, `fromAccountType`, plus paging/sort. | `StandingInstructionReadPlatformService.retrieveAll(dto)` | `READ_STANDINGINSTRUCTION` |
| `GET` | `/v1/standinginstructions/{id}` | Retrieve a single instruction; supports `associations=all,transactions,template`. | `retrieveOne(id)` + optional `accountTransfersReadPlatformService.retrieveByStandingInstruction(...)` | `READ_STANDINGINSTRUCTION` |

Unknown `command` values on the PUT endpoint raise `UnrecognizedQueryParamException`. Command dispatch goes through:

```java
private static final CommandHandlerRegistry<String, Long, String, CommandWrapper> COMMAND_HANDLER_REGISTRY = new CommandHandlerRegistry<>(
    Map.of(
        CommandParameterUtil.UPDATE_COMMAND_VALUE,
        (id, json) -> new CommandWrapperBuilder().updateStandingInstruction(id).withJson(json).build(),
        CommandParameterUtil.DELETE_COMMAND_VALUE,
        (id, json) -> new CommandWrapperBuilder().deleteStandingInstruction(id).withJson(json).build()));
```

## Transfer types

`transferType` codes (`StandingInstructionType` enum):

| Code | Meaning |
| --- | --- |
| `1` | Account Transfer |
| `2` | Dues amount transfer (loan repayment) |

`fromAccountType` / `toAccountType` reuse the `PortfolioAccountType` enum (savings vs loan).

## Request body — create

The deserialiser binds to `StandingInstructionCreationRequest`:

```json
{
  "name": "Monthly rent",
  "fromOfficeId": 1,
  "fromClientId": 1,
  "fromAccountType": 2,
  "fromAccountId": 12,
  "toOfficeId": 1,
  "toClientId": 2,
  "toAccountType": 2,
  "toAccountId": 24,
  "transferType": 1,
  "priority": 1,
  "instructionType": 1,
  "status": 1,
  "amount": 500.00,
  "validFrom": "01 March 2024",
  "validTill": "01 March 2025",
  "recurrenceType": 1,
  "recurrenceFrequency": 2,
  "recurrenceInterval": 1,
  "recurrenceOnMonthDay": "01 March",
  "locale": "en",
  "dateFormat": "dd MMMM yyyy",
  "monthDayFormat": "dd MMMM"
}
```

## Request body — update / delete

`StandingInstructionUpdatesRequest` accepts any of the modifiable fields above plus `status`:

```json
{
  "amount": 600.00,
  "locale": "en"
}
```
