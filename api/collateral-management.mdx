`CollateralManagementApiResource` is the JAX-RS resource that exposes Apache Fineract's global **collateral product catalogue**. Each row in `m_collateral_management` is a template — a kind of collateral (vehicle, jewellery, real estate, etc.) — with a base price, currency, quality and unit type. Client-side inventory and loan-side reservations both reference rows owned by this resource.

This page covers only the catalogue resource. For the per-client inventory see [client-collateral-management](/api/client-collateral-management) and for loan-side linkage see [loan-collateral-management](/api/loan-collateral-management). Higher-level subsystem notes live under [/portfolio/collateral-management](/portfolio/collateral-management).

## Source

- **File:** `fineract-provider/src/main/java/org/apache/fineract/portfolio/collateralmanagement/api/CollateralManagementApiResource.java`
- **Class path annotation:** `@Path("/v1/collateral-management")`
- **OpenAPI tag:** `Collateral Management`
- **Spring stereotype:** `@Component` (registered as a CXF JAX-RS provider)

The resource is constructor-injected (`@RequiredArgsConstructor`) with:

- `DefaultToApiJsonSerializer<CollateralManagementData>` — payload serializer for the product template
- `DefaultToApiJsonSerializer<CurrencyData>` — serializer used by the template endpoint
- `PortfolioCommandSourceWritePlatformService` — accepts the `CommandWrapper` envelopes built by `CommandWrapperBuilder`
- `PlatformSecurityContext` — used to call `authenticatedUser().validateHasReadPermission(...)`
- `CollateralManagementReadPlatformService` — read store
- `CurrencyReadPlatformService` — for the template (`/template`) endpoint

## Endpoints

| Method | Path | Description | Command / Handler | Permission |
| ------ | ---- | ----------- | ----------------- | ---------- |
| POST | `/v1/collateral-management` | Create a new collateral product template. | `CommandWrapperBuilder.createCollateral()` → `CREATE_COLLATERAL_PRODUCT` | `CREATE_COLLATERAL_PRODUCT` (maker-checker capable) |
| GET | `/v1/collateral-management` | List every collateral product template. | `collateralManagementReadPlatformService.getAllCollateralProducts()` | Read permission `COLLATERAL_PRODUCT` |
| GET | `/v1/collateral-management/{collateralId}` | Retrieve a single collateral product by ID. | `collateralManagementReadPlatformService.getCollateralProduct(collateralId)` | Read permission `COLLATERAL_PRODUCT` |
| GET | `/v1/collateral-management/template` | Return the list of platform currencies for use when building UI for a new product. | `currencyReadPlatformService.retrieveAllPlatformCurrencies()` | None (must be authenticated) |
| PUT | `/v1/collateral-management/{collateralId}` | Update a collateral product template. | `CommandWrapperBuilder.updateCollateralProduct(collateralId)` → `UPDATE_COLLATERAL_PRODUCT` | `UPDATE_COLLATERAL_PRODUCT` (maker-checker) |
| DELETE | `/v1/collateral-management/{collateralId}` | Soft-delete a product template. | `CommandWrapperBuilder.deleteCollateralProduct(collateralId)` → `DELETE_COLLATERAL_PRODUCT` | `DELETE_COLLATERAL_PRODUCT` (maker-checker) |

Permission strings are sourced from `CollateralManagementJsonInputParams.COLLATERAL_PRODUCT_READ_PERMISSION`.

## Request / response examples

### Create product template

`POST /v1/collateral-management`

```json
{
  "name": "Gold Bar 1oz",
  "quality": "Pure",
  "basePrice": "2000",
  "unitType": "Ounce",
  "pctToBase": "80",
  "currency": "USD",
  "locale": "en"
}
```

The handler builds:

```java
new CommandWrapperBuilder()
    .createCollateral()
    .withJson(apiJsonSerializerService.serialize(collateralManagementProductRequest))
    .build();
```

Response (`CommandProcessingResult`):

```json
{
  "resourceId": 17,
  "officeId": null,
  "changes": {},
  "subResourceId": null
}
```

### Retrieve a product

`GET /v1/collateral-management/17`

```json
{
  "id": 17,
  "name": "Gold Bar 1oz",
  "quality": "Pure",
  "basePrice": 2000.00,
  "unitType": "Ounce",
  "pctToBase": 80.00,
  "currency": {
    "code": "USD",
    "name": "US Dollar",
    "decimalPlaces": 2,
    "displaySymbol": "$"
  }
}
```

### Template (/api/currencies)

`GET /v1/collateral-management/template`

Returns a `List<CurrencyData>` for populating the **currency** field of a new product:

```json
[
  { "code": "USD", "name": "US Dollar", "decimalPlaces": 2 },
  { "code": "EUR", "name": "Euro", "decimalPlaces": 2 }
]
```

### Update

`PUT /v1/collateral-management/17`

```json
{
  "name": "Gold Bar 1oz (refined)",
  "basePrice": "2100",
  "pctToBase": "85",
  "locale": "en"
}
```

The handler calls `updateCollateralProduct(collateralId)` to wrap the JSON in an `UPDATE_COLLATERAL_PRODUCT` command. The response surfaces the modified fields as a `changes` map.

### Delete

`DELETE /v1/collateral-management/17`

```json
{ "resourceId": 17, "changes": {} }
```

## Data carriers

- **Request:** `CollateralManagementProductRequest` (POST), `CollateralProductRequest` (PUT). Both are plain DTOs deserialized by Jackson.
- **Read response:** `CollateralManagementData` — joined with `CurrencyData` for the `currency` field.
- **Write response:** `CommandProcessingResult` from `commandsSourceWritePlatformService.logCommandSource(...)`. The Swagger module describes `PostCollateralManagementProductResponse`, `PutCollateralProductResponse` and `DeleteCollateralProductResponse` (see `CollateralManagementApiResourceSwagger`).

## Permissions

All read endpoints call:

```java
this.context.authenticatedUser()
    .validateHasReadPermission(CollateralManagementJsonInputParams.COLLATERAL_PRODUCT_READ_PERMISSION.getValue());
```

Write endpoints do not validate permissions inline — they delegate to `PortfolioCommandSourceWritePlatformService.logCommandSource` which enforces the appropriate `CREATE_*` / `UPDATE_*` / `DELETE_*` permissions and feeds the maker-checker queue when configured.
