`ClientsApiResource` is the largest write surface in Fineract — it covers the entire client lifecycle (pending → active → close), inter-office transfers, staff assignment, default savings account binding, bulk import, and external-id-keyed mirror routes for every operation. This page enumerates every `@GET`/`@POST`/`@PUT`/`@DELETE` method from source, lists the underlying command handlers, and shows representative payloads.

## Source

```
fineract-provider/src/main/java/org/apache/fineract/portfolio/client/api/ClientsApiResource.java
```

| Annotation | Value |
| --- | --- |
| `@Path` | `/v1/clients` |
| `@Component` | yes (auto-discovered by [`JerseyConfig`](/api/jersey-paths)) |
| `@Tag` name | `Clients` |

The class injects:

- `PlatformSecurityContext context` — for permission checks.
- `ClientReadPlatformService clientReadPlatformService` — all GET endpoints.
- `PortfolioCommandSourceWritePlatformService commandsSourceWritePlatformService` — all writes funnel through here.
- `DefaultToApiJsonSerializer<ClientData> toApiJsonSerializer` — and several siblings for transfer, accounts, obligee.
- `BulkImportWorkbookService` / `BulkImportWorkbookPopulatorService` — Excel import/export.

## Permissions

Every read calls `context.authenticatedUser().validateHasReadPermission(ClientApiConstants.CLIENT_RESOURCE_NAME)` (resource string `CLIENT`). Writes go through `CommandWrapper` permission codes — `CREATE_CLIENT`, `UPDATE_CLIENT`, `DELETE_CLIENT`, `ACTIVATE_CLIENT`, `CLOSE_CLIENT`, etc. — checked by the [command pipeline](/command/synchronous-command-processing) before the handler runs.

## Endpoint inventory

### Numeric-id routes

| HTTP | Path | Description | Command / Read service |
| --- | --- | --- | --- |
| `GET` | `/v1/clients/template` | Dropdown options for the create form | `clientReadPlatformService.retrieveTemplate(...)` |
| `GET` | `/v1/clients` | Paged list with filters | `clientReadPlatformService.retrieveAll(SearchParameters)` |
| `GET` | `/v1/clients/{clientId}` | Fetch one client | `clientReadPlatformService.retrieveOne(id)` |
| `POST` | `/v1/clients` | Create client | `createClient` → `ClientWritePlatformService.createClient` |
| `PUT` | `/v1/clients/{clientId}` | Update basic fields | `updateClient` → `ClientWritePlatformService.updateClient` |
| `DELETE` | `/v1/clients/{clientId}` | Hard-delete a pending client | `deleteClient` → `ClientWritePlatformService.deleteClient` |
| `POST` | `/v1/clients/{clientId}?command=...` | Lifecycle command dispatch (see table below) | varies by `command` |
| `GET` | `/v1/clients/{clientId}/accounts` | Associated loans / savings / shares | `accountDetailsReadPlatformService.retrieveClientAccountDetails` |
| `GET` | `/v1/clients/{clientId}/obligeedetails` | Guarantor obligation snapshot | `GuarantorReadPlatformService.retrieveObligeeDetails` |
| `GET` | `/v1/clients/{clientId}/transferproposaldate` | Earliest legal date to propose a transfer | `clientReadPlatformService.retrieveTransferTemplate` |

### External-id mirror routes

Every numeric-id route has a twin keyed on `externalId`. The implementation resolves the external id to a numeric `clientId` (throws `ClientNotFoundException` if missing) and delegates to the same private helper.

| HTTP | Path | Description |
| --- | --- | --- |
| `GET` | `/v1/clients/external-id/{externalId}` | Fetch one client by external id |
| `PUT` | `/v1/clients/external-id/{externalId}` | Update by external id |
| `DELETE` | `/v1/clients/external-id/{externalId}` | Delete by external id |
| `POST` | `/v1/clients/external-id/{externalId}` | Lifecycle command dispatch |
| `GET` | `/v1/clients/external-id/{externalId}/accounts` | Associated accounts |
| `GET` | `/v1/clients/external-id/{externalId}/obligeedetails` | Obligee snapshot |
| `GET` | `/v1/clients/external-id/{externalId}/transferproposaldate` | Transfer template |

## The `command=` dispatch table

`POST /v1/clients/{clientId}?command=X` goes through `evaluateCommand(...)`. The full source list:

| `command=` | CommandWrapperBuilder call | Effect |
| --- | --- | --- |
| `activate` | `activateClient(id)` | Pending → Active |
| `close` | `closeClient(id)` | Active → Closed |
| `reject` | `rejectClient(id)` | Pending → Rejected (requires `rejectionDate`, `rejectionReasonId`) |
| `withdraw` | `withdrawClient(id)` | Pending → Withdrawn (requires `withdrawalDate`, `withdrawalReasonId`) |
| `reactivate` | `reActivateClient(id)` | Closed → Active |
| `undoRejection` | `undoRejection(id)` | Rejected → Pending |
| `undoWithdrawal` | `undoWithdrawal(id)` | Withdrawn → Pending |
| `assignStaff` | `assignClientStaff(id)` | Bind loan officer (body: `staffId`) |
| `unassignStaff` | `unassignClientStaff(id)` | Remove loan officer binding |
| `updateSavingsAccount` | `updateClientSavingsAccount(id)` | Set default savings account |
| `proposeTransfer` | `proposeClientTransfer(id)` | Initiate inter-office transfer (origin branch) |
| `proposeAndAcceptTransfer` | `proposeAndAcceptClientTransfer(id)` | One-shot transfer when caller has data scope over both branches |
| `withdrawTransfer` | `withdrawClientTransferRequest(id)` | Cancel a pending transfer (origin branch) |
| `acceptTransfer` | `acceptClientTransfer(id)` | Destination branch accepts |
| `rejectTransfer` | `rejectClientTransfer(id)` | Destination branch rejects |

Unrecognised `command` value → `UnrecognizedQueryParamException` (HTTP 400) with the allowed list embedded in the body.

## Source excerpt — create

```java
@POST
@Consumes(MediaType.APPLICATION_JSON)
@Produces(MediaType.APPLICATION_JSON)
public String create(final String apiRequestBodyAsJson) {
    final CommandWrapper commandRequest = new CommandWrapperBuilder()
            .createClient()
            .withJson(apiRequestBodyAsJson)
            .build();
    final CommandProcessingResult result =
        commandsSourceWritePlatformService.logCommandSource(commandRequest);
    return toApiJsonSerializer.serialize(result);
}
```

## Write request bodies

The Swagger-friendly DTOs live alongside the resource:

- `ClientsApiResourceSwagger.PostClientsRequest` — fields documented in `ClientApiConstants`. Mandatory: `officeId`, `firstname`+`lastname` OR `fullname`, `active` (+ `activationDate` if active). Optional: `externalId`, `accountNo`, `groupId`, `staffId`, `mobileNo`, `emailAddress`, `dateOfBirth`, `genderId`, `clientTypeId`, `clientClassificationId`, `savingsProductId`, `legalFormId`, `address`, `clientNonPersonDetails`.
- `ClientsApiResourceSwagger.PutClientsClientIdRequest` — same fields, all optional; `changes` map in the response reflects what actually mutated.
- `ClientsApiResourceSwagger.PostClientsClientIdRequest` — the union body for all `command=` actions.
