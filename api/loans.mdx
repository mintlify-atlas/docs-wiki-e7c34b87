The Loans API is the master entry point Apache Fineract clients call to manage the loan application lifecycle — from `Submitted and pending approval` through approval, disbursal, repayment and closure — including state transitions, delinquency actions, GLIM (Group Loan Individual Monitoring) flows, and Excel-based bulk import.

## Source

| Aspect | Value |
| --- | --- |
| Resource class | `org.apache.fineract.portfolio.loanaccount.api.LoansApiResource` |
| File | `fineract-provider/src/main/java/org/apache/fineract/portfolio/loanaccount/api/LoansApiResource.java` |
| JAX-RS `@Path` | `/v1/loans` |
| Swagger tag | `Loans` |
| Permission resource | `LOAN` (read/write) and `DELINQUENCY_TAGS`, `DELINQUENCY_ACTION` (delinquency endpoints) |
| Primary read service | `LoanReadPlatformService` |
| Command source | `PortfolioCommandSourceWritePlatformService` |

The resource is constructed via Lombok `@RequiredArgsConstructor` and wires more than thirty collaborators including `LoanReadPlatformService`, `LoanProductReadPlatformService`, `LoanScheduleCalculationPlatformService`, `GuarantorReadPlatformService`, `DelinquencyReadPlatformService`, `LoanSummaryProviderDelegate`, `LoanCapitalizedIncomeBalanceRepository`, `LoanApprovedAmountHistoryRepository`, `GLIMAccountInfoReadPlatformService`, `BulkImportWorkbookService` and `BulkImportWorkbookPopulatorService`.

## State transition commands

`POST /v1/loans/{loanId}?command={cmd}` dispatches through `CommandParameterUtil.is(commandParam, ...)` in `stateTransitions(...)`. Recognised values:

| `command` | Command handler | Outcome |
| --- | --- | --- |
| `reject` | `rejectLoanApplication(loanId)` | Reject pending application. |
| `withdrawnByApplicant` | `withdrawLoanApplication(loanId)` | Applicant withdraws. |
| `approve` | `approveLoanApplication(loanId)` | Approve a pending application. |
| `disburse` | `disburseLoanApplication(loanId)` | Disburse approved loan. |
| `disburseToSavings` | `disburseLoanToSavingsApplication(loanId)` | Disburse principal into a linked savings account. |
| `disburseWithoutAutoDownPayment` | `disburseWithoutAutoDownPayment(loanId)` | Disburse without auto-creating a down-payment transaction. |
| `undoapproval` | `undoLoanApplicationApproval(loanId)` | Reverse approval. |
| `undodisbursal` | `undoLoanApplicationDisbursal(loanId)` | Reverse the full disbursal. |
| `undolastdisbursal` | `undoLastDisbursalLoanApplication(loanId)` | Reverse only the last tranche (multi-disburse). |
| `assignloanofficer` | `assignLoanOfficer(loanId)` | Assign a staff member as loan officer. |
| `unassignloanofficer` | `unassignLoanOfficer(loanId)` | Remove the loan-officer assignment. |
| `recoverGuarantees` | `recoverFromGuarantor(loanId)` | Trigger guarantee recovery from posted guarantors. |
| `assigndelinquency` | `assignDelinquency(loanId)` | Manually assign a delinquency range. |
| `contractTermination` | `applyContractTermination(loanId)` | Terminate the contract (progressive loans). |
| `undoContractTermination` | `undoContractTermination(loanId)` | Reverse a contract termination. |

Unknown `command` values raise `UnrecognizedQueryParamException("command", commandParam)`.

## Request / response shapes

Swagger types live in `LoansApiResourceSwagger`. The most-used schemas:

- `PostLoansRequest` — used by `POST /v1/loans`. Mandatory: `clientId`/`groupId`, `productId`, `principal`, `loanTermFrequency`, `loanTermFrequencyType`, `numberOfRepayments`, `repaymentEvery`, `repaymentFrequencyType`, `interestRatePerPeriod`, `interestRateFrequencyType`, `amortizationType`, `interestType`, `interestCalculationPeriodType`, `transactionProcessingStrategyCode`, `expectedDisbursementDate`, `submittedOnDate`, `loanType`.
- `PutLoansLoanIdRequest` — used by `PUT /v1/loans/{loanId}` modifies the same envelope but is rejected unless the loan is still `Submitted and pending approval`.
- `PostLoansLoanIdRequest` — used by approve/reject/disburse/withdraw state transitions. Typical fields: `approvedOnDate`, `expectedDisbursementDate`, `approvedLoanAmount`, `transactionAmount`, `actualDisbursementDate`, `note`, `locale`, `dateFormat`.
- `PutLoansApprovedAmountRequest` — used by `PUT /v1/loans/{loanId}/approved-amount`. Fields: `approvedLoanAmount`, `note`.
- `PostLoansDelinquencyActionRequest` — `{ action, startDate, endDate, locale, dateFormat }`.

Example state-transition request (`POST /v1/loans/123?command=approve`):

```json
{
  "approvedOnDate": "20 September 2025",
  "approvedLoanAmount": 5000.00,
  "expectedDisbursementDate": "01 October 2025",
  "note": "Approved by branch manager",
  "locale": "en",
  "dateFormat": "dd MMMM yyyy"
}
```

Response (all command-driven write endpoints):

```json
{
  "officeId": 1,
  "clientId": 42,
  "loanId": 123,
  "resourceId": 123,
  "changes": { "status": { "id": 200, "code": "loanStatusType.approved" } }
}
```

## Field reference

The class-level `@Tag` Javadoc enumerates the meaning of every input parameter. Selected highlights:

- `accountNo` — auto-generated if omitted at create time.
- `externalId` — caller-provided unique reference.
- `principal` — disbursement amount.
- `loanTermFrequency` × `loanTermFrequencyType` (0=Days, 1=Weeks, 2=Months, 3=Years).
- `numberOfRepayments` × `repaymentEvery` × `repaymentFrequencyType` (0=Days, 1=Weeks, 2=Months).
- `interestRatePerPeriod` × `interestRateFrequencyType` (2=Per month, 3=Per year).
- `amortizationType` (0=Equal principal payments, 1=Equal installments).
- `interestType` (0=Declining balance, 1=Flat).
- `interestCalculationPeriodType` (0=Daily, 1=Same as repayment period) and `allowPartialPeriodInterestCalculation`.
- `transactionProcessingStrategyCode` — payment-application strategy code (`mifos-standard-strategy`, `principal-interest-penalties-fees-order-strategy`, `interest-principal-penalties-fees-order-strategy`, `early-repayment-strategy`, `rbi-india-strategy`, `progressive-loan-strategy`, …).
- `loanType` — one of `individual`, `group`, `jlg`.

## Permissions

Endpoint authorisation is enforced via `PlatformSecurityContext.authenticatedUser().validateHasReadPermission(...)` / `validateHasCreatePermission(...)` and through the `CommandWrapperBuilder` action code that is mapped to a permission inside `commandsource`. For most endpoints the resource code is `LOAN`; delinquency tag history uses `DELINQUENCY_TAGS`; delinquency-action writes use `DELINQUENCY_ACTION`.
