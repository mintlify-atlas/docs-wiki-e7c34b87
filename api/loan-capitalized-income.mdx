Capitalized income is the progressive-loan mechanism by which fees and discounts that should be recognised over the life of a loan (rather than at origination) are amortised across the schedule. The `LoanCapitalizedIncomeApiResource` is a read-only endpoint set that exposes the capitalised-income balance for a loan plus the per-transaction allocation data required by the General Ledger.

| Aspect | Value |
| --- | --- |
| Resource class | `org.apache.fineract.portfolio.loanaccount.api.LoanCapitalizedIncomeApiResource` |
| File | `fineract-progressive-loan/src/main/java/org/apache/fineract/portfolio/loanaccount/api/LoanCapitalizedIncomeApiResource.java` |
| JAX-RS `@Path` | `/v1/loans` |
| Swagger tag | `Loan Capitalized Income` |
| Permission resource | `LOAN` |
| Read services | `CapitalizedIncomeBalanceReadService`, `LoanAmortizationAllocationService`, `LoanReadPlatformService` |

The constructor injects `PlatformSecurityContext`, `CapitalizedIncomeBalanceReadService`, `LoanAmortizationAllocationService`, `DefaultToApiJsonSerializer<LoanAmortizationAllocationData>`, `ApiRequestParameterHelper` and `LoanReadPlatformService`.

Whitelisted response data set: `loanId`, `loanExternalId`, `baseLoanTransactionId`, `baseLoanTransactionDate`, `baseLoanTransactionAmount`, `unrecognizedAmount`, `chargedOffAmount`, `adjustmentAmount`, `amortizationMappings`.

## Endpoints

### Capitalized income summary

| Method | Path | Status | Description |
| --- | --- | --- | --- |
| `GET` | `/v1/loans/{loanId}/deferredincome` | **deprecated** | Legacy alias — returns a single `LoanCapitalizedIncomeData`. |
| `GET` | `/v1/loans/external-id/{loanExternalId}/deferredincome` | **deprecated** | External-id variant of the legacy alias. |
| `GET` | `/v1/loans/{loanId}/capitalized-incomes` | current | Returns `List<CapitalizedIncomeDetails>` with per-entry recognition status. |
| `GET` | `/v1/loans/external-id/{loanExternalId}/capitalized-incomes` | current | External-id variant. |

Both *legacy* endpoints are flagged `@Operation(deprecated = true, ...)` and delegate to `CapitalizedIncomeBalanceReadService.fetchLoanCapitalizedIncomeData(loanId)`. New integrations should use the `/capitalized-incomes` variants which call `fetchLoanCapitalizedIncomeDetails(loanId)`.

### Per-transaction allocation data

| Method | Path |
| --- | --- |
| `GET` | `/v1/loans/{loanId}/capitalized-incomes/{loanTransactionId}` |
| `GET` | `/v1/loans/external-id/{loanExternalId}/capitalized-incomes/{loanTransactionId}` |
| `GET` | `/v1/loans/{loanId}/capitalized-incomes/external-id/{loanTransactionExternalId}` |
| `GET` | `/v1/loans/external-id/{loanExternalId}/capitalized-incomes/external-id/{loanTransactionExternalId}` |

All four delegate to the shared `retrieveCapitalizedIncomeAllocationData(...)` private:

1. Validates `LOAN` read permission via `PlatformSecurityContext.authenticatedUser().validateHasReadPermission("LOAN")`.
2. Resolves loan and transaction ids (`LoanReadPlatformService.getResolvedLoanId(...)`, `getResolvedLoanTransactionId(...)`).
3. Calls `LoanAmortizationAllocationService.retrieveLoanAmortizationAllocationsForCapitalizedIncomeTransaction(resolvedLoanTransactionId, resolvedLoanId)`.
4. Honours `?fields=` query filtering via `ApiRequestParameterHelper`.

## Response shapes

### `/capitalized-incomes` list

```json
[
  {
    "loanId": 123,
    "transactionId": 555,
    "transactionDate": "2026-04-01",
    "totalAmount": 1000.00,
    "unrecognizedAmount": 750.00,
    "chargedOffAmount": 0.00,
    "adjustmentAmount": 0.00,
    "amortizationMappings": [
      { "installment": 1, "amount": 100.00, "recognitionDate": "2026-05-01" },
      { "installment": 2, "amount": 100.00, "recognitionDate": "2026-06-01" }
    ]
  }
]
```

### Allocation data

`GET /v1/loans/{loanId}/capitalized-incomes/{loanTransactionId}` returns a `LoanAmortizationAllocationData`:

```json
{
  "loanId": 123,
  "loanExternalId": "loan-001",
  "baseLoanTransactionId": 555,
  "baseLoanTransactionDate": "2026-04-01",
  "baseLoanTransactionAmount": 1000.00,
  "unrecognizedAmount": 750.00,
  "chargedOffAmount": 0.00,
  "adjustmentAmount": 0.00,
  "amortizationMappings": [
    { "installment": 1, "amount": 100.00, "recognitionDate": "2026-05-01" }
  ]
}
```

Example queries (from the Swagger annotations):

```
/loans/1/capitalized-incomes/1
/loans/1/capitalized-incomes/1?fields=baseLoanTransaction,unrecognizedAmount
/loans/external-id/1/capitalized-incomes/1
```

### Legacy `/deferredincome`

Returns a `LoanCapitalizedIncomeData` aggregate (used by older UI):

```json
{
  "loanId": 123,
  "totalCapitalizedAmount": 1000.00,
  "totalRecognizedAmount": 250.00,
  "totalUnrecognizedAmount": 750.00,
  "totalAdjustmentAmount": 0.00,
  "totalChargedOffAmount": 0.00
}
```

## Permissions

All endpoints invoke `context.authenticatedUser().validateHasReadPermission("LOAN")`. No write endpoints are exposed — capitalised-income transactions are created via the standard [Loan Transactions](/api/loan-transactions) `POST .../transactions?command=capitalizedIncome` (the command constant is exported from `LoanTransactionsApiResource`).
