The Audits API exposes Apache Fineract's command-source audit trail. Every non-read REST call is logged to `m_portfolio_command_source` with the maker, the action name, the entity it targeted, the request JSON, the result, and timestamps. The Audits resource is the read-side query for that table — it supports filtering by action name, entity, resource id, maker/checker, office, group, client, loan, savings account, and date ranges, plus pagination and a search-template helper for UI dropdowns.

## Source

| Aspect | Value |
| --- | --- |
| Resource class | `org.apache.fineract.commands.api.AuditsApiResource` |
| File | `fineract-provider/src/main/java/org/apache/fineract/commands/api/AuditsApiResource.java` |
| JAX-RS `@Path` | `/v1/audits` |
| Swagger tag | `Audits` |
| Read service | `AuditReadPlatformService` |
| Filter DTO | `AuditRequest` (`@BeanParam`) |
| SQL helper | `SQLBuilder` |
| Permission constant | `"AUDIT"` |

The resource enforces `validateHasReadPermission("AUDIT")` on every endpoint. Data scope: a non-head-office user only sees audits within their office scope; head-office users see everything, including audits without an office binding (e.g. loan product changes).

## Endpoints

| Method | Path | Description | Command / handler | Permission |
| --- | --- | --- | --- | --- |
| `GET` | `/v1/audits` | Search and page through audit rows. Supports `paged`, `offset`, `limit`, `orderBy`, `sortOrder`, plus all `AuditRequest` filters. | `AuditReadPlatformService.retrieveAuditEntries(...)` / `retrievePaginatedAuditEntries(...)` | `READ_AUDIT` / `ALL_FUNCTIONS_READ` / `ALL_FUNCTIONS` |
| `GET` | `/v1/audits/{auditId}` | Retrieve one audit row by id. | `AuditReadPlatformService.retrieveAuditEntry(auditId)` | `READ_AUDIT` |
| `GET` | `/v1/audits/searchtemplate` | Return search-template metadata (action names, entity names, app users) scoped to the requestor. | `AuditReadPlatformService.retrieveSearchTemplate("audit")` | `READ_AUDIT` |

## Filtering

`AuditRequest` (bound via `@BeanParam`) supports the following query parameters; each is translated into a `WHERE` clause by `SQLBuilder.addNonNullCriteria(...)`:

| Query param | Column |
| --- | --- |
| `actionName` | `aud.action_name = ?` |
| `entityName` | `aud.entity_name like '<value>%'` |
| `resourceId` | `aud.resource_id = ?` |
| `makerId` | `aud.maker_id = ?` |
| `checkerId` | `aud.checker_id = ?` |
| `makerDateTimeFrom` / `makerDateTimeTo` | `aud.made_on_date >= ?` and `>=`/`<=` on `made_on_date_utc` (subquery, `OR`-joined) |
| `checkerDateTimeFrom` / `checkerDateTimeTo` | Same pattern on `checked_on_date` and `checked_on_date_utc` |
| `status` | `aud.status = ?` |
| `officeId` | `aud.office_id = ?` |
| `groupId` | `aud.group_id = ?` |
| `clientId` | `aud.client_id = ?` |
| `loanId` | `aud.loan_id = ?` |
| `savingsAccountId` | `aud.savings_account_id = ?` |

Pagination is controlled by the standard parameters; setting `paged=true` switches to `retrievePaginatedAuditEntries(...)` which returns a `Page<AuditData>` instead of a flat list.

`includeJson=true` adds the request and response JSON bodies to each row — handy when debugging, expensive over large windows.

## Examples

```bash
# All audits, default sort, top 50
curl -u mifos:password -H 'fineract-platform-tenantid: default' \
  'https://fineract.example.org/fineract-provider/api/v1/audits'

# Filtered by maker time range
curl -u mifos:password -H 'fineract-platform-tenantid: default' \
  'https://fineract.example.org/fineract-provider/api/v1/audits?makerDateTimeFrom=2024-04-01+00:00:00&makerDateTimeTo=2024-04-15+23:59:59'

# Search for a single resource's history
curl -u mifos:password -H 'fineract-platform-tenantid: default' \
  'https://fineract.example.org/fineract-provider/api/v1/audits?entityName=LOAN&resourceId=1042&includeJson=true'
```

## Single audit lookup

```bash
curl -u mifos:password -H 'fineract-platform-tenantid: default' \
  'https://fineract.example.org/fineract-provider/api/v1/audits/5821'
```

Returns the same `AuditData` shape as the list endpoint, with `commandAsJson` populated unconditionally — single-row lookups always include the JSON payload.

## Search template

```bash
curl -u mifos:password -H 'fineract-platform-tenantid: default' \
  'https://fineract.example.org/fineract-provider/api/v1/audits/searchtemplate'
```

```json
{
  "appUsers":     [ { "id": 1, "username": "mifos" }, { "id": 2, "username": "alice" } ],
  "actionNames":  ["APPROVE","CREATE","DELETE","DISBURSE","UPDATE",...],
  "entityNames":  ["CLIENT","LOAN","SAVINGSACCOUNT",...],
  "processingResults": ["PROCESSED","AWAITING_APPROVAL","REJECTED","ERROR"]
}
```

`appUsers` is data-scoped — a branch user only sees colleagues from the same office. The list helps UIs populate filter dropdowns without having to call `/v1/users` separately.

## Sorting and pagination

| Query param | Default | Notes |
| --- | --- | --- |
| `paged` | `false` | When `true`, return a `Page<AuditData>` with `totalFilteredRecords`. |
| `offset` | `0` | Skip N rows. |
| `limit` | impl-defined | Page size. |
| `orderBy` | `aud.id` (descending) | Column to sort by. |
| `sortOrder` | `DESC` | `ASC` / `DESC`. |

Sorting columns must be valid SQL identifiers; the `SqlValidator` used by [Scheduler Jobs](/api/scheduler-jobs) is not applied here, so wrap user input in your own allow-list before passing to `orderBy`.

## Status codes in audit rows

`processingResult` matches the lifecycle of a command:

| Value | Meaning |
| --- | --- |
| `PROCESSED` | Applied immediately. |
| `AWAITING_APPROVAL` | Pending maker-checker — visible in [Maker Checkers](/api/maker-checkers). |
| `REJECTED` | Maker-checker reject. |
| `ERROR` | Handler threw and the row was preserved with the error JSON. |

## Permissions

| Permission | Required for |
| --- | --- |
| `READ_AUDIT` | All endpoints. |
| `ALL_FUNCTIONS_READ` / `ALL_FUNCTIONS` | Wildcard alternatives. |

The resource's class-level note in the Swagger tag is the authoritative source: "Permissions: To search and look at audit entries a user needs to be attached to a role that has one of the ALL_FUNCTIONS, ALL_FUNCTIONS_READ or READ_AUDIT permissions."
