The Internal Progressive Loan API is a **test-only** maintenance surface for the `ProgressiveLoanInterestScheduleModel` — the precomputed interest schedule Apache Fineract's progressive loan engine caches per loan. Tests use it to inspect the model after a transaction, force a fresh recomputation (reprocessing all transactions in the process), or drop the cached row entirely so the next read regenerates from scratch.

<Warning>
This resource is guarded by `@Profile(FineractProfiles.TEST)`. It is only registered when the Spring profile `test` is active, every operation logs a `DO NOT USE THIS IN PRODUCTION!` banner, and the operation descriptions themselves repeat the warning. Production deployments must omit `test` from `SPRING_PROFILES_ACTIVE`.
</Warning>

## Endpoints

| Method | Path | Description | Command / handler | Permission |
| --- | --- | --- | --- | --- |
| `GET` | `/v1/internal/loan/progressive/{loanId}/model` | Read the cached `ProgressiveLoanInterestScheduleModel` for a progressive loan. | `InterestScheduleModelRepositoryWrapper.readProgressiveLoanInterestScheduleModel(loanId, loanConfigurationDetails, installmentAmountInMultiplesOf)` | None (test profile guard) |
| `POST` | `/v1/internal/loan/progressive/{loanId}/model` | Regenerate the schedule (reprocessing transactions) and persist the resulting model. Runs in a Spring `@Transactional` boundary. | `LoanScheduleService.regenerateScheduleWithReprocessingTransactions(loan)` → `InterestScheduleModelRepositoryWrapper.writeInterestScheduleModel(loan, model)` | None (test profile guard) |
| `DELETE` | `/v1/internal/loan/progressive/{loanId}/model` | Remove the persisted model row for the loan. Returns the affected row id. | `InterestScheduleModelRepositoryWrapper.removeByLoanId(loanId)` | None (test profile guard) |

All three operations validate that the loan is a progressive loan first — non-progressive loans raise `IllegalArgumentException("The loan is not progressive.")` (the `DELETE` variant skips the check because it just removes a row keyed by `loanId`).

## Reading the model

```bash
curl -u mifos:password \
  -H 'fineract-platform-tenantid: default' \
  'https://fineract.example.org/fineract-provider/api/v1/internal/loan/progressive/1042/model'
```

The handler:

```java
Loan loan = loanRepository.findOneWithNotFoundDetection(loanId);
if (!loan.isProgressiveSchedule()) {
    throw new IllegalArgumentException("The loan is not progressive.");
}
ILoanConfigurationDetails loanConfigurationDetails = LoanConfigurationDetailsMapper.map(loan);
return writePlatformService.readProgressiveLoanInterestScheduleModel(loanId, loanConfigurationDetails,
        loan.getLoanProductRelatedDetail().getInstallmentAmountInMultiplesOf()).orElse(null);
```

A `null` body means no cached model exists yet — typically the next public schedule read regenerates one.

## Updating the model

```bash
curl -X POST -u mifos:password \
  -H 'fineract-platform-tenantid: default' \
  'https://fineract.example.org/fineract-provider/api/v1/internal/loan/progressive/1042/model'
```

The handler is `@Transactional`. It loads the loan, asserts it is progressive, regenerates the schedule via `LoanScheduleService.regenerateScheduleWithReprocessingTransactions(loan)` — the same routine [Internal COB](/api/internal-cob)'s `/loan-reprocess` calls — then persists the resulting model:

```java
ProgressiveLoanInterestScheduleModel model = reprocessTransactionsAndGetModel(loan);
return writePlatformService.writeInterestScheduleModel(loan, model);
```

The response is the freshly written model, useful for asserting "this transaction flipped the schedule shape exactly like so".

## Deleting the model

```bash
curl -X DELETE -u mifos:password \
  -H 'fineract-platform-tenantid: default' \
  'https://fineract.example.org/fineract-provider/api/v1/internal/loan/progressive/1042/model'
```

```json
1042
```

`removeByLoanId` returns the deleted row id (or zero if no row existed). Tests use this to force the engine to recompute from cold cache on the next call.

## When to use which endpoint

| Goal | Endpoint |
| --- | --- |
| Inspect the current model | `GET .../model` |
| Force a recompute after seeding transactions, then read it | `POST .../model` |
| Confirm cold-start behaviour | `DELETE .../model` then a public schedule read |

The `POST` path matches the production behaviour that runs whenever the loan engine notices its model is stale — the test surface lets the test trigger it on demand.
