`LoanOriginatorApiResource` is the JAX-RS resource that maintains the **loan originator catalogue** in Apache Fineract. A *loan originator* is an external broker / partner / agent through whom a loan was sourced — used for revenue sharing, regulatory reporting and origination-channel analytics. The records are referenced by [LoanOriginatorsApiResource](/api/loan-originators) when attaching to a specific loan.

The whole loan-origination module is feature-flagged: the bean is only registered when `fineract.module.loan-origination.enabled=true`.

## Source

- **File:** `fineract-loan-origination/src/main/java/org/apache/fineract/portfolio/loanorigination/api/LoanOriginatorApiResource.java`
- **Class path annotation:** `@Path("/v1/loan-originators")`
- **OpenAPI tag:** `Loan Originators` — *"Manage loan originator details for revenue sharing and reporting"*
- **Spring stereotype:** `@Component`
- **Conditional:** `@ConditionalOnProperty("fineract.module.loan-origination.enabled" = "true")`

Constructor-injected dependencies:

- `PlatformSecurityContext context`
- `LoanOriginatorReadPlatformService loanOriginatorReadPlatformService`
- `PortfolioCommandSourceWritePlatformService commandsSourceWritePlatformService`

Static config:

```java
// LoanOriginatorApiConstants.RESOURCE_NAME = "LOAN_ORIGINATOR"
```

## Endpoints

### Read

| Method | Path | Description | Command / Handler | Permission |
| ------ | ---- | ----------- | ----------------- | ---------- |
| GET | `/v1/loan-originators` | List all loan originators. | `loanOriginatorReadPlatformService.retrieveAll()` | `READ_LOAN_ORIGINATOR` |
| GET | `/v1/loan-originators/template` | Template for new-originator UI (status options, channel options). | `loanOriginatorReadPlatformService.retrieveTemplate()` | `READ_LOAN_ORIGINATOR` |
| GET | `/v1/loan-originators/{originatorId}` | Retrieve by internal id. | `loanOriginatorReadPlatformService.retrieveById(originatorId)` | `READ_LOAN_ORIGINATOR` |
| GET | `/v1/loan-originators/external-id/{externalId}` | Retrieve by external id. | `loanOriginatorReadPlatformService.retrieveByExternalId(externalId)` | `READ_LOAN_ORIGINATOR` |

### Write

| Method | Path | Description | Command / Handler | Permission |
| ------ | ---- | ----------- | ----------------- | ---------- |
| POST | `/v1/loan-originators` | Create a loan originator. | `CommandWrapperBuilder.createLoanOriginator()` → `CREATE_LOAN_ORIGINATOR` | `CREATE_LOAN_ORIGINATOR` |
| PUT | `/v1/loan-originators/{originatorId}` | Update by internal id. | `CommandWrapperBuilder.updateLoanOriginator(originatorId)` → `UPDATE_LOAN_ORIGINATOR` | `UPDATE_LOAN_ORIGINATOR` |
| PUT | `/v1/loan-originators/external-id/{externalId}` | Update by external id (id resolved via `resolveIdByExternalId`). | As above. | `UPDATE_LOAN_ORIGINATOR` |
| DELETE | `/v1/loan-originators/{originatorId}` | Delete by internal id. | `CommandWrapperBuilder.deleteLoanOriginator(originatorId)` → `DELETE_LOAN_ORIGINATOR` | `DELETE_LOAN_ORIGINATOR` |
| DELETE | `/v1/loan-originators/external-id/{externalId}` | Delete by external id. | As above. | `DELETE_LOAN_ORIGINATOR` |

Delete fails (HTTP 403) when the originator is currently mapped to one or more loans — detach those mappings via [LoanOriginatorsApiResource](/api/loan-originators) first.

## Request / response examples

### Create

`POST /v1/loan-originators`

```json
{
  "externalId": "ORIG-001",
  "name": "Acme Brokerage",
  "status": "ACTIVE",
  "commissionRate": 0.005,
  "contactName": "Jane Doe",
  "contactEmail": "jane@acme.example",
  "locale": "en"
}
```

Handler:

```java
final CommandWrapper commandRequest = new CommandWrapperBuilder()
    .createLoanOriginator()
    .withJson(apiRequestBodyAsJson)
    .build();
return this.commandsSourceWritePlatformService.logCommandSource(commandRequest);
```

Response (`CommandProcessingResult` → Swagger `PostLoanOriginatorsResponse`):

```json
{
  "resourceId": 7,
  "resourceExternalId": "ORIG-001",
  "changes": {}
}
```

Returns HTTP 403 when the supplied `externalId` is already in use (`Duplicate external ID`).

### List

### Retrieve

`GET /v1/loan-originators/7`

```json
{
  "id": 7,
  "externalId": "ORIG-001",
  "name": "Acme Brokerage",
  "status": "ACTIVE",
  "commissionRate": 0.005,
  "contactName": "Jane Doe",
  "contactEmail": "jane@acme.example"
}
```

`GET /v1/loan-originators/external-id/ORIG-001` returns the same payload.

### Update

`PUT /v1/loan-originators/7`

```json
{
  "commissionRate": 0.006,
  "locale": "en"
}
```

Response:

```json
{
  "resourceId": 7,
  "changes": { "commissionRate": 0.006 }
}
```

### Delete

`DELETE /v1/loan-originators/7`

```json
{
  "resourceId": 7,
  "changes": {}
}
```

`403` when the originator is still mapped to any loan.

## Data carriers

- **Requests:** raw JSON (no DTO binding); the resource passes the body straight into the command wrapper. Swagger types in `LoanOriginatorApiResourceSwagger`:
  - `PostLoanOriginatorsRequest`, `PutLoanOriginatorsRequest`
- **Read responses:** `LoanOriginatorData`, `LoanOriginatorTemplateData`. Swagger: `GetLoanOriginatorsResponse`, `GetLoanOriginatorTemplateResponse`.
- **Write responses:** `CommandProcessingResult` envelopes. Swagger: `PostLoanOriginatorsResponse`, `PutLoanOriginatorsResponse`, `DeleteLoanOriginatorsResponse`.

## Permissions

All read endpoints call:

```java
this.context.authenticatedUser()
    .validateHasReadPermission(LoanOriginatorApiConstants.RESOURCE_NAME);
```

Writes delegate to `PortfolioCommandSourceWritePlatformService` against `CREATE_LOAN_ORIGINATOR`, `UPDATE_LOAN_ORIGINATOR` and `DELETE_LOAN_ORIGINATOR`.
