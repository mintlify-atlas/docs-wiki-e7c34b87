`ProvisioningCategoryApiResource` manages the named buckets used by the loan-loss provisioning engine. Each category — typically Standard, Sub-standard, Doubtful, Loss — represents a bracket of overdue days; the [provisioning criteria](/api/provisioning-criteria) layer on top of these categories to decide what percentage of outstanding to provision and which GL accounts to post against.

Injected collaborators:

- `PortfolioCommandSourceWritePlatformService commandsSourceWritePlatformService`

## Permissions

The resource calls `platformSecurityContext.authenticatedUser()` only to ensure a user is bound to the thread — there is no explicit `validateHasReadPermission` for read; access is gated only by being authenticated. The write handlers (`CREATE_PROVISIONCATEGORY`, `UPDATE_PROVISIONCATEGORY`, `DELETE_PROVISIONCATEGORY`) check permissions themselves.

## Endpoint inventory

| HTTP | Path | Description | Command / Read service |
| --- | --- | --- | --- |
| `GET` | `/v1/provisioningcategory` | List all categories ordered by `categoryRank` | `provisioningCategoryReadPlatformService.retrieveAllProvisionCategories()` |
| `POST` | `/v1/provisioningcategory` | Create category | `createProvisioningCategory` |
| `PUT` | `/v1/provisioningcategory/{categoryId}` | Update category | `updateProvisioningCategory(categoryId)` |
| `DELETE` | `/v1/provisioningcategory/{categoryId}` | Delete a category not referenced by any criteria | `deleteProvisioningCategory(categoryId)` |

There is no single-fetch `GET /{id}` — the read service returns the full small list in one go and the UI filters client-side.

## Source excerpt

```java
@Path("/v1/provisioningcategory")
@Component
@Tag(name = "Provisioning Category")
@RequiredArgsConstructor
public class ProvisioningCategoryApiResource {

    @GET
    public List<ProvisioningCategoryData> retrieveAll() {
        platformSecurityContext.authenticatedUser();
        return provisioningCategoryReadPlatformService.retrieveAllProvisionCategories();
    }

    @POST
    public CommandProcessingResult createProvisioningCategory(final String apiRequestBodyAsJson) {
        platformSecurityContext.authenticatedUser();
        CommandWrapper cw = new CommandWrapperBuilder()
            .createProvisioningCategory()
            .withJson(apiRequestBodyAsJson).build();
        return commandsSourceWritePlatformService.logCommandSource(cw);
    }
}
```

`categoryRank` orders the categories for display; it is set automatically on creation (max + 1).

## Request body — POST/PUT

| Field | Required | Notes |
| --- | --- | --- |
| `categoryName` | yes | Unique among categories |
| `description` | no | Free text |

## Delete semantics

`DELETE /v1/provisioningcategory/{id}` fails with `error.msg.provisionincategory.in.use` if any active criterion references the category — drop the criterion first.
