The Collaterals API is the legacy per-loan collateral surface in Apache Fineract. Each `Collateral` is anchored to a single loan, references a `LoanCollateral` code value, and carries a value, description, and currency. The endpoints only operate while a loan is still in the "submitted and pending approval" state — once a loan is approved, collateral entries become immutable here.

A newer subsystem (`CollateralManagementApiResource` under `/v1/collateral-management`) handles reusable client-level collateral; the legacy resource documented on this page remains in place for compatibility with older clients and saved integrations.

## Endpoints

| Method | Path | Description | Command / read handler | Permission |
| --- | --- | --- | --- | --- |
| `GET` | `/v1/loans/{loanId}/collaterals/template` | Template containing allowed `LoanCollateral` code values for the loan currency. | `CodeValueReadPlatformService.retrieveCodeValuesByCode("LoanCollateral")` → `CollateralData.template(...)` | `READ_COLLATERAL` |
| `GET` | `/v1/loans/{loanId}/collaterals` | List collaterals attached to the loan. | `CollateralReadPlatformService.retrieveCollateralsForValidLoan(loanId)` | `READ_COLLATERAL` |
| `GET` | `/v1/loans/{loanId}/collaterals/{collateralId}` | Retrieve a single collateral; `?template=true` overlays allowed types. | `CollateralReadPlatformService.retrieveCollateral(loanId, collateralId)` | `READ_COLLATERAL` |
| `POST` | `/v1/loans/{loanId}/collaterals` | Add a collateral entry to a pending-approval loan. | `CommandWrapperBuilder.createCollateral(loanId)` → `CREATE_COLLATERAL` | `CREATE_COLLATERAL` |
| `PUT` | `/v1/loans/{loanId}/collaterals/{collateralId}` | Update a collateral entry. | `updateCollateral(loanId, collateralId)` → `UPDATE_COLLATERAL` | `UPDATE_COLLATERAL` |
| `DELETE` | `/v1/loans/{loanId}/collaterals/{collateralId}` | Remove a collateral entry (only on unapproved loans). | `deleteCollateral(loanId, collateralId)` → `DELETE_COLLATERAL` | `DELETE_COLLATERAL` |

Response data parameters: `id`, `type`, `value`, `description`, `allowedCollateralTypes`, `currency`.

## Request body — create

The payload binds to `LoansLoanIdCollateralsRequest`:

```json
{
  "collateralTypeId": 11,
  "value": 1500.00,
  "description": "Cattle - 1 cow",
  "locale": "en"
}
```

`collateralTypeId` references a code value belonging to the `LoanCollateral` code.

## Request body — update

`LoansLoandIdCollateralsCollateralIdRequest`:

```json
{
  "value": 1700.00,
  "description": "Cattle - updated valuation",
  "locale": "en"
}
```

## Response — retrieve

```json
{
  "id": 3,
  "type": { "id": 11, "name": "Cattle" },
  "value": 1500.00,
  "description": "Cattle - 1 cow",
  "currency": { "code": "USD", "name": "US Dollar", "decimalPlaces": 2 }
}
```

When `?template=true` is supplied, the response also includes `allowedCollateralTypes` drawn from the `LoanCollateral` code values.

## Response — write

```json
{
  "officeId": 1,
  "clientId": 1,
  "loanId": 12,
  "resourceId": 3,
  "changes": { "value": 1700.00 }
}
```

## Operational notes

- Adding/removing collateral after loan approval is rejected at the command-handler level — managers must use the new collateral-management API to attach reusable collateral to an active loan.
- The `LoanCollateral` code is system-defined; new collateral types are added via [/api/code-values](/api/code-values).
