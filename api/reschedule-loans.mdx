The Reschedule Loans API drives the request → preview → approve/reject workflow Apache Fineract uses to re-project the repayment schedule of an **approved or active** loan. Unlike the [Loan Schedule](/api/loan-schedule) endpoints which apply only to pending applications, a `LoanRescheduleRequest` records who asked for the change, lets a second user preview the new schedule, and then approves or rejects it.

## Request / response shapes

### Create

`POST /v1/rescheduleloans` body type `PostCreateRescheduleLoansRequest`:

```json
{
  "loanId": 123,
  "rescheduleFromDate": "01 June 2026",
  "submittedOnDate": "20 May 2026",
  "rescheduleReasonId": 18,
  "rescheduleReasonComment": "Borrower lost income",
  "graceOnPrincipal": 1,
  "graceOnInterest": 1,
  "extraTerms": 3,
  "newInterestRate": 8.5,
  "adjustedDueDate": "15 July 2026",
  "locale": "en",
  "dateFormat": "dd MMMM yyyy"
}
```

Key fields:

- `rescheduleFromDate` — first installment whose due-date is adjusted.
- `adjustedDueDate` — new due date for that installment (date-shift only).
- `graceOnPrincipal` / `graceOnInterest` — add `n` grace periods.
- `extraTerms` — append `n` additional installments to the end.
- `newInterestRate` — apply a new interest rate from `rescheduleFromDate`.

### Approve

`POST /v1/rescheduleloans/{scheduleId}?command=approve`:

```json
{
  "approvedOnDate": "21 May 2026",
  "locale": "en",
  "dateFormat": "dd MMMM yyyy"
}
```

### Reject

`POST /v1/rescheduleloans/{scheduleId}?command=reject`:

```json
{
  "rejectedOnDate": "21 May 2026",
  "locale": "en",
  "dateFormat": "dd MMMM yyyy"
}
```

### Preview projected schedule

`GET /v1/rescheduleloans/{scheduleId}?command=previewLoanReschedule` returns the `LoanScheduleData` that would be applied if the request were approved. Useful for sanity-checking new totals before approval.

### Retrieve request

`GET /v1/rescheduleloans/{scheduleId}` returns a `LoanRescheduleRequestData`:

```json
{
  "id": 12,
  "loanId": 123,
  "statusEnum": { "id": 100, "code": "loanStatus.submitted.and.pending.approval" },
  "rescheduleFromInstallment": 2,
  "rescheduleFromDate": "2026-06-01",
  "rescheduleReasonCodeValue": { "id": 18, "name": "Loss of income" },
  "rescheduleReasonComment": "Borrower lost income",
  "submittedOnDate": "2026-05-20",
  "submittedByUser": { "id": 1, "username": "loan-officer-a" }
}
```

### List

`GET /v1/rescheduleloans?command=pending&loanId=123` accepts any of the command tokens declared on `RescheduleLoansApiConstants`:

- `all` — every reschedule request
- `pending` — submitted but not yet approved/rejected
- `approve` — approved
- `reject` — rejected

Unknown commands raise `UnrecognizedQueryParamException("command", ...)`.

## Workflow summary

1. **Create** — `POST /v1/rescheduleloans` records a `LoanRescheduleRequest` in `Submitted and pending approval`.
2. **Preview** — `GET /v1/rescheduleloans/{scheduleId}?command=previewLoanReschedule` lets reviewers inspect the projected schedule without committing.
3. **Approve** / **Reject** — `POST /v1/rescheduleloans/{scheduleId}?command=approve|reject` either applies the variation to the live `Loan` schedule (creating `LoanTermVariations`) or marks the request rejected.

## Permissions

All read endpoints invoke `platformSecurityContext.authenticatedUser().validateHasReadPermission("LOANRESCHEDULEREQUEST")`. Writes go through `PortfolioCommandSourceWritePlatformService.logCommandSource(...)` which maps the builder action codes to the `CREATE_LOANRESCHEDULEREQUEST` / `APPROVE_LOANRESCHEDULEREQUEST` / `REJECT_LOANRESCHEDULEREQUEST` permissions.
