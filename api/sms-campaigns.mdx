The SMS Campaigns resource is the bulk counterpart to [`/v1/sms`](/api/sms). A campaign couples a **business rule** (a SQL `stretchy report` that produces recipient rows + merge fields) with a message template and a recurrence; the scheduler dispatches per-recipient messages into the SMS outbox at each tick. Lifecycle is managed with explicit `activate / close / reactivate` commands.

## Source

- **File**: `fineract-provider/src/main/java/org/apache/fineract/infrastructure/campaigns/sms/api/SmsCampaignApiResource.java`
- **Base path**: `@Path("/v1/smscampaigns")`
- **Permission entity**: `SMS_CAMPAIGN` (`SmsCampaignConstants.RESOURCE_NAME`)
- **Tag**: (none — `@Tag` is not declared)

Reads use `SmsCampaignReadPlatformService`; writes use `PortfolioCommandSourceWritePlatformService` with `createSmsCampaign` / `updateSmsCampaign` / `activateSmsCampaign` / `closeSmsCampaign` / `reactivateSmsCampaign` / `deleteSmsCampaign` builders. Preview runs `SmsCampaignWritePlatformService.previewMessage` directly without going through the command bus.

## Endpoints

| Method | Path | Description | Command builder / Handler | Permission |
| ------ | ---- | ----------- | ------------------------- | ---------- |
| GET | `/v1/smscampaigns` | Paged list of campaigns | `retrieveAll(SearchParameters)` | `READ_SMS_CAMPAIGN` |
| GET | `/v1/smscampaigns/{resourceId}` | Retrieve one campaign | `retrieveOne` | `READ_SMS_CAMPAIGN` |
| GET | `/v1/smscampaigns/template` | Form scaffolding + available business rules and trigger types | `retrieveTemplate(CampaignType.SMS)` | `READ_SMS_CAMPAIGN` |
| POST | `/v1/smscampaigns` | Create a campaign (body = `SmsCampaignCreationDto`) | `createSmsCampaign` | `CREATE_SMS_CAMPAIGN` |
| PUT | `/v1/smscampaigns/{campaignId}` | Update (body = `SmsCampaignUpdateDto`) | `updateSmsCampaign(id)` | `UPDATE_SMS_CAMPAIGN` |
| POST | `/v1/smscampaigns/{campaignId}?command=activate\|close\|reactivate` | Lifecycle action (body = `SmsCampaignHandlerDto`) | `activateSmsCampaign` / `closeSmsCampaign` / `reactivateSmsCampaign` | `ACTIVATE_SMS_CAMPAIGN` / `CLOSE_SMS_CAMPAIGN` / `REACTIVATE_SMS_CAMPAIGN` |
| POST | `/v1/smscampaigns/preview` | Render preview rows without persisting | `SmsCampaignWritePlatformService.previewMessage` | `READ_SMS_CAMPAIGN` |
| DELETE | `/v1/smscampaigns/{campaignId}` | Delete a **closed** campaign | `deleteSmsCampaign(id)` | `DELETE_SMS_CAMPAIGN` |

The `@Operation` for DELETE notes: *"Only closed SMS Campaigns can be deleted"* — the command handler rejects deletes of active campaigns.

## Request body — create (`SmsCampaignCreationDto`)

Mandatory fields per the `@Operation`: `campaignName`, `campaignType`, `triggerType`, `providerId`, `runReportId`, `message`. `paramValue` is required when the chosen report has bound parameters.

```json
{
  "campaignName": "Overdue Loan SMS",
  "campaignType": 1,
  "triggerType": 1,
  "providerId": 1,
  "runReportId": 12,
  "paramValue": "{\"officeId\":\"1\",\"loanStatusId\":\"300\"}",
  "message": "Hi {{firstname}}, your loan {{loanAccountNo}} is overdue by {{daysOverdue}} days.",
  "recurrenceStartDate": "2024 03 05 09 00 00",
  "recurrence": "FREQ=DAILY;INTERVAL=1",
  "locale": "en",
  "dateFormat": "yyyy MM dd HH mm ss"
}
```

| Field | Meaning |
| ----- | ------- |
| `campaignType` / `triggerType` | 1 = Direct, 2 = Schedule, 3 = Triggered |
| `providerId` | SMS provider id from `s_sms_camp_provider` (registered SMS bridges/gateways) |
| `runReportId` | stretchy report id producing the recipient rows |
| `message` | Mustache-style template; tokens are columns from the SQL projection |
| `recurrence` | RRULE — required for `campaignType=2` |

## Preview

`POST /v1/smscampaigns/preview`

```json
{
  "runReportId": 12,
  "paramValue": "{\"officeId\":\"1\",\"loanStatusId\":\"300\"}",
  "message": "Hi {{firstname}}, your loan {{loanAccountNo}} is overdue."
}
```

Response (`CampaignPreviewData`):

```json
{
  "campaignMessage": "Hi Jane, your loan 4500-001 is overdue.",
  "campaignParams": "{\"officeId\":\"1\",\"loanStatusId\":\"300\"}"
}
```

## Lifecycle

```
draft ──activate──▶ active ──close──▶ closed ──reactivate──▶ active
                                          └──delete──▶ removed
```

```
POST /v1/smscampaigns/17?command=activate
POST /v1/smscampaigns/17?command=close
POST /v1/smscampaigns/17?command=reactivate
DELETE /v1/smscampaigns/17        # only when closed
```

## Subsystem cross-links

- **[SMS](/api/sms)** — campaigns enqueue messages here; the outbox is the actual unit of dispatch.
- **[Hooks](/api/hooks)** — `SMS Bridge` / Twilio templates dispatch the messages.
- **[Reports](/api/reports)** / **[Run Reports](/api/run-reports)** — the `runReportId` references a stretchy report definition.
- **[Email Campaigns](/api/email-campaigns)** — same machinery on the email channel.

## Notes

- The same stretchy report can back both SMS and Email campaigns — the business-rule abstraction is channel-agnostic.
- Triggered campaigns (`triggerType=3`) are fired by [Hooks](/api/hooks) on specific events; the recurrence fields are ignored.
- `SmsCampaignHandlerDto` is used by the lifecycle POST mainly to carry an optional `closureReason` or `reactivationDate`; pass `{}` if no extra payload is needed.

## Curl reference

Create a campaign:

```bash
curl -u mifos:password \
  -H "Fineract-Platform-TenantId: default" \
  -H "Content-Type: application/json" \
  -X POST https://example.org/fineract-provider/api/v1/smscampaigns \
  -d '{
    "campaignName": "Active loan reminder",
    "campaignType": 1,
    "campaignTrigger": 2,
    "providerId": 1,
    "runReportId": 7,
    "paramValue": "{\"officeId\":\"1\",\"loanStatus\":\"300\"}",
    "message": "Hi {{client_display_name}}, your repayment of {{installment_amount}} is due on {{due_date}}.",
    "recurrenceStartDate": "2024-04-15 08:00:00",
    "recurrence": "FREQ=DAILY;INTERVAL=1",
    "dateFormat": "yyyy-MM-dd HH:mm:ss",
    "locale": "en"
  }'
```

Activate:

```bash
curl -u mifos:password \
  -H "Fineract-Platform-TenantId: default" \
  -H "Content-Type: application/json" \
  -X POST "https://example.org/fineract-provider/api/v1/smscampaigns/9?command=activate" \
  -d '{ "activationDate": "2024-04-15", "dateFormat": "yyyy-MM-dd", "locale": "en" }'
```

Preview without persisting:

```bash
curl -u mifos:password \
  -H "Fineract-Platform-TenantId: default" \
  -H "Content-Type: application/json" \
  -X POST https://example.org/fineract-provider/api/v1/smscampaigns/preview \
  -d '{
    "businessRuleId": 7,
    "paramValue": "{\"officeId\":\"1\"}",
    "message": "Hi {{client_display_name}}, your repayment of {{installment_amount}} is due."
  }'
```

## Trigger types

`triggerType` (`SmsCampaignTriggerType`):

| Id | Meaning | Required fields |
| --- | --- | --- |
| 1 | Direct (one-shot run) | `activationDate` |
| 2 | Schedule (recurring) | `recurrenceStartDate`, `recurrence` |
| 3 | Triggered (hook event) | `triggerEntityType`, `triggerActionType` |

## Lifecycle states (`statusType`)

| Id | State | Allowed transitions |
| --- | --- | --- |
| 100 | Pending | → 200 (activate) |
| 200 | Active | → 300 (close) |
| 300 | Closed | → 200 (reactivate), → delete |

Calling `DELETE` on an `active` row returns `error.msg.sms.campaign.cannot.be.deleted.in.this.state`.

## Error responses

| HTTP | Code | Cause |
| --- | --- | --- |
| 400 | `error.msg.campaign.message.cannot.be.blank` | Empty body template. |
| 400 | `error.msg.campaign.runReportId.cannot.be.blank` | Missing business rule. |
| 400 | `error.msg.campaign.cannot.be.activated.in.this.state` | Wrong-state activate. |
| 404 | `error.msg.sms.campaign.id.invalid` | `campaignId` not found. |
