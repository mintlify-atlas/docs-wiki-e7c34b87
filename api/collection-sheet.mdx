The Collection Sheet API generates the daily collection roster that Apache Fineract loan officers use to process repayments and mandatory deposits for individual loans on a given meeting date, then accepts the same payload back to post bulk repayments and savings deposits in a single command. The endpoint is command-routed via the `command` query parameter.

## Source

| Aspect | Value |
| --- | --- |
| Resource class | `org.apache.fineract.portfolio.collectionsheet.api.CollectionSheetApiResource` |
| File | `fineract-provider/src/main/java/org/apache/fineract/portfolio/collectionsheet/api/CollectionSheetApiResource.java` |
| JAX-RS `@Path` | `/v1/collectionsheet` |
| Swagger tag | `Collection Sheet` |
| Permission code | `CollectionSheetConstants.COLLECTIONSHEET_RESOURCE_NAME` (`COLLECTIONSHEET`) |
| Read service | `CollectionSheetReadPlatformService` |
| Command builder | `CommandWrapperBuilder.saveIndividualCollectionSheet()` |
| Command constants | `GENERATE_COLLECTION_SHEET_COMMAND_VALUE` (`generateCollectionSheet`), `SAVE_COLLECTION_SHEET_COMMAND_VALUE` (`saveCollectionSheet`) |

## Endpoints

| Method | Path | Description | Command / read handler | Permission |
| --- | --- | --- | --- | --- |
| `POST` | `/v1/collectionsheet?command=generateCollectionSheet` | Build the individual collection sheet for the given office and meeting date. | `CollectionSheetReadPlatformService.generateIndividualCollectionSheet(JsonQuery)` | `READ_COLLECTIONSHEET` |
| `POST` | `/v1/collectionsheet?command=saveCollectionSheet` | Post bulk repayments and mandatory deposits from the supplied sheet. | `CommandWrapperBuilder.saveIndividualCollectionSheet()` → `SAVE_COLLECTIONSHEET` | `SAVE_COLLECTIONSHEET` |

Any other `command` value short-circuits the method and returns an empty `200 OK`. The handler checks the command parameter via `CommandParameterUtil.is(commandParam, ...)`.

## Request body — generate

The payload binds to `CollectionSheetRequest`:

```json
{
  "officeId": 1,
  "staffId": 7,
  "transactionDate": "01 March 2024",
  "locale": "en",
  "dateFormat": "dd MMMM yyyy"
}
```

## Request body — save

```json
{
  "transactionDate": "01 March 2024",
  "actualDisbursementDate": "01 March 2024",
  "locale": "en",
  "dateFormat": "dd MMMM yyyy",
  "bulkRepaymentTransactions": [
    { "loanId": 12, "transactionAmount": 250.00, "paymentTypeId": 1 }
  ],
  "bulkSavingsDueTransactions": [
    { "savingsId": 24, "transactionAmount": 50.00, "paymentTypeId": 1 }
  ],
  "clientsAttendance": [
    { "clientId": 1, "attendanceType": 1 }
  ]
}
```

`attendanceType` maps to the standard meeting attendance code values (`1` Present, `2` Absent, `3` Approved, etc.).

## Response — generate

```json
{
  "dueDate": [2024, 3, 1],
  "loanProducts": [ { "id": 1, "name": "Standard Loan" } ],
  "savingsProducts": [ { "id": 1, "name": "Voluntary Savings" } ],
  "groups": [
    {
      "clients": [
        {
          "clientId": 1,
          "clientName": "Alice",
          "loans": [ { "loanId": 12, "accountId": "0000012", "totalDue": 250.00 } ],
          "savings": [ { "savingsId": 24, "accountId": "0000024", "dueAmount": 50.00 } ]
        }
      ]
    }
  ]
}
```

## Response — save

A `CommandProcessingResult`:

```json
{
  "officeId": 1,
  "resourceId": 0,
  "changes": {}
}
```
