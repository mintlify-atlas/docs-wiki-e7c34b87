The Two-Factor Configuration API exposes the tenant-level knobs that govern OTP-based two-factor authentication in Apache Fineract — token live times, the OTP length and validity window, available delivery methods (email / SMS), and SMTP/SMS provider configuration. The actual sign-in handshake lives at [Two Factor](/api/two-factor); this resource is where operators tune the policy that handshake enforces.

## Source

| Aspect | Value |
| --- | --- |
| Resource class | `org.apache.fineract.infrastructure.security.api.TwoFactorConfigurationApiResource` |
| File | `fineract-security/src/main/java/org/apache/fineract/infrastructure/security/api/TwoFactorConfigurationApiResource.java` |
| JAX-RS `@Path` | `/v1/twofactor/configure` |
| Default `@Consumes` / `@Produces` | `application/json` |
| Conditional bean | `@ConditionalOnProperty("fineract.security.2fa.enabled")` |
| Read service | `TwoFactorConfigurationService.retrieveAll()` |
| Write pipeline | `PortfolioCommandSourceWritePlatformService` via `CommandWrapperBuilder.updateTwoFactorConfiguration()` |
| Permission constant | `"TWOFACTOR_CONFIG"` |

## Endpoints

| Method | Path | Description | Command / handler | Permission |
| --- | --- | --- | --- | --- |
| `GET` | `/v1/twofactor/configure` | Return the full tenant two-factor configuration map. | `TwoFactorConfigurationService.retrieveAll()` | `READ_TWOFACTOR_CONFIG` |
| `PUT` | `/v1/twofactor/configure` | Update one or more configuration keys. Body is a JSON object of key→value pairs. | `CommandWrapperBuilder.updateTwoFactorConfiguration()` | `UPDATE_TWOFACTOR_CONFIG` (under maker-checker) |

The handlers are compact because all configuration logic lives in the service:

```java
@GET
public String retrieveAll() {
    this.context.authenticatedUser().validateHasReadPermission(RESOURCE_NAME_FOR_PERMISSIONS);
    Map<String, Object> configurationMap = configurationService.retrieveAll();
    return toApiJsonSerializer.serialize(configurationMap);
}

@PUT
public String updateConfiguration(final String apiRequestBodyAsJson) {
    final CommandWrapper commandRequest = new CommandWrapperBuilder().updateTwoFactorConfiguration()
            .withJson(apiRequestBodyAsJson).build();
    final CommandProcessingResult result = this.commandsSourceWritePlatformService.logCommandSource(commandRequest);
    return this.toApiJsonSerializer.serialize(result);
}
```

`RESOURCE_NAME_FOR_PERMISSIONS` is the literal string `"TWOFACTOR_CONFIG"`.

## Reading the configuration

```bash
curl -u mifos:password \
  -H 'fineract-platform-tenantid: default' \
  'https://fineract.example.org/fineract-provider/api/v1/twofactor/configure'
```

```json
{
  "otp-delivery-email-enable":     true,
  "otp-delivery-email-subject":    "Fineract one-time password",
  "otp-delivery-email-body":       "Your token is {{ token }}",
  "otp-delivery-sms-enable":       false,
  "otp-delivery-sms-provider":     "default",
  "otp-delivery-sms-text":         "Your token is {{ token }}",
  "otp-token-length":              6,
  "otp-token-live-time":           300,
  "access-token-live-time":        1800,
  "access-token-live-time-extended": 86400
}
```

Keys map to entries in `c_external_service_properties` / `m_two_factor_configuration` depending on Fineract version. The integer values (`*-live-time*`) are in seconds.

## Updating the configuration

```bash
curl -X PUT -u mifos:password \
  -H 'fineract-platform-tenantid: default' \
  -H 'Content-Type: application/json' \
  'https://fineract.example.org/fineract-provider/api/v1/twofactor/configure' \
  -d '{
        "otp-delivery-sms-enable": true,
        "otp-token-live-time": 180,
        "access-token-live-time-extended": 43200
      }'
```

```json
{
  "resourceId": null,
  "changes": {
    "otp-delivery-sms-enable": true,
    "otp-token-live-time": 180,
    "access-token-live-time-extended": 43200
  }
}
```

Keys absent from the payload remain unchanged. Because the resource goes through `PortfolioCommandSourceWritePlatformService.logCommandSource(...)`, the update participates in maker-checker when `UPDATE_TWOFACTOR_CONFIG` is flagged in [Permissions](/api/permissions). Pending changes are visible through [Maker Checkers](/api/maker-checkers).

## Configuration keys at a glance

| Key | Purpose |
| --- | --- |
| `otp-delivery-email-enable` | Allow OTPs to be delivered via SMTP. |
| `otp-delivery-email-subject` / `otp-delivery-email-body` | Templates rendered with `{{ token }}`. |
| `otp-delivery-sms-enable` | Allow OTPs to be delivered via SMS. |
| `otp-delivery-sms-provider` | Provider name (e.g. `default`, `twilio`). |
| `otp-delivery-sms-text` | SMS body template. |
| `otp-token-length` | Number of digits in the OTP. |
| `otp-token-live-time` | OTP validity, seconds. |
| `access-token-live-time` | Default session token lifetime, seconds. |
| `access-token-live-time-extended` | Extended session lifetime used when `extendedToken=true` is passed to `POST /v1/twofactor`. |

The exact set of keys is whatever `TwoFactorConfigurationService` exposes for the running version.

## Permissions

| Permission | Required for |
| --- | --- |
| `READ_TWOFACTOR_CONFIG` | `GET /v1/twofactor/configure` (validated by `validateHasReadPermission("TWOFACTOR_CONFIG")`). |
| `UPDATE_TWOFACTOR_CONFIG` | `PUT /v1/twofactor/configure` (command-handler permission, maker-checkerable). |
| `ALL_FUNCTIONS` | Wildcard alternative. |

Grant `READ_TWOFACTOR_CONFIG` to a "security operator" role and reserve `UPDATE_TWOFACTOR_CONFIG` for maker-checker workflows so that token-lifetime changes are reviewed before they take effect.
