The Credit Bureau Integration resource is the **runtime** side of the credit-bureau subsystem. While [`/v1/CreditBureauConfiguration`](/api/creditbureau-configuration) handles enrolment and credentials, the endpoints here perform the actual report lookups, upload pre-fetched reports, persist results to the local database, and let you list/delete saved reports per bureau.

## Source

- **File**: `fineract-provider/src/main/java/org/apache/fineract/infrastructure/creditbureau/api/CreditBureauIntegrationApiResource.java`
- **Base path**: `@Path("/v1/creditBureauIntegration")`
- **Tag**: `Credit Bureau Integration`

Reads use `CreditReportReadPlatformService`; writes are mostly command-sourced via `PortfolioCommandSourceWritePlatformService`. The multipart upload of an externally-fetched report goes directly to `CreditReportWritePlatformService.addCreditReport` (bypassing the command bus) because the payload is a binary file.

## Endpoints

| Method | Path | Description | Command builder / Handler | Permission |
| ------ | ---- | ----------- | ------------------------- | ---------- |
| POST | `/v1/creditBureauIntegration/creditReport` | Fetch a fresh report from the configured bureau (synchronous bureau call) | `CommandWrapperBuilder.getCreditReport` | `CREATE_CREDITREPORT` / per-bureau permissions |
| POST | `/v1/creditBureauIntegration/addCreditReport` | Multipart upload of an externally produced report file | `CreditReportWritePlatformService.addCreditReport` | typically `CREATE_CREDITREPORT` |
| POST | `/v1/creditBureauIntegration/saveCreditReport` | Persist a previously-fetched (in-memory) report into the local store | `CommandWrapperBuilder.saveCreditReport(creditBureauId, nationalId)` | `CREATE_CREDITREPORT` |
| GET | `/v1/creditBureauIntegration/creditReport/{creditBureauId}` | List saved reports for a bureau | `CreditReportReadPlatformService.retrieveCreditReportDetails` | authenticated user |
| DELETE | `/v1/creditBureauIntegration/deleteCreditReport/{creditBureauId}` | Delete saved reports for a bureau | `CommandWrapperBuilder.deleteCreditReport(id)` | `DELETE_CREDITREPORT` |

The list GET only calls `context.authenticatedUser()`; access control is left to the per-bureau permissions enforced inside the read service.

## Request bodies

### Fetch a report

`POST /v1/creditBureauIntegration/creditReport`

```json
{
  "creditBureauID": "1",
  "loanProductId": "5",
  "loanApplicationId": "120",
  "nationalIdentificationNumber": "12345678",
  "userName": "operator1"
}
```

The body is captured by the framework as a `Map<String, Object>` (`@RequestParam("params")`) and serialised to JSON before being passed to the command. Response includes the bureau-supplied report as the `creditReport` field on `CreditReportData`:

```json
{
  "officeId": null,
  "clientId": null,
  "resourceId": 0,
  "changes": {
    "creditReport": "{ ... bureau JSON ... }"
  }
}
```

### Persist a fetched report

`POST /v1/creditBureauIntegration/saveCreditReport?creditBureauId=1&nationalId=12345678`

The body is the JSON returned from the fetch step:

```json
{ "score": 712, "tradeLines": [/* ... */] }
```

### Multipart upload (when fetched outside Fineract)

`POST /v1/creditBureauIntegration/addCreditReport?creditBureauId=1`

`Content-Type: multipart/form-data; boundary=...`

Form parts:

| Part | Description |
| ---- | ----------- |
| `file` | The bureau-supplied PDF / JSON / XML report blob. |

Response is a plain string returned by `addCreditReport` (typically `"File uploaded successfully"` or an error message).

### List saved reports for a bureau

`GET /v1/creditBureauIntegration/creditReport/1`

```json
[
  { "id": 8, "creditBureauId": 1, "nrc": "12345678", "creditReport": "{\"score\":712,...}" },
  { "id": 9, "creditBureauId": 1, "nrc": "98765432", "creditReport": "{\"score\":640,...}" }
]
```

`RESPONSE_DATA_PARAMETERS` in source: `id`, `creditBureauId`, `nrc`, `creditReport`.

### Delete saved reports

`DELETE /v1/creditBureauIntegration/deleteCreditReport/1`

```json
{ "nrc": "12345678" }
```

When `nrc` is omitted, all saved reports for the bureau are removed.

## Subsystem cross-links

- **[Credit Bureau Configuration](/api/creditbureau-configuration)** — enrol the bureau and store its credentials before calling these endpoints.
- **[Loans](/api/loans)** — when a loan-product → bureau mapping has `isCreditCheckMandatory=true`, the loan workflow invokes `getCreditReport` automatically at appraisal time.
- **[Documents](/api/documents)** — large PDF reports may be alternatively attached as client documents.

## Notes

- The `fetchCreditReport` endpoint is synchronous — the bureau call happens inline and the HTTP response only returns after the bureau replies. Configure a reasonable HTTP timeout in your reverse proxy.
- `saveCreditReport` stores the raw JSON exactly as supplied; downstream consumers parse it. This intentionally keeps Fineract bureau-agnostic.
- The integration is **per-bureau**: there is no aggregated endpoint that pulls from multiple bureaus at once. Loop over `creditBureauId`s on the client side if you need a fan-out.
