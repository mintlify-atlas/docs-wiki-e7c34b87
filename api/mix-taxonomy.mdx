The Mix Taxonomy resource exposes the static list of XBRL elements Apache Fineract knows how to render in a MIX-format report. Each element represents one financial or operational fact (e.g. `GLP`, `NumberOfActiveBorrowers`, `OperatingExpense`) that the [`/v1/mixmapping`](/api/mix-taxonomy-mapping) configuration binds to GL accounts so [`/v1/mixreport`](/api/mix-report) can compute its value.

## Source

- **File**: `fineract-mix/src/main/java/org/apache/fineract/mix/api/MixTaxonomyApiResource.java`
- **Base path**: `@Path("/v1/mixtaxonomy")`
- **Tag**: `Mix Taxonomy`

The resource is read-only — the taxonomy is loaded from `m_mix_taxonomy` (seeded by the Fineract migration scripts in `fineract-mix`).

## Endpoints

| Method | Path | Description | Handler | Permission |
| ------ | ---- | ----------- | ------- | ---------- |
| GET | `/v1/mixtaxonomy` | List every taxonomy element | `MixTaxonomyReadService.retrieveAll` | authenticated user |

There are no create/update/delete endpoints: the XBRL taxonomy is a vendor-controlled schema that you adopt rather than edit.

## Response shape

The handler returns `List<MixTaxonomyData>`; the JSON shape is therefore a top-level array:

```json
[
  {
    "id": 1,
    "name": "Assets",
    "namespace": "mix:Assets",
    "dimension": null,
    "type": "monetaryItemType",
    "description": "Total assets at end of period."
  },
  {
    "id": 14,
    "name": "GrossLoanPortfolio",
    "namespace": "mix:GLP",
    "dimension": null,
    "type": "monetaryItemType",
    "description": "Outstanding principal balance of all loans."
  },
  {
    "id": 27,
    "name": "NumberOfActiveBorrowers",
    "namespace": "mix:NumberOfActiveBorrowers",
    "dimension": null,
    "type": "integerItemType",
    "description": "Number of borrowers with at least one active loan."
  }
]
```

| Field | Meaning |
| ----- | ------- |
| `id` | Primary key in `m_mix_taxonomy`; used by the mapping resource. |
| `name` | Human-readable label. |
| `namespace` | XBRL local name with `mix:` prefix — the tag that appears in the rendered XML. |
| `dimension` | Optional XBRL dimension (e.g. `urban / rural`). |
| `type` | XBRL data type: `monetaryItemType`, `integerItemType`, `decimalItemType`, `stringItemType`. |
| `description` | Tooltip text shown by the configuration UI. |

## Example

`GET /v1/mixtaxonomy`

Use this output to build the configuration UI for [`/v1/mixmapping`](/api/mix-taxonomy-mapping): for each element, the operator chooses the GL identifier or chart-of-accounts node that should drive its value.

## Subsystem cross-links

- **[Mix Taxonomy Mapping](/api/mix-taxonomy-mapping)** — assign GL identifiers to these elements.
- **[Mix Report](/api/mix-report)** — render an XBRL document using the mapping.
- **[GL Accounts](/api/gl-accounts)** — the chart of accounts whose ids the mapping references.

## Notes

- The taxonomy is the MIX Market 2010 schema (`mx-cor` namespace). It is intentionally not editable through the API; updates require a database migration.
- The `fineract-mix` Gradle module must be on the runtime classpath; absent that, the resource is not registered.
