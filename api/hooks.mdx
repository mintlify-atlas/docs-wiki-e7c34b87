The Hooks resource is Apache Fineract's mechanism for triggering custom code on portfolio events. A hook binds a **template** (Web hook, Twilio, processor, etc.) to a set of events (`{entityName, actionName}` pairs) and a key/value `config` blob. When the matching event is committed, the hook engine invokes the template — for the built-in `Web` template, that means POSTing the command JSON to the configured URL.

## Source

- **File**: `fineract-provider/src/main/java/org/apache/fineract/infrastructure/hooks/api/HookApiResource.java`
- **Base path**: `@Path("/v1/hooks")`
- **Permission entity**: `HOOK` (`HookApiConstants.HOOK_RESOURCE_NAME`)
- **Tag**: `Hooks`

Reads go through `HookReadPlatformService`; writes are command-sourced via `PortfolioCommandSourceWritePlatformService` and the `createHook` / `updateHook` / `deleteHook` builders in `CommandWrapperBuilder`.

## Endpoints

| Method | Path | Description | Command handler | Permission |
| ------ | ---- | ----------- | --------------- | ---------- |
| GET | `/v1/hooks` | List all registered hooks | `HookReadPlatformService.retrieveAllHooks` | `READ_HOOK` |
| GET | `/v1/hooks/{hookId}` | Retrieve one; `?template=true` embeds template and groupings | `HookReadPlatformService.retrieveHook` + `retrieveNewHookDetails` | `READ_HOOK` |
| GET | `/v1/hooks/template` | Empty hook shell + list of available templates, events and groupings | `HookReadPlatformService.retrieveNewHookDetails(null)` | `READ_HOOK` |
| POST | `/v1/hooks` | Create a hook | `CommandWrapperBuilder.createHook` → entity `HOOK`, action `CREATE` | `CREATE_HOOK` |
| PUT | `/v1/hooks/{hookId}` | Update fields, events and config | `CommandWrapperBuilder.updateHook(id)` → action `UPDATE` | `UPDATE_HOOK` |
| DELETE | `/v1/hooks/{hookId}` | Remove a hook | `CommandWrapperBuilder.deleteHook(id)` → action `DELETE` | `DELETE_HOOK` |

## Request body — create

Per the `@Operation` description:

| Field | Type | Description |
| ----- | ---- | ----------- |
| `name` | string, required | Template name. Pick from the `templates` array of `/hooks/template`. |
| `isActive` | boolean | Whether the hook fires. |
| `events` | array | List of `{ actionName, entityName }` pairs that trigger this hook. |
| `config` | object/hash, required | Template-specific settings (e.g. `payloadURL`, `contentType` for the Web template). |
| `templateId` | long, optional | UGD (User Generated Document) template id, when the hook needs to render a payload. |
| `displayName` | string, optional | UI label. |

### Example — create a webhook on loan disbursement

`POST /v1/hooks`

```json
{
  "name": "Web",
  "displayName": "Disbursement notifier",
  "isActive": true,
  "events": [
    { "actionName": "DISBURSE", "entityName": "LOAN" }
  ],
  "config": {
    "Content Type": "json",
    "Payload URL": "https://ops.example.org/fineract/webhook/disbursement"
  }
}
```

Response:

```json
{
  "officeId": null,
  "clientId": null,
  "resourceId": 5,
  "changes": {}
}
```

## Available templates

`GET /v1/hooks/template` enumerates them. The shipped templates are typically:

| Template | Purpose | Required `config` |
| -------- | ------- | ------------------ |
| `Web` | Generic webhook POST | `Payload URL`, `Content Type` (`json` or `form`) |
| `SMS Bridge` | Forward to an external SMS gateway | `endpoint`, `tenantId`, `mifosToken` |
| `Twilio` | Send SMS via Twilio | `accountSid`, `authToken`, `from` |

Custom templates can be registered through the hooks SPI (`HookSource` / `HookProcessor` extensions).

## Response data — `HookData`

```json
{
  "id": 5,
  "name": "Web",
  "displayName": "Disbursement notifier",
  "isActive": true,
  "createdAt": "2024-03-04T08:55:11",
  "updatedAt": "2024-03-04T08:55:11",
  "templateId": null,
  "templateName": "Web",
  "events": [
    { "actionName": "DISBURSE", "entityName": "LOAN" }
  ],
  "config": [
    { "fieldName": "Content Type", "fieldValue": "json" },
    { "fieldName": "Payload URL", "fieldValue": "https://ops.example.org/fineract/webhook/disbursement" }
  ]
}
```

With `?template=true`, the payload is enriched with `templates`, `groupings` (list of available event groups) and `events` (the full set of `{entityName, actionName}` pairs the platform can emit).

## Notes

- Hooks fire **after** a command is committed; failures in the hook do not roll back the transaction. They are retried on a separate schedule by the `HookProcessor`.
- `GET /v1/hooks/{id}?template=true` returns both the saved hook and the empty template merged together — useful for rendering an edit form pre-populated with the current values.
- Updating a hook with only `isActive: false` is the supported way to "pause" a webhook without losing its configuration.

## Curl reference

List hooks:

```bash
curl -u mifos:password \
  -H "Fineract-Platform-TenantId: default" \
  https://example.org/fineract-provider/api/v1/hooks
```

Create a hook:

```bash
curl -u mifos:password \
  -H "Fineract-Platform-TenantId: default" \
  -H "Content-Type: application/json" \
  -X POST https://example.org/fineract-provider/api/v1/hooks \
  -d '{
    "name": "Web",
    "displayName": "Loan disbursed → ops bus",
    "isActive": true,
    "events": [{ "actionName": "DISBURSE", "entityName": "LOAN" }],
    "config": {
      "Content Type": "json",
      "Payload URL": "https://ops.example.org/fineract/webhook/disbursement"
    }
  }'
```

Pause a hook in place (preserves config and events):

```bash
curl -u mifos:password \
  -H "Fineract-Platform-TenantId: default" \
  -H "Content-Type: application/json" \
  -X PUT https://example.org/fineract-provider/api/v1/hooks/5 \
  -d '{ "isActive": false }'
```

## Processor implementations

`HookProcessorProvider` resolves which `HookProcessor` bean to invoke for each saved hook by template name. The Apache Fineract codebase ships these processors under `org.apache.fineract.infrastructure.hooks.processor`:

| Template | Processor bean | Notes |
| --- | --- | --- |
| `Web` | `webHookProcessor` (`WebHookProcessor`) | Generic POST through a Retrofit-built `WebHookService`. |
| `SMS Bridge` | `messageGatewayHookProcessor` (`MessageGatewayHookProcessor`) | Forwards to a Mifos-CN message gateway. |
| `Twilio` | `twilioHookProcessor` (`TwilioHookProcessor`) | Drives the Twilio REST API. |
| `Elastic Search` | `elasticSearchHookProcessor` (`ElasticSearchHookProcessor`) | Indexes the payload into an ES cluster. |

Each processor implements the `HookProcessor` SPI — third parties can register additional `@Component` beans of that interface to add channels.

## Event delivery semantics

- Hooks fire **after** the originating command commits. The processor runs in a separate worker and a failed delivery does **not** roll back the business transaction.
- The payload posted to a `Web` hook is the JSON of the matched `CommandWrapper` plus the response body produced by the command handler. `actionName` and `entityName` are echoed in HTTP headers (`X-Fineract-Event`, `X-Fineract-Entity`).
- Retries are the responsibility of each processor; the default `WebHookProcessor` does **not** retry on HTTP 5xx — operators that need at-least-once delivery should front the hook URL with their own retry queue or sidecar.
- `displayName` is metadata only; the engine routes purely on the saved `(actionName, entityName)` events.

## Validation rules

| Error | When |
| --- | --- |
| `400 BAD_REQUEST` (`error.msg.hook.name.cannot.be.blank`) | `name` missing in the create body. |
| `400 BAD_REQUEST` (`error.msg.hook.events.cannot.be.empty`) | `events` array missing or empty. |
| `400 BAD_REQUEST` (`error.msg.hook.config.cannot.be.empty`) | `config` missing required fields for the chosen template. |
| `403 FORBIDDEN` | Caller lacks `READ_HOOK` / `CREATE_HOOK` / `UPDATE_HOOK` / `DELETE_HOOK`. |
| `404 NOT_FOUND` (`HookNotFoundException`) | `hookId` does not exist. |

## Operational tips

- Keep the receiver idempotent: the same `(commandId, hookId)` pair may be retried by an upstream queue and the engine does not deduplicate.
- Use distinct hooks per environment by setting `displayName` to include the target (`prod-ops-bus`, `staging-ops-bus`) — that field surfaces in the run logs and makes troubleshooting easier.
- The `Web` template uses the URL exactly as configured; HTTPS endpoints must present a certificate the JVM trusts. To add a private CA, set `javax.net.ssl.trustStore` on the Fineract JVM.
- For high-volume events (`REPAYMENT`, `CREATE_TRANSACTION`), prefer routing the hook to a thin sidecar that fan-outs to downstream consumers — the hook engine should not be the back-pressure point.
