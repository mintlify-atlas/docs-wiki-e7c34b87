`WorkingCapitalLoanProductApiResource` is the JAX-RS resource that maintains **Working Capital Loan Product** templates in Apache Fineract. A WCL product is a distinct product type from the classic `LoanProduct` — it carries WCL-specific knobs such as `periodPaymentRate`, `npvDayCount`, payment allocation rules and amortization-only configuration. Every WCL application submitted through [WorkingCapitalLoanApiResource](/api/working-capital-loans) references one of these products.

The implementation class name in source is `WorkingCapitalLoanProductApiResource` (singular). The task brief used the plural alias *WorkingCapitalLoanProductsApiResource* — this page documents the actual resource.

- **OpenAPI tag:** `Working Capital Loan Products` — *"A Working Capital Loan Product is a template that is used when creating a Working Capital loan. This is a separate product type from standard loan products."*

## Endpoints

| Method | Path | Description | Command / Handler | Permission |
| ------ | ---- | ----------- | ----------------- | ---------- |
| GET | `/v1/working-capital-loan-products` | List all WCL products. | `readPlatformService.retrieveAllWorkingCapitalLoanProducts()` | Read permission `WORKINGCAPITALLOANPRODUCT` |
| GET | `/v1/working-capital-loan-products/template` | Template for a new WCL product. | `readPlatformService.retrieveNewWorkingCapitalLoanProductDetails()` | Read permission `WORKINGCAPITALLOANPRODUCT` |
| GET | `/v1/working-capital-loan-products/{productId}` | Retrieve a WCL product. `?template=true` merges in defaults from the template. | `readPlatformService.retrieveWorkingCapitalLoanProduct(productId)` + optional `applyTemplate(...)` | Read permission `WORKINGCAPITALLOANPRODUCT` |
| GET | `/v1/working-capital-loan-products/external-id/{externalProductId}` | Retrieve by external id. | Resolves id via `resolveProductId(externalId)` then `retrieveWorkingCapitalLoanProduct(...)` | Read permission `WORKINGCAPITALLOANPRODUCT` |
| POST | `/v1/working-capital-loan-products` | Create a WCL product. Mandatory: name, shortName, currencyCode, digitsAfterDecimal, inMultiplesOf, amortizationType, npvDayCount, principal, periodPaymentRate, repaymentEvery, repaymentFrequencyType. Optional: externalId, fundId, startDate, closeDate, description, delinquencyBucketClassification, minPrincipal, maxPrincipal, minPeriodPaymentRate, maxPeriodPaymentRate, discount, paymentAllocation, allowAttributeOverrides. | `CommandWrapperBuilder.createWorkingCapitalLoanProduct()` → `CREATE_WORKINGCAPITALLOANPRODUCT` | `CREATE_WORKINGCAPITALLOANPRODUCT` |
| PUT | `/v1/working-capital-loan-products/{productId}` | Update a WCL product. | `CommandWrapperBuilder.updateWorkingCapitalLoanProduct(productId)` → `UPDATE_WORKINGCAPITALLOANPRODUCT` | `UPDATE_WORKINGCAPITALLOANPRODUCT` |
| PUT | `/v1/working-capital-loan-products/external-id/{externalProductId}` | Same update, keyed by external id. | As above, after id resolution. | `UPDATE_WORKINGCAPITALLOANPRODUCT` |
| DELETE | `/v1/working-capital-loan-products/{productId}` | Delete a WCL product (only if not referenced by any loan). | `CommandWrapperBuilder.deleteWorkingCapitalLoanProduct(productId)` → `DELETE_WORKINGCAPITALLOANPRODUCT` | `DELETE_WORKINGCAPITALLOANPRODUCT` |
| DELETE | `/v1/working-capital-loan-products/external-id/{externalProductId}` | Same delete, keyed by external id. | As above. | `DELETE_WORKINGCAPITALLOANPRODUCT` |

External-id endpoints resolve `productId` through:

```java
private Long resolveProductId(final ExternalId externalProductId) {
    try {
        return readPlatformService
            .retrieveWorkingCapitalLoanProductByExternalId(externalProductId)
            .getId();
    } catch (Exception e) {
        return null;
    }
}
```

A `null` resolution raises `WorkingCapitalLoanProductNotFoundException` (HTTP 404).

## Data carriers

- **Requests:** raw JSON (no DTO binding) — validated by `WorkingCapitalLoanProductFromApiJsonDeserializer`. Swagger types in `WorkingCapitalLoanProductApiResourceSwagger`:
  - `PostWorkingCapitalLoanProductsRequest`
  - `PutWorkingCapitalLoanProductsProductIdRequest`
- **Read responses:** `WorkingCapitalLoanProductData`, with `applyTemplate(...)` overlay.
- **Write responses:** `CommandProcessingResult` envelopes (Swagger: `PostWorkingCapitalLoanProductsResponse`, `PutWorkingCapitalLoanProductsProductIdResponse`, `DeleteWorkingCapitalLoanProductsProductIdResponse`, `GetWorkingCapitalLoanProductsResponse`, `GetWorkingCapitalLoanProductsTemplateResponse`).

## Permissions

All read endpoints call:

```java
this.context.authenticatedUser().validateHasReadPermission(RESOURCE_NAME_FOR_PERMISSIONS);
```

Writes are enforced by `PortfolioCommandSourceWritePlatformService` against the `CREATE_*` / `UPDATE_*` / `DELETE_*` actions above.
