Apache Fineract uses a single uniform lookup mechanism — `Code` /
`CodeValue` — for every system enumeration that is editable at runtime
(client types, loan purposes, payment types, gender, marital status, etc.).
The `infrastructure.codes` package owns the entities, repositories, the
constants enum, the read services and the two `CodesApiResource` /
`CodeValuesApiResource` resources (the resources themselves live in
`fineract-provider`). This page is the reference for the package.

## `Code` — the category

```java
@Entity
@Table(name = "m_code",
       uniqueConstraints = { @UniqueConstraint(columnNames = { "code_name" }, name = "code_name") })
public class Code extends AbstractPersistableCustom<Long> {

    @Column(name = "code_name", length = 100)
    private String name;

    @Column(name = "is_system_defined")
    private boolean systemDefined;

    @OneToMany(fetch = FetchType.EAGER, cascade = CascadeType.ALL, mappedBy = "code", orphanRemoval = true)
    private Set<CodeValue> values;

    public static Code fromJson(final JsonCommand command) {
        final String name = command.stringValueOfParameterNamed("name");
        return new Code().setName(name);
    }

    public Map<String, Object> update(final JsonCommand command) {
        if (this.systemDefined) {
            throw new SystemDefinedCodeCannotBeChangedException();
        }
        final Map<String, Object> actualChanges = new LinkedHashMap<>(1);
        if (command.isChangeInStringParameterNamed("name", this.name)) {
            final String newValue = command.stringValueOfParameterNamed("name");
            actualChanges.put("name", newValue);
            this.name = StringUtils.defaultIfEmpty(newValue, null);
        }
        return actualChanges;
    }

    public boolean remove(final CodeValue codeValueToDelete) { return this.values.remove(codeValueToDelete); }
}
```

Properties:

- **`m_code`** holds one row per category.
- `code_name` is unique (constraint name `code_name`).
- `is_system_defined` distinguishes codes the platform ships with — those
  cannot be renamed or removed. The `update` method throws
  `SystemDefinedCodeCannotBeChangedException` for such codes.
- `values` is an eager `@OneToMany(orphanRemoval = true)` to `CodeValue`. A
  removed `CodeValue` instance is cascade‑deleted from the database.

## `CodeValue` — a single value in the category

```java
@Entity
@Table(name = "m_code_value",
       uniqueConstraints = { @UniqueConstraint(columnNames = { "code_id", "code_value" },
                                               name = "code_value_duplicate") })
public class CodeValue extends AbstractPersistableCustom<Long> {

    @Column(name = "code_value", length = 100)  private String  label;
    @Column(name = "order_position")            private int     position;
    @Column(name = "code_description")          private String  description;
    @ManyToOne                                  @JoinColumn(name = "code_id", nullable = false)
                                                private Code    code;
    @Column(name = "is_active")                 private boolean isActive;
    @Column(name = "is_mandatory")              private boolean mandatory;

    public static CodeValue fromJson(final Code code, final JsonCommand command) {
        final String  label       = command.stringValueOfParameterNamed(NAME.getValue());
        Integer       position    = command.integerValueSansLocaleOfParameterNamed(POSITION.getValue());
        String        description = command.stringValueOfParameterNamed(DESCRIPTION.getValue());
        Boolean       isActiveObj = command.booleanObjectValueOfParameterNamed(IS_ACTIVE.getValue());
        boolean       isActive    = isActiveObj == null ? true : isActiveObj;
        if (position == null) position = 0;
        Boolean       mandatory   = command.booleanPrimitiveValueOfParameterNamed(IS_MANDATORY.getValue());
        return new CodeValue().setCode(code).setLabel(StringUtils.defaultIfEmpty(label, null))
                .setPosition(position).setDescription(description).setActive(isActive).setMandatory(mandatory);
    }
    // ...
}
```

| Column | Purpose |
| --- | --- |
| `code_value` | The display label (`"Passport"`, `"Joint"`, `"Cash"`). |
| `order_position` | Sort order in UI dropdowns. |
| `code_description` | Optional long description. |
| `code_id` | FK to `m_code`. |
| `is_active` | Hide from selection UIs without removing the row. |
| `is_mandatory` | The platform treats this value as mandatory in flows that consult the category (used by e.g. KYC required documents). |

`(code_id, code_value)` is uniquely constrained — two values within the
same category cannot have the same label.

## `CodeConstants.CodevalueJSONinputParams`

```java
public enum CodevalueJSONinputParams {
    CODEVALUE_ID("id"),

    POSITION("position"),
    DESCRIPTION("description"),
    IS_ACTIVE("isActive"),
    IS_MANDATORY("isMandatory");
```

Exposes a `getAllValues()` method that returns the full set, used by the
JSON deserializer for `checkForUnsupportedParameters`. The strings are the
JSON keys accepted by `POST /v1/codes/{id}/codevalues`.

## Repositories

`CodeRepository` and `CodeValueRepository` are Spring Data JPA repositories.
`CodeValueRepository` adds the platform‑specific finders most domains need:

```java
public interface CodeValueRepository extends JpaRepository<CodeValue, Long>,
                                              JpaSpecificationExecutor<CodeValue> {

    Collection<CodeValue> findByCodeName(String code);
    CodeValue findByCodeNameAndId(String code, Long id);
    Collection<CodeValue> findByCodeNameAndLabel(String code, String label);
    CodeValue             findByCodeNameAndLabelOrderByCodeValueAsc(String code, String label);
}
```

### `CodeValueRepositoryWrapper`

The wrapper adds the "find or throw" semantics every handler uses:

```java
@Service
public class CodeValueRepositoryWrapper {

    private final CodeValueRepository repository;

    public CodeValue findOneWithNotFoundDetection(final Long id) {
        return this.repository.findById(id)
                .orElseThrow(() -> new CodeValueNotFoundException(id));
    }
    // findOneByCodeNameAndIdWithNotFoundDetection, ...
}
```

`CodeValueNotFoundException` extends
`AbstractPlatformResourceNotFoundException` and is mapped to `404` by
[`PlatformResourceNotFoundExceptionMapper`](/core/exception-mappers).

## Read services

| Service | Purpose |
| --- | --- |
| `CodeReadPlatformService` | `retrieveAllCodes()`, `retrieveCode(id)` — returns `CodeData` DTOs. |
| `CodeValueReadPlatformService` | `retrieveCodeValuesByCode(codeName)`, `retrieveCodeValue(id)` — returns `CodeValueData` DTOs filtered by `is_active`. |

The read services rely on Spring caching (`@Cacheable` with cache names
`codes` and `codeValues`) so high‑volume read endpoints (template lookups,
client creation forms) do not re‑hit the DB on every request. The cache is
evicted by `@CacheEvict(value = "codeValues", allEntries = true)` on the
write services. The active `CacheManager` is the
[runtime‑delegating one](/core/cache-infrastructure) so caching can be
turned off without redeploying.

## `CodesApiResource`

Top‑level resource at `/v1/codes` (in `fineract-provider`):

```java
@Path("/v1/codes")
@Component
@RequiredArgsConstructor
public class CodesApiResource {

    private static final Set<String> RESPONSE_DATA_PARAMETERS = Set.of("id", "name", "systemDefined");
    private static final String RESOURCE_NAME_FOR_PERMISSIONS = "CODE";

    private final PlatformSecurityContext                   context;
    private final CodeReadPlatformService                   readPlatformService;
    private final DefaultToApiJsonSerializer<CodeData>      toApiJsonSerializer;
    private final ApiRequestParameterHelper                 apiRequestParameterHelper;
    private final PortfolioCommandSourceWritePlatformService commandsSourceWritePlatformService;

    @GET public String retrieveCodes(@Context UriInfo uriInfo) { … }
    @POST public String createCode(String apiRequestBodyAsJson) { … }
    @GET @Path("{codeId}") public String retrieveCode(@PathParam("codeId") Long codeId) { … }
    @PUT @Path("{codeId}") public String updateCode(@PathParam("codeId") Long codeId, String body) { … }
    @DELETE @Path("{codeId}") public String deleteCode(@PathParam("codeId") Long codeId) { … }
}
```

| Method | Path | Behaviour |
| --- | --- | --- |
| `GET` | `/v1/codes` | Lists every code. |
| `POST` | `/v1/codes` | Creates a user‑defined code (always `systemDefined=false`). Routed through `CommandWrapperBuilder().createCode()`. |
| `GET` | `/v1/codes/{codeId}` | Returns one code. |
| `PUT` | `/v1/codes/{codeId}` | Updates the name. Rejected with `SystemDefinedCodeCannotBeChangedException` for system codes. |
| `DELETE` | `/v1/codes/{codeId}` | Deletes a user‑defined code. |

Every write goes through
`PortfolioCommandSourceWritePlatformService.logCommandSource(commandRequest)` —
the same maker‑checker pipeline as everywhere else.

## `CodeValuesApiResource`

Resource at `/v1/codes/{codeId}/codevalues`:

| Method | Path | Behaviour |
| --- | --- | --- |
| `GET` | `/v1/codes/{codeId}/codevalues` | Lists active values for the code. |
| `POST` | `/v1/codes/{codeId}/codevalues` | Creates a code value (subject to the `(code_id, code_value)` unique constraint). |
| `GET` | `/v1/codes/{codeId}/codevalues/{codeValueId}` | Returns one code value. |
| `PUT` | `/v1/codes/{codeId}/codevalues/{codeValueId}` | Updates label, position, description, active flag, mandatory flag. |
| `DELETE` | `/v1/codes/{codeId}/codevalues/{codeValueId}` | Removes the row (orphan‑removed from the parent `Code.values` collection). |

The duplicate‑value path triggers `PlatformDataIntegrityException` via the
`code_value_duplicate` constraint → `403` via
`PlatformDataIntegrityExceptionMapper`.

## Examples of system‑defined codes

System codes are seeded by Liquibase and have `is_system_defined = true`.
Common examples (not exhaustive):

| Code name | Used by |
| --- | --- |
| `Customer Identifier` | Client → identification documents (passport, drivers license, national ID). |
| `ClientType` | Client metadata; also drives `AccountNumberPrefixType.CLIENT_TYPE`. |
| `ClientClassification` | Client classification dropdown. |
| `Gender` | Client gender. |
| `MaritalStatus` | Client demographics. |
| `LoanPurpose` | Loan creation purpose drop‑down. |
| `LoanCollateral` | Loan collateral types. |
| `PaymentType` | Transaction payment instruments (cash, cheque, bank transfer). Note: `PaymentType` is **also** modelled by a dedicated entity (`m_payment_type`); the legacy code‑value path remains for compatibility. |
| `AddressType` / `Country` / `State` | Used by the address sub‑resource when `enable-address` is on. |

The seed migrations live under
`fineract-provider/src/main/resources/db/changelog/tenant/parts/` — search
for `m_code` and `m_code_value`.

## Mapper

`CodeValueMapper` is a MapStruct mapper that converts `CodeValue` rows into
`CodeValueData` DTOs. Pattern is unremarkable; the mapper is referenced
explicitly by the read services so JPA fetch shape and DTO shape stay
de‑coupled.

## End‑to‑end: client picks an identification document type

```mermaid
sequenceDiagram
    autonumber
    participant UI as Web UI
    participant API as ClientsApiResource template
    participant CodesAPI as CodeValuesApiResource
    participant Read as CodeValueReadPlatformService
    participant Cache as Spring cache (codeValues)
    participant DB as m_code + m_code_value

    UI->>API: GET /clients/template
    API->>Read: retrieveCodeValuesByCode("Customer Identifier")
    Read->>Cache: lookup
    alt cache hit
        Cache-->>Read: cached list
    else cache miss
        Read->>DB: SELECT v.* FROM m_code_value v JOIN m_code c ON c.id = v.code_id WHERE c.code_name='Customer Identifier' AND v.is_active=true ORDER BY v.order_position
        DB-->>Read: rows
        Read->>Cache: store
    end
    Read-->>API: List<CodeValueData>
    API-->>UI: template JSON
    Note over UI: User picks "Passport"; UI POSTs identification record with codeValueId
```

## Common pitfalls

<AccordionGroup>
  <Accordion title="Editing a system-defined code">
    Throws `SystemDefinedCodeCannotBeChangedException` — checked **inside**
    `Code.update(JsonCommand)`. The mapper for this exception is
    `PlatformDomainRuleExceptionMapper` returning `403`. If you need to
    edit the *values* of a system code, that is allowed (only the parent
    `Code` row is protected, not its `CodeValue`s).
  </Accordion>
  <Accordion title="Querying by codeName needs the exact spelling">
    `CodeValueRepository.findByCodeName(...)` is case‑sensitive. The
    canonical names are the strings inserted by the Liquibase seeds — use
    them verbatim.
  </Accordion>
  <Accordion title="Inactive values disappear from the read API">
    `retrieveCodeValuesByCode` filters by `is_active=true`. Set
    `is_active=false` to hide a value from new transactions while keeping
    historical references intact.
  </Accordion>
  <Accordion title="Caching is global per JVM">
    The `codes` and `codeValues` caches are not per tenant — Spring's
    cache key uses the method arguments. A method called
    `retrieveCodeValuesByCode("Gender")` returns the same cached entry
    regardless of the calling tenant. This works because both tenants run
    the same Liquibase seed; for tenant‑specific overrides flip the cache
    type to `NO_CACHE` (see [cache infrastructure](/core/cache-infrastructure)).
  </Accordion>
</AccordionGroup>
