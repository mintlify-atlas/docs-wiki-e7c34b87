This page is the catalog for `org.apache.fineract.infrastructure.core` in
Apache Fineract's `fineract-core` Gradle module. Every other doc in the
`core/*` section drills into one of the subpackages listed here. Use this page
to find the class that owns a concern, then jump to the focused page for the
behavioural detail.

Top‑level subpackages (in source order):

```text
infrastructure/core/
├── annotation/       diagnostics/        jersey/        service/
├── aop/              domain/             jpa/           validator/
├── api/              exception/          logging/
├── boot/             exceptionmapper/    persistence/
├── component/        filters/            serialization/
├── condition/        config/             data/
```

In addition, the loose class
`org.apache.fineract.infrastructure.DataIntegrityErrorHandler` is documented
here because it lives in the parent `infrastructure` package and is consumed by
every write handler — see [`DataIntegrityErrorHandler.java`](#dataintegrityerrorhandler).

## `DataIntegrityErrorHandler`

Lives one package up at
`org.apache.fineract.infrastructure.DataIntegrityErrorHandler` and is included
here because it is consumed across the platform. Every write handler that
catches a JPA `DataIntegrityViolationException` delegates to it:

```java
// fineract-core/src/main/java/org/apache/fineract/infrastructure/DataIntegrityErrorHandler.java
@Component
@Slf4j
public class DataIntegrityErrorHandler {

    public void handleDataIntegrityIssues(final JsonCommand command, final Throwable realCause, final Exception dve,
            final String msgType, final String msgDescription) {
        if (realCause.getMessage().contains("external_id")) {
            final String externalId = command.stringValueOfParameterNamed("externalId");
            throw new PlatformDataIntegrityException("error.msg." + msgType + ".duplicate.externalId",
                    msgDescription + ": externalId `" + externalId + "` already exists", "externalId", externalId);
        }
        log.error("Error occured.", dve);
        throw ErrorHandler.getMappable(dve, "error.msg." + msgType + ".unknown.data.integrity.issue",
                "Unknown data integrity issue with resource: " + msgDescription);
    }
}
```

The handler turns a raw constraint violation into a domain‑appropriate
`PlatformDataIntegrityException`, which is then projected to `403` by
[`PlatformDataIntegrityExceptionMapper`](/core/exception-mappers).
